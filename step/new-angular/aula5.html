Serviços, Injeção de Dependências e HTTP

Serviços no Angular

1. O que são serviços?

Serviços no Angular são classes criadas para concentrar lógicas que não devem ficar diretamente dentro dos componentes.

Em vez de colocar toda a lógica da aplicação dentro do arquivo do componente, podemos separar responsabilidades e criar serviços para organizar melhor o projeto.

Assim, o componente fica mais focado na interface, enquanto o serviço fica responsável por tarefas como buscar dados, compartilhar informações, aplicar regras de negócio e se comunicar com APIs.

2. Para que servem os serviços?

  • Centralizar regras de negócio
  • Compartilhar dados entre componentes
  • Consumir APIs externas
  • Organizar melhor o código
  • Evitar repetição de lógica

3. Exemplo simples de uso de serviço

Imagine uma aplicação de estoque. Em vez de cadastrar e listar produtos diretamente em vários componentes, podemos criar um serviço responsável por manipular esses dados.

Dessa forma, qualquer componente que precisar dos produtos poderá usar o mesmo serviço.

Injeção de Dependências (Dependency Injection)

4. O que é Injeção de Dependências?

Injeção de Dependências é um mecanismo que permite que uma classe receba outra classe pronta para uso, sem precisar criá-la manualmente.

No Angular, isso acontece com muita frequência quando um componente precisa utilizar um serviço.

Em vez de o componente criar o serviço com new, o próprio Angular faz esse trabalho automaticamente.

5. Ideia principal da Injeção de Dependências

A ideia é desacoplar o código. Ou seja, fazer com que uma classe não fique fortemente presa à criação de outra.

Isso facilita:

  • Organização
  • Reutilização
  • Manutenção
  • Testes

6. Objetivos e tarefas da Injeção de Dependências

  • Fornecer instâncias prontas para uso
  • Reduzir acoplamento entre classes
  • Facilitar troca de implementações
  • Melhorar a manutenção do sistema
  • Permitir reaproveitamento de lógica

7. Como isso acontece no Angular?

Quando um serviço é registrado corretamente, o Angular consegue injetá-lo no componente por meio do construtor. 


constructor(private produtoService: ProdutoService) {}

Nesse exemplo, o Angular cria e entrega uma instância do ProdutoService para o componente.

Logo, o componente pode usar os métodos do serviço sem precisar se preocupar em criar manualmente esse objeto.

Exemplos de uso de serviços e DI

9. Situações comuns de uso

  • Buscar dados de uma API
  • Salvar informações temporárias
  • Compartilhar dados entre páginas
  • Fazer login e autenticação
  • Aplicar regras de negócio

10. Exemplo de serviço simples


export class ProdutoService {
  produtos = ['Notebook', 'Mouse', 'Teclado'];

  listarProdutos() {
    return this.produtos;
  }
}

Nesse exemplo, o serviço possui uma lista de produtos e um método para retorná-la.

11. Exemplo de componente usando o serviço


export class ListaProdutos {
  produtos: any = [];

  constructor(private produtoService: ProdutoService) {
    this.produtos = this.produtoService.listarProdutos();
  }
}

O componente recebe o serviço por injeção de dependência e usa o método listarProdutos() para obter os dados.

Uso de requisições HTTP

12. O que são requisições HTTP?

Requisições HTTP são chamadas feitas pela aplicação para se comunicar com servidores e APIs.

Com elas, podemos:

  • Buscar dados
  • Cadastrar dados
  • Atualizar informações
  • Remover registros

13. Principais métodos HTTP

Método Função
GET Buscar dados
POST Criar dados
PUT Atualizar dados
DELETE Excluir dados

14. Como o Angular faz requisições HTTP?

O Angular utiliza o HttpClient para fazer chamadas HTTP.

Esse recurso normalmente é usado dentro de serviços, e não diretamente dentro dos componentes.

15. Exemplo de uso de HttpClient


import { HttpClient } from '@angular/common/http';
import { Injectable } from '@angular/core';

@Injectable({
  providedIn: 'root'
})
export class ProdutoApiService {
  constructor(private http: HttpClient) {}

  listarProdutos() {
    return this.http.get('https://fakestoreapi.com/products');
  }
}

Nesse caso, o serviço faz uma requisição GET para uma API e devolve os dados para quem o utilizar.

16. Exemplo de componente consumindo a API


import { Component } from '@angular/core';
import { ProdutoApiService } from '../services/produto-api.service';

@Component({
  selector: 'app-produtos-api',
  imports: [],
  templateUrl: './produtos-api.html',
  styleUrl: './produtos-api.css'
})
export class ProdutosApi {
  produtos: any = [];

  constructor(private produtoApiService: ProdutoApiService) {}

  ngOnInit() {
    this.produtoApiService.listarProdutos().subscribe((dados: any) => {
      this.produtos = dados;
    });
  }
}

Aqui, o componente pede os dados ao serviço e, quando a resposta chega, salva os dados na variável produtos.

Observable (RxJS)

Um Observable é uma estrutura que representa um fluxo de dados ao longo do tempo. Ou seja, ele pode emitir vários valores de forma assíncrona.

No Angular, os Observables são muito utilizados principalmente em requisições HTTP, eventos e operações assíncronas.

2. Características principais

  • Pode emitir vários valores ao longo do tempo
  • Funciona de forma assíncrona
  • Precisa de um subscribe() para ser executado
  • Pode ser cancelado (unsubscribe)

3. Exemplo simples de Observable


import { Observable } from 'rxjs';

const meuObservable = new Observable(observer => {
  observer.next('Primeiro valor');
  observer.next('Segundo valor');
  observer.complete();
});

meuObservable.subscribe(valor => {
  console.log(valor);
});

4. Observable no Angular (HttpClient)

Quando usamos o HttpClient, o Angular retorna um Observable. 


listarProdutos(): Observable<Product[]> {
  return this.http.get<Product[]>(this.apiUrl);
}

Isso significa que a requisição só será executada quando fizermos o subscribe. 


this.produtoService.listarProdutos().subscribe(dados => {
  this.produtos = dados;
});

5. Ciclo de um Observable

  • next: emite um valor
  • error: emite um erro
  • complete: finaliza o fluxo

6. Diferença entre Observable e Promise

  • Promise: retorna apenas um valor
  • Observable: pode retornar vários valores
  • Observable: é cancelável

7. Async Pipe

Quando usamos subscribe() no componente, precisamos:

  • Controlar quando os dados chegam
  • Atualizar variáveis manualmente
  • Nos preocupar com unsubscribe (em alguns casos) 

produtos: Product[] = [];

ngOnInit() {
  this.produtoService.listarProdutos().subscribe(dados => {
    this.produtos = dados;
  });
}

Esse modelo funciona, mas aumenta o acoplamento e a responsabilidade do componente.

O async pipe resolve isso automaticamente no template.

  • Faz o subscribe automaticamente
  • Atualiza a view quando os dados chegam
  • Faz unsubscribe automaticamente

Ou seja, ele remove a necessidade de manipular manualmente Observables no componente. 


import { Component } from '@angular/core';
import { Observable } from 'rxjs';
import { ProdutoService, Product } from '../services/produto.service';

@Component({
  selector: 'app-produtos',
  templateUrl: './produtos.html'
})
export class ProdutosComponent {
  produtos$: Observable<Product[]>;

  constructor(private produtoService: ProdutoService) {
    this.produtos$ = this.produtoService.listarProdutos();
  }
}

Aqui não usamos subscribe. Apenas guardamos o Observable.

Template (.html)

<h1>Lista de Produtos</h1>

<ul>
  <li *ngFor="let produto of produtos$ | async">
    {{ produto.title }} - {{ produto.price | currency:'BRL' }}
  </li>
</ul>

O | async pega o Observable, faz o subscribe e entrega os dados diretamente para o template.

4. Exemplo com loading e controle de estado


<h1>Produtos</h1>

<ul *ngIf="produtos$ | async as produtos; else loading">
  <li *ngFor="let produto of produtos">
    {{ produto.title }}
  </li>
</ul>

<ng-template #loading>
  <p>Carregando produtos...</p>
</ng-template>

Aqui usamos "as produtos" para armazenar o resultado do async pipe e reutilizar sem chamar novamente.

5. Quando usar Async Pipe

  • Quando estiver trabalhando com Observable
  • Quando quiser código mais limpo
  • Quando quiser evitar vazamento de memória

6. Quando NÃO usar

  • Quando precisar manipular dados antes de exibir (usar subscribe + lógica)
  • Quando precisar combinar múltiplos fluxos complexos no componente

Resumo direto

  • subscribe: controle manual
  • async pipe: controle automático (melhor para UI)

Signal no Angular

O signal é um novo mecanismo reativo do Angular que permite armazenar estado e reagir automaticamente às mudanças desse estado.

  • Substitui parcialmente o uso de Observables em cenários simples
  • Permite leitura reativa com ()
  • Atualização com .set()

  const contador = signal(0);
  
  contador.set(1);
  console.log(contador()); // 1

Passo 2 — Inserir a lógica do serviço

Abra o arquivo produto.ts que está dentro da pasta services e insira: 


import { Injectable } from '@angular/core';
import { HttpClient } from '@angular/common/http';
import { Observable } from 'rxjs';

export interface Rating {
  rate: number;
  count: number;
}

export interface Product {
  id: number;
  title: string;
  price: number;
  description: string;
  category: string;
  image: string;
  rating: Rating;
}

@Injectable({
  providedIn: 'root',
})
export class Produto {
  private readonly apiUrl = 'https://fakestoreapi.com/products';

  constructor(private http: HttpClient) {}

  listarProdutos(): Observable<Product[]> {
    return this.http.get<Product[]>(this.apiUrl);
  }

  buscarProdutoPorId(id: string | number): Observable<Product> {
    return this.http.get<Product>(`${this.apiUrl}/${id}`);
  }
}

Passo 4 — Inserir a lógica da lista de produtos

Abra o arquivo lista-produtos.ts e insira:


import { Component, OnInit, signal } from '@angular/core';
import { CommonModule } from '@angular/common';
import { RouterModule } from '@angular/router';
import { Produto } from '../../services/produto';

export interface Rating {
  rate: number;
  count: number;
}

export interface Product {
  id: number;
  title: string;
  price: number;
  description: string;
  category: string;
  image: string;
  rating: Rating;
}

@Component({
  selector: 'app-lista-produtos',
  imports: [CommonModule, RouterModule],
  templateUrl: './lista-produtos.html',
  styleUrl: './lista-produtos.css',
})
export class ListaProdutos implements OnInit {
  produtos = signal<Product[]>([]);
  carregando = signal(true);

  constructor(private produtoService: Produto) {}

  ngOnInit(): void {
    this.produtoService.listarProdutos().subscribe({
      next: (dados: Product[]) => {
        this.produtos.set(dados);
        this.carregando.set(false);
      },
      error: (erro: any) => {
        console.error(erro);
        this.carregando.set(false);
      },
    });
  }
}

Agora abra o arquivo lista-produtos.html e insira: 


<h1>Lista de Produtos</h1>

@if (carregando()) {
  <p>Carregando produtos...</p>
} @else {
  <ul>
    @for (produto of produtos(); track produto.id) {
      <li>
        {{ produto.title }} -
        <a [routerLink]="['/produto', produto.id]">Ver detalhes</a>
      </li>
    }
  </ul>
}

Passo 5 — Inserir a lógica do detalhe do produto

Abra o arquivo detalhe-produto.ts e insira:


import { Component, OnInit, signal } from '@angular/core';
import { CommonModule } from '@angular/common';
import { ActivatedRoute, RouterModule } from '@angular/router';
import { Produto, Product } from '../../services/produto';

@Component({
  selector: 'app-detalhe-produto',
  standalone: true,
  imports: [CommonModule, RouterModule],
  templateUrl: './detalhe-produto.html',
  styleUrl: './detalhe-produto.css',
})
export class DetalheProduto implements OnInit {
  produto = signal<Product | null>(null);
  carregando = signal(true);
  erro = signal(false);

  constructor(
    private route: ActivatedRoute,
    private produtoService: Produto,
  ) {}

  get produtoDetalhe(): Product | null {
    return this.produto();
  }

  ngOnInit(): void {
    const id = this.route.snapshot.paramMap.get('id');

    if (!id) {
      console.error('ID do produto não encontrado na rota');
      this.carregando.set(false);
      this.erro.set(true);
      return;
    }

    this.produtoService.buscarProdutoPorId(id).subscribe({
      next: (dados) => {
        this.produto.set(dados);
        this.carregando.set(false);
      },
      error: (erro) => {
        console.error('Erro ao buscar produto:', erro);
        this.carregando.set(false);
        this.erro.set(true);
      },
    });
  }
}

Passo 6 — Inserir o template do detalhe do produto

Agora abra o arquivo detalhe-produto.html e insira:

Passo 7 — Configurar o HttpClient

Para que o Angular consiga fazer requisições HTTP, precisamos ativar o provider do HttpClient na configuração da aplicação.

No arquivo principal de configuração da aplicação, app.config.ts, insira algo como: 


import { ApplicationConfig } from '@angular/core';
import { provideRouter } from '@angular/router';
import { provideHttpClient } from '@angular/common/http';
import { routes } from './app.routes';

export const appConfig: ApplicationConfig = {
  providers: [
    provideRouter(routes),
    provideHttpClient()
  ]
};
