Guardiões de Rota (Route Guards)

O que são Guardiões de Rota?

Guardiões de rota (Route Guards) são mecanismos do Angular que permitem controlar o acesso às rotas da aplicação.

Eles são usados para decidir se um usuário pode ou não acessar uma determinada página.

Para que servem?

  • Proteger páginas privadas
  • Validar autenticação
  • Controlar permissões (admin, usuário comum, etc)
  • Evitar acesso indevido

Exemplo prático

Você pode impedir que um usuário acesse a rota /painel-adm caso ele não esteja logado.

Tipos de Guards

  • CanActivate → controla acesso à rota
  • CanActivateChild → protege rotas filhas
  • CanDeactivate → impede sair da página
  • Resolve → carrega dados antes da rota
  • CanMatch → controla se rota pode ser carregada

Angular moderno (IMPORTANTE)

No Angular atual, usamos funções ao invés de classes para criar guards.

Isso é feito usando CanActivateFn e a função inject().

Exemplo de Guard moderno


import { CanActivateFn } from '@angular/router';

export const authGuard: CanActivateFn = () => {
  return true;
};

Usando inject()

O inject() substitui o constructor para injeção de dependência. 


import { inject } from '@angular/core';
import { Router } from '@angular/router';

export const authGuard: CanActivateFn = () => {
  const router = inject(Router);

  const logado = false;

  if (!logado) {
    router.navigate(['/login']);
    return false;
  }

  return true;
};

Aplicando o guard na rota


import { authGuard } from './guards/auth.guard';

export const routes = [
  {
    path: 'painel-adm',
    loadComponent: () =>
      import('./painel-adm/painel-adm').then(m => m.PainelAdm),
    canActivate: [authGuard]
  }
];

Agora essa rota só será acessada se o guard permitir.

Resumo

  • Guards protegem rotas
  • Podem bloquear acesso
  • São usados para autenticação e autorização
  • Hoje usamos funções (CanActivateFn)
  • inject() substitui constructor

Passo a Passo: Criando um Guard de Autenticação

Passo 0 — Criar branch e acessar a branch

Acesse novamente o repositório no GitHub e crie agora uma nova branch baseada na branch aula5 com o nome abaixo: 


aula6-guards

No terminal, execute o comando abaixo:


git fetch
git checkout aula6-guards

Passo 1 — Criar o guard

No terminal, execute o comando abaixo:


ng g guard guards/auth

Passo 2 — Implementar o guard

No arquivo auth.guard.ts, insira o seguinte código:


import { CanActivateFn } from '@angular/router';
import { inject } from '@angular/core';
import { Router } from '@angular/router';

export const authGuard: CanActivateFn = () => {
  const router = inject(Router);

  const token = localStorage.getItem('token');

  if (!token) {
    router.navigate(['/login']);
    return false;
  }

  return true;
};

Passo 3 — Aplicar na rota

No arquivo app.routes.ts, insira o seguinte código:


import { Routes } from '@angular/router';
import { authGuard } from './guards/auth-guard';

export const routes: Routes = [
  {
    path: '',
    redirectTo: 'produtos',
    pathMatch: 'full',
  },
  {
    path: 'produtos',
    loadComponent: () =>
      import('./pages/lista-produtos/lista-produtos').then(
        (m) => m.ListaProdutos
      ),
  },
  {
    path: 'produto/:id',
    loadComponent: () =>
      import('./pages/detalhe-produto/detalhe-produto').then(
        (m) => m.DetalheProduto
      ),
  },
  {
    path: 'painel-adm',
    loadComponent: () =>
      import('./painel-adm/painel-adm').then(
        (m) => m.PainelAdm
      ),
    canActivate: [authGuard],
  },
  {
    path: '**',
    redirectTo: 'produtos',
  }
];

Passo 4 — Rodar aplicação


ng serve

Passo 5 - Simule login

No navegador abra o console e execute o comando abaixo:

Assim, vamos inserir o token para simular o login, fazendo com que a rota fique acessível. 


localStorage.setItem('token', '123456');

Agora, Acesse a rota /painel-adm com o token no localStorage, e veja se terá acesso a rota.

Passo 6 - Simule logout

No navegador abra o console e execute o comando abaixo:

Assim, removeremos o token para simular o logout, fazendo com que a rota fique inacessível. 


localStorage.removeItem('token');

Agora, Acesse a rota /painel-adm sem o token no localStorage, e veja se terá acesso a rota.

Passo 7 — Teste a rota protegida

Acesse a rota /painel-adm sem estar logado para verificar o funcionamento do guard.

git add .

Dever de Casa

Proteger rota detalhes-produto se usuário for admin

Implemente um guard chamado role que verifica se o usuário for administrador, ele acessa a página detalhes-produto, caso contrário, ele é redirecionado para a página de lista de produtos

Você deve fazer o localStorage receber a propriedade isAdmin e colocar um valor true ou false

Obs: Suba o código para o Git após finalizar

Gabarito: 1. Inserir no localStorage a flag isAdmin no navegador 
  localStorage.setItem('isAdmin', 'true');
2. Criar o guard de role - arquivo role.guard.ts 

export const roleGuard: CanActivateFn = () => {
  const router = inject(Router);
  const isAdmin = localStorage.getItem('isAdmin') === 'true';
  if (!isAdmin) {
    router.navigate(['/produtos']);
    return false;
  }
  return true;
};
