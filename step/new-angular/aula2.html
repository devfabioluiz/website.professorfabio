Diretivas, Data Binding e Pipes

Diretivas no Angular

1. O que são diretivas?

Diretivas são instruções especiais do Angular usadas dentro do HTML para modificar a estrutura, a aparência ou o comportamento dos elementos da página.

Em outras palavras, as diretivas permitem que o Angular “faça algo” com um elemento HTML.

Elas são muito importantes porque ajudam a deixar a interface dinâmica, isto é, capaz de reagir aos dados e às ações do usuário.

2. Para que servem as diretivas?

  • Exibir ou esconder elementos
  • Repetir elementos em listas
  • Aplicar estilos dinamicamente
  • Alterar classes CSS
  • Controlar comportamento da interface

3. Tipos de diretivas

No Angular, de forma geral, trabalhamos com três grupos principais:

  • Diretivas estruturais
  • Diretivas de atributo
  • Diretivas personalizadas

4. Diretivas estruturais

As diretivas estruturais alteram a estrutura do DOM, ou seja, elas podem criar ou remover elementos da tela.

As mais comuns são:

  • *ngIf → mostra ou esconde elementos
  • *ngFor → repete elementos
  • *ngSwitch → escolhe entre várias opções

Exemplo com *ngIf

O *ngIf renderiza um elemento somente se a condição for verdadeira. 


<p *ngIf="estaLogado">Usuário autenticado com sucesso.</p>

Nesse caso, o parágrafo só será exibido se a variável estaLogado tiver valor verdadeiro.

Exemplo com *ngFor

O *ngFor é utilizado para repetir um trecho do HTML para cada item de uma lista. 


// No componente temos a seguinte variável: produtos = ['Notebook', 'Teclado', 'Mouse'];
<ul>
<li *ngFor="let produto of produtos">
    {{ produto }}
  </li>
</ul>

Aqui, para cada item do array produtos, um elemento li será criado.

Exemplo com *ngSwitch

O *ngSwitch é útil quando queremos exibir conteúdos diferentes conforme o valor de uma variável. 


// No componente temos a variável: perfil = 'admin'; Porém poderia ser 'cliente' ou outro valor.
<div [ngSwitch]="perfil">
  <p *ngSwitchCase="'admin'">Administrador do sistema</p>
  <p *ngSwitchCase="'cliente'">Cliente da aplicação</p>
  <p *ngSwitchDefault>Perfil não identificado</p>
</div>

5. Diretivas de atributo

As diretivas de atributo alteram a aparência ou o comportamento de um elemento já existente, mas sem remover ou recriar esse elemento.

As mais comuns são:

  • [ngClass] → adiciona ou remove classes CSS
  • [ngStyle] → aplica estilos inline dinamicamente

Exemplo com [ngClass]


<p [ngClass]="{ 'ativo': estaAtivo, 'inativo': !estaAtivo }">
  Status do usuário
</p>

Se estaAtivo for verdadeiro, a classe ativo será aplicada. Caso contrário, a classe inativo será aplicada.

Exemplo com [ngStyle]


<p [ngStyle]="{ 'color': corTexto, 'font-size': tamanhoFonte }">
  Texto estilizado dinamicamente
</p>

Nesse caso, o Angular aplica os estilos com base nos valores das variáveis declaradas no componente.

Ligação de dados no Angular (Data Binding)

6. O que é Data Binding?

Data Binding é o mecanismo que permite conectar os dados do componente com o template HTML.

É graças ao Data Binding que conseguimos exibir valores na tela, capturar eventos do usuário e sincronizar informações entre a interface e a lógica da aplicação.

7. Tipos principais de Data Binding

  • Interpolação
  • Property Binding
  • Event Binding
  • Two-Way Data Binding

8. Interpolação

A interpolação é usada para exibir valores do componente no HTML.


<h1>{{ titulo }}</h1>
<p>Bem-vindo, {{ nomeUsuario }}</p>

As expressões dentro de {{ }} são avaliadas pelo Angular e exibidas no template.

9. Property Binding

O Property Binding é usado para enviar valores do componente para propriedades de elementos HTML. 


// No componente temos as variáveis:
imagemUrl = 'caminho/para/imagem.jpg';
descricaoImagem = 'Descrição da imagem';
botaoDesabilitado = true;

<img [src]="imagemUrl" [alt]="descricaoImagem" />
<button [disabled]="botaoDesabilitado">Salvar</button>

Aqui, o Angular preenche as propriedades do HTML com os valores das variáveis do componente.

10. Event Binding

O Event Binding permite capturar eventos da interface, como clique, digitação, foco e outros. 


<button (click)="mostrarMensagem()">Clique aqui</button>

Quando o botão for clicado, o método mostrarMensagem() será executado.

11. Two-Way Data Binding

O Two-Way Data Binding faz uma ligação em dois sentidos: do componente para a interface e da interface para o componente.

Esse recurso é muito usado em formulários.


// No componente temos a variável: nome = '';

<input [(ngModel)]="nome" />
<p>Nome digitado: {{ nome }}</p>

Nesse exemplo, tudo o que for digitado no campo de entrada será automaticamente refletido na variável nome, e o valor atualizado também aparecerá no parágrafo.

Para usar ngModel, normalmente precisamos importar o FormsModule na propriedade imports do componente.

Pipes em uma aplicação Angular

12. O que são Pipes?

Pipes são recursos do Angular usados para transformar dados no template antes de exibi-los ao usuário.

Em vez de alterar o valor original no componente, o pipe modifica apenas a forma como ele será apresentado na interface.

13. Para que servem os Pipes?

  • Formatar textos
  • Formatar datas
  • Formatar moedas
  • Formatar números
  • Organizar dados para exibição

14. Sintaxe de um Pipe


{{ valor | nomeDoPipe }}

O símbolo | indica que o valor será transformado pelo pipe.

15. Exemplos de Pipes nativos

  • uppercase → deixa o texto em maiúsculo
  • lowercase → deixa o texto em minúsculo
  • titlecase → primeira letra maiúscula
  • date → formata datas
  • currency → formata valores monetários
  • number → formata números
  • percent → formata porcentagem

Exemplo com uppercase


<p>{{ nome | uppercase }}</p>

Exemplo com date


<p>{{ hoje | date:'dd/MM/yyyy' }}</p>

Exemplo com currency


<p>{{ preco | currency:'BRL' }}</p>

16. Encadeando Pipes

Também podemos aplicar mais de um pipe em sequência.


<p>{{ nomeCompleto | lowercase | titlecase }}</p>

Passo a Passo: Aplicando diretivas, data binding e pipes

Passo 0 — Criar uma nova branch no GitHub

Acesse novamente o repositório no GitHub e crie agora uma nova branch com o nome: 


aula2-diretivas-binding-pipes

Depois, no terminal do VSCode, dentro da pasta do projeto, execute: 


git fetch
git checkout aula2-diretivas-binding-pipes

Passo 1 — Criar um novo componente

Agora vamos criar um componente para praticar os conceitos desta aula. 


ng g c painel

Passo 2 — Inserir a lógica do componente

Abra o arquivo painel.ts e insira o código abaixo: 


import { Component } from '@angular/core';
import { CommonModule } from '@angular/common';
import { FormsModule } from '@angular/forms';

@Component({
  selector: 'app-painel',
  imports: [CommonModule, FormsModule],
  templateUrl: './painel.html',
  styleUrl: './painel.css',
})
export class Painel {
  nome = 'Fabio';
  estaLogado = true;
  botaoDesabilitado = false;
  perfil = 'admin';
  corTexto = 'blue';
  tamanhoFonte = '22px';
  hoje = new Date();
  preco = 199.9;

  produtos = ['Notebook', 'Teclado', 'Mouse', 'Monitor'];

  mostrarMensagem() {
    alert('Botão clicado com sucesso!');
  }

  alternarLogin() {
    this.estaLogado = !this.estaLogado;
  }
}

Passo 3 — Inserir o template HTML do componente

Abra o arquivo painel.html e insira o código abaixo: 


<h1>{{ nome | uppercase }}</h1>

<input [(ngModel)]="nome" placeholder="Digite seu nome" />

<p *ngIf="estaLogado">Usuário logado no sistema</p>
<p *ngIf="!estaLogado">Usuário não está logado</p>

<button (click)="alternarLogin()">Alternar login</button>
<button (click)="mostrarMensagem()" [disabled]="botaoDesabilitado">
  Exibir mensagem
</button>

<h2>Lista de produtos</h2>
<ul>
  <li *ngFor="let produto of produtos">
    {{ produto }}
  </li>
</ul>

<div [ngSwitch]="perfil">
  <p *ngSwitchCase="'admin'">Perfil Administrador</p>
  <p *ngSwitchCase="'cliente'">Perfil Cliente</p>
  <p *ngSwitchDefault>Perfil desconhecido</p>
</div>

<p [ngStyle]="{ 'color': corTexto, 'font-size': tamanhoFonte }">
  Texto estilizado dinamicamente



<p>Data atual: {{ hoje | date:'dd/MM/yyyy' }}</p>
<p>Preço do produto: {{ preco | currency:'BRL' }}</p>

Passo 4 — Inserir o componente no template principal

Entre em app.ts e insira o código abaixo na parte de imports.


import { Component, signal } from '@angular/core';
import { RouterOutlet } from '@angular/router';
import { Exemplo } from './exemplo/exemplo';
import { Painel } from './painel/painel';
@Component({
    selector: 'app-root', 
    imports: [RouterOutlet, Exemplo, Painel],
    templateUrl: './app.html',
    styleUrl: './app.css'
})

Perceba que toda vez que criarmos um componente, precisamos importá-lo no arquivo principal da aplicação caso queiramos usá-lo dentro do template do app.html

Agora entre no arquivo app.html, apague o conteúdo atual e insira: 


<app-painel></app-painel>

Passo 5 — Entendendo o que foi feito

Nesse exemplo, aplicamos vários conceitos importantes:

  • Interpolação para exibir variáveis
  • Two-Way Data Binding com ngModel
  • Event Binding com click
  • Property Binding com disabled
  • *ngIf para condição
  • *ngFor para repetição
  • ngSwitch para múltiplas condições
  • ngStyle para estilo dinâmico
  • Pipes para formatação dos dados

Passo 6 — Rodar a aplicação

No terminal execute:


ng serve

Acesse:


http://localhost:4200

Você verá um exemplo prático reunindo diretivas, ligação de dados e pipes em uma mesma interface.

Passo 7 — Testar alterações

Faça alguns testes na aplicação:

  • Digite outro nome no campo de entrada
  • Veja o nome sendo atualizado em tempo real
  • Clique no botão para alternar o estado de login
  • Observe os produtos sendo renderizados em lista
  • Veja a data e o valor monetário formatados
