Diretivas no Angular

1. O que são diretivas?

Diretivas são instruções especiais do Angular usadas dentro do HTML para modificar a estrutura, a aparência ou o comportamento dos elementos da página.

Em outras palavras, as diretivas permitem que o Angular “faça algo” com um elemento HTML. Elas são muito importantes porque ajudam a deixar a interface dinâmica, isto é, capaz de reagir aos dados e às ações do usuário.

Em Angular moderno, parte das antigas diretivas estruturais evoluiu para uma nova forma chamada control flow, utilizando:

  • @if
  • @for
  • @switch

Essas estruturas tornam o código mais legível e mais próximo da lógica de programação tradicional.

2. Para que servem?

  • Controlar exibição de elementos
  • Renderizar listas
  • Alterar estilos dinamicamente
  • Controlar estados da interface

3. Exemplos de diretivas

Exemplo com @if


@if (estaLogado) {
  <p>Usuário autenticado</p>
} @else {
  <p>Usuário não autenticado</p>
}

Exemplo com @for


// componente
produtos = [
  { id: 1, nome: 'Notebook' },
  { id: 2, nome: 'Teclado' },
  { id: 3, nome: 'Mouse' }
];

<ul>
  @for (produto of produtos; track produto.id) {
    <li>{{ produto.nome }}</li>
  } @empty {
    <li>Nenhum produto disponível</li>
  }
</ul>

Exemplo com @switch


@switch (perfil) {
  @case ('admin') {
    <p>Administrador</p>
  }
  @case ('cliente') {
    <p>Cliente</p>
  }
  @default {
    <p>Perfil desconhecido</p>
  }
}

5. Diretivas de atributo

As diretivas de atributo alteram a aparência ou o comportamento de um elemento já existente, mas sem remover ou recriar esse elemento. As mais comuns são:

  • [ngClass] → adiciona ou remove classes CSS
  • [ngStyle] → aplica estilos inline dinamicamente

Exemplo com ngClass


<p [ngClass]="{
  'online': estaLogado,
  'offline': !estaLogado
}">
  Status do usuário
</p>

Exemplo com ngStyle


<p [ngStyle]="{
  'color': estaLogado ? 'green' : 'red'
}">
  Situação de acesso
</p>

Hora de Praticar

Passo 0 — Criar branch

Acesse novamente o repositório no GitHub e crie agora uma nova branch baseada na branch aula1-angular com o nome: 


aula2-diretivas

Passo 1 — Criar componente


ng g c aula2

Passo 2 — Código do componente

Entre em aula2.ts e substitua o conteúdo existente por este código abaixo: 


import { Component } from '@angular/core';
import { CommonModule } from '@angular/common';

@Component({
  selector: 'app-aula2',
  standalone: true,
  imports: [CommonModule],
  templateUrl: './aula2.html',
  styleUrl: './aula2.css',
})
export class Aula2 {
  estaLogado = true;
  perfil = 'admin';

  produtos = [
    { id: 1, nome: 'Notebook' },
    { id: 2, nome: 'Teclado' },
    { id: 3, nome: 'Mouse' },
  ];

  alternarLogin() {
    this.estaLogado = !this.estaLogado;
  }
}

Passo 3 — Template

Agora entre em aula2.html e substitua o conteúdo existente por este código abaixo: 


@if (estaLogado) {
  <p>Logado</p>
} @else {
  <p>Não logado</p>
}

<button (click)="alternarLogin()">
  Alternar
</button>

<ul>
  @for (produto of produtos; track produto.id) {
    <li>{{ produto.nome }}</li>
  }
</ul>

@switch (perfil) {
  @case ('admin') {
    <p>Admin</p>
  }
  @default {
    <p>Outro</p>
  }
}

<p [ngClass]="{ 'logado': estaLogado }">
  Classe dinâmica
</p>

Passo 4 - Adicionando estilo ao componente

Entre em aula2.css e substitua o conteúdo existente por este código abaixo: 


.logado {
  color: green;
}

Passo 5 — Usar no app

Entre em app.ts e insira o componente Aula2 na parte de imports tal como no código abaixo. 


import { Component, signal } from '@angular/core';
import { Aula2 } from './aula2/aula2';
@Component({
  selector: 'app-root',
  imports: [Aula2],
  templateUrl: './app.html',
  styleUrl: './app.css',
})
export class App {
  protected readonly title = signal('minha-aplicacao-angular');
}

Agora precisamos importá-lo no arquivo principal da aplicação caso queiramos usá-lo dentro do template do app.html. Logo, devemos acesar o arquivo app.html, apagar o conteúdo atual dentro dele e insirir este: 


<app-aula2></app-aula2>

Passo 6 — Rodar


ng s
