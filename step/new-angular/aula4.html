Roteamento e Navegação no Angular

O que é navegação no Angular?

Navegação é o processo de mudar de uma tela para outra dentro da aplicação.

Em aplicações Angular, isso normalmente é feito com rotas.

O que são rotas?

Rotas definem qual componente será exibido quando um determinado caminho da URL for acessado.

Exemplo:

  • /home → página inicial
  • /produtos → página de produtos
  • /detalhe/1 → detalhe de um produto

Exemplo de rotas


import { Routes } from '@angular/router';

export const routes: Routes = [
  {
    path: '',
    loadComponent: () =>
      import('./home/home').then(m => m.Home)
  }
];

Esse formato é bastante usado no Angular moderno porque melhora a organização e ajuda na performance.

Router Outlet

Para que as rotas funcionem corretamente, precisamos indicar no template onde o componente da rota será exibido.

Isso é feito com a diretiva router-outlet. 


<router-outlet></router-outlet>

Sem o router-outlet, a navegação pode até mudar a URL, mas o conteúdo da rota não será renderizado na tela.

Navegação pelo template

Podemos navegar entre páginas usando a diretiva routerLink. 


<a routerLink="/produtos">Ir para produtos</a>

Navegação pelo TypeScript

Também podemos navegar por código usando o serviço Router. 


import { Router } from '@angular/router';

constructor(private router: Router) {}

irParaProdutos() {
  this.router.navigate(['/produtos']);
}

Transferência de dados entre rotas

Existem várias formas de transferir dados de uma página para outra no Angular.

  • Parâmetros de rota
  • Query params
  • State de navegação
  • Uso de serviços compartilhados

Exemplo com parâmetro de rota


{ path: 'produto/:id', component: ProdutoDetalhe }
          

<a [routerLink]="['/produto', 10]">Ver produto 10</a>

Nesse caso, o número 10 será enviado na URL como identificador do produto.

Lendo o parâmetro da rota

Assim conseguimos recuperar o valor enviado na rota.


import { ActivatedRoute } from '@angular/router';

constructor(private route: ActivatedRoute) {}

ngOnInit() {
  this.route.paramMap.subscribe(params => {
    const id = params.get('id');
    console.log(id);
  });
}

Dessa forma, o componente reage automaticamente às mudanças nos parâmetros da URL.

Enviando query params

Os query params são muito usados para filtros, paginação, ordenação e buscas. 


this.router.navigate(['/produtos'], {
  queryParams: {
    pagina: 2,
    categoria: 'eletronicos'
  }
});

A URL pode ficar parecida com esta:


/produtos?pagina=2&categoria=eletronicos

Lendo query params


this.route.queryParams.subscribe(params => {
  console.log(params['pagina']);
  console.log(params['categoria']);
});

Essa técnica é muito útil quando queremos manter filtros e estado da tela na URL.

Enviando dados com state

Outra forma de enviar dados entre páginas é usar o state da navegação.

Essa abordagem é útil quando não queremos expor todas as informações na URL. 


this.router.navigate(['/detalhe'], {
  state: {
    produto: {
      id: 1,
      nome: 'Notebook'
    }
  }
});

Recebendo dados enviados com state


import { Router } from '@angular/router';

constructor(private router: Router) {}

ngOnInit() {
  const navigation = this.router.getCurrentNavigation();
  const produto = navigation?.extras.state?.['produto'];

  console.log(produto);
}

Essa estratégia pode ser útil para transferir objetos temporários entre telas durante a navegação.

Usando o resolver

Outra funcionalidade importante é o resolver, usado para carregar dados antes que a rota seja exibida.

Isso ajuda a evitar que a página carregue vazia enquanto espera dados externos. 


import { ResolveFn } from '@angular/router';

export const produtoResolver: ResolveFn = () => {
  return fetch('https://api.exemplo.com/produto/1')
    .then(res => res.json());
};
          

{
  path: 'produto/:id',
  component: ProdutoDetalheComponent,
  resolve: {
    produto: produtoResolver
  }
}

Nesse caso, a rota pode esperar os dados serem carregados antes de mostrar o componente ao usuário.

Resumo

O sistema de rotas do Angular é responsável por controlar a navegação entre páginas e também permite transferir informações de diferentes formas.

Entre os principais recursos, destacam-se:

  • Definição de rotas com Routes
  • Exibição de telas com router-outlet
  • Navegação com routerLink e Router
  • Envio de dados com parâmetros de rota
  • Uso de query params
  • Transferência de objetos com state
  • Compartilhamento de informações com serviços
  • Proteção de rotas com guards
  • Pré-carregamento de dados com resolvers
  • Uso de componentes standalone e lazy loading moderno

Esses recursos tornam a navegação mais organizada, flexível e adequada para aplicações modernas.

Passo a Passo: Criando exemplo prático com serviço, HTTP e navegação

Passo 0 — Criar uma nova branch no GitHub

Acesse novamente o repositório no GitHub e crie uma nova branch baseada na branch aula3 com o seguinte nome: 


aula4-navegacao

Depois, no terminal do VSCode, execute:

 
git fetch
git checkout aula4-navegacao

Passo 1 — Criar os componentes

Vamos criar dois componentes: um para listar produtos e outro para mostrar o detalhe de um produto.

Perceba que usamos o pages/. Isso serve para criar uma pasta para os componentes de página. Assim, toda vez que precisarmos criar um novo componente de página, podemos organizá-lo dentro dessa pasta.


ng g c pages/lista-produtos
ng g c pages/detalhe-produto

Passo 2 — Configurar as rotas

Agora abra o arquivo de rotas do projeto, no caso o arquivo app.routes.ts e insira o código abaixo: 


import { Routes } from '@angular/router';
import { ListaProdutos } from './pages/lista-produtos/lista-produtos';
import { DetalheProduto } from './pages/detalhe-produto/detalhe-produto';

export const routes: Routes = [
  { path: '', component: ListaProdutos },
  { path: 'produto/:id', component: DetalheProduto },
];

Passo 3 — Importar RouterOutlet

No componente principal da aplicação, no caso o app.ts, adicione os imports necessários para o roteamento funcionar. 


import { Component } from '@angular/core';
//adicione o RouterOutlet no imports
import { RouterOutlet } from '@angular/router';

@Component({
  selector: 'app-root',
  imports: [RouterOutlet],
  templateUrl: './app.html',
  styleUrl: './app.css'
})
export class App {}

Passo 4 — Atualizar o app.html para receber as rotas

Agora no arquivo app.html, apague tudo que está lá e deixe apenas o código abaixo:


<router-outlet></router-outlet>

Passo 5 — Criar o template da lista de produtos

Agora abra o arquivo lista-produtos.html e insira:


<h1>Lista de Produtos</h1>

<ul>
  <li>
    <a [routerLink]="['/produto', 10]">Ver detalhes</a>
  </li>
</ul>

Lista de Produtos

Por fim nesse etapa, atualize o lista-produtos.ts com o código abaixo:


import { Component } from '@angular/core';
import { RouterModule } from '@angular/router';

@Component({
  selector: 'app-lista-produtos',
  standalone: true,
  imports: [RouterModule],
  templateUrl: './lista-produtos.html',
  styleUrl: './lista-produtos.css',
})
export class ListaProdutos {
  produtos = [];
}

Passo 7 — Inserir a lógica do detalhe do produto

Abra o arquivo detalhe-produto.ts e insira:


import { Component } from '@angular/core';
import { CommonModule } from '@angular/common';
import { ActivatedRoute, RouterModule } from '@angular/router';

@Component({
  selector: 'app-detalhe-produto',
  imports: [CommonModule, RouterModule],
  templateUrl: './detalhe-produto.html',
  styleUrl: './detalhe-produto.css',
})
export class DetalheProduto {
  public id: string | null = null;
  constructor(private route: ActivatedRoute) {}

  ngOnInit() {
    this.id = this.route.snapshot.paramMap.get('id');
    console.log('ID do produto selecionado na rota:', this.id);
  }
}

Agora abra o arquivo detalhe-produto.html e insira:


<p>Este é a página de detalhes do produto {{ id }}</p>

Passo 8 — Rodar a aplicação

No terminal, execute:


ng serve

Acesse:


http://localhost:4200

Acesse as rotas http://localhost:4200/produtos e http://localhost:4200/produto/:id. Inserindo em :id o número do produto que deseja visualizar.

Por exemplo: http://localhost:4200/produto/10.
