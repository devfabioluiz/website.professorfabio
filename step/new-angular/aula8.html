Formulários

O que são formulários?

Formulários são estruturas usadas para capturar dados do usuário.

Em praticamente toda aplicação web usamos formulários em situações como:

  • Login
  • Cadastro
  • Busca
  • Filtros
  • Envio de informações

No Angular, existem duas formas principais de trabalhar com formulários:

  • Template-driven forms
  • Reactive forms

Template-driven forms

Nos template-driven forms, a maior parte da lógica do formulário fica no HTML.

Essa abordagem é mais simples e costuma ser usada em cenários menores ou introdutórios.

Para utilizar esse tipo de formulário, normalmente precisamos importar o FormsModule.

Exemplo simples com ngModel


import { Component } from '@angular/core';
import { FormsModule } from '@angular/forms';

@Component({
  selector: 'app-exemplo-form',
  imports: [FormsModule],
  templateUrl: './exemplo-form.html'
})
export class ExemploForm {
  nome = '';
}
          

<input type="text" [(ngModel)]="nome" name="nome" />
<p>Nome digitado: {{ nome }}</p>

Nesse exemplo, usamos [(ngModel)] para fazer a ligação entre o valor digitado no campo e a variável do componente.

Reactive forms

Nos reactive forms, a estrutura do formulário é criada no TypeScript.

Essa é a abordagem mais robusta e muito usada em aplicações reais, porque oferece mais controle sobre:

  • Validações
  • Estrutura do formulário
  • Manipulação de estados
  • Leitura dos valores

Para trabalhar com esse modelo, normalmente usamos o ReactiveFormsModule.

Principais estruturas dos Reactive Forms

  • FormGroup → representa o formulário
  • FormControl → representa um campo
  • Validators → valida os campos

Exemplo de estrutura com FormGroup


import { FormControl, FormGroup } from '@angular/forms';

formulario = new FormGroup({
  nome: new FormControl(''),
  email: new FormControl('')
});

Nesse caso, criamos um formulário com dois campos: nome e email.

Validações com Validators

O Angular oferece validadores prontos para verificarmos se os dados do formulário estão corretos.

Alguns validadores comuns são:

  • Validators.required
  • Validators.email
  • Validators.minLength()
  • Validators.maxLength()

import { Validators } from '@angular/forms';

formulario = new FormGroup({
  nome: new FormControl('', [
    Validators.required,
    Validators.minLength(3)
  ]),
  email: new FormControl('', [
    Validators.required,
    Validators.email
  ])
});

Exibindo erros no template

Podemos usar as informações do próprio formulário para mostrar mensagens de erro ao usuário. 


@if (formulario.get('email')?.invalid && formulario.get('email')?.touched) {
  <p>Informe um email válido.</p>
}

Assim, a mensagem aparece apenas quando o campo estiver inválido e já tiver sido tocado pelo usuário.

Estados importantes de um formulário

Estado Significado
valid O campo ou formulário está válido
invalid O campo ou formulário está inválido
touched O usuário tocou no campo
untouched O usuário ainda não tocou no campo
dirty O valor foi alterado
pristine O valor ainda não foi alterado

Resumo

  • Formulários capturam dados do usuário
  • Angular possui template-driven forms e reactive forms
  • FormsModule é usado com ngModel
  • ReactiveFormsModule é usado com FormGroup e FormControl
  • Validators ajudam a validar os dados
  • Podemos mostrar erros diretamente no template

import { Routes } from '@angular/router';

Passo 3 — Entendendo o que vamos usar

Neste exemplo vamos utilizar:

  • ReactiveFormsModule para ativar formulários reativos
  • FormGroup para representar o formulário
  • FormControl para representar os campos
  • Validators para validar os dados

Passo 4 — Criar a lógica do componente

Agora abra o arquivo cadastro-usuario.ts e substitua o conteúdo pelo código abaixo: 


import { Component } from '@angular/core';
import { CommonModule } from '@angular/common';
import {
  FormControl,
  FormGroup,
  ReactiveFormsModule,
  Validators,
} from '@angular/forms';
import { RouterModule } from '@angular/router';

@Component({
  selector: 'app-cadastro-usuario',
  imports: [CommonModule, ReactiveFormsModule, RouterModule],
  templateUrl: './cadastro-usuario.html',
  styleUrl: './cadastro-usuario.css',
})
export class CadastroUsuario {
  mensagem = '';

  formulario = new FormGroup({
    nome: new FormControl('', [
      Validators.required,
      Validators.minLength(3),
    ]),
    email: new FormControl('', [
      Validators.required,
      Validators.email,
    ]),
    senha: new FormControl('', [
      Validators.required,
      Validators.minLength(6),
    ]),
  });

  enviarFormulario(): void {
    if (this.formulario.invalid) {
      this.formulario.markAllAsTouched();
      return;
    }

    console.log('Dados enviados:', this.formulario.value);
    this.mensagem = 'Cadastro realizado com sucesso!';
    this.formulario.reset();
  }

  get nome() {
    return this.formulario.get('nome');
  }

  get email() {
    return this.formulario.get('email');
  }

  get senha() {
    return this.formulario.get('senha');
  }
}

Passo 5 — Criar o template do formulário

Agora abra o arquivo cadastro-usuario.html e insira:


<h1>Cadastro de Usuário</h1>

<form [formGroup]="formulario" (ngSubmit)="enviarFormulario()">
  <div>
    <label for="nome">Nome</label>
    <input id="nome" type="text" formControlName="nome" />

    @if (nome?.invalid && nome?.touched) {
      <p>O nome é obrigatório e deve ter pelo menos 3 caracteres.</p>
    }
  </div>

  <div>
    <label for="email">Email</label>
    <input id="email" type="email" formControlName="email" />

    @if (email?.invalid && email?.touched) {
      <p>Informe um email válido.</p>
    }
  </div>

  <div>
    <label for="senha">Senha</label>
    <input id="senha" type="password" formControlName="senha" />

    @if (senha?.invalid && senha?.touched) {
      <p>A senha é obrigatória e deve ter pelo menos 6 caracteres.</p>
    }
  </div>

  <button type="submit">Cadastrar</button>
</form>

@if (mensagem) {
  <p>{{ mensagem }}</p>
}

<br />
<a routerLink="/produtos">Voltar para produtos</a>

Passo 7 — Entendendo o TypeScript do formulário

No TypeScript, criamos um FormGroup com três campos: nome, email e senha.

Cada campo foi criado com FormControl e recebeu suas próprias validações.

O método enviarFormulario() verifica se o formulário está inválido. Se estiver, marcamos todos os campos como tocados com markAllAsTouched().

Se estiver válido, mostramos os dados no console e exibimos uma mensagem de sucesso.

Passo 8 — Entendendo o HTML do formulário

No template usamos:

  • [formGroup] para ligar o HTML ao formulário
  • formControlName para ligar cada campo
  • (ngSubmit) para tratar o envio do formulário
  • @ if para mostrar mensagens de erro

Passo 11 — Exemplo rápido de template-driven form

Além do reactive forms, o Angular também suporta template-driven forms.

Veja abaixo um exemplo simples apenas para comparação:


import { Component } from '@angular/core';
import { FormsModule } from '@angular/forms';

@Component({
  selector: 'app-exemplo-template-form',
  imports: [FormsModule],
  templateUrl: './exemplo-template-form.html'
})
export class ExemploTemplateForm {
  nome = '';
}
          

<input type="text" [(ngModel)]="nome" name="nome" />
<p>Nome digitado: {{ nome }}</p>

Esse modelo é mais simples, porém para aplicações maiores o reactive forms costuma ser mais organizado.

Dever de Casa

Criar um formulário reativo de login

Crie um componente chamado login-usuario contendo um formulário reativo com os campos email e senha.

O campo email deve ser obrigatório e validar se o valor informado é realmente um email.

O campo senha deve ser obrigatório e ter no mínimo 6 caracteres.

Ao enviar o formulário corretamente, mostre os dados no console.

Obs: Não se esqueça de subir as alterações para o Git na branch dessa aula após terminar o dever.

Gabarito:

1. Criar o componente 


ng g c pages/login-usuario

2. Arquivo login-usuario.ts 


import { Component } from '@angular/core';
import { CommonModule } from '@angular/common';
import {
  FormControl,
  FormGroup,
  ReactiveFormsModule,
  Validators,
} from '@angular/forms';

@Component({
  selector: 'app-login-usuario',
  imports: [CommonModule, ReactiveFormsModule],
  templateUrl: './login-usuario.html',
  styleUrl: './login-usuario.css',
})
export class LoginUsuario {
  formulario = new FormGroup({
    email: new FormControl('', [
      Validators.required,
      Validators.email,
    ]),
    senha: new FormControl('', [
      Validators.required,
      Validators.minLength(6),
    ]),
  });

  enviarFormulario(): void {
    if (this.formulario.invalid) {
      this.formulario.markAllAsTouched();
      return;
    }

    console.log(this.formulario.value);
  }

  get email() {
    return this.formulario.get('email');
  }

  get senha() {
    return this.formulario.get('senha');
  }
}

3. Arquivo login-usuario.html 


<h1>Login de Usuário</h1>

<form [formGroup]="formulario" (ngSubmit)="enviarFormulario()">
  <div>
    <label for="email">Email</label>
    <input id="email" type="email" formControlName="email" />

    @if (email?.invalid && email?.touched) {
      <p>Informe um email válido.</p>
    }
  </div>

  <div>
    <label for="senha">Senha</label>
    <input id="senha" type="password" formControlName="senha" />

    @if (senha?.invalid && senha?.touched) {
      <p>A senha deve ter pelo menos 6 caracteres.</p>
    }
  </div>

  <button type="submit">Entrar</button>
</form>

4. Exemplo de rota no app.routes.ts 


{
  path: 'login-usuario',

    import('./pages/login-usuario/login-usuario').then(

5. Lembrete importante

No componente é necessário importar o ReactiveFormsModule, senão o Angular não reconhecerá diretivas como [formGroup] e formControlName.
