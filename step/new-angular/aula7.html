Ciclo de Vida de Componentes e Passagem de Dados

Comunicação entre Componentes no Angular

Em aplicações Angular, é muito comum precisarmos trocar informações entre componentes.

Por exemplo:

  • Um componente pai envia dados para um componente filho
  • Um componente filho avisa ao pai que algo aconteceu
  • Dois componentes compartilham informações por meio de um serviço

As principais formas de comunicação entre componentes são:

  • @Input() → pai envia dados para o filho
  • @Output() → filho envia eventos para o pai
  • Serviços → compartilhamento entre componentes

Usando Input

O @Input() permite que um componente receba dados de outro componente.

Normalmente usamos Input quando um componente pai precisa enviar uma informação para um componente filho.

Exemplo conceitual

Imagine que temos um componente pai com o nome de um usuário e queremos mostrar esse nome em um componente filho.

Componente filho com Input


import { Component, Input } from '@angular/core';

@Component({
  selector: 'app-filho',
  templateUrl: './filho.html',
  styleUrl: './filho.css'
})
export class Filho {
  @Input() nome = '';
}

Template do componente filho


<p>Nome recebido do componente pai: {{ nome }}</p>

Componente pai enviando o dado


<app-filho [nome]="nomeUsuario"></app-filho>

Nesse caso, o componente pai envia o valor da variável nomeUsuario para a propriedade nome do filho.

Usando Output

O @Output() permite que um componente filho envie um evento para o componente pai.

Isso é muito útil quando o filho precisa avisar ao pai que alguma ação aconteceu, como clicar em um botão, remover um item ou salvar dados.

Componente filho com Output


import { Component, EventEmitter, Output } from '@angular/core';

@Component({
  selector: 'app-filho',
  templateUrl: './filho.html',
  styleUrl: './filho.css'
})
export class Filho {
  @Output() mensagemEnviada = new EventEmitter<string>();

  enviarMensagem() {
    this.mensagemEnviada.emit('Olá, pai!');
  }
}

Template do componente filho


<button (click)="enviarMensagem()">Enviar mensagem para o pai</button>

Componente pai ouvindo o evento


<app-filho (mensagemEnviada)="receberMensagem($event)"></app-filho>
          

mensagem = '';

receberMensagem(texto: string) {
  this.mensagem = texto;
}

Quando o botão do filho for clicado, o pai receberá o valor enviado no método receberMensagem().

Input e Output juntos

Em muitos casos, usamos Input e Output juntos.

O pai envia dados para o filho, e o filho responde enviando eventos de volta.

Isso cria uma comunicação organizada e bastante comum em aplicações Angular.

Outras formas de passar dados entre componentes

1. De pai para filho com Input

É a forma mais direta quando existe relação hierárquica entre os componentes.

2. De filho para pai com Output

É usada quando o filho precisa notificar o pai sobre alguma ação.

3. Entre componentes sem relação direta usando serviço

Quando dois componentes não têm relação direta de pai e filho, uma forma comum de comunicação é usar um serviço compartilhado. 


import { Injectable } from '@angular/core';

@Injectable({
  providedIn: 'root'
})
export class MensagemService {
  mensagem = 'Mensagem compartilhada';
}

Nesse caso, múltiplos componentes podem usar o mesmo serviço e acessar os mesmos dados.

4. Por rotas e parâmetros

Outra forma de passar dados é por rotas, query params ou state de navegação, como vimos em aulas anteriores.

Ciclo de Vida de Componentes no Angular

Todo componente Angular passa por etapas desde o momento em que é criado até o momento em que é destruído.

Essas etapas são chamadas de ciclo de vida do componente.

O Angular disponibiliza métodos especiais, chamados de hooks do ciclo de vida, para executarmos lógicas em momentos específicos.

Para que serve o ciclo de vida?

  • Executar lógica quando o componente inicia
  • Reagir a alterações em dados recebidos
  • Manipular dados quando a interface estiver pronta
  • Limpar recursos quando o componente for destruído

Principais hooks do ciclo de vida

Hook Função
ngOnChanges Executa quando um valor recebido por Input muda
ngOnInit Executa após a criação do componente
ngDoCheck Permite verificações manuais de mudança
ngAfterViewInit Executa após a view estar pronta
ngOnDestroy Executa antes do componente ser destruído

Exemplo de ngOnInit

O ngOnInit é muito utilizado para inicializar dados, chamar serviços e preparar o componente. 


import { Component, OnInit } from '@angular/core';

@Component({
  selector: 'app-exemplo',
  template: '<p>Componente iniciado</p>'
})
export class ExemploComponent implements OnInit {
  ngOnInit(): void {
    console.log('Componente iniciado com ngOnInit');
  }
}

Exemplo de ngOnChanges

O ngOnChanges é útil quando o componente recebe dados de outro componente através de @Input(). 


import { Component, Input, OnChanges, SimpleChanges } from '@angular/core';

@Component({
  selector: 'app-filho',
  template: '<p>{{ nome }}</p>'
})
export class FilhoComponent implements OnChanges {
  @Input() nome = '';

  ngOnChanges(changes: SimpleChanges): void {
    console.log('Mudanças recebidas:', changes);
  }
}

Exemplo de ngOnDestroy

O ngOnDestroy é usado para limpar intervalos, subscriptions e outros recursos ativos. 


import { Component, OnDestroy } from '@angular/core';

@Component({
  selector: 'app-relogio',
  template: '<p>Relógio ativo</p>'
})
export class RelogioComponent implements OnDestroy {
  intervalo: any;

  constructor() {
    this.intervalo = setInterval(() => {
      console.log('Executando intervalo...');
    }, 1000);
  }

  ngOnDestroy(): void {
    clearInterval(this.intervalo);
    console.log('Componente destruído');
  }
}

Resumo

  • O ciclo de vida controla o comportamento do componente
  • ngOnInit é usado para inicialização
  • ngOnChanges reage a mudanças em Inputs
  • ngOnDestroy limpa recursos
  • @Input() envia dados do pai para o filho
  • @Output() envia eventos do filho para o pai
  • Serviços ajudam a compartilhar dados entre componentes

Passo a Passo: Criando comunicação entre componentes com Input e Output

Passo 0 — Criar branch e acessar a branch

Acesse novamente o repositório no GitHub e crie agora uma nova branch baseada na branch aula6-guards com o nome abaixo: 


aula7-ciclo-vida-input-output

No terminal, execute o comando abaixo:


git fetch

Passo 1 — Criar os componentes

Vamos criar dois componentes: um componente pai e um componente filho. 


ng g c pages/componente-pai
ng g c components/componente-filho

Passo 2 — Configurar a rota do componente pai

Agora abra o arquivo app.routes.ts e adicione a rota abaixo: 


import { Routes } from '@angular/router';
import { authGuard } from './guards/auth.guard';

export const routes: Routes = [
  {
    path: '',
    redirectTo: 'produtos',
    pathMatch: 'full',
  },
  {
    path: 'produtos',
    loadComponent: () =>
      import('./pages/lista-produtos/lista-produtos').then(
        (m) => m.ListaProdutos
      ),
  },
  {
    path: 'produto/:id',
    loadComponent: () =>
      import('./pages/detalhe-produto/detalhe-produto').then(
        (m) => m.DetalheProduto
      ),
  },
  {
    path: 'painel-adm',
    loadComponent: () =>
      import('./painel-adm/painel-adm').then(
        (m) => m.PainelAdm
      ),
    canActivate: [authGuard],
  },
  {
    path: 'componente-pai',
    loadComponent: () =>
      import('./pages/componente-pai/componente-pai').then(
        (m) => m.ComponentePai
      ),
  },
  {
    path: '**',
    redirectTo: 'produtos',
  }
];

Passo 3 — Criar a lógica do componente filho

Agora abra o arquivo componente-filho.ts e insira:


import {
  Component,
  EventEmitter,
  Input,
  OnChanges,
  Output,
  SimpleChanges
} from '@angular/core';

@Component({
  selector: 'app-componente-filho',
  imports: [],
  templateUrl: './componente-filho.html',
  styleUrl: './componente-filho.css'
})
export class ComponenteFilho implements OnChanges {
  @Input() nome = '';
  @Output() mensagemEnviada = new EventEmitter<string>();

  ngOnChanges(changes: SimpleChanges): void {
    console.log('Valor recebido pelo filho mudou:', changes);
  }

  enviarMensagem(): void {
    this.mensagemEnviada.emit('Mensagem enviada do filho para o pai');
  }
}

Passo 4 — Criar o template do componente filho

Agora abra o arquivo componente-filho.html e insira:


<h2>Componente Filho</h2>
<p>Nome recebido do pai: {{ nome }}</p>

<button (click)="enviarMensagem()">
  Enviar mensagem para o pai
</button>

Passo 5 — Criar a lógica do componente pai

Agora abra o arquivo componente-pai.ts e insira:


import { Component, OnDestroy, OnInit } from '@angular/core';
import { CommonModule } from '@angular/common';
import { RouterModule } from '@angular/router';
import { ComponenteFilho } from '../../components/componente-filho/componente-filho';

@Component({
  selector: 'app-componente-pai',
  imports: [CommonModule, RouterModule, ComponenteFilho],
  templateUrl: './componente-pai.html',
  styleUrl: './componente-pai.css'
})
export class ComponentePai implements OnInit, OnDestroy {
  nomeUsuario = 'Fabio';
  mensagemRecebida = '';

  ngOnInit(): void {
    console.log('Componente pai iniciado');
  }

  ngOnDestroy(): void {
    console.log('Componente pai destruído');
  }

  receberMensagem(mensagem: string): void {
    this.mensagemRecebida = mensagem;
  }

  alterarNome(): void {
    this.nomeUsuario = 'Novo nome enviado pelo pai';
  }
}

Passo 6 — Criar o template do componente pai

Agora abra o arquivo componente-pai.html e insira:


<h1>Componente Pai</h1>

<p>Mensagem recebida do filho: {{ mensagemRecebida }}</p>

<button (click)="alterarNome()">Alterar nome enviado ao filho</button>

<hr />

<app-componente-filho
  [nome]="nomeUsuario"
  (mensagemEnviada)="receberMensagem($event)"
></app-componente-filho>

<br />
<br />

<a routerLink="/produtos">Voltar para produtos</a>

Passo 7 — Entendendo o que foi feito

O componente pai envia dados ao filho usando:


[nome]="nomeUsuario"

Já o componente filho envia eventos de volta ao pai usando:


(mensagemEnviada)="receberMensagem($event)"

Além disso:

  • O pai usa ngOnInit e ngOnDestroy
  • O filho usa ngOnChanges para reagir ao Input
  • O filho dispara um evento com EventEmitter

Passo 8 — Rodar a aplicação


ng serve

Depois, acesse a rota abaixo no navegador:


http://localhost:4200/componente-pai

Passo 9 — Testar a comunicação entre componentes

Ao abrir a página, o componente filho receberá o nome enviado pelo pai.

Ao clicar no botão do pai, o valor enviado pelo Input mudará, e o ngOnChanges do filho será executado.

Ao clicar no botão do filho, ele enviará uma mensagem para o pai usando Output.

Dever de Casa

Criar comunicação entre componente pai e componente filho

Crie um componente filho que receba uma propriedade chamada produto via @Input().

Esse componente deve exibir o nome e o preço do produto.

Além disso, o componente filho deve possuir um botão para avisar ao pai que o produto foi selecionado, utilizando @Output().

Quando o evento for disparado, o componente pai deve exibir na tela uma mensagem informando qual produto foi selecionado.

Obs: Suba o código para o Git após finalizar.

Gabarito:

1. Arquivo produto-filho.ts 


import { Component, EventEmitter, Input, Output } from '@angular/core';

@Component({
  selector: 'app-produto-filho',
  imports: [],
  templateUrl: './produto-filho.html',
  styleUrl: './produto-filho.css'
})
export class ProdutoFilho {
  @Input() produto: any;
  @Output() produtoSelecionado = new EventEmitter<string>();

  selecionarProduto(): void {
    this.produtoSelecionado.emit(this.produto.nome);
  }
}

2. Arquivo produto-filho.html 


<p>Produto: {{ produto.nome }}</p>
<p>Preço: {{ produto.preco }}</p>
<button (click)="selecionarProduto()">Selecionar produto</button>

3. Arquivo do componente pai 


produto = {
  nome: 'Notebook',
  preco: 3500
};

mensagem = '';

receberProdutoSelecionado(nomeProduto: string): void {
  this.mensagem = `Produto selecionado: ${nomeProduto}`;
}

4. Template do componente pai 


<p>{{ mensagem }}</p>

<app-produto-filho
  [produto]="produto"
  (produtoSelecionado)="receberProdutoSelecionado($event)"
></app-produto-filho>
