8. Chamadas síncronas e assíncronas

Uma chamada síncrona bloqueia o fluxo da aplicação até terminar. Já uma chamada assíncrona permite que a aplicação continue executando outras tarefas enquanto aguarda o resultado. 


const fs = require('fs');

console.log('Início');

const conteudo = fs.readFileSync('arquivo.txt', 'utf-8');
console.log(conteudo);

console.log('Fim');

No exemplo acima, o Node.js espera a leitura terminar antes de seguir. 


const fs = require('fs');

console.log('Início');

fs.readFile('arquivo.txt', 'utf-8', (erro, conteudo) => {
  if (erro) {
    console.log('Erro ao ler arquivo:', erro.message);
    return;
  }

  console.log(conteudo);
});

console.log('Fim');

No segundo exemplo, a leitura é assíncrona. O programa continua sua execução enquanto o arquivo é lido.

9. Buffers

Buffers são áreas de memória usadas para lidar com dados binários. Eles são úteis quando trabalhamos com arquivos, imagens, vídeos, uploads ou dados recebidos pela rede. 


const buffer = Buffer.from('Olá, Node.js');

console.log(buffer);
console.log(buffer.toString());

10. Streams

Streams permitem processar dados em partes, sem carregar tudo de uma vez na memória. Isso é importante para arquivos grandes, vídeos, uploads e respostas HTTP extensas. 


const fs = require('fs');

const leitura = fs.createReadStream('entrada.txt');
const escrita = fs.createWriteStream('saida.txt');

leitura.pipe(escrita);

11. Objetos globais

O Node.js possui objetos e funções globais que podem ser usados sem importação direta.

  • console: exibe mensagens no terminal
  • process: traz informações do processo em execução
  • __dirname: caminho da pasta atual
  • __filename: caminho do arquivo atual
  • setTimeout: executa algo depois de um tempo
  • setInterval: executa algo repetidamente

console.log('Diretório atual:', __dirname);
console.log('Arquivo atual:', __filename);
console.log('Sistema operacional:', process.platform);

12. Módulo Web no Node.js

O Node.js possui o módulo http, que permite criar servidores web sem instalar bibliotecas externas. 


const http = require('http');

const servidor = http.createServer((req, res) => {
  res.writeHead(200, { 'Content-Type': 'text/plain; charset=utf-8' });
  res.end('Servidor Node.js funcionando!');
});

servidor.listen(3000, () => {
  console.log('Servidor rodando em http://localhost:3000');
});

Embora seja possível criar aplicações usando apenas o módulo http, na prática usamos frameworks como o Express para organizar melhor rotas, requisições, respostas e middlewares.

13. O que é Express?

Express é um framework para Node.js que facilita a criação de aplicações web e APIs. Ele simplifica o trabalho com rotas, middlewares, arquivos estáticos, formulários, cookies, sessões e integração com bancos de dados.

14. Request e Response

Em uma aplicação Express, cada rota normalmente recebe dois objetos principais: req e res.

  • req: representa a requisição feita pelo cliente.
  • res: representa a resposta que será enviada pelo servidor. 

app.get('/saudacao', (req, res) => {
  res.send('Olá! Essa resposta veio do servidor.');
});

15. Roteamento

Roteamento é o mecanismo usado para definir o que a aplicação deve fazer quando recebe uma requisição em determinado endereço e método HTTP. 


app.get('/tarefas', listarTarefas);
app.post('/tarefas', criarTarefa);
app.put('/tarefas/:id', atualizarTarefa);
app.delete('/tarefas/:id', removerTarefa);

16. Cookies e sessões

Cookies são pequenas informações armazenadas no navegador do usuário. Sessões são usadas para manter informações do usuário entre várias requisições, como um login ativo.

Em aplicações reais, é comum guardar apenas um identificador no cookie e manter os dados sensíveis no servidor.

17. Banco de dados relacional

Em um banco relacional, os dados são organizados em tabelas. Cada tabela possui colunas e linhas. Esse modelo é útil quando precisamos representar entidades e relações, como usuários, tarefas, produtos, pedidos e pagamentos.

Nesta aula, usaremos PostgreSQL para armazenar tarefas de uma pequena aplicação de controle pessoal.

18. Arquitetura REST

REST é um estilo de arquitetura para criação de serviços web. Em uma API REST, usamos recursos, URLs, métodos HTTP e códigos de status para organizar a comunicação entre cliente e servidor.

  • GET: buscar dados
  • POST: criar dados
  • PUT: atualizar dados
  • DELETE: remover dados

19. Códigos de status HTTP

  • 200: requisição bem-sucedida
  • 201: recurso criado com sucesso
  • 400: erro nos dados enviados pelo cliente
  • 404: recurso não encontrado
  • 500: erro interno no servidor

20. Escalabilidade em Node.js

Escalar uma aplicação significa prepará-la para lidar com mais usuários, mais requisições e mais processamento. No Node.js, podemos usar processos filhos, múltiplas instâncias, balanceadores de carga, filas, cache e separação de responsabilidades.

O módulo child_process permite criar processos separados usando recursos como exec, spawn e fork. 


const { exec } = require('child_process');

exec('node -v', (erro, stdout, stderr) => {
  if (erro) {
    console.log('Erro:', erro.message);
    return;
  }

  console.log('Versão do Node:', stdout);
});
