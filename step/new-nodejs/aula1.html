Node.js

Definição

O que é o Node.js

Node.js é uma plataforma que permite executar JavaScript fora do navegador. Com ele, podemos criar aplicações backend, APIs, scripts, servidores web, ferramentas de linha de comando e sistemas que lidam com arquivos, eventos, conexões HTTP e bancos de dados.

Antes do Node.js, o JavaScript era usado principalmente no navegador. Com o Node.js, a mesma linguagem também pode ser usada no servidor.

Para que serve o Node.js?

  • Criar servidores web
  • Criar APIs REST
  • Processar formulários
  • Interagir com bancos de dados
  • Ler e escrever arquivos
  • Trabalhar com eventos
  • Executar tarefas em linha de comando
  • Criar sistemas escaláveis

Características principais do Node.js

  • Assíncrono: evita travar a aplicação enquanto uma tarefa demora.
  • Orientado a eventos: reage a acontecimentos, como requisições, cliques, leituras de arquivos e conexões.
  • Não bloqueante: consegue lidar com várias operações sem esperar uma terminar para iniciar outra.
  • Baseado em JavaScript: permite usar a mesma linguagem no frontend e no backend.
  • Possui grande ecossistema: usa o npm para instalar bibliotecas prontas.

Arquitetura orientada a eventos

No Node.js, muitas ações acontecem por meio de eventos. Uma requisição HTTP chegando ao servidor, um arquivo sendo lido ou uma conexão sendo encerrada são exemplos de eventos.

Em vez de executar tudo de forma linear e bloqueante, o Node.js registra funções que serão chamadas quando determinado evento ocorrer. 


const EventEmitter = require('events');

const eventos = new EventEmitter();

eventos.on('pedidoCriado', (pedido) => {
  console.log('Novo pedido recebido:', pedido);
});

eventos.emit('pedidoCriado', {
  id: 1,
  cliente: 'Maria',
  produto: 'Café'
});

Nesse exemplo, primeiro registramos o evento pedidoCriado. Depois, usamos emit para disparar esse evento.

Gerenciador de pacotes npm

O npm é o gerenciador de pacotes do Node.js. Ele é usado para instalar bibliotecas, criar projetos, executar scripts e controlar dependências. 


npm init -y
npm install express
npm install nodemon --save-dev

O arquivo package.json registra informações do projeto, dependências instaladas e comandos personalizados.

REPL do Node.js

O REPL é uma interface interativa do Node.js. A sigla significa Read, Evaluate, Print, Loop. Ele permite testar comandos JavaScript diretamente no terminal. 


node
          

2 + 2
'Node.js'.toUpperCase()
const nome = 'João'
nome

Para sair do REPL, pressione Ctrl + C duas vezes ou digite .exit.

Passo a Passo: Criando uma aplicação Node.js simples com HTML

Passo 1 — Iniciar o projeto Node.js

Dentro da pasta do projeto, ainda no terminal, execute o comando abaixo para criar o arquivo package.json: 


npm init -y

Passo 2 — Criar o arquivo principal

Agora crie o arquivo index.js. Ele será o arquivo principal da nossa aplicação. 


type nul > index.js

Passo 3 — Inserir o código da aplicação

Abra o arquivo index.js e insira o código abaixo: 


const http = require('http');

const produtos = [
  'Notebook',
  'Mouse',
  'Teclado',
  'Monitor'
];

const server = http.createServer((req, res) => {
  const listaProdutos = produtos
    .map((produto) => `<li>${produto}</li>`)
    .join('');

  const html = `
    <!DOCTYPE html>
    <html lang="pt-br">
    <head>
      <meta charset="UTF-8">
      <meta name="viewport" content="width=device-width, initial-scale=1.0">
      <title>Aplicação Node.js</title>
    </head>
    <body>
      <h1>Lista de Produtos</h1>

      <ul>
        ${listaProdutos}
      </ul>
    </body>
    </html>
  `;

  res.writeHead(200, { 'Content-Type': 'text/html; charset=utf-8' });
  res.end(html);
});

server.listen(3000, () => {
  console.log('Servidor rodando em http://localhost:3000');
});

🤓 Explicando o código 📝

Quando utilizamos const http = require('http');, estamos importando um módulo nativo do Node.js responsável por permitir a criação de servidores web. Esse módulo implementa o protocolo HTTP, que é o mesmo utilizado pelos navegadores para acessar sites.

Com o http, conseguimos criar aplicações backend que recebem requisições e retornam respostas, como páginas HTML, dados JSON ou textos simples.

🔹 O que o require('http') faz

O require é utilizado para importar módulos no Node.js. Quando fazemos require('http'), estamos carregando o módulo HTTP nativo e armazenando suas funcionalidades dentro da variável http.

A partir disso, podemos usar métodos como http.createServer() para criar servidores.

🔹 Criando um servidor

O principal uso do módulo HTTP é criar um servidor que escuta requisições. Isso é feito com o método createServer. 


const http = require('http');

const server = http.createServer((req, res) => {
  res.end('Olá mundo');
});

server.listen(3000);

🔹 Entendendo req e res

Dentro do createServer, temos dois parâmetros importantes:

  • req (request): representa a requisição feita pelo usuário
  • res (response): representa a resposta que o servidor irá enviar

Com o req, conseguimos acessar informações como a URL e o método da requisição. Com o res, conseguimos enviar dados de volta para o navegador.

🔹 Enviando uma resposta

Para enviar uma resposta ao navegador, utilizamos o método res.end(). Ele finaliza a requisição e envia o conteúdo. 


res.end('Resposta enviada com sucesso');

🔹 Definindo tipo de conteúdo (HTML)

Quando queremos retornar HTML, precisamos informar o tipo da resposta usando headers: 


res.writeHead(200, {
  'Content-Type': 'text/html; charset=utf-8'
});

🔹 Exemplo com HTML

Abaixo um exemplo simples de um servidor que retorna uma página HTML: 


const http = require('http');

const server = http.createServer((req, res) => {

  res.writeHead(200, {
    'Content-Type': 'text/html; charset=utf-8'
  });

  res.end(`
    <h1>Minha aplicação Node</h1>
    <p>Funcionando com HTTP</p>
  `);
});

server.listen(3000, () => {
  console.log('Servidor rodando em http://localhost:3000');
});

🔹 Como funciona na prática

  1. O servidor é criado com createServer
  2. Ele fica escutando uma porta (ex: 3000)
  3. Quando alguém acessa, a função é executada
  4. O servidor envia uma resposta com res.end()

🔹 Quando usar o módulo HTTP

  • Para aprender como funciona o backend na base
  • Para criar servidores simples
  • Para entender requisições e respostas HTTP

📌 Observação importante

Apesar de ser muito útil para aprendizado, o módulo http sozinho não é ideal para aplicações grandes, pois não possui recursos como roteamento organizado ou middlewares. Por isso, em projetos reais, utilizamos frameworks como Express ou Fastify.

Mesmo assim, entender o http é essencial, pois ele é a base de tudo que acontece no backend com Node.js.

Passo 4 — Entendendo o que foi feito

Nesse exemplo, criamos uma aplicação Node.js simples usando:

  • http para criar um servidor web
  • array para armazenar uma lista de produtos
  • map para transformar os produtos em itens HTML
  • template string para montar o HTML dentro do JavaScript
  • res.end para enviar o HTML como resposta ao navegador

No terminal, execute:

Acesse no navegador:

Você verá uma página HTML simples renderizando uma lista de produtos.

Faça alguns testes na aplicação:

  • Modifique o texto do h1
  • Atualize a página no navegador e veja as mudanças
