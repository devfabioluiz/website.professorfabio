Introdução ao desenvolvimento backend com Node.js

1. Objetivo da aula

Esta aula tem como objetivo capacitar o estudante no desenvolvimento de soluções backend utilizando a plataforma Node.js, com ênfase na escolha adequada de mecanismos e estruturas para resolver problemas típicos do desenvolvimento web.

Ao final, o estudante deverá compreender como criar um servidor, organizar rotas, processar dados enviados por formulários, interagir com banco de dados e criar uma API RESTful usando Express.

2. O que é Node.js?

Node.js é uma plataforma que permite executar JavaScript fora do navegador. Com ele, podemos criar aplicações backend, APIs, scripts, servidores web, ferramentas de linha de comando e sistemas que lidam com arquivos, eventos, conexões HTTP e bancos de dados.

Antes do Node.js, o JavaScript era usado principalmente no navegador. Com o Node.js, a mesma linguagem também pode ser usada no servidor.

3. Para que serve o Node.js?

  • Criar servidores web
  • Criar APIs REST
  • Processar formulários
  • Interagir com bancos de dados
  • Ler e escrever arquivos
  • Trabalhar com eventos
  • Executar tarefas em linha de comando
  • Criar sistemas escaláveis

4. Características principais do Node.js

  • Assíncrono: evita travar a aplicação enquanto uma tarefa demora.
  • Orientado a eventos: reage a acontecimentos, como requisições, cliques, leituras de arquivos e conexões.
  • Não bloqueante: consegue lidar com várias operações sem esperar uma terminar para iniciar outra.
  • Baseado em JavaScript: permite usar a mesma linguagem no frontend e no backend.
  • Possui grande ecossistema: usa o npm para instalar bibliotecas prontas.

5. Arquitetura orientada a eventos

No Node.js, muitas ações acontecem por meio de eventos. Uma requisição HTTP chegando ao servidor, um arquivo sendo lido ou uma conexão sendo encerrada são exemplos de eventos.

Em vez de executar tudo de forma linear e bloqueante, o Node.js registra funções que serão chamadas quando determinado evento ocorrer. 


const EventEmitter = require('events');

const eventos = new EventEmitter();

eventos.on('pedidoCriado', (pedido) => {
  console.log('Novo pedido recebido:', pedido);
});

eventos.emit('pedidoCriado', {
  id: 1,
  cliente: 'Maria',
  produto: 'Café'
});

Nesse exemplo, primeiro registramos o evento pedidoCriado. Depois, usamos emit para disparar esse evento.

6. Gerenciador de pacotes npm

O npm é o gerenciador de pacotes do Node.js. Ele é usado para instalar bibliotecas, criar projetos, executar scripts e controlar dependências. 


npm init -y
npm install express
npm install nodemon --save-dev

O arquivo package.json registra informações do projeto, dependências instaladas e comandos personalizados.

7. REPL do Node.js

O REPL é uma interface interativa do Node.js. A sigla significa Read, Evaluate, Print, Loop. Ele permite testar comandos JavaScript diretamente no terminal. 


node
          

2 + 2
'Node.js'.toUpperCase()
const nome = 'João'
nome

Para sair do REPL, pressione Ctrl + C duas vezes ou digite .exit.

8. Chamadas síncronas e assíncronas

Uma chamada síncrona bloqueia o fluxo da aplicação até terminar. Já uma chamada assíncrona permite que a aplicação continue executando outras tarefas enquanto aguarda o resultado. 


const fs = require('fs');

console.log('Início');

const conteudo = fs.readFileSync('arquivo.txt', 'utf-8');
console.log(conteudo);

console.log('Fim');

No exemplo acima, o Node.js espera a leitura terminar antes de seguir. 


const fs = require('fs');

console.log('Início');

fs.readFile('arquivo.txt', 'utf-8', (erro, conteudo) => {
  if (erro) {
    console.log('Erro ao ler arquivo:', erro.message);
    return;
  }

  console.log(conteudo);
});

console.log('Fim');

No segundo exemplo, a leitura é assíncrona. O programa continua sua execução enquanto o arquivo é lido.

9. Buffers

Buffers são áreas de memória usadas para lidar com dados binários. Eles são úteis quando trabalhamos com arquivos, imagens, vídeos, uploads ou dados recebidos pela rede. 


const buffer = Buffer.from('Olá, Node.js');

console.log(buffer);
console.log(buffer.toString());

10. Streams

Streams permitem processar dados em partes, sem carregar tudo de uma vez na memória. Isso é importante para arquivos grandes, vídeos, uploads e respostas HTTP extensas. 


const fs = require('fs');

const leitura = fs.createReadStream('entrada.txt');
const escrita = fs.createWriteStream('saida.txt');

leitura.pipe(escrita);

11. Objetos globais

O Node.js possui objetos e funções globais que podem ser usados sem importação direta.

  • console: exibe mensagens no terminal
  • process: traz informações do processo em execução
  • __dirname: caminho da pasta atual
  • __filename: caminho do arquivo atual
  • setTimeout: executa algo depois de um tempo
  • setInterval: executa algo repetidamente

console.log('Diretório atual:', __dirname);
console.log('Arquivo atual:', __filename);
console.log('Sistema operacional:', process.platform);

12. Módulo Web no Node.js

O Node.js possui o módulo http, que permite criar servidores web sem instalar bibliotecas externas. 


const http = require('http');

const servidor = http.createServer((req, res) => {
  res.writeHead(200, { 'Content-Type': 'text/plain; charset=utf-8' });
  res.end('Servidor Node.js funcionando!');
});

servidor.listen(3000, () => {
  console.log('Servidor rodando em http://localhost:3000');
});

Embora seja possível criar aplicações usando apenas o módulo http, na prática usamos frameworks como o Express para organizar melhor rotas, requisições, respostas e middlewares.

13. O que é Express?

Express é um framework para Node.js que facilita a criação de aplicações web e APIs. Ele simplifica o trabalho com rotas, middlewares, arquivos estáticos, formulários, cookies, sessões e integração com bancos de dados.

14. Request e Response

Em uma aplicação Express, cada rota normalmente recebe dois objetos principais: req e res.

  • req: representa a requisição feita pelo cliente.
  • res: representa a resposta que será enviada pelo servidor. 

app.get('/saudacao', (req, res) => {
  res.send('Olá! Essa resposta veio do servidor.');
});

15. Roteamento

Roteamento é o mecanismo usado para definir o que a aplicação deve fazer quando recebe uma requisição em determinado endereço e método HTTP. 


app.get('/tarefas', listarTarefas);
app.post('/tarefas', criarTarefa);
app.put('/tarefas/:id', atualizarTarefa);
app.delete('/tarefas/:id', removerTarefa);

16. Cookies e sessões

Cookies são pequenas informações armazenadas no navegador do usuário. Sessões são usadas para manter informações do usuário entre várias requisições, como um login ativo.

Em aplicações reais, é comum guardar apenas um identificador no cookie e manter os dados sensíveis no servidor.

17. Banco de dados relacional

Em um banco relacional, os dados são organizados em tabelas. Cada tabela possui colunas e linhas. Esse modelo é útil quando precisamos representar entidades e relações, como usuários, tarefas, produtos, pedidos e pagamentos.

Nesta aula, usaremos PostgreSQL para armazenar tarefas de uma pequena aplicação de controle pessoal.

18. Arquitetura REST

REST é um estilo de arquitetura para criação de serviços web. Em uma API REST, usamos recursos, URLs, métodos HTTP e códigos de status para organizar a comunicação entre cliente e servidor.

  • GET: buscar dados
  • POST: criar dados
  • PUT: atualizar dados
  • DELETE: remover dados

19. Códigos de status HTTP

  • 200: requisição bem-sucedida
  • 201: recurso criado com sucesso
  • 400: erro nos dados enviados pelo cliente
  • 404: recurso não encontrado
  • 500: erro interno no servidor

20. Escalabilidade em Node.js

Escalar uma aplicação significa prepará-la para lidar com mais usuários, mais requisições e mais processamento. No Node.js, podemos usar processos filhos, múltiplas instâncias, balanceadores de carga, filas, cache e separação de responsabilidades.

O módulo child_process permite criar processos separados usando recursos como exec, spawn e fork. 


const { exec } = require('child_process');

exec('node -v', (erro, stdout, stderr) => {
  if (erro) {
    console.log('Erro:', erro.message);
    return;
  }

  console.log('Versão do Node:', stdout);
});

Passo 8 — Criar o controller de tarefas

Crie o arquivo src/controllers/tarefasController.js. Ele concentrará as regras para listar, criar, atualizar e remover tarefas. 


const pool = require('../database/connection');

async function listarTarefas(req, res) {
  try {
    const resultado = await pool.query('SELECT * FROM tarefas ORDER BY id ASC');
    res.status(200).json(resultado.rows);
  } catch (erro) {
    res.status(500).json({ mensagem: 'Erro ao listar tarefas' });
  }
}

async function buscarTarefaPorId(req, res) {
  try {
    const { id } = req.params;

    const resultado = await pool.query('SELECT * FROM tarefas WHERE id = $1', [id]);

    if (resultado.rows.length === 0) {
      return res.status(404).json({ mensagem: 'Tarefa não encontrada' });
    }

    res.status(200).json(resultado.rows[0]);
  } catch (erro) {
    res.status(500).json({ mensagem: 'Erro ao buscar tarefa' });
  }
}

async function criarTarefa(req, res) {
  try {
    const { titulo, descricao } = req.body;

    if (!titulo) {
      return res.status(400).json({ mensagem: 'O título é obrigatório' });
    }

    const resultado = await pool.query(
      'INSERT INTO tarefas (titulo, descricao) VALUES ($1, $2) RETURNING *',
      [titulo, descricao]
    );

    res.status(201).json(resultado.rows[0]);
  } catch (erro) {
    res.status(500).json({ mensagem: 'Erro ao criar tarefa' });
  }
}

async function atualizarTarefa(req, res) {
  try {
    const { id } = req.params;
    const { titulo, descricao, concluida } = req.body;

    const resultado = await pool.query(
      'UPDATE tarefas SET titulo = $1, descricao = $2, concluida = $3 WHERE id = $4 RETURNING *',
      [titulo, descricao, concluida, id]
    );

    if (resultado.rows.length === 0) {
      return res.status(404).json({ mensagem: 'Tarefa não encontrada' });
    }

    res.status(200).json(resultado.rows[0]);
  } catch (erro) {
    res.status(500).json({ mensagem: 'Erro ao atualizar tarefa' });
  }
}

async function removerTarefa(req, res) {
  try {
    const { id } = req.params;

    const resultado = await pool.query('DELETE FROM tarefas WHERE id = $1 RETURNING *', [id]);

    if (resultado.rows.length === 0) {
      return res.status(404).json({ mensagem: 'Tarefa não encontrada' });
    }

    res.status(200).json({ mensagem: 'Tarefa removida com sucesso' });
  } catch (erro) {
    res.status(500).json({ mensagem: 'Erro ao remover tarefa' });
  }
}

module.exports = {
  listarTarefas,
  buscarTarefaPorId,
  criarTarefa,
  atualizarTarefa,
  removerTarefa
};

Passo 10 — Criar o servidor Express

Crie o arquivo src/server.js. Ele será o ponto de entrada da aplicação. 


const express = require('express');
const cookieParser = require('cookie-parser');
const session = require('express-session');
const multer = require('multer');
const path = require('path');

const tarefasRoutes = require('./routes/tarefasRoutes');

const app = express();
const upload = multer({ dest: 'uploads/' });

app.use(express.json());
app.use(express.urlencoded({ extended: true }));
app.use(cookieParser());

app.use(
  session({
    secret: 'segredo-da-aula-nodejs',
    resave: false,
    saveUninitialized: false
  })
);

app.use(express.static(path.join(__dirname, '..', 'public')));

app.use('/api', tarefasRoutes);

app.get('/', (req, res) => {
  res.send('API de tarefas funcionando!');
});

app.post('/login', (req, res) => {
  const { nome } = req.body;

  if (!nome) {
    return res.status(400).send('Informe um nome para entrar');
  }

  req.session.usuario = nome;
  res.cookie('ultimoAcesso', new Date().toLocaleString('pt-BR'));
  res.send(`Sessão criada para ${nome}`);
});

app.get('/perfil', (req, res) => {
  if (!req.session.usuario) {
    return res.status(401).send('Usuário não está logado');
  }

  res.send(`Usuário logado: ${req.session.usuario}`);
});

app.post('/upload', upload.single('arquivo'), (req, res) => {
  res.json({
    mensagem: 'Arquivo enviado com sucesso',
    arquivo: req.file
  });
});

app.listen(3000, () => {
  console.log('Servidor rodando em http://localhost:3000');
});

Passo 11 — Criar uma página HTML simples para formulário

Crie o arquivo public/index.html. Essa página representará um cliente web simples enviando dados para o servidor. 


<!doctype html>
<html lang="pt-BR">
  <head>
    <meta charset="UTF-8" />
    <title>Cadastro de Tarefas</title>
  </head>
  <body>
    <h1>Cadastro de Tarefas</h1>

    <form id="formTarefa">
      <input type="text" id="titulo" placeholder="Título da tarefa" required />
      <br /><br />

      <textarea id="descricao" placeholder="Descrição da tarefa"></textarea>
      <br /><br />

      <button type="submit">Cadastrar</button>
    </form>

    <h2>Tarefas</h2>
    <ul id="listaTarefas"></ul>

    <script>
      async function carregarTarefas() {
        const resposta = await fetch('/api/tarefas');
        const tarefas = await resposta.json();

        const lista = document.getElementById('listaTarefas');
        lista.innerHTML = '';

        tarefas.forEach((tarefa) => {
          const item = document.createElement('li');
          item.textContent = `${tarefa.id} - ${tarefa.titulo}`;
          lista.appendChild(item);
        });
      }

      document.getElementById('formTarefa').addEventListener('submit', async (evento) => {
        evento.preventDefault();

        const titulo = document.getElementById('titulo').value;
        const descricao = document.getElementById('descricao').value;

        await fetch('/api/tarefas', {
          method: 'POST',
          headers: {
            'Content-Type': 'application/json'
          },
          body: JSON.stringify({ titulo, descricao })
        });

        document.getElementById('titulo').value = '';
        document.getElementById('descricao').value = '';

        carregarTarefas();
      });

      carregarTarefas();
    </script>
  </body>
</html>

5. Criar um pequeno cliente Node.js para consumir a API

Crie um arquivo chamado src/cliente.js que faça uma requisição para listar tarefas usando fetch.

Esse exercício mostra que o Node.js pode ser usado tanto no servidor quanto como cliente consumindo serviços web.

Gabarito: 

async function buscarTarefas() {
  try {
    const resposta = await fetch('http://localhost:3000/api/tarefas');
    const tarefas = await resposta.json();

    console.log(tarefas);
  } catch (erro) {
    console.log('Erro ao buscar tarefas:', erro.message);
  }
}

buscarTarefas();

node src/cliente.js
