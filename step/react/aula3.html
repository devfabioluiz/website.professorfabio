Props e Passagem de dados (Pai/Filho)

Tópicos do nosso plano de curso abordados nessa aula: Props, Interação entre componentes filhos e pais.

Props

Props (abreviação de "properties") são uma forma de passar dados de um componente pai para um componente filho em React. 

              
import React from 'react';

const Saudacao = (props) => {
  return (
    <div>
      <h1>Olá, {props.nome}!</h1>
    </div>
  );
};

export default Saudacao;

🤓 Explicando o código 📝

O componente Saudacao recebe props como parâmetro e acessa props.nome para exibir o valor passado. Este é o exemplo mais básico de props em React: o componente pai passa um atributo nome="valor" e o filho o renderiza.

PropTypes

PropTypes é um recurso do React que permite validar as props que um componente recebe. Com ele, você pode definir o tipo esperado de cada prop (string, número, função, etc.) e indicar se ela é obrigatória ou opcional.

Por que usar PropTypes?

  • Documentação automática: Fica claro quais props um componente espera e de que tipo.
  • Prevenção de erros: O React exibe avisos no console se uma prop receber um tipo inesperado.
  • Manutenibilidade: Facilita a vida de quem for usar ou dar manutenção no componente.

Exemplo prático

import React from 'react';
import PropTypes from 'prop-types';

const Saudacao = ({ nome, idade }) => {
  return (
    <div>
      <h1>Olá, {nome}!</h1>
      {idade && <p>Você tem {idade} anos.</p>}
    </div>
  );
};

Saudacao.propTypes = {
  nome: PropTypes.string.isRequired,
  idade: PropTypes.number,
};

export default Saudacao;

🤓 Explicando o código 📝

Saudacao.propTypes é um objeto onde cada chave é o nome da prop e o valor é a validação. PropTypes.string exige que nome seja uma string, e .isRequired torna a prop obrigatória. PropTypes.number para idade é opcional (sem isRequired).

Para usar PropTypes, é necessário instalar o pacote separado: npm install prop-types (a partir do React 15.5, PropTypes foi movido para um pacote próprio).

Principais validadores

  • PropTypes.string — deve ser string
  • PropTypes.number — deve ser número
  • PropTypes.bool — deve ser booleano
  • PropTypes.func — deve ser função
  • PropTypes.array — deve ser array
  • PropTypes.object — deve ser objeto
  • PropTypes.oneOf(['a', 'b']) — deve ser um dos valores
  • PropTypes.oneOfType([...]) — deve ser um dos tipos

Passagem de dados entre componentes (Pai/Filho)

A comunicação entre componentes no React é fundamental para construir aplicações dinâmicas. Aqui veremos como passar dados de um componente pai para um filho e vice-versa.

Passando Dados do Pai para o Filho

A passagem de dados do pai para o filho é realizada por meio de props (propriedades). Veja o exemplo abaixo: 

import React from 'react';

const Child = ({ message }) => {
  return <p>{message}</p>;
}

const Parent = () => {
  return <Child message="Olá, do componente Pai!" />;
}

export default Parent;

🤓 Explicando o código 📝

Aqui temos dois componentes no mesmo arquivo: Child recebe message via desestruturação e o exibe. Parent renderiza Child passando a prop message="Olá, do componente Pai!".

Essa é a comunicação unidirecional (pai → filho) típica do React. Os dados fluem de cima para baixo na árvore de componentes.

Vantagens

  • Simples e direto para componentes hierárquicos.
  • Propriedades podem ser usadas como valores estáticos ou dinâmicos.

Desvantagens

  • Torna-se complexo em hierarquias profundas (pode ser resolvido com Context API ou bibliotecas de gerenciamento de estado).

2. Passando Dados do Filho para o Pai

Para enviar dados do filho para o pai, o pai deve passar uma função como prop para o filho. O filho pode chamar essa função com os dados desejados. 

import React, { useState } from 'react';

Child = ({ sendDataToParent }) => {
  return (
    <button onClick={() => sendDataToParent('Dados do Filho')}>Enviar</button>
  );
}

Parent = () => {
  const [data, setData] = useState('');

  const handleDataFromChild = (childData) => {
    setData(childData);
  };

  return (
    <div>
      <p>Dados recebidos: {data}</p>
      <Child sendDataToParent={handleDataFromChild} />;
    </div>
  );
}

export default Parent;

🤓 Explicando o código 📝

Para comunicação filho → pai, o pai passa uma função como prop (sendDataToParent). O filho a invoca com os dados desejados (sendDataToParent('Dados do Filho')).

No pai, handleDataFromChild recebe o dado e atualiza o estado data via setData, fazendo o React re-renderizar automaticamente o componente com o novo valor.

Vantagens

  • Permite que o pai reaja a eventos do filho.
  • Flexível para construir interações dinâmicas.

Desvantagens

  • Requer mais configuração do que a passagem pai para filho.

Passo 1 — Criar o componente Filho

Crie o arquivo src/components/Filho.jsx:


const Filho = ({ message, sendDataToParent }) => {
  return (
    <div>
      <p>Mensagem do pai: {message}</p>
      <button onClick={() => sendDataToParent('Dados do Filho')}>
        Enviar dados para o pai
      </button>
    </div>
  );
};

export default Filho;

🤓 Explicando o código 📝

O Filho recebe duas props: message (string exibida no parágrafo) e sendDataToParent (função chamada no clique do botão). A seta onClick={() => sendDataToParent('Dados do Filho')} cria uma função anônima que invoca sendDataToParent com os dados.

Passo 2 — Criar o componente Pai

Crie o arquivo src/components/Pai.jsx:


import { useState } from 'react';
import Filho from './Filho';

const Pai = () => {
  const [dadosDoFilho, setDadosDoFilho] = useState('');

  const receberDados = (dados) => {
    setDadosDoFilho(dados);
  };

  return (
    <div>
      <h2>Componente Pai</h2>
      <p>Dados recebidos do filho: {dadosDoFilho}</p>
      <Filho message="Olá do Pai!" sendDataToParent={receberDados} />
    </div>
  );
};

export default Pai;

🤓 Explicando o código 📝

O Pai gerencia o estado dadosDoFilho e passa para o Filho duas props: message (string) e sendDataToParent (função receberDados). Quando o Filho chama essa função, o estado do Pai é atualizado e a mensagem aparece na tela.

Este padrão de elevação de estado (lifting state up) é fundamental no React: o estado fica no componente pai, e o filho o modifica através de funções passadas via props.

Passo 3 — Renderizar o componente no App

No src/App.jsx, importe e renderize o componente Pai: 


import Pai from './components/Pai';

function App() {
  return (
    <div>
      <Pai />
    </div>
  );
}

export default App;

🤓 Explicando o código 📝

O App.jsx importa apenas o Pai, que por sua vez importa e renderiza o Filho internamente. Toda a lógica de comunicação pai/filho fica encapsulada dentro do componente Pai.

Passo 4 — Rodar o projeto e testar


npm run dev

Observe que o Pai envia uma mensagem para o Filho via prop message, e o Filho envia dados de volta para o Pai chamando a função sendDataToParent.

Formulário com comunicação Pai/Filho

Crie um componente FormularioFilho que contenha um campo de input e um botão. Quando o botão for clicado, o valor do input deve ser enviado para o Pai e exibido na tela.

O Pai deve gerenciar o estado do dado recebido e exibi-lo.

Obs: Não se esqueça de subir as alterações para o Git na branch dessa aula após terminar o dever.

Gabarito:

Componente FormularioFilho.jsx: 


import React, { useState } from 'react';

const FormularioFilho = ({ enviarDados }) => {
  const [inputValue, setInputValue] = useState('');

  const handleSubmit = () => {
    enviarDados(inputValue);
    setInputValue('');
  };

  return (
    <div>
      <input
        type="text"
        value={inputValue}
        onChange={(e) => setInputValue(e.target.value)}
        placeholder="Digite algo"
      />
      <button onClick={handleSubmit}>Enviar</button>
    </div>
  );
};

export default FormularioFilho;

🤓 Explicando o código 📝

O FormularioFilho gerencia seu próprio estado inputValue para o campo de texto. Ao clicar em "Enviar", ele chama enviarDados(inputValue) (função recebida do pai) e limpa o input. Este é um padrão comum de componente controlado em React.

Componente PaiFormulario.jsx: 


import React, { useState } from 'react';
import FormularioFilho from './FormularioFilho';

const PaiFormulario = () => {
  const [dados, setDados] = useState('');

  return (
    <div>
      <h2>Dados recebidos: {dados}</h2>
      <FormularioFilho enviarDados={setDados} />
    </div>
  );
};

export default PaiFormulario;

🤓 Explicando o código 📝

O PaiFormulario é mais simples que o Pai da aula: ele passa a própria função setDados diretamente como prop enviarDados. Isso funciona porque setDados já é uma função que atualiza o estado. É um padrão útil para casos simples.

App.jsx:


import React from 'react';
import PaiFormulario from './components/PaiFormulario';

function App() {
  return (
    <div>
      <PaiFormulario />
    </div>
  );
}

export default App;

🤓 Explicando o código 📝

O App.jsx do gabarito segue o mesmo padrão das aulas anteriores: importa o componente principal e o renderiza. A comunicação pai/filho fica toda encapsulada dentro de PaiFormulario.
