O que é React?

React é uma biblioteca JavaScript criada pelo Facebook em 2013 para construir interfaces de usuário (UI) dinâmicas e reativas. Ele é amplamente utilizado no desenvolvimento de aplicações web e móveis devido à sua flexibilidade, alta performance e facilidade de manutenção.

Para que Serve o React?

O React é usado para:

  • Criar Componentes Reutilizáveis: Permite encapsular partes da interface como botões ou caixas de texto.
  • Gerenciar Estados: Facilita o gerenciamento de dados dinâmicos e sua sincronização com a interface.
  • Renderizar Interfaces de Forma Reativa: Atualiza a interface automaticamente quando os dados mudam.
  • Criar Aplicações Escaláveis: A abordagem baseada em componentes torna projetos mais fáceis de manter e expandir.

Como o React Funciona?

O React segue um modelo declarativo, onde você descreve o que deve ser exibido, enquanto ele cuida de como atualizá-lo.

Principais Conceitos

  • Componentes: Peças fundamentais da interface, como botões ou formulários.
  • JSX: Extensão de sintaxe que combina HTML e JavaScript.
  • Virtual DOM: Representação leve do DOM que torna as atualizações mais eficientes. O React utiliza um Virtual DOM para gerenciar mudanças na interface. Em vez de atualizar o DOM real (relembre ou pesquise sobre o que é DOM no JavaScript) diretamente, ele mantém uma cópia virtual e atualiza apenas os elementos necessários, melhorando a performance.
  • Hooks: Recursos como useState e useEffect para gerenciar estado e ciclo de vida. (veremos melhor nas próximas aulas)
  • Servidores de Desenvolvimento: Ferramentas como Vite criam um ambiente local para desenvolvimento.
  • Node.js e npm/yarn: Gerenciam pacotes e bibliotecas do projeto.
  • Transpilers (Babel): Convertem código moderno em versões compatíveis com navegadores.
  • Empacotadores (Webpack/Vite): Empacotam os arquivos para otimizar a entrega.

Live Reload e HMR

Quando você salva uma alteração no código, o servidor de desenvolvimento recarrega automaticamente a página no navegador. Ferramentas como o Vite ou o Create React App integram o Live Reload automaticamente.

Hot Module Replacement (HMR): Uma evolução do live reload, onde apenas o código alterado é recarregado no navegador, preservando o estado atual da aplicação. Isso é especialmente útil para acelerar o desenvolvimento, pois evita que a interface seja reiniciada completamente.

Configurando o Ambiente com Vite

O Vite é uma ferramenta moderna para construção de aplicações web. Ele oferece uma experiência de desenvolvimento rápida e simples.

Passo a Passo

  1. Instale o Node.js: Certifique-se de que o Node.js está instalado em sua máquina.
  2. Crie o projeto:

    Nesse exemplo iremos criar um projeto React com o Vite o qual será chamado de my-react-app (nome que você pode colocar o que quiser) 

    
npm create vite@latest my-react-app --template react
  3. Escolha a opção de criar um projeto React com o Vite usando JavaScript
  4. Acesse a pasta do projeto: 
    
cd my-react-app
  5. Instale as dependências: 
    
npm install
  6. Inicie o servidor de desenvolvimento: 
    
npm run dev

Após seguir esses passos, você poderá acessar sua aplicação no navegador pelo endereço exibido no terminal (geralmente http://localhost:5173).

Arquitetura do React

A arquitetura do React é baseada no conceito de componentes reutilizáveis. Uma aplicação React geralmente segue a seguinte estrutura de pastas:

Estrutura de Pastas


my-react-app/
├── public/           # Arquivos estáticos
├── src/              # Código fonte da aplicação
│   ├── components/   # Componentes reutilizáveis
│   ├── pages/        # Páginas da aplicação
│   ├── assets/       # Arquivos de mídia e estilos
│   ├── App.jsx       # Componente principal
│   └── index.jsx     # Ponto de entrada da aplicação
├── package.json      # Configuração do projeto
└── vite.config.js    # Configuração do Vite

- node_modules/: Contém todas as dependências instaladas via NPM ou Yarn.

- public/: Armazena arquivos estáticos, como index.html, o ponto de entrada para o navegador.

- src/: Contém o código-fonte, incluindo os componentes e a lógica do aplicativo.

Exemplo de Componente

Nesse código podemos perceber que temos um componente chamado Welcome o qual é uma Arrow Function, podendo também ser uma função em JavaScript ou uma classe em JavaScript (veremos melhor na próxima aula a diferença entre eles), que recebe um parâmetro name. O componente retorna um JSX que exibe uma mensagem de bem-vindo com o nome fornecido. 


import React from 'react';

const Welcome = ({ name }) => (
  <div>
    <h1>Bem-vindo, {name}!</h1>
  </div>
);

export default Welcome;

🤓 Explicando o código 📝

O componente Welcome é declarado como uma arrow function que recebe um objeto props como parâmetro. Usamos a desestruturação { name } para extrair diretamente a propriedade name sem precisar escrever props.name.

O JSX retornado usa chaves { } para interpolar a variável name dentro do texto do <h1>, gerando uma mensagem personalizada. O export default Welcome permite que este componente seja importado em outros arquivos da aplicação.

Passo 1 — Criar projeto Vite


npm create vite@latest my-react-app --template react
cd my-react-app
npm install

Passo 2 — Criar o componente Welcome

Crie um arquivo Welcome.jsx dentro da pasta src/components e insira o código abaixo:


import React from 'react';

const Welcome = ({ name }) => (
  <div>
    <h1>Bem-vindo, {name}!</h1>
  </div>
);

export default Welcome;

🤓 Explicando o código 📝

Este é o mesmo componente Welcome visto anteriormente, agora sendo criado dentro da pasta src/components — prática recomendada para organizar componentes reutilizáveis. A estrutura é idêntica: uma arrow function que recebe { name } via desestruturação e renderiza um JSX com a saudação.

Note que o Vite já configura o projeto com suporte a JSX, então não precisamos nos preocupar com transpilação manual — o Babel é integrado automaticamente.

Passo 3 — Renderizar o componente no App

Abra o arquivo src/App.jsx e substitua o conteúdo pelo código abaixo:


import React from 'react';
import Welcome from './components/Welcome';

function App() {
  return (
    <div>
      <Welcome name="Fabio" />
    </div>
  );
}

export default App;

🤓 Explicando o código 📝

O App.jsx é o componente raiz da aplicação. Aqui importamos o Welcome de ./components/Welcome e o utilizamos como uma tag JSX <Welcome name="Fabio" />.

A prop name="Fabio" é passada para o componente Welcome, que a recebe via desestruturação e a exibe no <h1>. Esse é o mecanismo fundamental de comunicação entre componentes no React: o pai (App) passa dados para o filho (Welcome) através de props.

🛠️ Entendendo os comandos Git 📂

git add . — Adiciona todos os arquivos modificados (o . representa o diretório atual) à área de staging (preparação). É como "marcar" quais arquivos farão parte do próximo commit.

git commit -m "mensagem" — Cria um commit (um ponto na história do repositório) com os arquivos que estão em staging. A flag -m permite escrever a mensagem descritiva diretamente no comando.

git push — Envia os commits locais para o repositório remoto no GitHub. Como a branch aula1-react foi criada localmente com checkout -b, o Git pode pedir para usar --set-upstream na primeira vez (em aulas seguintes veremos isso).

import React from 'react';
import BoasVindas from './components/BoasVindas';

function App() {
  return (
    <div>
      <BoasVindas nome="Fabio" />
    </div>
  );
}

export default App;

🤓 Explicando o código 📝

O App.jsx do gabarito segue a mesma estrutura da aula: importa o componente filho (BoasVindas) e o renderiza passando a prop nome. Perceba que o caminho do import reflete a organização de pastas: ./components/BoasVindas.
