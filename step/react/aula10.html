Context API (hook useContext)

A Context API no React é uma ferramenta que permite compartilhar valores e estados entre componentes sem a necessidade de passar props manualmente por toda a árvore de componentes. Ela é ideal para gerenciar dados globais, como temas, autenticação ou configurações de idioma. O contexto é criado usando React.createContext(), e seus valores são fornecidos aos componentes filhos através do Provider. Os componentes que precisam acessar esses valores utilizam o hook useContext. A Context API simplifica a gestão de estados globais em aplicações React, eliminando a complexidade de "prop drilling". 


import React, { createContext, useContext } from 'react';

const TemaContext = createContext('claro');

function Painel() {
  const tema = useContext(TemaContext);
  return <div>Tema atual: {tema}</div>;
}

function App() {
  return (
    <TemaContext.Provider value="escuro">
      <Painel />
    </TemaContext.Provider>
  );
}

🤔 Explicando o código

createContext cria um objeto de contexto que pode ser compartilhado entre componentes. O Provider (ex: TemaContext.Provider) envolve a árvore de componentes e fornece o valor do contexto. O hook useContext permite que qualquer componente filho acesse esse valor sem precisar recebê-lo via props — eliminando o prop drilling.

Redux

Redux e uma ferramenta popular para gerenciamento de estado em aplicações React. Ele proporciona uma estrutura de dados unificada que pode ser facilmente compartilhada entre componentes e acionados por ações.

Exemplo
              
import React from 'react';
import { createStore } from 'redux';

const initialState = { count: 0 };  
const reducer = (state = initialState, action) => {
  switch (action.type) {
    case 'INCREMENT':
      return { ...state, count: state.count + 1 };
    case 'DECREMENT':
      return { ...state, count: state.count - 1 };
    default:
      return state;
  }
};

const store = createStore(reducer);

const Contador = () => {
  const state = store.getState();
  return (
    <>
      <p>Contagem: {state.count}</p>
      <button onClick={() => store.dispatch({ type: 'INCREMENT' })}>+</button>
      <button onClick={() => store.dispatch({ type: 'DECREMENT' })}>-</button>
    </>
  );
};

export default Contador;

🤔 Explicando o código

O Redux gerencia o estado global da aplicação através de uma store centralizada. Um reducer é uma função pura que recebe o estado atual e uma action e retorna um novo estado. As ações são objetos com uma propriedade type e são enviadas via store.dispatch(). O componente lê o estado com store.getState().

useReducer

Hook para gerenciamento de estado complexo, similar ao Redux, com dispatch e reducer.

Exemplo

import React, { useReducer } from 'react';

const initialState = { count: 0 };

function reducer(state, action) {
  switch (action.type) {
    case 'increment':
      return { count: state.count + 1 };
    case 'decrement':
      return { count: state.count - 1 };
    default:
      return state;
  }
}

function Contador() {
  const [state, dispatch] = useReducer(reducer, initialState);

  return (
    <>;
      <p>;Contagem: {state.count}</p>;
      <button onClick={() =>; dispatch({ type: 'increment' })}>;+</button>;
      <button onClick={() =>; dispatch({ type: 'decrement' })}>;-</button>;
    </>;
  );
}

🤔 Explicando o código

O hook useReducer é uma alternativa ao useState para gerenciar estados complexos. Ele recebe uma função reducer e um estado inicial, retornando um par [state, dispatch]. O dispatch envia ações para o reducer, que calcula o novo estado com base no tipo da ação. É especialmente útil quando o próximo estado depende de múltiplos valores ou de lógica condicional.

Passo 2 — Criar o ThemeContext

Crie o arquivo src/ThemeContext.jsx:


import { createContext, useState, useContext } from 'react';

const ThemeContext = createContext();

export const ThemeProvider = ({ children }) => {
  const [theme, setTheme] = useState('light');

  const toggleTheme = () => {
    setTheme(prev => (prev === 'light' ? 'dark' : 'light'));
  };

  return (
    <ThemeContext.Provider value={{ theme, toggleTheme }}>
      {children}
    </ThemeContext.Provider>
  );
};

export const useTheme = () => useContext(ThemeContext);

🤔 Explicando o código

createContext() cria um contexto vazio. O ThemeProvider é um componente que utiliza o ThemeContext.Provider para disponibilizar o estado do tema e a função toggleTheme para toda a árvore de componentes filhos. O hook customizado useTheme encapsula useContext(ThemeContext), facilitando o consumo do contexto em qualquer componente.

Passo 3 — Criar um componente que use o tema

Crie o arquivo src/ThemeSwitcher.jsx:


import { useTheme } from './ThemeContext';

const ThemeSwitcher = () => {
  const { theme, toggleTheme } = useTheme();

  return (
    <div style={{
      background: theme === 'light' ? '#fff' : '#333',
      color: theme === 'light' ? '#000' : '#fff',
      padding: '20px',
      minHeight: '100vh',
    }}>
      <h1>Tema atual: {theme}</h1>
      <button onClick={toggleTheme}>
        Alternar para {theme === 'light' ? 'dark' : 'light'}
      </button>
    </div>
  );
};

export default ThemeSwitcher;

🤔 Explicando o código

O ThemeSwitcher consome o contexto através do hook useTheme(), que retorna o tema atual e a função toggleTheme. O estilo do componente é alterado dinamicamente conforme o valor de theme (light ou dark), e o botão dispara a alternância do tema. Tudo isso sem que nenhuma prop seja passada manualmente — o contexto faz o compartilhamento automaticamente.

Passo 4 — Usar o Provider no App.jsx


import { ThemeProvider } from './ThemeContext';
import ThemeSwitcher from './ThemeSwitcher';

const App = () => {
  return (
    <ThemeProvider>
      <ThemeSwitcher />
    </ThemeProvider>
  );
};

export default App;

🤔 Explicando o código

O App envolve o componente ThemeSwitcher com o ThemeProvider. Esse é o padrão Provider da Context API: o provedor deve estar no nível mais alto da árvore de componentes que precisam acessar o contexto. Assim, qualquer componente descendente do ThemeProvider consegue consumir o tema sem a necessidade de props intermediárias.

Criar uma Todo List com useReducer

Crie uma branch chamada dever-de-casa-aula10 e nela desenvolva:

  1. Crie um componente TodoList.jsx que use useReducer para gerenciar uma lista de tarefas.
  2. Implemente as ações: ADD_TASK, TOGGLE_TASK e REMOVE_TASK.
  3. Renderize um input para adicionar tarefas e a lista com opção de marcar como concluída e remover.

Obs: Não se esqueça de subir o projeto para o Git na branch do dever após terminar.

Gabarito:

TodoList.jsx:


import { useReducer, useState } from 'react';

const reducer = (state, action) => {
  switch (action.type) {
    case 'ADD_TASK':
      return [...state, { id: Date.now(), text: action.payload, completed: false }];
    case 'TOGGLE_TASK':
      return state.map(task =>
        task.id === action.payload
          ? { ...task, completed: !task.completed }
          : task
      );
    case 'REMOVE_TASK':
      return state.filter(task => task.id !== action.payload);
    default:
      return state;
  }
};

const TodoList = () => {
  const [tasks, dispatch] = useReducer(reducer, []);
  const [input, setInput] = useState('');

  const addTask = () => {
    if (input.trim()) {
      dispatch({ type: 'ADD_TASK', payload: input });
      setInput('');
    }
  };

  return (
    <div>
      <h1>Lista de Tarefas</h1>
      <input
        value={input}
        onChange={(e) => setInput(e.target.value)}
        placeholder="Nova tarefa"
      />
      <button onClick={addTask}>Adicionar</button>
      <ul>
        {tasks.map(task => (
          <li key={task.id}
            style={{ textDecoration: task.completed ? 'line-through' : 'none' }}
          >
            {task.text}
            <button onClick={() => dispatch({ type: 'TOGGLE_TASK', payload: task.id })}>
              {task.completed ? 'Desmarcar' : 'Concluir'}
            </button>
            <button onClick={() => dispatch({ type: 'REMOVE_TASK', payload: task.id })}>
              Remover
            </button>
          </li>
        ))}
      </ul>
    </div>
  );
};

export default TodoList;

🤔 Explicando o código

O hook useReducer gerencia o array de tarefas. O reducer trata três ações: ADD_TASK adiciona uma nova tarefa com Date.now() como ID único, TOGGLE_TASK alterna a propriedade completed da tarefa correspondente ao ID, e REMOVE_TASK filtra a tarefa pelo ID removendo-a da lista. O useState é usado separadamente para controlar o valor do input.

App.jsx:


import TodoList from './TodoList';

const App = () => {
  return <TodoList />;
};

export default App;

🤔 Explicando o código

O App.jsx importa e renderiza o componente TodoList, que contém toda a lógica de gerenciamento de tarefas com useReducer. A estrutura é simples porque o useReducer encapsula tanto o estado quanto as ações dentro do próprio componente TodoList.
