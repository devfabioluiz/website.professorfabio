Requisições HTTP no React

Tópicos do nosso plano de curso abordados nessa aula: Fundamentos do HTTP, Métodos HTTP (GET, POST, PUT, DELETE, PATCH), REST API, fetch, Axios, React Query, SWR, Obtendo dados do servidor em React.

Existem várias formas de realizar requisições HTTP no React. São elas:

O que é HTTP?

HTTP significa Hypertext Transfer Protocol (Protocolo de Transferência de Hipertexto). É o protocolo fundamental utilizado para a comunicação entre clientes (navegadores) e servidores na web.

A comunicação acontece através de requisições (requests) e respostas (responses). O cliente envia uma requisição para o servidor, que processa e devolve uma resposta.

Uma requisição HTTP é composta por:

  • URL — o endereço do recurso que queremos acessar
  • Método — a ação que queremos realizar (GET, POST, etc.)
  • Cabeçalhos (headers) — informações adicionais como formato dos dados, autenticação, etc.
  • Corpo (body) — dados enviados ao servidor (comum em POST, PUT, PATCH)

Métodos HTTP

Os métodos HTTP indicam a ação que desejamos realizar ao fazer uma requisição. Os principais são:

  • GET — obter dados (o mais comum, usado nas APIs públicas como JSONPlaceholder)
  • POST — criar novos recursos no servidor
  • PUT — atualizar recursos existentes por completo
  • PATCH — atualização parcial de um recurso
  • DELETE — remover recursos do servidor
Método Função Possui corpo? Exemplo
GET Obter dados Não /users
POST Criar recurso Sim /users
PUT Atualizar recurso Sim /users/1
PATCH Atualização parcial Sim /users/1
DELETE Remover recurso Não /users/1

Códigos de Resposta HTTP

Quando o servidor responde a uma requisição, ele envia um código de status indicando o resultado da operação. Os códigos são agrupados em cinco classes:

  • 1xx — Informativo: a requisição foi recebida e o processamento continua.
  • 2xx — Sucesso: a requisição foi recebida, compreendida e aceita com sucesso. Exemplos: 200 OK, 201 Created.
  • 3xx — Redirecionamento: o cliente precisa realizar ações adicionais para completar a requisição. Exemplos: 301 Moved Permanently, 302 Found.
  • 4xx — Erro do Cliente: a requisição contém erro ou não pode ser processada. Exemplos: 400 Bad Request, 404 Not Found.
  • 5xx — Erro do Servidor: o servidor falhou ao processar a requisição. Exemplo: 500 Internal Server Error.

O que é REST API?

REST (Representational State Transfer) é um estilo arquitetural para projetar APIs web. Uma API que segue os princípios REST é chamada de API RESTful.

As principais características de uma API RESTful são:

  • Usa os métodos HTTP de forma padronizada (GET para ler, POST para criar, etc.)
  • As URLs representam recursos de forma clara: /users, /users/1, /posts, /products/123
  • As respostas geralmente são no formato JSON
  • Não mantém estado da sessão (stateless) — cada requisição contém todas as informações necessárias

A API que utilizaremos nesta aula — JSONPlaceholder — é um exemplo de REST API gratuita para testes e prototipação. Ela disponibiliza recursos como /users, /posts, /comments, etc.

Usando fetch nativo do JavaScript


const App = () => {
  const [data, setData] = useState(null);

  useEffect(() => {
    fetch('https://api.example.com/data')
      .then((response) => response.json())
      .then((data) => setData(data))
      .catch((error) => console.error('Error fetching data:', error));
  }, []);

   return (
    <div>
      <h1>Data:</h1>
      <pre>{data ? JSON.stringify(data, null, 2) : 'Loading...'}</pre>
    </div>
  );

}

export default App;

🤓 Explicando o código 📝

Este exemplo usa o hook useState para armazenar os dados retornados pela API. O useEffect com array de dependências vazio executa a requisição apenas uma vez, quando o componente é montado. O método fetch nativo do JavaScript faz a chamada HTTP, e response.json() converte a resposta para objeto JavaScript. Enquanto os dados não chegam, exibimos "Loading..." usando um operador ternário.

Vantagens

  • Nativo no JavaScript; nenhuma biblioteca externa necessária.
  • Compatível com qualquer ambiente moderno.
  • Ideal para chamadas simples.

Desvantagens

  • Não inclui timeout por padrão.
  • Tratamento de erros mais trabalhoso.
  • Pode ser verboso para lógicas repetitivas.

Usando Axios

            
import React, { useEffect, useState } from 'react';
import axios from 'axios';
const App = () => {
  const [data, setData] = useState(null);
  useEffect(() => {
    axios.get('https://api.example.com/data')
      .then((response) => setData(response.data))
      .catch((error) => console.error('Error fetching data:', error));
  }, []);
  return (
    <div>
      <h1>Data:</h1>
      <pre>{data ? JSON.stringify(data, null, 2) : 'Loading...'}</pre>
    </div>
  );
}
export default App;

🤓 Explicando o código 📝

Este exemplo utiliza a biblioteca Axios para fazer a requisição GET. Diferente do fetch, o Axios já retorna os dados em response.data, eliminando a necessidade de chamar response.json(). A sintaxe é mais concisa e o Axios oferece recursos como timeout e interceptadores embutidos, facilitando o tratamento global de requisições.

Vantagens

  • Sintaxe mais concisa que fetch.
  • Suporte nativo a JSON, timeout e cancelamento de requisições.
  • Interceptadores para manipulação global de requisições e respostas.

Desvantagens

  • Adiciona uma dependência externa.
  • Leve aumento no tamanho do bundle.

Usando react-query (TanStack Query)

React Query é uma biblioteca para gerenciamento de estados relacionados a dados assíncronos em aplicações React. Ela simplifica tarefas como o fetch de dados, caching, sincronização e atualização em tempo real, reduzindo a necessidade de gerenciar manualmente estados de carregamento, erro e sucesso. Com React Query, dados são armazenados em cache e revalidados automaticamente, melhorando a performance da aplicação. A biblioteca também oferece suporte a funcionalidades avançadas, como pré-busca de dados, paginação e invalidação de cache, tornando-se uma escolha poderosa para lidar com APIs e dados assíncronos de forma eficiente. 

import React from 'react';
import { useQuery } from '@tanstack/react-query';
const App = () => {
  const { data, isLoading, error } = useQuery(['data'], () =>
    fetch('https://api.example.com/data').then((res) => res.json())
  );
  if (isLoading) return <div>Loading...</div>;
  if (error) return <div>Error: {error.message}</div>;
  return (
    <div>
      <h1>Data:</h1>
      <pre>{JSON.stringify(data, null, 2)}</pre>
    </div>
  );
}
export default App;

🤓 Explicando o código 📝

O React Query (TanStack Query) gerencia o estado assíncrono de forma declarativa. O hook useQuery recebe uma chave única para identificar a query e uma função que retorna uma Promise. Ele retorna os estados data, isLoading e error automaticamente, eliminando a necessidade de gerenciar manualmente cada estado com useState e useEffect. O cache e a revalidação são feitos automaticamente pela biblioteca.

Vantagens

  • Cache de dados automático e revalidação em segundo plano.
  • Gerenciamento eficiente de estado de requisições.
  • Ótimo suporte a SSR (Server-Side Rendering) (Veremos esse tema em aulas futuras).

Desvantagens

  • Curva de aprendizado mais longa para iniciantes.
  • Adiciona uma dependência externa.

Instale e veja como funciona em: Documentação do TanStack Query

Usando SWR

SWR (stale-while-revalidate) é uma biblioteca para React que facilita o gerenciamento de dados remotos, especialmente na interação com APIs. Baseada no conceito de "stale-while-revalidate", ela retorna dados armazenados em cache enquanto busca novas informações em segundo plano, garantindo uma experiência rápida e responsiva para o usuário.

SWR simplifica a implementação de funcionalidades como cache, revalidação automática, sincronização de dados, e suporte a SSR (Server-Side Rendering). É ideal para aplicações que requerem dados em tempo real ou atualizações frequentes, oferecendo uma abordagem leve e eficiente para o gerenciamento de dados assíncronos. 

import useSWR from 'swr';
const fetcher = (url) => fetch(url).then((res) => res.json());
const App = () => {
  const { data, error } = useSWR('https://api.example.com/data', fetcher);
  if (error) return <div>Error: {error.message}</div>;
  if (!data) return <div>Loading...</div>;
  return (
    <div>
      <h1>Data:</h1>
      <pre>{JSON.stringify(data, null, 2)}</pre>
    </div>
  );
}
export default App;

🤓 Explicando o código 📝

O SWR (stale-while-revalidate) é uma biblioteca leve para gerenciamento de dados remotos. O hook useSWR recebe a URL da API e uma função fetcher que define como os dados serão buscados. O SWR retorna os dados em cache enquanto revalida em segundo plano, garantindo uma experiência rápida. Ele também fornece os estados data e error para controlar a exibição.

Vantagens

  • Cache integrado e revalidação automática.
  • API simples e declarativa.
  • Suporte a SSR e ISR. (Veremos esse tema em aulas futuras).

Desvantagens

  • Adiciona uma dependência externa.
  • Funcionalidades avançadas (ex.:interceptadores) exigem configuração adicional.

Instale e veja como funciona em: Documentação do SWR

Passo 2 — Fazer requisição GET com fetch

No componente App.jsx, use useEffect e fetch para buscar dados da JSONPlaceholder API: 


import { useState, useEffect } from 'react';

function App() {
  const [users, setUsers] = useState([]);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {
    fetch('https://jsonplaceholder.typicode.com/users')
      .then((response) => {
        if (!response.ok) {
          throw new Error('Erro na requisição');
        }
        return response.json();
      })
      .then((data) => {
        setUsers(data);
        setLoading(false);
      })
      .catch((err) => {
        setError(err.message);
        setLoading(false);
      });
  }, []);

  if (loading) return <p>Carregando...</p>;
  if (error) return <p>Erro: {error}</p>;

  return (
    <div>
      <h1>Lista de Usuários</h1>
      <ul>
        {users.map((user) => (
          <li key={user.id}>
            {user.name} - {user.email}
          </li>
        ))}
      </ul>
    </div>
  );
}

export default App;

🤓 Explicando o código 📝

Neste componente, usamos três estados: users para armazenar a lista de usuários, loading para controlar o carregamento, e error para capturar erros. Dentro do useEffect, fazemos uma requisição GET com fetch para a JSONPlaceholder API. Verificamos se a resposta é ok com response.ok, e em caso de erro lançamos uma exceção. Os dados são armazenados no estado users e exibidos em uma lista não ordenada com map.

Passo 3 — Fazer requisição POST com fetch

O método POST é utilizado para enviar dados ao servidor e criar novos recursos. Vamos criar um formulário que envia um novo post para a JSONPlaceholder API usando fetch: 


import { useState } from 'react';

function App() {
  const [title, setTitle] = useState('');
  const [body, setBody] = useState('');
  const [response, setResponse] = useState(null);

  const handleSubmit = (e) => {
    e.preventDefault();

    fetch('https://jsonplaceholder.typicode.com/posts', {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
      },
      body: JSON.stringify({
        title,
        body,
        userId: 1,
      }),
    })
      .then((res) => res.json())
      .then((data) => setResponse(data))
      .catch((err) => console.error('Erro:', err));
  };

  return (
    <div>
      <h1>Criar Post</h1>
      <form onSubmit={handleSubmit}>
        <input
          type="text"
          placeholder="Título"
          value={title}
          onChange={(e) => setTitle(e.target.value)}
        />
        <textarea
          placeholder="Conteúdo"
          value={body}
          onChange={(e) => setBody(e.target.value)}
        />
        <button type="submit">Enviar</button>
      </form>

      {response && (
        <div>
          <h3>Post criado com sucesso!</h3>
          <pre>{JSON.stringify(response, null, 2)}</pre>
        </div>
      )}
    </div>
  );
}

export default App;

🤓 Explicando o código 📝

Neste exemplo, usamos o método POST do fetch para enviar dados ao servidor. Passamos um objeto de configuração com method: 'POST', os headers indicando que estamos enviando JSON (Content-Type: application/json), e o body com os dados convertidos para JSON via JSON.stringify(). O servidor JSONPlaceholder simula a criação e retorna o recurso criado com um id gerado automaticamente.

Refatore o App.jsx para usar Axios:


import { useState, useEffect } from 'react';
import axios from 'axios';

function App() {
  const [users, setUsers] = useState([]);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {
    axios
      .get('https://jsonplaceholder.typicode.com/users')
      .then((response) => {
        setUsers(response.data);
        setLoading(false);
      })
      .catch((err) => {
        setError(err.message);
        setLoading(false);
      });
  }, []);

  if (loading) return <p>Carregando...</p>;
  if (error) return <p>Erro: {error}</p>;

  return (
    <div>
      <h1>Lista de Usuários (Axios)</h1>
      <ul>
        {users.map((user) => (
          <li key={user.id}>
            {user.name} - {user.email} - {user.phone}
          </li>
        ))}
      </ul>
    </div>
  );
}

export default App;

🤓 Explicando o código 📝

A refatoração com Axios mantém a mesma lógica do exemplo anterior, mas simplifica o código. axios.get() retorna uma Promise cujo resultado está em response.data — não precisamos mais chamar response.json(). O tratamento de erro com .catch() continua similar, mas o Axios já trata automaticamente erros de rede e HTTP como exceções, tornando o código mais limpo.

Crie o componente UserList.jsx:


import { useState, useEffect } from 'react';
import axios from 'axios';

const UserList = () => {
  const [users, setUsers] = useState([]);
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    axios
      .get('https://jsonplaceholder.typicode.com/users')
      .then((response) => {
        setUsers(response.data);
        setLoading(false);
      })
      .catch(() => setLoading(false));
  }, []);

  if (loading) return <p>Carregando...</p>;

  return (
    <div>
      <h2>Usuários</h2>
      {users.map((user) => (
        <div key={user.id} className="user-card">
          <h3>{user.name}</h3>
          <p>Email: {user.email}</p>
          <p>Cidade: {user.address.city}</p>
        </div>
      ))}
    </div>
  );
};

export default UserList;

Criamos um componente separado UserList para organizar melhor o código. Ele contém toda a lógica de busca e exibição dos usuários, incluindo os estados de carregamento. Cada usuário é exibido em um div com classe user-card, mostrando nome, email e cidade. Separar componentes é uma boa prática no React para manter o código modular e reutilizável.

Dever de Casa

Buscar e exibir posts da JSONPlaceholder

Crie um componente PostList que busque a lista de posts da API https://jsonplaceholder.typicode.com/posts e os exiba em formato de cartões.

Requisitos do dever de casa:

  • Usar Axios para fazer a requisição GET
  • Exibir o title e o body de cada post
  • Mostrar o userId como badge no cartão
  • Adicionar um input de busca que filtre os posts pelo título
  • Exibir "Nenhum post encontrado" quando o filtro não tiver resultados
  • Tratar estados de loading e erro

Obs: Não se esqueça de subir as alterações para o Git na branch dessa aula após terminar o dever.

Gabarito: 

import { useState, useEffect } from 'react';
import axios from 'axios';

const PostList = () => {
  const [posts, setPosts] = useState([]);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);
  const [search, setSearch] = useState('');

  useEffect(() => {
    axios
      .get('https://jsonplaceholder.typicode.com/posts')
      .then((response) => {
        setPosts(response.data);
        setLoading(false);
      })
      .catch((err) => {
        setError(err.message);
        setLoading(false);
      });
  }, []);

  const filteredPosts = posts.filter((post) =>
    post.title.toLowerCase().includes(search.toLowerCase())
  );

  if (loading) return <p>Carregando posts...</p>;
  if (error) return <p>Erro: {error}</p>;

  return (
    <div>
      <h2>Posts</h2>
      <input
        type="text"
        placeholder="Buscar por título..."
        value={search}
        onChange={(e) => setSearch(e.target.value)}
      />

      {filteredPosts.length === 0 ? (
        <p>Nenhum post encontrado</p>
      ) : (
        filteredPosts.map((post) => (
          <div key={post.id} className="post-card">
            <span className="badge">User {post.userId}</span>
            <h3>{post.title}</h3>
            <p>{post.body}</p>
          </div>
        ))
      )}
    </div>
  );
};

export default PostList;

🤓 Explicando o código 📝

O componente PostList busca os posts da API JSONPlaceholder usando Axios. Ele gerencia os estados de carregamento, erro e dados, além de um estado search para filtrar os posts pelo título. O filtro é feito com o método filter combinado com includes para busca case-insensitive. Se não houver resultados, exibe "Nenhum post encontrado". Cada post é exibido em um cartão com o userId como badge, o título e o corpo do post.
