1. O que são componentes de classe?

Antes da introdução dos Hooks no React 16.8 (2019), os componentes de classe eram a única forma de usar estado e ciclo de vida em componentes React. Mesmo hoje, é comum encontrar código legado que utiliza classes, e entender esse padrão é fundamental para manter e migrar projetos antigos.

Um componente de classe é uma classe JavaScript que estende React.Component e deve implementar, no mínimo, o método render(), que retorna o JSX a ser exibido.

Exemplo básico: HelloWorld com classe
              
import React, { Component } from 'react';

class HelloWorld extends Component {
  render() {
    return <h1>Olá, mundo!</h1>;
  }
}

export default HelloWorld;

🤓 Explicando o código 📝

A classe HelloWorld estende Component (importado do React) e implementa o método render(), que retorna um elemento JSX. Esse é o componente mais simples possível usando classe. Para usá-lo, basta importá-lo e inseri-lo no JSX como <HelloWorld />.

2. Estado em componentes de classe

Em componentes funcionais usamos useState para gerenciar estado. Em classes, o estado é um objeto armazenado em this.state, geralmente inicializado no constructor. Para atualizá-lo, usamos sempre o método this.setState() — nunca atribua diretamente (this.state.x = valor), pois isso não dispara uma re-renderização.

Exemplo prático: Contador com classe
              
import React, { Component } from 'react';

class Contador extends Component {
  constructor(props) {
    super(props);
    this.state = {
      count: 0,
    };
  }

  incrementar = () => {
    this.setState({ count: this.state.count + 1 });
  };

  decrementar = () => {
    this.setState({ count: this.state.count - 1 });
  };

  render() {
    return (
      <div>
        <h1>Contador: {this.state.count}</h1>
        <button onClick={this.incrementar}>+</button>
        <button onClick={this.decrementar}>-</button>
      </div>
    );
  }
}

export default Contador;

🤓 Explicando o código 📝

No constructor, inicializamos this.state com um objeto contendo count: 0. Os métodos incrementar e decrementar usam this.setState() para atualizar o estado — isso dispara uma nova chamada ao render(), atualizando a界面. Note o uso de arrow functions para definir os métodos, o que elimina a necessidade de bind manual no constructor.

3. Métodos do ciclo de vida

Os métodos de ciclo de vida são hooks nativos das classes que nos permitem executar código em momentos específicos da existência do componente. Os três principais são:

  • componentDidMount() — executado após o componente renderizar pela primeira vez. Ideal para requisições a APIs, subscriptions e inicializações.
  • componentDidUpdate(prevProps, prevState) — executado após uma atualização (quando props ou state mudam). Útil para reagir a mudanças comparando valores anteriores com atuais.
  • componentWillUnmount() — executado antes do componente ser removido do DOM. Usado para cleanup: limpar timers, cancelar subscriptions, remover event listeners.
Exemplo prático: Relógio com classe
              
import React, { Component } from 'react';

class Relogio extends Component {
  constructor(props) {
    super(props);
    this.state = {
      hora: new Date().toLocaleTimeString(),
    };
  }

  componentDidMount() {
    this.timerID = setInterval(() => {
      this.setState({ hora: new Date().toLocaleTimeString() });
    }, 1000);
  }

  componentDidUpdate(prevProps, prevState) {
    console.log('Relógio atualizado:', this.state.hora);
  }

  componentWillUnmount() {
    clearInterval(this.timerID);
  }

  render() {
    return <h2>{this.state.hora}</h2>;
  }
}

export default Relogio;

🤓 Explicando o código 📝

No componentDidMount(), iniciamos um setInterval que atualiza o estado com a hora atual a cada 1 segundo. O timer é armazenado em this.timerID para ser limpo posteriormente. componentDidUpdate exibe um log no console sempre que o estado muda. componentWillUnmount() limpa o setInterval com clearInterval, evitando vazamento de memória quando o componente for removido.

4. Manipulação de eventos em classes

Em componentes de classe, os event handlers são definidos como métodos da classe. Existem duas abordagens comuns para garantir que o this dentro do handler aponte para a instância correta:

  • Bind manual no constructor: this.handleClick = this.handleClick.bind(this);
  • Arrow functions (class fields): handleClick = () => { ... } — mais conciso e recomendado.
Exemplo: Formulário com classe
              
import React, { Component } from 'react';

class Formulario extends Component {
  constructor(props) {
    super(props);
    this.state = {
      nome: '',
      email: '',
    };
  }

  handleChange = (e) => {
    this.setState({ [e.target.name]: e.target.value });
  };

  handleSubmit = (e) => {
    e.preventDefault();
    alert(`Nome: ${this.state.nome}, Email: ${this.state.email}`);
  };

  render() {
    return (
      <form onSubmit={this.handleSubmit}>
        <input
          name="nome"
          value={this.state.nome}
          onChange={this.handleChange}
          placeholder="Nome"
        />
        <input
          name="email"
          value={this.state.email}
          onChange={this.handleChange}
          placeholder="Email"
        />
        <button type="submit">Enviar</button>
      </form>
    );
  }
}

export default Formulario;

🤓 Explicando o código 📝

O método handleChange é uma arrow function, então o this se refere à instância da classe. Usamos [e.target.name] para atualizar dinamicamente a propriedade correta do estado com base no name do input. handleSubmit previne o recarregamento da página com e.preventDefault() e exibe os dados em um alert.

5. Comparação: Classe vs Funcional

A tabela abaixo mostra o mesmo componente (um contador) implementado nos dois estilos:

Componente de Classe Componente Funcional 

class Contador extends Component {
  constructor(props) {
    super(props);
    this.state = { count: 0 };
  }

  incrementar = () => {
    this.setState({ count: this.state.count + 1 });
  };

  render() {
    return (
      <div>
        <p>{this.state.count}</p>
        <button onClick={this.incrementar}>+</button>
      </div>
    );
  }
}
                    

function Contador() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <p>{count}</p>
      <button onClick={() => setCount(count + 1)}>+</button>
    </div>
  );
}

Quando usar cada um? O estilo funcional com Hooks é o padrão moderno do React e deve ser preferido para novos projetos. Componentes de classe ainda aparecem em código legado e em projetos que não foram migrados. Saber trabalhar com ambos é essencial para um desenvolvedor React completo.

6. Passo a Passo: Converter um componente funcional para classe

Vamos pegar o componente Contador que criamos na aula 2 (versão funcional) e reescrevê-lo como componente de classe.

Versão funcional original (aula 2):

import React, { useState } from 'react';

function Contador() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <p>Você clicou {count} vezes</p>
      <button onClick={() => setCount(count + 1)}>
        Clique aqui
      </button>
    </div>
  );
}

export default Contador;
Versão equivalente em classe:

import React, { Component } from 'react';

class Contador extends Component {
  constructor(props) {
    super(props);
    this.state = {
      count: 0,
    };
  }

  handleClick = () => {
    this.setState({ count: this.state.count + 1 });
  };

  render() {
    return (
      <div>
        <p>Você clicou {this.state.count} vezes</p>
        <button onClick={this.handleClick}>
          Clique aqui
        </button>
      </div>
    );
  }
}

export default Contador;

🤓 Explicando o código 📝

As principais diferenças na conversão: (1) useState(0) vira this.state = { count: 0 } no constructor. (2) setCount vira this.setState. (3) A variável count vira this.state.count no JSX. (4) O event handler é um método da classe (handleClick) em vez de uma função inline.

8. Dever de Casa

Criar um Cronômetro com Componente de Classe

Crie uma branch chamada dever-de-casa-aula13 e nela desenvolva:

  1. Crie um componente de classe Cronometro que exiba minutos, segundos e milissegundos.
  2. Use componentDidMount para iniciar um setInterval que atualiza o tempo.
  3. Use componentDidUpdate para exibir no console o tempo atualizado a cada tick.
  4. Use componentWillUnmount para limpar o intervalo quando o componente for desmontado.
  5. Adicione botões de Iniciar, Pausar e Resetar.

Obs: Não se esqueça de subir o projeto para o Git na branch do dever após terminar.

Gabarito:

Cronometro.jsx:


import React, { Component } from 'react';

class Cronometro extends Component {
  constructor(props) {
    super(props);
    this.state = {
      milisegundos: 0,
      ativo: false,
    };
  }

  componentDidMount() {
    console.log('Cronômetro montado!');
  }

  componentDidUpdate(prevProps, prevState) {
    if (prevState.milisegundos !== this.state.milisegundos) {
      console.log('Tempo:', this.formatarTempo());
    }
  }

  componentWillUnmount() {
    clearInterval(this.intervalo);
    console.log('Cronômetro desmontado!');
  }

  iniciar = () => {
    if (!this.state.ativo) {
      this.setState({ ativo: true });
      this.intervalo = setInterval(() => {
        this.setState({ milisegundos: this.state.milisegundos + 10 });
      }, 10);
    }
  };

  pausar = () => {
    this.setState({ ativo: false });
    clearInterval(this.intervalo);
  };

  resetar = () => {
    this.pausar();
    this.setState({ milisegundos: 0 });
  };

  formatarTempo() {
    const ms = this.state.milisegundos;
    const minutos = Math.floor(ms / 60000);
    const segundos = Math.floor((ms % 60000) / 1000);
    const centesimos = Math.floor((ms % 1000) / 10);
    return `${String(minutos).padStart(2, '0')}:${String(segundos).padStart(2, '0')}:${String(centesimos).padStart(2, '0')}`;
  }

  render() {
    return (
      <div style={{ textAlign: 'center', padding: '20px' }}>
        <h1>{this.formatarTempo()}</h1>
        <button onClick={this.iniciar}>Iniciar</button>
        <button onClick={this.pausar}>Pausar</button>
        <button onClick={this.resetar}>Resetar</button>
      </div>
    );
  }
}

export default Cronometro;

🤓 Explicando o código 📝

O Cronometro usa componentDidMount apenas para logging. O timer é iniciado no método iniciar com setInterval a cada 10ms, incrementando milisegundos. componentDidUpdate loga o tempo formatado sempre que os milissegundos mudam. componentWillUnmount garante que o intervalo seja limpo. A formatação converte milissegundos em MM:SS:CC.

App.jsx:


import Cronometro from './Cronometro';

function App() {
  return (
    <div>
      <h1>Cronômetro</h1>
      <Cronometro />
    </div>
  );
}

export default App;
