Testando Aplicações React

Tópicos do nosso plano de curso abordados nessa aula: Por que testar, Tipos de testes, Jest, React Testing Library, Exemplos práticos.

1. Por que testar aplicações React?

Testes são uma parte fundamental do desenvolvimento de software moderno. Eles garantem que o código funciona conforme o esperado, especialmente em aplicações React que evoluem rapidamente com novos componentes e funcionalidades.

  • Garantia de funcionamento: Testes verificam que cada componente e função se comporta corretamente em diferentes cenários.
  • Previnem regressões: Quando você altera ou adiciona código, os testes existentes avisam se algo que funcionava antes quebrou.
  • Facilitam refatoração e manutenção: Com testes, você pode reestruturar o código com confiança, sabendo que os testes validarão o comportamento.
  • Documentam o comportamento esperado: Testes funcionam como uma documentação viva que descreve o que cada parte da aplicação deve fazer.

2. Tipos de testes

Existem diferentes níveis de testes, cada um com um objetivo específico. Em aplicações React, os mais comuns são:

  • Testes unitários: Testam componentes, funções ou hooks de forma isolada, sem depender de outras partes do sistema. São rápidos e fáceis de escrever.
  • Testes de integração: Testam a interação entre dois ou mais componentes, verificando se eles se comunicam corretamente (por exemplo, um formulário que atualiza uma lista).
  • Testes end-to-end (E2E): Testam o fluxo completo da aplicação no navegador, simulando ações do usuário como cliques, digitação e navegação. Ferramentas como Cypress e Playwright são usadas para esse tipo de teste.

Nesta aula, focaremos nos testes unitários e de integração, que são a base para qualquer aplicação React.

3. Jest — O framework de testes padrão do React

Jest é um framework de testes criado pelo Facebook, amplamente utilizado no ecossistema React. Ele já vem configurado com o Create React App e pode ser facilmente adicionado ao Vite com plugins como vite-plugin-testing-library e jest.

Principais características do Jest:

  • Funções globais: Não é necessário importar funções como describe, test / it, expect — elas estão disponíveis globalmente.
  • Matchers: O Jest oferece diversos matchers para verificar valores:
    • toBe(valor) — comparação exata (usando ===)
    • toEqual(valor) — comparação profunda de objetos e arrays
    • toBeInTheDocument() — verifica se um elemento está no DOM (via React Testing Library)
    • toHaveBeenCalled() — verifica se uma função mock foi chamada
  • Mocks: O Jest permite criar funções falsas com jest.fn() para simular comportamentos e verificar chamadas.

4. React Testing Library — Biblioteca para testar componentes React

Enquanto o Jest é o executor de testes, a React Testing Library (RTL) é a biblioteca que nos ajuda a renderizar componentes e interagir com eles em ambiente simulado.

O grande diferencial da RTL é o foco em testar o comportamento do usuário, não os detalhes de implementação. Isso significa que você testa o que o usuário vê e com o que ele interage, e não métodos internos ou estado do componente.

Principais funções da React Testing Library:

  • render(componente) — renderiza o componente em um DOM virtual
  • screen.getByText("texto") — busca um elemento pelo texto visível
  • screen.getByRole("botao") — busca por papel (role) semântico, como button
  • screen.getByPlaceholderText("texto") — busca um input pelo placeholder
  • screen.getByTestId("id") — busca por data-testid (último recurso)
  • fireEvent — dispara eventos como clique, digitação, submit (versão mais simples)
  • userEvent — simula eventos de forma mais realista (requer instalação separada)

5. Exemplo prático 1: Testando um componente simples (Contador)

Vamos começar com um componente Contador que possui dois botões para incrementar e decrementar um valor.

Componente Contador (Contador.jsx)


import React, { useState } from 'react';

const Contador = () => {
  const [contador, setContador] = useState(0);

  return (
    <div>
      <h2>Contador: {contador}</h2>
      <button onClick={() => setContador(contador + 1)}>Incrementar</button>
      <button onClick={() => setContador(contador - 1)}>Decrementar</button>
    </div>
  );
};

export default Contador;

🤓 Explicando o código 📝

O componente Contador usa o hook useState para gerenciar o estado contador, iniciado em 0. O JSX exibe o valor atual em um <h2> e dois botões que, ao serem clicados, chamam setContador para incrementar ou decrementar o valor.

Teste do Contador (Contador.test.jsx)


import React from 'react';
import { render, screen, fireEvent } from '@testing-library/react';
import Contador from './Contador';

test('deve incrementar e decrementar o contador', () => {
  render(<Contador />);
  
  // Verifica valor inicial
  expect(screen.getByText('Contador: 0')).toBeInTheDocument();
  
  // Clica no botão de incrementar
  fireEvent.click(screen.getByText('Incrementar'));
  expect(screen.getByText('Contador: 1')).toBeInTheDocument();
  
  // Clica no botão de decrementar
  fireEvent.click(screen.getByText('Decrementar'));
  expect(screen.getByText('Contador: 0')).toBeInTheDocument();
});

🤓 Explicando o código 📝

1. render(<Contador />) — renderiza o componente em um DOM virtual para teste.

2. screen.getByText('Contador: 0') — busca um elemento que contenha exatamente o texto "Contador: 0". O matcher toBeInTheDocument() verifica se ele existe no DOM.

3. fireEvent.click(elemento) — dispara um evento de clique no elemento encontrado. Após clicar em "Incrementar", esperamos que o texto mude para "Contador: 1". Em seguida, clicamos em "Decrementar" e esperamos voltar para "Contador: 0".

6. Exemplo prático 2: Testando um formulário

Agora vamos testar um formulário que coleta o nome do usuário e o envia através de uma função onSubmit.

Componente Formulario (Formulario.jsx)


import React, { useState } from 'react';

const Formulario = ({ onSubmit }) => {
  const [nome, setNome] = useState('');

  const handleSubmit = (e) => {
    e.preventDefault();
    onSubmit(nome);
  };

  return (
    <form onSubmit={handleSubmit}>
      <input
        type="text"
        placeholder="Digite seu nome"
        value={nome}
        onChange={(e) => setNome(e.target.value)}
      />
      <button type="submit">Enviar</button>
    </form>
  );
};

export default Formulario;

🤓 Explicando o código 📝

O componente Formulario recebe uma prop onSubmit, que é uma função chamada quando o formulário é enviado. O estado nome é atualizado a cada digitação no input. Ao submeter (clicar no botão "Enviar" ou pressionar Enter), handleSubmit previne o recarregamento da página e chama onSubmit(nome) com o valor digitado.

Teste do Formulario (Formulario.test.jsx)


import React from 'react';
import { render, screen, fireEvent } from '@testing-library/react';
import Formulario from './Formulario';

test('deve chamar onSubmit com o nome digitado', () => {
  const onSubmit = jest.fn();
  render(<Formulario onSubmit={onSubmit} />);
  
  // Digita no input
  const input = screen.getByPlaceholderText('Digite seu nome');
  fireEvent.change(input, { target: { value: 'Maria' } });
  
  // Clica no botão de enviar
  fireEvent.click(screen.getByText('Enviar'));
  
  // Verifica se onSubmit foi chamado com o valor correto
  expect(onSubmit).toHaveBeenCalledWith('Maria');
});

🤓 Explicando o código 📝

1. jest.fn() — cria uma função mock (falsa) que registra todas as chamadas recebidas, permitindo verificar depois se foi chamada e com quais argumentos.

2. fireEvent.change(input, { target: { value: 'Maria' } }) — simula a digitação no input, alterando seu valor para "Maria".

3. expect(onSubmit).toHaveBeenCalledWith('Maria') — verifica se a função onSubmit foi chamada exatamente com o argumento 'Maria', confirmando que o formulário está funcionando corretamente.

7. Dever de Casa

Criar testes para um componente Lista de Tarefas

Você recebeu o componente ListaTarefas abaixo. Crie testes para verificar seu funcionamento: 


import React, { useState } from 'react';

const ListaTarefas = () => {
  const [tarefas, setTarefas] = useState([]);
  const [input, setInput] = useState('');

  const adicionar = () => {
    if (input.trim() === '') return;
    setTarefas([...tarefas, input]);
    setInput('');
  };

  const remover = (index) => {
    setTarefas(tarefas.filter((_, i) => i !== index));
  };

  return (
    <div>
      <h2>Lista de Tarefas</h2>
      <input
        type="text"
        value={input}
        onChange={(e) => setInput(e.target.value)}
        placeholder="Nova tarefa"
      />
      <button onClick={adicionar}>Adicionar</button>
      <ul>
        {tarefas.map((tarefa, index) => (
          <li key={index}>
            {tarefa}
            <button onClick={() => remover(index)}>Remover</button>
          </li>
        ))}
      </ul>
    </div>
  );
};

export default ListaTarefas;

Tarefa: Escreva os seguintes testes:

  1. Testar que a lista começa vazia (não exibe nenhum <li>)
  2. Testar que é possível adicionar uma tarefa digitando no input e clicando em "Adicionar"
  3. Testar que é possível remover uma tarefa clicando no botão "Remover"
  4. Testar que tarefas vazias (só espaços) não são adicionadas

Dica: Use screen.getByText para buscar elementos pelo texto visível e screen.getByPlaceholderText para o input.

Gabarito:

Arquivo ListaTarefas.test.jsx:


import React from 'react';
import { render, screen, fireEvent } from '@testing-library/react';
import ListaTarefas from './ListaTarefas';

test('deve iniciar com a lista vazia', () => {
  render(<ListaTarefas />);
  
  const lista = screen.queryAllByRole('listitem');
  expect(lista).toHaveLength(0);
});

test('deve adicionar uma tarefa', () => {
  render(<ListaTarefas />);
  
  const input = screen.getByPlaceholderText('Nova tarefa');
  fireEvent.change(input, { target: { value: 'Estudar React' } });
  
  fireEvent.click(screen.getByText('Adicionar'));
  
  expect(screen.getByText('Estudar React')).toBeInTheDocument();
});

test('deve remover uma tarefa', () => {
  render(<ListaTarefas />);
  
  const input = screen.getByPlaceholderText('Nova tarefa');
  fireEvent.change(input, { target: { value: 'Tarefa para remover' } });
  fireEvent.click(screen.getByText('Adicionar'));
  
  expect(screen.getByText('Tarefa para remover')).toBeInTheDocument();
  
  fireEvent.click(screen.getByText('Remover'));
  
  expect(screen.queryByText('Tarefa para remover')).not.toBeInTheDocument();
});

test('não deve adicionar tarefas vazias', () => {
  render(<ListaTarefas />);
  
  const input = screen.getByPlaceholderText('Nova tarefa');
  fireEvent.change(input, { target: { value: '   ' } });
  fireEvent.click(screen.getByText('Adicionar'));
  
  const lista = screen.queryAllByRole('listitem');
  expect(lista).toHaveLength(0);
});

🤓 Explicando o código 📝

Teste 1: Usamos screen.queryAllByRole('listitem') em vez de getByRole porque queryAllByRole retorna um array vazio se nenhum elemento for encontrado (não lança erro), permitindo verificar que a lista tem comprimento 0.

Teste 2 e 3: Simulamos a digitação no input com fireEvent.change, clicamos em "Adicionar" e verificamos se o texto aparece. Para remover, clicamos em "Remover" e usamos queryByText com .not.toBeInTheDocument() para confirmar que o elemento não está mais no DOM.

Teste 4: Verificamos que digitar apenas espaços não adiciona itens à lista (graças à validação input.trim() === '' no componente).
