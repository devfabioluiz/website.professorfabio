React Router

O roteamento no React é gerenciado principalmente pela biblioteca React Router. Ele permite criar interfaces de várias páginas em aplicações Single-Page Application (SPA), onde a navegação é feita dinamicamente sem recarregar a página.

Rode o seguinte comando no seu terminal npm install react-router-dom

Roteamento Básico

          
//crie um arquivo chamado Home.jsx
const Home = () => {
  return <h1>Bem-vindo à Home</h1>
};

export default Home;

🤓 Explicando o código 📝

O componente Home é um componente funcional simples que retorna um JSX com um título. Ele representa a página inicial nos exemplos de roteamento básico e será renderizado quando a URL corresponder à raiz do site.

          
//crie um arquivo chamado About.jsx
const About = () => {
  return <h1>Sobre Nós</h1>
};

export default About;

🤓 Explicando o código 📝

O componente About é um componente funcional que retorna um título "Sobre Nós". Ele será renderizado quando o usuário navegar para a rota /about, demonstrando como diferentes componentes podem ser associados a diferentes URLs no React Router.

          
import { BrowserRouter, Routes, Route } from 'react-router-dom';
import Home from "./components/Home";
import About from "./components/About";

const App = () => {
  return (
    <BrowserRouter>
      <Routes>
        <Route path="/" element={<Home />} />
        <Route path="/about" element={<About />} />
      </Routes>
    </BrowserRouter>
  );
}

export default App;

🤓 Explicando o código 📝

O componente App.jsx é o ponto central do roteamento. O BrowserRouter deve envolver toda a aplicação para que o React Router funcione corretamente. O componente Routes funciona como um switch que verifica cada Route filho e renderiza apenas o primeiro que corresponde à URL atual. Cada Route possui um atributo path (caminho da URL) e element (componente a ser renderizado).

Redirecionamento

          
import { BrowserRouter, Routes, Route } from 'react-router-dom';

const App = () => {
  return (
    <BrowserRouter>
      <Routes>
        <Route path="/" element={<Navigate to="/home" />} />
        <Route path="/" element={<Home />} />
        <Route path="/about" element={<About />} />
      </Routes>
    </BrowserRouter>
  );
}

export default App;

🤓 Explicando o código 📝

O componente Navigate permite redirecionar o usuário de uma rota para outra automaticamente. Neste exemplo, quando a URL for "/", o usuário será imediatamente redirecionado para "/home". Isso é útil para criar URLs amigáveis ou redirecionar usuários não autenticados.

Rotas Dinâmicas

Utiliza parâmetros na URL para acessar dados específicos.

            
//Crie um arquivo chamado User.jsx
const User = () =>  {
  const { id } = useParams();
  return <h1>Usuário ID: {id}</h1>;
}

🤓 Explicando o código 📝

O componente User utiliza o hook useParams para acessar o parâmetro :id definido na rota. O useParams retorna um objeto com todos os parâmetros da URL. A variável id é extraída diretamente e exibida no JSX, permitindo criar páginas dinâmicas que se adaptam ao valor informado na URL.

            
import { BrowserRouter, Routes, Route, useParams } from 'react-router-dom';

const App = () => => {
  return (
    <BrowserRouter>
      <Routes>
        <Route path="/user/:id" element={<User />} />
      </Routes>
    </BrowserRouter>
  );
}

🤓 Explicando o código 📝

Neste exemplo, a rota /user/:id utiliza :id como um parâmetro dinâmico da URL. O valor informado após /user/ é capturado pelo hook useParams dentro do componente User. Por exemplo, ao acessar /user/42, a variável id receberá o valor "42", permitindo buscar e exibir dados específicos daquele usuário.

Navegação Programática

Permite alterar rotas dinamicamente usando o hook useNavigate. 

            
import { useNavigate } from 'react-router-dom';

const Home = () => {
  const navigate = useNavigate();
  return <button onClick={() => navigate('/about')}>Ir para Sobre</button>
}

export default Home;

🤓 Explicando o código 📝

O hook useNavigate retorna uma função navigate que permite redirecionar o usuário programaticamente, ou seja, através de código JavaScript (como após o clique de um botão ou conclusão de uma ação). Isso é diferente do componente Link, que exige um clique em um elemento âncora. A navegação programática é ideal para fluxos como login, cadastro ou submissão de formulários.

Rotas Aninhadas

Organiza rotas hierarquicamente com o componente Outlet.

Inclui um onde serão renderizados os componentes das rotas filhas (Profile ou Settings). 

    
import { Outlet } from 'react-router-dom';

// Crie um arquivo chamado Dashboard.jsx
const Dashboard = () => {
  return (
    <div>
      <h1>Dashboard </h1>
      <Outlet />
    </div>
  );
}

export default Dashboard;

🤓 Explicando o código 📝

O componente Dashboard utiliza o componente Outlet como um placeholder para renderizar os componentes das rotas filhas. O Outlet é substituído pelo componente filho ativo no momento. Isso permite criar layouts compartilhados onde a estrutura principal permanece visível enquanto o conteúdo muda dinamicamente.

    
// Crie um arquivo chamado Profile.jsx
const Profile = () => {
  return  <h1>Perfil </h1>;
}

export default Profile;

🤓 Explicando o código 📝

O componente Profile é um componente funcional que exibe um título "Perfil". Ele será renderizado dentro do componente Dashboard através do Outlet quando a rota filha profile for acessada.

    

// Crie um arquivo chamado Settings.jsx
const Settings = () => {
  return <h1>Configurações</h1>
}

export default Settings;

🤓 Explicando o código 📝

O componente Settings é um componente funcional simples que retorna um título "Configurações". Ele será renderizado como rota filha dentro do Dashboard, aparecendo no lugar do Outlet quando o usuário navegar para /dashboard/settings.

    
import { BrowserRouter, Routes, Route } from 'react-router-dom';
import Dashboard from "./components/Dashboard";
import Profile from "./components/Profile";
import Settings from "./components/Settings";

const App = () => {
  return (
    <BrowserRouter>
      <Routes>
        <Route path="/dashboard" element={ <Dashboard />}>
          <Route path="profile" element={ <Profile />} />
          <Route path="settings" element={ <Settings />} />
        </Route>
      </Routes>
    </BrowserRouter>
  );
}

export default App;

🤓 Explicando o código 📝

A configuração de rotas aninhadas é feita colocando rotas Route como filhas de outra Route. Neste exemplo, Dashboard é o layout principal (com o Outlet), e Profile e Settings são rotas filhas que serão renderizadas dentro de Dashboard. As URLs resultantes são /dashboard, /dashboard/profile e /dashboard/settings.

Logo, temos como definidas as rotas:

  • /dashboard: Renderiza o componente Dashboard.
  • /dashboard/profile: Renderiza Profile dentro de Dashboard (graças ao <Outlet />).
  • /dashboard/settings: Renderiza Settings dentro de Dashboard.

Rotas Protegidas

Controle de acesso baseado em autenticação

            
import { useNavigate } from "react-router-dom";

//crie um arquivo chamado Login.jsx
const Login = () => {
  const navigate = useNavigate();
  return (
    <button onClick={() => navigate("/home")}>Clique para se logar</button>
  );
};

export default Login;


          
            
import { useNavigate } from "react-router-dom";

//crie um arquivo chamado Login.jsx
const Login = () => {
  const navigate = useNavigate();
  return (
    <button onClick={() => navigate("/home")}>Clique para se logar</button>
  );
};
export default Login;

🤓 Explicando o código 📝

O componente Login utiliza o hook useNavigate para redirecionar o usuário após uma ação. No exemplo, ao clicar no botão "Clique para se logar", a função navigate("/home") é chamada, levando o usuário para a página inicial. Esta é uma forma de navegação programática, diferente do Link que exige um clique em um elemento âncora.

            
import { Navigate } from "react-router-dom";

//Crie um arquivo chamado PrivateRoute.jsx
const PrivateRoute = ({ children }) => {
  const isAuthenticated = true; // Simulação de autenticação
  return isAuthenticated ? children : <Navigate to="/login" />
}

🤓 Explicando o código 📝

O componente PrivateRoute atua como um guardião de rotas. Ele recebe um prop children (o componente a ser protegido) e verifica se o usuário está autenticado. Se estiver autenticado (isAuthenticated = true), renderiza o children normalmente. Caso contrário, redireciona para a página de login usando Navigate. Esse padrão é amplamente utilizado em sistemas com autenticação.

            
import { BrowserRouter, Routes, Route, Navigate } from 'react-router-dom';

const App = () => {
  return (
    <BrowserRouter>
      <Routes>
        <Route
          path="/home"
          element={
            <PrivateRoute>
              <Home />
            </PrivateRoute>
          }
        />
        <Route path="/login" element={<Login />} />
      </Routes>
    </BrowserRouter>
  );
}

🤓 Explicando o código 📝

O App.jsx configura as rotas protegidas envolvendo os componentes que exigem autenticação com o componente PrivateRoute. Se o usuário não estiver autenticado, ele é redirecionado para /login. A rota /login não possui proteção, permitindo que qualquer usuário acesse a página de login.

Agora faça um teste de colocar isAuthenticated como false e veja o resultado

Passo 3 — Configurar o Router no App.jsx


import { BrowserRouter, Routes, Route } from 'react-router-dom';
import Home from './Home';
import About from './About';
import Contact from './Contact';

const App = () => {
  return (
    <BrowserRouter>
      <Routes>
        <Route path="/" element={<Home />} />
        <Route path="/about" element={<About />} />
        <Route path="/contact" element={<Contact />} />
      </Routes>
    </BrowserRouter>
  );
};

export default App;

🤓 Explicando o código 📝

O App.jsx configura o roteamento básico da aplicação. BrowserRouter é o componente que encapsula toda a aplicação e gerencia o histórico de navegação. Routes define um conjunto de rotas que serão avaliadas da mais específica para a mais genérica. Cada Route associa um path (caminho da URL) a um element (componente React). Quando a URL corresponde ao path, o componente especificado é renderizado.

Passo 4 — Adicionar navegação com Links

Crie um componente Navbar.jsx:


import { Link } from 'react-router-dom';

const Navbar = () => {
  return (
    <nav>
      <Link to="/">Home</Link>
      <Link to="/about">Sobre</Link>
      <Link to="/contact">Contato</Link>
    </nav>
  );
};

export default Navbar;

🤓 Explicando o código 📝

O componente Navbar utiliza o componente Link do React Router para criar links de navegação. Diferente de uma tag <a> tradicional, o Link não recarrega a página — ele apenas atualiza a URL e renderiza o componente correspondente no Routes. Cada Link possui o atributo to que define o caminho da rota: "/" para Home, "/about" para Sobre e "/contact" para Contato.

Atualize o App.jsx para incluir a Navbar:


import { BrowserRouter, Routes, Route } from 'react-router-dom';
import Home from './Home';
import About from './About';
import Contact from './Contact';
import Navbar from './Navbar';

const App = () => {
  return (
    <BrowserRouter>
      <Navbar />
      <Routes>
        <Route path="/" element={<Home />} />
        <Route path="/about" element={<About />} />
        <Route path="/contact" element={<Contact />} />
      </Routes>
    </BrowserRouter>
  );
};

export default App;

🤓 Explicando o código 📝

O componente App.jsx agora inclui o componente Navbar acima do sistema de rotas. Isso significa que a barra de navegação será exibida em todas as páginas da aplicação, pois ela está fora do componente Routes. O BrowserRouter continua encapsulando tudo, permitindo que tanto a Navbar quanto as rotas tenham acesso ao contexto de roteamento.

Dever de Casa

Criar um blog com rotas dinâmicas

Crie uma branch chamada dever-de-casa-aula9 e nela desenvolva:

  1. Crie um componente Post.jsx que use useParams para exibir o título e conteúdo de um post baseado no ID da URL.
  2. Crie um componente Blog.jsx que liste links para posts (ex: /post/1, /post/2, /post/3).
  3. Configure as rotas no App.jsx.

Obs: Não se esqueça de subir o projeto para o Git na branch do dever após terminar.

Gabarito:

1. Blog.jsx:


import { Link } from 'react-router-dom';

const posts = [
  { id: 1, titulo: 'React é incrível' },
  { id: 2, titulo: 'Aprendendo React Router' },
  { id: 3, titulo: 'Hooks na prática' },
];

const Blog = () => {
  return (
    <div>
      <h1>Blog</h1>
      <ul>
        {posts.map(post => (
          <li key={post.id}>
            <Link to={`/post/${post.id}`}>{post.titulo}</Link>
          </li>
        ))}
      </ul>
    </div>
  );
};

export default Blog;

🤓 Explicando o código 📝

O componente Blog utiliza o componente Link do React Router para criar navegação entre páginas sem recarregar o navegador. A lista de posts é percorrida com map, gerando um item de lista <li> para cada post. Cada Link aponta para uma rota dinâmica /post/{id}, permitindo que o usuário clique e seja direcionado para a página detalhada de cada post.

2. Post.jsx:


import { useParams } from 'react-router-dom';

const posts = [
  { id: 1, titulo: 'React é incrível', conteudo: 'React é uma biblioteca para construir interfaces...' },
  { id: 2, titulo: 'Aprendendo React Router', conteudo: 'React Router permite navegação SPA...' },
  { id: 3, titulo: 'Hooks na prática', conteudo: 'Hooks como useState e useEffect são essenciais...' },
];

const Post = () => {
  const { id } = useParams();
  const post = posts.find(p => p.id === Number(id));

  if (!post) return <h2>Post não encontrado</h2>;

  return (
    <div>
      <h1>{post.titulo}</h1>
      <p>{post.conteudo}</p>
    </div>
  );
};

export default Post;

🤓 Explicando o código 📝

O componente Post utiliza o hook useParams para capturar o parâmetro :id da URL. Internamente, ele mantém uma lista de posts com id, titulo e conteudo. A função find localiza o post cujo id corresponde ao parâmetro da URL (convertido para número). Caso o post não exista, exibe uma mensagem de erro; caso contrário, renderiza o título e o conteúdo do post encontrado.

3. App.jsx — rotas atualizadas:


import { BrowserRouter, Routes, Route } from 'react-router-dom';
import Blog from './Blog';
import Post from './Post';

const App = () => {
  return (
    <BrowserRouter>
      <Routes>
        <Route path="/" element={<Blog />} />
        <Route path="/post/:id" element={<Post />} />
      </Routes>
    </BrowserRouter>
  );
};

export default App;

🤓 Explicando o código 📝

O componente App.jsx configura o roteador principal da aplicação. O BrowserRouter envolve toda a árvore de componentes e gerencia o histórico de navegação. Dentro dele, o componente Routes define um sistema de roteamento exclusivo: a primeira rota path="/" renderiza o Blog como página inicial, e a segunda rota path="/post/:id" usa um parâmetro dinâmico :id para identificar qual post específico será exibido através do componente Post.
