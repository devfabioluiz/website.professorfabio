O que é PL/SQL?

PL/SQL (Procedural Language/Structured Query Language) é a linguagem de programação procedural proprietária da Oracle Corporation. Ela estende a linguagem SQL padrão com construções típicas de linguagens de programação como variáveis, condicionais, loops, exceções, procedimentos, funções, pacotes e triggers.

Enquanto o SQL puro é uma linguagem declarativa (você diz o que quer, não como obter), o PL/SQL adiciona a capacidade de escrever lógica procedural diretamente no banco de dados. Isso significa que você pode criar programas completos que executam dentro do servidor Oracle, reduzindo tráfego de rede e centralizando a lógica de negócio.

Características Fundamentais do PL/SQL

  • Integração total com SQL: comandos SQL podem ser usados diretamente dentro de blocos PL/SQL, incluindo SELECT, INSERT, UPDATE, DELETE, MERGE
  • Execução no servidor: todo o código PL/SQL é executado dentro do Oracle Database, eliminando round-trips entre aplicação e banco
  • Desempenho: o Oracle compila e armazena o bytecode na SGA (System Global Area) para execução otimizada
  • Portabilidade: código PL/SQL é portável entre diferentes sistemas operacionais onde o Oracle roda
  • Tipagem forte: variáveis e parâmetros são fortemente tipados, reduzindo erros em tempo de execução
  • Programação modular: suporta procedures, functions, packages e triggers para organização do código

Diferença entre SQL e PL/SQL

SQL PL/SQL
Linguagem declarativa (diz O QUE fazer) Linguagem procedural (diz COMO fazer)
Executa um comando por vez Executa blocos inteiros de código
Não possui variáveis Possui variáveis, constantes e tipos
Não possui controle de fluxo Possui IF, CASE, LOOP, WHILE, FOR
Não possui tratamento de erros Possui sistema completo de exceções
Retorna conjuntos de dados Pode processar linha a linha com cursores

Estrutura do Bloco PL/SQL

Todo programa PL/SQL é composto por blocos. A estrutura fundamental de um bloco é: 


DECLARE
  -- Seção de declarações (opcional)
  -- Variáveis, constantes, cursores, exceções
BEGIN
  -- Seção executável (obrigatória)
  -- Código procedural e comandos SQL
EXCEPTION
  -- Seção de tratamento de exceções (opcional)
  -- Captura e tratamento de erros
END;
/
Seção Obrigatória? Propósito
DECLARE Não Declara variáveis, constantes, cursores, tipos e exceções
BEGIN Sim Contém o código a ser executado (comandos SQL, lógica procedural)
EXCEPTION Não Captura e trata erros que ocorrem na seção BEGIN
END; Sim Finaliza o bloco. O / é necessário para executar em SQL*Plus/SQL Developer
Exemplo — Primeiro Bloco PL/SQL:

DECLARE
  v_nome VARCHAR2(100);
BEGIN
  SELECT nome INTO v_nome
  FROM funcionarios
  WHERE id = 1;

  DBMS_OUTPUT.PUT_LINE('Nome do funcionário: ' || v_nome);
EXCEPTION
  WHEN NO_DATA_FOUND THEN
    DBMS_OUTPUT.PUT_LINE('Funcionário não encontrado');
END;
/

Antes de executar blocos PL/SQL, habilite a saída do DBMS_OUTPUT no SQL*Plus ou SQL Developer: 


-- No SQL*Plus ou SQL Developer:
SET SERVEROUTPUT ON;

Variáveis e Constantes

Declaração de Variáveis

As variáveis em PL/SQL devem ser declaradas na seção DECLARE antes de serem usadas: 


DECLARE
  v_nome         VARCHAR2(100);        -- String de até 100 caracteres
  v_idade        NUMBER(3);            -- Número de até 3 dígitos
  v_salario      NUMBER(10,2);         -- Número com 2 casas decimais
  v_ativo        BOOLEAN;              -- TRUE, FALSE ou NULL
  v_data_nasc    DATE;                 -- Data
  v_data_hora    TIMESTAMP;            -- Data e hora com fração de segundos
  v_observacao   CLOB;                 -- Texto longo (Character Large Object)
BEGIN
  v_nome := 'Maria Silva';
  v_idade := 28;
  v_salario := 5500.00;
  v_ativo := TRUE;
  v_data_nasc := DATE '1998-03-15';
  v_data_hora := SYSTIMESTAMP;

  DBMS_OUTPUT.PUT_LINE('Nome: ' || v_nome);
  DBMS_OUTPUT.PUT_LINE('Salário: R$ ' || TO_CHAR(v_salario, '999G990D00'));
END;
/

Constantes

Use a palavra-chave CONSTANT para declarar valores imutáveis:


DECLARE
  c_limite_credito CONSTANT NUMBER(10,2) := 10000.00;
  c_nome_empresa   CONSTANT VARCHAR2(50) := 'Oracle Corporation';
  v_compra         NUMBER(10,2) := 7500.00;
BEGIN
  IF v_compra > c_limite_credito THEN
    DBMS_OUTPUT.PUT_LINE('Compra excede o limite de crédito!');
  ELSE
    DBMS_OUTPUT.PUT_LINE('Compra aprovada.');
  END IF;
END;
/

Atributo %TYPE — Ancoragem de Tipo

O atributo %TYPE permite declarar uma variável com o mesmo tipo de dados de uma coluna de tabela ou de outra variável. Isso torna o código mais robusto: se a definição da coluna mudar, a variável acompanha automaticamente. 


DECLARE
  v_nome    funcionarios.nome%TYPE;      -- Mesmo tipo da coluna nome
  v_salario funcionarios.salario%TYPE;    -- Mesmo tipo da coluna salario
  v_id      funcionarios.id%TYPE := 5;    -- Com valor inicial
  v_salario_novo v_salario%TYPE;          -- Mesmo tipo de outra variável
BEGIN
  SELECT nome, salario INTO v_nome, v_salario
  FROM funcionarios
  WHERE id = v_id;

  DBMS_OUTPUT.PUT_LINE(v_nome || ' - Salário atual: R$ ' || v_salario);
END;
/

Escopo e Atribuição com :=


DECLARE
  v_global NUMBER := 100;
BEGIN
  DBMS_OUTPUT.PUT_LINE('Valor externo: ' || v_global);

  DECLARE
    v_global NUMBER := 50;  -- Esta variável SOMBREIA a externa
  BEGIN
    DBMS_OUTPUT.PUT_LINE('Valor interno: ' || v_global);  -- Mostra 50
  END;

  DBMS_OUTPUT.PUT_LINE('Valor externo novamente: ' || v_global);  -- Mostra 100
END;
/

Estruturas Condicionais

IF / ELSIF / ELSE

O PL/SQL suporta condicionais encadeadas com a sintaxe clássica:


DECLARE
  v_nota NUMBER(3) := 85;
  v_conceito VARCHAR2(20);
BEGIN
  IF v_nota >= 90 THEN
    v_conceito := 'A - Excelente';
  ELSIF v_nota >= 80 THEN
    v_conceito := 'B - Bom';
  ELSIF v_nota >= 70 THEN
    v_conceito := 'C - Regular';
  ELSIF v_nota >= 60 THEN
    v_conceito := 'D - Insuficiente';
  ELSE
    v_conceito := 'F - Reprovado';
  END IF;

  DBMS_OUTPUT.PUT_LINE('Nota: ' || v_nota || ' - Conceito: ' || v_conceito);
END;
/
Exemplo com validação de negócio:

DECLARE
  v_salario    NUMBER(10,2) := 3500.00;
  v_bonus      NUMBER(10,2);
  v_departamento VARCHAR2(30) := 'TI';
BEGIN
  IF v_salario > 10000 THEN
    v_bonus := v_salario * 0.05;  -- 5%
  ELSIF v_salario > 5000 AND v_departamento = 'TI' THEN
    v_bonus := v_salario * 0.10;  -- 10% para TI com salário médio
  ELSIF v_salario > 5000 THEN
    v_bonus := v_salario * 0.07;  -- 7%
  ELSE
    v_bonus := v_salario * 0.12;  -- 12% para salários baixos
  END IF;

  DBMS_OUTPUT.PUT_LINE('Bônus calculado: R$ ' || TO_CHAR(v_bonus, '999G990D00'));
END;
/

CASE Expression (com retorno)

A expressão CASE retorna um valor e pode ser usada dentro de uma atribuição:


DECLARE
  v_dia_semana VARCHAR2(20) := 'Monday';
  v_traducao   VARCHAR2(20);
BEGIN
  v_traducao := CASE v_dia_semana
    WHEN 'Monday'    THEN 'Segunda-feira'
    WHEN 'Tuesday'   THEN 'Terça-feira'
    WHEN 'Wednesday' THEN 'Quarta-feira'
    WHEN 'Thursday'  THEN 'Quinta-feira'
    WHEN 'Friday'    THEN 'Sexta-feira'
    ELSE 'Fim de semana'
  END;

  DBMS_OUTPUT.PUT_LINE(v_dia_semana || ' = ' || v_traducao);
END;
/

CASE Statement (com ações)

O comando CASE permite executar blocos de código dentro de cada condição:


DECLARE
  v_opcao NUMBER := 2;
  v_resultado VARCHAR2(100);
BEGIN
  CASE v_opcao
    WHEN 1 THEN
      v_resultado := 'Cadastrar novo usuário';
      DBMS_OUTPUT.PUT_LINE('Executando rotina de cadastro...');
    WHEN 2 THEN
      v_resultado := 'Alterar dados do usuário';
      DBMS_OUTPUT.PUT_LINE('Abrindo formulário de edição...');
    WHEN 3 THEN
      v_resultado := 'Excluir usuário';
      DBMS_OUTPUT.PUT_LINE('Solicitando confirmação de exclusão...');
    ELSE
      v_resultado := 'Opção inválida';
      DBMS_OUTPUT.PUT_LINE('Nenhuma ação correspondente.');
  END CASE;

  DBMS_OUTPUT.PUT_LINE('Ação selecionada: ' || v_resultado);
END;
/
Exemplo — CASE com condição de busca (Searched CASE):

DECLARE
  v_salario NUMBER(10,2) := 8200.00;
  v_faixa   VARCHAR2(20);
BEGIN
  CASE
    WHEN v_salario > 20000 THEN v_faixa := 'Executivo';
    WHEN v_salario > 10000 THEN v_faixa := 'Sênior';
    WHEN v_salario > 5000  THEN v_faixa := 'Pleno';
    WHEN v_salario > 2000  THEN v_faixa := 'Júnior';
    ELSE v_faixa := 'Estagiário';
  END CASE;

  DBMS_OUTPUT.PUT_LINE('Salário: R$ ' || v_salario || ' — Faixa: ' || v_faixa);
END;
/

Estruturas de Repetição (Loops)

LOOP Básico (com EXIT WHEN)

O LOOP simples repete infinitamente até encontrar um EXIT ou EXIT WHEN: 


DECLARE
  v_contador NUMBER := 1;
BEGIN
  LOOP
    DBMS_OUTPUT.PUT_LINE('Iteração: ' || v_contador);
    v_contador := v_contador + 1;
    EXIT WHEN v_contador > 5;
  END LOOP;

  DBMS_OUTPUT.PUT_LINE('Loop concluído. Contador final: ' || v_contador);
END;
/

WHILE LOOP

A condição é verificada antes de cada iteração. O loop pode executar zero vezes:


DECLARE
  v_numero NUMBER := 10;
  v_soma   NUMBER := 0;
BEGIN
  WHILE v_numero > 0 LOOP
    v_soma := v_soma + v_numero;
    DBMS_OUTPUT.PUT_LINE('Número: ' || v_numero || ' — Soma parcial: ' || v_soma);
    v_numero := v_numero - 2;
  END LOOP;

  DBMS_OUTPUT.PUT_LINE('Soma total: ' || v_soma);
END;
/

FOR LOOP (Numérico)

O FOR LOOP itera automaticamente sobre um intervalo. A variável de controle é declarada implicitamente e é somente leitura dentro do loop: 


DECLARE
  v_fatorial NUMBER := 1;
BEGIN
  FOR i IN 1..10 LOOP
    v_fatorial := v_fatorial * i;
    DBMS_OUTPUT.PUT_LINE(i || '! = ' || v_fatorial);
  END LOOP;
END;
/
FOR LOOP com REVERSE:

BEGIN
  FOR i IN REVERSE 10..1 LOOP
    DBMS_OUTPUT.PUT_LINE('Contagem regressiva: ' || i);
  END LOOP;

  DBMS_OUTPUT.PUT_LINE('Lançamento!');
END;
/
Exemplo prático — Loop com inserção em tabela:

CREATE TABLE log_execucao (
  id        NUMBER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  mensagem   VARCHAR2(200),
  data_hora  TIMESTAMP DEFAULT SYSTIMESTAMP
);

DECLARE
  v_inicio TIMESTAMP := SYSTIMESTAMP;
BEGIN
  FOR i IN 1..5 LOOP
    INSERT INTO log_execucao (mensagem)
    VALUES ('Registro ' || i || ' inserido às ' || TO_CHAR(SYSTIMESTAMP, 'HH24:MI:SS'));

    DBMS_OUTPUT.PUT_LINE('Inserido registro ' || i);
  END LOOP;

  COMMIT;
  DBMS_OUTPUT.PUT_LINE('Processo concluído. Duração: ' ||
    TO_CHAR(EXTRACT(SECOND FROM (SYSTIMESTAMP - v_inicio)), '90D00') || ' segundos');
END;
/

Nested Loops e Labels


DECLARE
  v_resultado VARCHAR2(200);
BEGIN
  <>
  FOR i IN 1..3 LOOP
    <>
    FOR j IN 1..3 LOOP
      IF i = 2 AND j = 2 THEN
        EXIT laco_externo;  -- Sai de ambos os loops
      END IF;
      DBMS_OUTPUT.PUT_LINE('i=' || i || ', j=' || j);
    END LOOP laco_interno;
  END LOOP laco_externo;

  DBMS_OUTPUT.PUT_LINE('Saiu dos loops em i=2, j=2');
END;
/

Palavra-chave INTO — SELECT dentro de PL/SQL

Em PL/SQL, um comando SELECT deve obrigatoriamente usar a cláusula INTO para armazenar o resultado em variáveis. Diferente do SQL puro (que retorna um conjunto de resultados), o SELECT em PL/SQL deve retornar exatamente uma linha. Se retornar zero linhas, ocorre a exceção NO_DATA_FOUND; se retornar mais de uma, ocorre TOO_MANY_ROWS.

Sintaxe e Exemplos:

DECLARE
  v_nome   VARCHAR2(100);
  v_salario NUMBER(10,2);
  v_id     NUMBER := 10;
BEGIN
  -- INTO simples
  SELECT nome, salario
  INTO v_nome, v_salario
  FROM funcionarios
  WHERE id = v_id;

  DBMS_OUTPUT.PUT_LINE('Funcionário: ' || v_nome || ' — Salário: ' || v_salario);

  -- INTO com função de agregação
  SELECT COUNT(*), AVG(salario), MAX(salario)
  INTO v_contador, v_media, v_maximo
  FROM funcionarios;

  DBMS_OUTPUT.PUT_LINE('Total: ' || v_contador ||
                       ' | Média: ' || v_media ||
                       ' | Máximo: ' || v_maximo);
END;
/
Comparação: SQL Puro vs PL/SQL com INTO:
SQL Puro PL/SQL com INTO
SELECT nome FROM funcionarios; SELECT nome INTO v_nome FROM funcionarios WHERE id = 1;
Retorna conjunto de resultados (result set) Armazena em variável(veis) PL/SQL
Pode retornar 0, 1 ou N linhas Deve retornar exatamente 1 linha (senão: exception)
Executado diretamente no cliente SQL Dentro de um bloco BEGIN...END
SELECT INTO com registro completo (%ROWTYPE):

DECLARE
  v_func funcionarios%ROWTYPE;
BEGIN
  SELECT * INTO v_func FROM funcionarios WHERE id = 1;

  DBMS_OUTPUT.PUT_LINE('Nome: ' || v_func.nome);
  DBMS_OUTPUT.PUT_LINE('Salário: ' || v_func.salario);
  DBMS_OUTPUT.PUT_LINE('Departamento: ' || v_func.departamento);
END;
/

Tratamento de Exceções

O PL/SQL possui um sistema robusto de tratamento de erros. Exceções são eventos que interrompem o fluxo normal de execução. Elas podem ser predefinidas pela Oracle, definidas pelo usuário ou mapeadas via PRAGMA EXCEPTION_INIT.

Exceções Predefinidas (Predefined Exceptions)

A Oracle fornece várias exceções nomeadas para os erros mais comuns:

Exceção Código Oracle Quando ocorre
NO_DATA_FOUND ORA-01403 SELECT INTO não encontrou nenhuma linha
TOO_MANY_ROWS ORA-01422 SELECT INTO retornou mais de uma linha
ZERO_DIVIDE ORA-01476 Divisão por zero
DUP_VAL_ON_INDEX ORA-00001 Tentativa de inserir valor duplicado em chave única
INVALID_NUMBER ORA-01722 Falha na conversão de string para número
VALUE_ERROR ORA-06502 Erro de truncamento, aritmético ou de conversão
LOGIN_DENIED ORA-01017 Usuário ou senha inválidos ao conectar
PROGRAM_ERROR ORA-06501 Erro interno PL/SQL
STORAGE_ERROR ORA-06500 PL/SQL ficou sem memória
TIMEOUT_ON_RESOURCE ORA-00051 Timeout ao aguardar recurso
CURSOR_ALREADY_OPEN ORA-06511 Tentativa de abrir cursor que já está aberto
INVALID_CURSOR ORA-01001 Operação em cursor que não está aberto
NOT_LOGGED_ON ORA-01012 Tentativa de operar sem estar conectado
Exemplo — Tratamento de múltiplas exceções:

DECLARE
  v_nome     VARCHAR2(100);
  v_salario  NUMBER(10,2);
  v_id       NUMBER := &id_funcionario;  -- Solicita valor ao usuário
BEGIN
  SELECT nome, salario INTO v_nome, v_salario
  FROM funcionarios
  WHERE id = v_id;

  DBMS_OUTPUT.PUT_LINE('Funcionário: ' || v_nome);
  DBMS_OUTPUT.PUT_LINE('Salário: R$ ' || v_salario);

  -- Simular cálculo que pode gerar divisão por zero
  DECLARE
    v_resultado NUMBER;
    v_divisor   NUMBER := 0;
  BEGIN
    v_resultado := 1000 / v_divisor;
    DBMS_OUTPUT.PUT_LINE('Resultado: ' || v_resultado);
  EXCEPTION
    WHEN ZERO_DIVIDE THEN
      DBMS_OUTPUT.PUT_LINE('Erro: Divisão por zero! Verifique o divisor.');
  END;

EXCEPTION
  WHEN NO_DATA_FOUND THEN
    DBMS_OUTPUT.PUT_LINE('Erro: Funcionário com ID ' || v_id || ' não encontrado.');
  WHEN TOO_MANY_ROWS THEN
    DBMS_OUTPUT.PUT_LINE('Erro: Múltiplos funcionários com o mesmo ID (isso não deveria acontecer).');
  WHEN OTHERS THEN
    DBMS_OUTPUT.PUT_LINE('Erro inesperado: ' || SQLERRM);
    DBMS_OUTPUT.PUT_LINE('Código do erro: ' || SQLCODE);
END;
/

Exceções Definidas pelo Usuário

Você pode criar suas próprias exceções para validar regras de negócio. O processo é: declarar a exceção, dispará-la com RAISE e capturá-la no bloco EXCEPTION.

Exemplo — Validação de salário:

DECLARE
  e_salario_invalido EXCEPTION;
  v_salario NUMBER(10,2) := -500.00;
BEGIN
  IF v_salario < 0 THEN
    RAISE e_salario_invalido;
  END IF;

  -- Só chega aqui se salário for >= 0
  DBMS_OUTPUT.PUT_LINE('Salário válido: R$ ' || v_salario);

EXCEPTION
  WHEN e_salario_invalido THEN
    DBMS_OUTPUT.PUT_LINE('Erro de negócio: Salário não pode ser negativo.');
    DBMS_OUTPUT.PUT_LINE('Valor informado: R$ ' || v_salario);
END;
/
Exemplo — RAISE_APPLICATION_ERROR:

Para propagar erros customizados para a aplicação cliente com código e mensagem específicos, use RAISE_APPLICATION_ERROR: 


DECLARE
  v_estoque   NUMBER := 5;
  v_pedido    NUMBER := 10;
BEGIN
  IF v_pedido > v_estoque THEN
    RAISE_APPLICATION_ERROR(
      -20001,
      'Estoque insuficiente. Disponível: ' || v_estoque ||
      ', Solicitado: ' || v_pedido
    );
  END IF;

  DBMS_OUTPUT.PUT_LINE('Pedido processado com sucesso!');

EXCEPTION
  WHEN OTHERS THEN
    DBMS_OUTPUT.PUT_LINE('Erro capturado: ' || SQLERRM);
    ROLLBACK;
END;
/

PRAGMA EXCEPTION_INIT — Mapeando Códigos Oracle para Exceções Nomeadas

Quando você precisa tratar um erro específico da Oracle que não tem uma exceção predefinida, use PRAGMA EXCEPTION_INIT para associar um código de erro Oracle a um nome de exceção que você define: 


DECLARE
  e_tabela_inexistente EXCEPTION;
  PRAGMA EXCEPTION_INIT(e_tabela_inexistente, -942);

  e_senha_expirada EXCEPTION;
  PRAGMA EXCEPTION_INIT(e_senha_expirada, -28001);

  v_contador NUMBER;
BEGIN
  -- Tentar consultar uma tabela que pode não existir
  SELECT COUNT(*) INTO v_contador FROM tabela_temporaria;

  DBMS_OUTPUT.PUT_LINE('Registros encontrados: ' || v_contador);

EXCEPTION
  WHEN e_tabela_inexistente THEN
    DBMS_OUTPUT.PUT_LINE('Erro ORA-00942: Tabela ou view não existe.');
    DBMS_OUTPUT.PUT_LINE('Execute o script de criação de tabelas primeiro.');

  WHEN e_senha_expirada THEN
    DBMS_OUTPUT.PUT_LINE('Erro ORA-28001: Sua senha expirou. Altere-a antes de continuar.');

  WHEN OTHERS THEN
    DBMS_OUTPUT.PUT_LINE('Erro não mapeado: Código ' || SQLCODE || ' — ' || SQLERRM);
END;
/

SQLCODE e SQLERRM

SQLCODE retorna o número do erro Oracle (negativo para erros, 0 para sucesso, +100 para NO_DATA_FOUND). SQLERRM retorna a mensagem de erro associada. Ambos são especialmente úteis no handler WHEN OTHERS: 


DECLARE
  v_codigo  NUMBER;
  v_mensagem VARCHAR2(500);
BEGIN
  -- Forçar um erro
  EXECUTE IMMEDIATE 'SELECT * FROM tabela_que_nao_existe';

EXCEPTION
  WHEN OTHERS THEN
    v_codigo := SQLCODE;
    v_mensagem := SQLERRM;

    DBMS_OUTPUT.PUT_LINE('=== LOG DE ERRO ===');
    DBMS_OUTPUT.PUT_LINE('Código: ' || v_codigo);
    DBMS_OUTPUT.PUT_LINE('Mensagem: ' || v_mensagem);

    INSERT INTO log_erros (codigo_erro, mensagem, data_hora)
    VALUES (v_codigo, v_mensagem, SYSTIMESTAMP);

    COMMIT;

    -- Re-lançar o erro para a aplicação
    RAISE;
END;
/

Cursores em PL/SQL

Um cursor é um ponteiro para a área de contexto privada (Private SQL Area) na memória PGA/SGA que contém informações sobre uma instrução SQL processada. Em PL/SQL, cursores permitem processar múltiplas linhas retornadas por uma consulta, linha por linha.

Tipos de Cursores

Tipo Descrição Uso
Cursor Implícito Criado automaticamente pelo Oracle para comandos SQL (SELECT INTO, INSERT, UPDATE, DELETE) Operações que retornam 1 linha ou comandos DML
Cursor Explícito Declarado e controlado pelo programador Consultas que retornam múltiplas linhas
Cursor FOR LOOP Sintaxe simplificada que gerencia automaticamente OPEN, FETCH e CLOSE Iteração simples sobre resultados de consulta
Cursor com Parâmetros Cursor explícito que aceita parâmetros de entrada Consultas reutilizáveis com filtros dinâmicos
REF CURSOR Ponteiro para cursor que pode ser passado entre programas Retornar resultados para aplicações cliente

Cursor Explícito — Passo a Passo

O ciclo de vida completo de um cursor explícito envolve 4 etapas:

  1. DECLARE: definir o cursor com sua consulta SQL
  2. OPEN: abrir o cursor (executar a consulta e preparar o conjunto de resultados)
  3. FETCH: buscar uma linha por vez para processamento
  4. CLOSE: fechar o cursor e liberar recursos

DECLARE
  -- 1. DECLARE: define o cursor
  CURSOR cur_funcionarios IS
    SELECT id, nome, salario, departamento
    FROM funcionarios
    WHERE status = 'A'
    ORDER BY salario DESC;

  -- Variáveis para armazenar cada linha
  v_id           funcionarios.id%TYPE;
  v_nome         funcionarios.nome%TYPE;
  v_salario      funcionarios.salario%TYPE;
  v_departamento funcionarios.departamento%TYPE;

  v_contador NUMBER := 0;
BEGIN
  -- 2. OPEN: abre o cursor e executa a query
  OPEN cur_funcionarios;

  LOOP
    -- 3. FETCH: busca a próxima linha
    FETCH cur_funcionarios INTO v_id, v_nome, v_salario, v_departamento;

    -- Verifica se ainda há linhas
    EXIT WHEN cur_funcionarios%NOTFOUND;

    v_contador := v_contador + 1;

    DBMS_OUTPUT.PUT_LINE(
      v_contador || '. ' || v_nome ||
      ' | Dept: ' || v_departamento ||
      ' | Salário: R$ ' || TO_CHAR(v_salario, '999G990D00')
    );
  END LOOP;

  -- 4. CLOSE: fecha o cursor
  CLOSE cur_funcionarios;

  DBMS_OUTPUT.PUT_LINE('Total de funcionários ativos: ' || v_contador);
END;
/

Atributos de Cursores

Cursores, tanto explícitos quanto implícitos, possuem atributos que fornecem informações sobre o estado da execução:

Atributo Retorna Quando usar
%FOUND TRUE se o último FETCH retornou uma linha Antes de processar dados do FETCH, para saber se há algo
%NOTFOUND TRUE se o último FETCH NÃO retornou uma linha Condição de saída de loops (EXIT WHEN NOTFOUND)
%ROWCOUNT Número de linhas processadas até o momento Contagem de registros afetados, progresso, logging
%ISOPEN TRUE se o cursor está aberto Verificar antes de tentar FETCH ou CLOSE
Exemplo demonstrando todos os atributos:

DECLARE
  CURSOR cur_produtos IS
    SELECT id, nome, preco
    FROM produtos
    WHERE estoque > 0;

  v_id    produtos.id%TYPE;
  v_nome  produtos.nome%TYPE;
  v_preco produtos.preco%TYPE;
BEGIN
  DBMS_OUTPUT.PUT_LINE('Cursor aberto? ' ||
    CASE WHEN cur_produtos%ISOPEN THEN 'Sim' ELSE 'Não' END);

  OPEN cur_produtos;

  DBMS_OUTPUT.PUT_LINE('Cursor aberto após OPEN? ' ||
    CASE WHEN cur_produtos%ISOPEN THEN 'Sim' ELSE 'Não' END);

  LOOP
    FETCH cur_produtos INTO v_id, v_nome, v_preco;
    EXIT WHEN cur_produtos%NOTFOUND;

    DBMS_OUTPUT.PUT_LINE(
      'Linha ' || cur_produtos%ROWCOUNT || ': ' ||
      v_nome || ' — R$ ' || v_preco
    );
  END LOOP;

  DBMS_OUTPUT.PUT_LINE('Total de produtos processados: ' || cur_produtos%ROWCOUNT);

  CLOSE cur_produtos;
  DBMS_OUTPUT.PUT_LINE('Cursor fechado.');
END;
/

Cursor FOR LOOP — Sintaxe Simplificada

O Cursor FOR LOOP é a forma mais prática e segura de iterar sobre cursores. O Oracle gerencia automaticamente OPEN, FETCH, CLOSE e a variável de registro é declarada implicitamente: 


DECLARE
  v_total NUMBER := 0;
BEGIN
  FOR reg IN (
    SELECT id, nome, salario, departamento
    FROM funcionarios
    WHERE status = 'A'
    ORDER BY salario DESC
  ) LOOP
    v_total := v_total + reg.salario;

    DBMS_OUTPUT.PUT_LINE(
      reg.nome || ' (' || reg.departamento || ') — R$ ' || reg.salario
    );
  END LOOP;

  DBMS_OUTPUT.PUT_LINE('Folha total: R$ ' || TO_CHAR(v_total, '999G999G990D00'));
END;
/
Cursor FOR LOOP com cursor nomeado:

DECLARE
  CURSOR cur_vendas_mes IS
    SELECT produto, SUM(valor) AS total_vendas
    FROM vendas
    WHERE EXTRACT(MONTH FROM data_venda) = EXTRACT(MONTH FROM SYSDATE)

  v_ranking NUMBER := 0;

    EXIT WHEN v_ranking >= 5;  -- Mostrar apenas top 5
  END LOOP;

%ROWTYPE e %TYPE — Ancoragem de Tipos

%ROWTYPE — Registro baseado em Tabela ou Cursor

%ROWTYPE declara uma variável do tipo registro que espelha exatamente a estrutura de uma linha de tabela, view ou cursor. Isso é extremamente útil para SELECT * INTO ou para iterar sobre cursores sem declarar múltiplas variáveis individuais.

%ROWTYPE baseado em Tabela:

DECLARE
  -- Declara uma variável com a mesma estrutura da tabela funcionarios
  v_func   funcionarios%ROWTYPE;
  v_func2  funcionarios%ROWTYPE;
BEGIN
  -- Preencher com SELECT *
  SELECT * INTO v_func
  FROM funcionarios
  WHERE id = 1;

  DBMS_OUTPUT.PUT_LINE('=== Funcionário 1 ===');
  DBMS_OUTPUT.PUT_LINE('ID: ' || v_func.id);
  DBMS_OUTPUT.PUT_LINE('Nome: ' || v_func.nome);
  DBMS_OUTPUT.PUT_LINE('Salário: R$ ' || v_func.salario);
  DBMS_OUTPUT.PUT_LINE('Departamento: ' || v_func.departamento);
  DBMS_OUTPUT.PUT_LINE('Data admissão: ' || TO_CHAR(v_func.data_admissao, 'DD/MM/YYYY'));

  -- Copiar um registro para outro (atribuição completa)
  v_func2 := v_func;
  v_func2.salario := v_func2.salario * 1.15;  -- Aumento de 15%

  DBMS_OUTPUT.PUT_LINE('=== Após aumento ===');
  DBMS_OUTPUT.PUT_LINE('Novo salário: R$ ' || v_func2.salario);
END;
/
%ROWTYPE baseado em Cursor:

DECLARE
  CURSOR cur_resumo IS
    SELECT d.nome AS departamento,
           COUNT(f.id) AS qtd_funcionarios,
           SUM(f.salario) AS folha_total,
           AVG(f.salario) AS media_salarial
    FROM departamentos d
    LEFT JOIN funcionarios f ON f.departamento_id = d.id
    GROUP BY d.nome
    ORDER BY folha_total DESC;

  -- %ROWTYPE baseado no cursor, não na tabela
  v_registro cur_resumo%ROWTYPE;
BEGIN
  OPEN cur_resumo;
  LOOP
    FETCH cur_resumo INTO v_registro;
    EXIT WHEN cur_resumo%NOTFOUND;

    DBMS_OUTPUT.PUT_LINE(
      v_registro.departamento || ' | ' ||
      v_registro.qtd_funcionarios || ' funcs | ' ||
      'Folha: R$ ' || TO_CHAR(v_registro.folha_total, '999G990D00') || ' | ' ||
      'Média: R$ ' || TO_CHAR(v_registro.media_salarial, '999G990D00')
    );
  END LOOP;
  CLOSE cur_resumo;
END;
/

%TYPE ancora uma variável ao tipo de uma coluna específica ou de outra variável. Já vimos exemplos anteriormente. Abaixo, uma demonstração mais completa: 


DECLARE

Comparação %ROWTYPE vs %TYPE

Aspecto %ROWTYPE %TYPE
O que ancora Linha inteira de tabela/view/cursor Coluna ou variável individual
Sintaxe var tabela%ROWTYPE; var tabela.coluna%TYPE;
Acesso var.coluna Direto: var
Quando usar SELECT * INTO, Cursor FOR LOOP Variáveis individuais, parâmetros
Atribuição entre registros Sim: reg1 := reg2; N/A (uma variável por vez)

Passo a Passo: Procedure PL/SQL Completa

Vamos construir uma procedure completa que ilustra todos os conceitos aprendidos: variáveis, cursores, estruturas condicionais, loops, tratamento de exceções, %ROWTYPE e %TYPE. Esta procedure percorre funcionários, calcula bônus baseado no salário e departamento, atualiza a tabela e registra logs. 


CREATE TABLE bonus_log (
  id             NUMBER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  funcionario_id NUMBER,
  nome           VARCHAR2(100),
  salario_antigo NUMBER(10,2),
  bonus          NUMBER(10,2),
  salario_novo   NUMBER(10,2),
  data_processo  TIMESTAMP DEFAULT SYSTIMESTAMP,
  status         VARCHAR2(20)
);

CREATE OR REPLACE PROCEDURE sp_calcular_bonus(
  p_percentual_base IN NUMBER DEFAULT 10,
  p_dept_foco       IN VARCHAR2 DEFAULT NULL
) IS
  CURSOR cur_func IS
    SELECT * FROM funcionarios
    WHERE status = 'A'
      AND (p_dept_foco IS NULL OR departamento = p_dept_foco)
    ORDER BY id;

  v_func          funcionarios%ROWTYPE;
  v_bonus_percent NUMBER(5,2);
  v_bonus_valor   NUMBER(10,2);
  v_salario_novo  NUMBER(10,2);
  v_processados   NUMBER := 0;
  v_erros         NUMBER := 0;

  e_salario_zerado EXCEPTION;
BEGIN
  DBMS_OUTPUT.PUT_LINE('=== INICIANDO CÁLCULO DE BÔNUS ===');
  DBMS_OUTPUT.PUT_LINE('Percentual base: ' || p_percentual_base || '%');

  OPEN cur_func;
  LOOP
    FETCH cur_func INTO v_func;
    EXIT WHEN cur_func%NOTFOUND;

    BEGIN
      -- Validar salário
      IF v_func.salario IS NULL OR v_func.salario <= 0 THEN
        RAISE e_salario_zerado;
      END IF;

      -- Calcular percentual de bônus baseado no departamento
      v_bonus_percent := p_percentual_base;

      IF v_func.departamento = 'TI' THEN
        v_bonus_percent := v_bonus_percent + 5;  -- +5% para TI
      ELSIF v_func.departamento = 'Vendas' THEN
        v_bonus_percent := v_bonus_percent + 8;  -- +8% para Vendas
      END IF;

      -- Funcionários com salário baixo recebem percentual maior
      IF v_func.salario < 3000 THEN
        v_bonus_percent := v_bonus_percent + 3;
      END IF;

      -- Calcular valores
      v_bonus_valor := v_func.salario * (v_bonus_percent / 100);
      v_salario_novo := v_func.salario + v_bonus_valor;

      -- Atualizar salário
      UPDATE funcionarios
      SET salario = v_salario_novo
      WHERE id = v_func.id;

      -- Registrar log de sucesso
      INSERT INTO bonus_log
        (funcionario_id, nome, salario_antigo, bonus, salario_novo, status)
      VALUES
        (v_func.id, v_func.nome, v_func.salario,
         v_bonus_valor, v_salario_novo, 'SUCESSO');

      v_processados := v_processados + 1;

      DBMS_OUTPUT.PUT_LINE(
        '✓ ' || v_func.nome ||
        ' | Dept: ' || v_func.departamento ||
        ' | Bônus: ' || v_bonus_percent || '%' ||
        ' | R$ ' || TO_CHAR(v_bonus_valor, '999G990D00') ||
        ' | Novo salário: R$ ' || TO_CHAR(v_salario_novo, '999G990D00')
      );

    EXCEPTION
      WHEN e_salario_zerado THEN
        v_erros := v_erros + 1;

        INSERT INTO bonus_log
          (funcionario_id, nome, salario_antigo, status)
        VALUES
          (v_func.id, v_func.nome, v_func.salario, 'ERRO: salário zerado');

        DBMS_OUTPUT.PUT_LINE(
          '✗ ' || v_func.nome || ' — ERRO: Salário inválido (R$ ' ||
          NVL(v_func.salario, 0) || ')'
        );

      WHEN OTHERS THEN
        v_erros := v_erros + 1;

        INSERT INTO bonus_log
          (funcionario_id, nome, salario_antigo, status)
        VALUES
          (v_func.id, v_func.nome, v_func.salario, 'ERRO: ' || SQLERRM);

        DBMS_OUTPUT.PUT_LINE(
          '✗ ' || v_func.nome || ' — ERRO: ' || SQLERRM
        );
    END;
  END LOOP;

  CLOSE cur_func;

  COMMIT;

  DBMS_OUTPUT.PUT_LINE('=== RESUMO ===');
  DBMS_OUTPUT.PUT_LINE('Funcionários processados com sucesso: ' || v_processados);
  DBMS_OUTPUT.PUT_LINE('Erros encontrados: ' || v_erros);
  DBMS_OUTPUT.PUT_LINE('Total de registros no cursor: ' || cur_func%ROWCOUNT);

EXCEPTION
  WHEN OTHERS THEN
    ROLLBACK;
    DBMS_OUTPUT.PUT_LINE('ERRO FATAL: ' || SQLERRM);
    RAISE;
END sp_calcular_bonus;
/

-- Executar a procedure
EXEC sp_calcular_bonus(p_percentual_base => 10, p_dept_foco => NULL);

-- Executar apenas para TI com 15% base
EXEC sp_calcular_bonus(15, 'TI');

-- Verificar logs
SELECT * FROM bonus_log ORDER BY data_processo DESC;

3. Criar bloco com LOOP e EXIT WHEN calculando fatorial

Crie um bloco PL/SQL que solicita um número via variável de substituição (&numero) e calcula o fatorial desse número usando um LOOP básico com EXIT WHEN. O programa deve validar se o número é positivo e exibir o resultado. Teste com os valores 0, 1, 5, 10 e um número negativo.

Gabarito: 

DECLARE
  v_numero    NUMBER := №
  v_fatorial  NUMBER := 1;
  v_contador  NUMBER := v_numero;

  e_numero_invalido EXCEPTION;
BEGIN
  -- Validar entrada
  IF v_numero < 0 THEN
    RAISE e_numero_invalido;
  END IF;

  -- Caso especial: fatorial de 0 é 1 (já inicializado)
  IF v_numero = 0 THEN
    DBMS_OUTPUT.PUT_LINE('0! = 1');
  ELSE
    -- Calcular fatorial com LOOP + EXIT WHEN
    LOOP
      v_fatorial := v_fatorial * v_contador;
      DBMS_OUTPUT.PUT_LINE('Multiplicando por ' || v_contador ||
        ' → parcial: ' || v_fatorial);

      v_contador := v_contador - 1;
      EXIT WHEN v_contador = 0;
    END LOOP;

    DBMS_OUTPUT.PUT_LINE(v_numero || '! = ' || v_fatorial);
  END IF;

EXCEPTION
  WHEN e_numero_invalido THEN
    DBMS_OUTPUT.PUT_LINE('Erro: O número deve ser >= 0.');
    DBMS_OUTPUT.PUT_LINE('Valor informado: ' || v_numero);
  WHEN VALUE_ERROR THEN
    DBMS_OUTPUT.PUT_LINE('Erro: Valor inválido. Digite um número inteiro.');
  WHEN OTHERS THEN
    DBMS_OUTPUT.PUT_LINE('Erro: ' || SQLERRM);
END;
