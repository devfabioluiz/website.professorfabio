Procedimentos Armazenados e Funções do Usuário no PostgreSQL

O PostgreSQL oferece dois mecanismos principais para encapsular lógica de negócio no lado do servidor: procedimentos armazenados (stored procedures) e funções definidas pelo usuário (UDFs — User-Defined Functions). Embora ambos permitam executar blocos de código no banco de dados, eles possuem diferenças importantes em relação a controle transacional, retorno de valores e formas de chamada.

Historicamente, antes do PostgreSQL 11 (lançado em 2018), o PostgreSQL só oferecia funções (CREATE FUNCTION). Os procedimentos armazenados (CREATE PROCEDURE) foram introduzidos na versão 11 para suprir a necessidade de controle de transações dentro do código — algo que funções nunca puderam fazer (COMMIT/ROLLBACK dentro de uma função é proibido no PostgreSQL).

1. Procedimentos Armazenados (Stored Procedures)

1.1 O que é um Procedimento Armazenado?

Um procedimento armazenado é um bloco nomeado de código PL/pgSQL (ou outra linguagem suportada) armazenado no catálogo do banco de dados. Ao contrário das funções, os procedimentos:

  • Não retornam valor com RETURN (embora possam ter parâmetros OUT que devolvem valores indiretamente)
  • Podem controlar transações — executar COMMIT, ROLLBACK, iniciar subtransações com SAVEPOINT
  • São chamados com CALL, não podem ser usados dentro de uma consulta SELECT
  • Foram introduzidos no PostgreSQL 11 (antes disso, apenas funções existiam)

1.2 Sintaxe CREATE OR REPLACE PROCEDURE


CREATE OR REPLACE PROCEDURE nome_procedimento(
    parametro1 TIPO [IN | OUT | INOUT] [DEFAULT valor],
    parametro2 TIPO [IN | OUT | INOUT] [DEFAULT valor],
    ...
)
LANGUAGE plpgsql -- ou LANGUAGE sql
AS $$
DECLARE
    -- declaração de variáveis locais (apenas para LANGUAGE plpgsql)
BEGIN
    -- corpo do procedimento
    -- pode conter COMMIT, ROLLBACK, SAVEPOINT
END;
$$;

1.3 Exemplo: Procedimento para Atualizar Estoque

Procedimento que recebe o ID do produto e a quantidade vendida, reduz o estoque e confirma a transação: 


-- Tabela de exemplo
CREATE TABLE produtos (
    id      SERIAL PRIMARY KEY,
    nome    VARCHAR(100) NOT NULL,
    estoque INTEGER NOT NULL DEFAULT 0,
    preco   NUMERIC(10,2) NOT NULL
);

-- Inserir dados de exemplo
INSERT INTO produtos (nome, estoque, preco) VALUES
    ('Notebook', 50, 3500.00),
    ('Mouse', 200, 89.90),
    ('Teclado', 150, 199.90),
    ('Monitor', 30, 1200.00);

-- Procedimento que atualiza estoque com controle de transação
CREATE OR REPLACE PROCEDURE atualizar_estoque(
    p_prod_id INT,
    p_qtd     INT
)
LANGUAGE plpgsql
AS $$
BEGIN
    -- Verifica se há estoque suficiente
    IF (SELECT estoque FROM produtos WHERE id = p_prod_id) < p_qtd THEN
        RAISE EXCEPTION 'Estoque insuficiente para o produto ID %', p_prod_id;
    END IF;

    -- Atualiza o estoque
    UPDATE produtos
    SET estoque = estoque - p_qtd
    WHERE id = p_prod_id;

    -- Confirma a transação (procedures podem fazer COMMIT!)
    COMMIT;
END;
$$;

1.4 Chamando o Procedimento com CALL

Diferentemente das funções, procedimentos são chamados com o comando CALL e não podem ser usados dentro de SELECT, WHERE ou qualquer expressão SQL: 


-- Chamar o procedimento para vender 5 unidades do produto ID 1
CALL atualizar_estoque(1, 5);

-- Verificar o estoque atualizado
SELECT id, nome, estoque FROM produtos;

-- Tentar vender mais do que há em estoque (vai disparar EXCEPTION)
CALL atualizar_estoque(1, 500);

-- Chamar dentro de um bloco DO (anônimo)
DO $$
BEGIN
    CALL atualizar_estoque(2, 10);
    RAISE NOTICE 'Estoque atualizado com sucesso!';
END;
$$;
Importante: Procedimentos no PostgreSQL só podem ser chamados com CALL. Tentar usar SELECT atualizar_estoque(1, 5); resultará em erro. Esta é uma diferença fundamental em relação a funções.

2. Parâmetros IN, OUT e INOUT

Os procedimentos e funções no PostgreSQL suportam três modos de parâmetros que controlam a direção do fluxo de dados:

Modo Descrição Comportamento
IN Entrada (padrão) Recebe valor do chamador. O valor dentro do procedimento é uma cópia; alterações não afetam o chamador.
OUT Saída Retorna valor ao chamador. O valor inicial é NULL. O procedimento/função deve atribuir um valor a este parâmetro antes de terminar.
INOUT Entrada e Saída Recebe valor do chamador e também retorna valor modificado ao chamador. Combina os comportamentos de IN e OUT.

2.1 Exemplo: Procedimento com Parâmetro OUT

Procedimento que calcula o total de vendas de um período e retorna pelo parâmetro OUT: 


-- Tabela de vendas para o exemplo
CREATE TABLE vendas (
    id          SERIAL PRIMARY KEY,
    produto_id  INT NOT NULL,
    quantidade  INT NOT NULL,
    valor_total NUMERIC(10,2) NOT NULL,
    data_venda  DATE NOT NULL DEFAULT CURRENT_DATE
);

-- Inserir dados de exemplo
INSERT INTO vendas (produto_id, quantidade, valor_total, data_venda) VALUES
    (1, 2, 7000.00, '2025-06-01'),
    (2, 5, 449.50, '2025-06-05'),
    (3, 3, 599.70, '2025-06-10'),
    (1, 1, 3500.00, '2025-06-15'),
    (4, 2, 2400.00, '2025-06-15');

-- Procedimento com parâmetro OUT
CREATE OR REPLACE PROCEDURE total_vendas_periodo(
    IN  p_data_inicio DATE,
    IN  p_data_fim    DATE,
    OUT p_total       NUMERIC
)
LANGUAGE plpgsql
AS $$
BEGIN
    SELECT COALESCE(SUM(valor_total), 0)
    INTO p_total
    FROM vendas
    WHERE data_venda BETWEEN p_data_inicio AND p_data_fim;
END;
$$;

-- Chamando o procedimento com parâmetro OUT
CALL total_vendas_periodo('2025-06-01', '2025-06-30', NULL);

-- No psql, o valor é exibido automaticamente.
-- Em outras interfaces, pode ser necessário capturar:
DO $$
DECLARE
    v_total NUMERIC := 0;
BEGIN
    CALL total_vendas_periodo('2025-06-01', '2025-06-30', v_total);
    RAISE NOTICE 'Total de vendas: R$ %', v_total;
END;
$$;

2.2 Exemplo: Procedimento com INOUT

Procedimento que aplica um percentual de desconto a um valor e retorna o resultado pelo mesmo parâmetro: 


CREATE OR REPLACE PROCEDURE aplicar_desconto(
    INOUT p_valor     NUMERIC,
    IN    p_percentual NUMERIC DEFAULT 10
)
LANGUAGE plpgsql
AS $$
BEGIN
    p_valor := p_valor * (1 - p_percentual / 100.0);
END;
$$;

-- Chamando o procedimento com INOUT
DO $$
DECLARE
    v_preco NUMERIC := 1500.00;
BEGIN
    CALL aplicar_desconto(v_preco, 15);
    RAISE NOTICE 'Preço com 15%% de desconto: R$ %', v_preco;
END;
$$;
Nota sobre CALL: Ao chamar um procedimento com parâmetros OUT/INOUT via CALL, você deve passar uma variável (não um literal) para receber o valor de saída. Em psql, se passar NULL, o resultado será exibido automaticamente. Em código PL/pgSQL, declare uma variável e passe-a.

3. Funções Definidas pelo Usuário (UDFs)

3.1 O que é uma Função Definida pelo Usuário?

Uma UDF (User-Defined Function) é um bloco de código armazenado que sempre retorna um valor (escalar, conjunto de linhas, ou tabela). Funções podem ser usadas em consultas SQL como se fossem funções nativas do banco.

Características principais das funções no PostgreSQL:

  • Sempre retornam um valor (RETURNS é obrigatório)
  • Não podem controlar transações (COMMIT/ROLLBACK são proibidos dentro de funções)
  • Podem ser usadas em SELECT, WHERE, ORDER BY, etc.
  • Suportam múltiplas linguagens: SQL, PL/pgSQL, C, Python, Perl, etc.

3.2 Função Escalar (Retorna um Único Valor)

Função que calcula o preço com desconto aplicado:


-- Função escalar que retorna NUMERIC
CREATE OR REPLACE FUNCTION calcular_desconto(
    preco      NUMERIC,
    percentual NUMERIC DEFAULT 10
)
RETURNS NUMERIC
LANGUAGE plpgsql
AS $$
BEGIN
    RETURN preco * (1 - percentual / 100.0);
END;
$$;

-- Usando a função em uma consulta SELECT
SELECT
    nome,
    preco,
    calcular_desconto(preco)        AS com_desconto_10,
    calcular_desconto(preco, 20)    AS com_desconto_20,
    preco - calcular_desconto(preco, 20) AS economia
FROM produtos;

-- Usando com WHERE
SELECT nome, preco
FROM produtos
WHERE calcular_desconto(preco, 25) < 2000;

-- Usando em ORDER BY
SELECT nome, preco, calcular_desconto(preco, 15) AS preco_final
FROM produtos
ORDER BY calcular_desconto(preco, 15) ASC;

3.3 Função com RETURNS TABLE (Retorna Múltiplas Linhas)

O RETURNS TABLE permite que uma função retorne um conjunto de linhas como se fosse uma tabela. É ideal para consultas parametrizadas reutilizáveis. 


-- Adicionar categoria aos produtos
ALTER TABLE produtos ADD COLUMN categoria VARCHAR(50);
UPDATE produtos SET categoria = 'Eletrônicos' WHERE id IN (1, 4);
UPDATE produtos SET categoria = 'Periféricos' WHERE id IN (2, 3);

-- Função que retorna tabela: lista produtos por categoria
CREATE OR REPLACE FUNCTION listar_por_categoria(cat VARCHAR)
RETURNS TABLE(
    id       INT,
    nome     VARCHAR,
    preco    NUMERIC,
    estoque  INT
)
LANGUAGE plpgsql
AS $$
BEGIN
    RETURN QUERY
    SELECT p.id, p.nome, p.preco, p.estoque
    FROM produtos p
    WHERE p.categoria = cat
    ORDER BY p.nome;
END;
$$;

-- Usando a função como se fosse uma tabela
SELECT * FROM listar_por_categoria('Eletrônicos');

-- Pode ser usada em JOIN também
SELECT f.nome, f.preco, COALESCE(v.total_vendido, 0) AS total_vendido
FROM listar_por_categoria('Eletrônicos') f
LEFT JOIN (
    SELECT produto_id, SUM(quantidade) AS total_vendido
    FROM vendas
    GROUP BY produto_id
) v ON f.id = v.produto_id;

3.4 RETURNS SETOF (Retornando Conjunto de Linhas)

RETURNS SETOF é uma alternativa ao RETURNS TABLE. Em vez de definir a estrutura na própria função, usa-se o tipo de uma tabela ou composite type existente: 


-- Função que retorna todas as linhas de uma tabela (SETOF)
CREATE OR REPLACE FUNCTION produtos_em_promocao(
    p_preco_max NUMERIC
)
RETURNS SETOF produtos
LANGUAGE plpgsql
AS $$
BEGIN
    RETURN QUERY
    SELECT *
    FROM produtos
    WHERE preco < p_preco_max
    ORDER BY preco ASC;
END;
$$;

-- Usando a função
SELECT nome, preco, categoria
FROM produtos_em_promocao(2000);

-- Outro exemplo: RETURNS SETOF com RECORD
CREATE OR REPLACE FUNCTION relatorio_vendas(
    p_data DATE
)
RETURNS SETOF RECORD
LANGUAGE plpgsql
AS $$
DECLARE
    rec RECORD;
BEGIN
    FOR rec IN
        SELECT p.nome, SUM(v.quantidade) AS qtd, SUM(v.valor_total) AS total
        FROM vendas v
        JOIN produtos p ON v.produto_id = p.id
        WHERE v.data_venda = p_data
        GROUP BY p.nome
    LOOP
        RETURN NEXT rec;
    END LOOP;
    RETURN;
END;
$$;

-- Ao usar RETURNS SETOF RECORD, é necessário especificar a estrutura na chamada:
SELECT * FROM relatorio_vendas('2025-06-15')
AS (nome VARCHAR, quantidade BIGINT, total NUMERIC);

3.5 Função com Múltiplos Parâmetros OUT

Uma função pode retornar múltiplos valores escalares usando parâmetros OUT ao invés de RETURNS TABLE: 


CREATE OR REPLACE FUNCTION estatisticas_produto(
    IN  p_prod_id   INT,
    OUT p_nome      VARCHAR,
    OUT p_estoque   INT,
    OUT p_total_vendas NUMERIC
)
LANGUAGE plpgsql
AS $$
BEGIN
    SELECT nome, estoque INTO p_nome, p_estoque
    FROM produtos WHERE id = p_prod_id;

    SELECT COALESCE(SUM(valor_total), 0) INTO p_total_vendas
    FROM vendas WHERE produto_id = p_prod_id;
END;
$$;

-- Chamando a função
SELECT * FROM estatisticas_produto(1);

4. Opções de LANGUAGE: SQL vs PL/pgSQL vs C

O PostgreSQL permite escrever funções em diversas linguagens. A escolha da linguagem afeta performance, capacidades e onde a função pode ser usada.

4.1 LANGUAGE SQL

Funções escritas em LANGUAGE SQL são as mais simples: consistem em uma ou mais instruções SQL. O otimizador do PostgreSQL pode inlinar (incorporar) essas funções diretamente na consulta, eliminando o custo de chamada. 


-- Função SQL pura (pode ser inlinada pelo otimizador)
CREATE OR REPLACE FUNCTION preco_com_imposto(preco NUMERIC)
RETURNS NUMERIC
LANGUAGE SQL
IMMUTABLE  -- dica para o otimizador
AS $$
    SELECT preco * 1.17;  -- 17% de imposto
$$;

-- Função SQL que retorna tabela
CREATE OR REPLACE FUNCTION produtos_caros(limite NUMERIC)
RETURNS TABLE(nome VARCHAR, preco NUMERIC)
LANGUAGE SQL
STABLE
AS $$
    SELECT nome, preco
    FROM produtos
    WHERE preco > limite
    ORDER BY preco DESC;
$$;

4.2 LANGUAGE plpgsql

PL/pgSQL é a linguagem procedural nativa do PostgreSQL. Oferece controle de fluxo (IF, LOOP, WHILE, FOR), cursores, tratamento de exceções, e variáveis. É a escolha padrão para lógica complexa. 


CREATE OR REPLACE FUNCTION classificar_preco(preco NUMERIC)
RETURNS VARCHAR
LANGUAGE plpgsql
AS $$
BEGIN
    IF preco < 100 THEN
        RETURN 'Barato';
    ELSIF preco < 1000 THEN
        RETURN 'Médio';
    ELSIF preco < 5000 THEN
        RETURN 'Caro';
    ELSE
        RETURN 'Premium';
    END IF;
END;
$$;

SELECT nome, preco, classificar_preco(preco) AS classificacao
FROM produtos;

4.3 Atributos do Otimizador: IMMUTABLE / STABLE / VOLATILE

Esses atributos informam ao planejador de consultas sobre o comportamento da função, permitindo otimizações importantes:

Atributo Significado Exemplo
IMMUTABLE Para os mesmos argumentos, sempre retorna o mesmo valor. Não modifica o banco. Não consulta tabelas. Função matemática pura: preco * 1.17
STABLE Dentro de uma mesma consulta, retorna o mesmo valor para os mesmos argumentos. Pode ler tabelas, mas não as modifica. Função que consulta uma tabela de configuração: SELECT taxa FROM config
VOLATILE (padrão) Pode retornar valores diferentes a cada chamada. Pode modificar o banco. Função que gera UUID: gen_random_uuid()

Marcar uma função como IMMUTABLE ou STABLE quando aplicável permite que o PostgreSQL a otimize em índices, consultas repetidas e paralelização.

4.4 Outros Atributos Importantes

Atributo Descrição
STRICT (RETURNS NULL ON NULL INPUT) Se qualquer argumento for NULL, a função não é executada e retorna NULL automaticamente. Economiza processamento.
SECURITY DEFINER A função executa com os privilégios do dono da função, não do usuário que a chamou. Útil para permitir acesso controlado a dados sensíveis.
SECURITY INVOKER (padrão) A função executa com os privilégios do usuário que a chamou.
PARALLEL SAFE Indica se a função pode ser executada em paralelo pelo worker de consulta paralela (SAFE, RESTRICTED, UNSAFE).
COST Custo estimado de execução (em unidades arbitrárias). Padrão: 100 para PL/pgSQL, 1 para SQL. Afeta a ordem de avaliação em WHERE. 

-- Função com múltiplos atributos
CREATE OR REPLACE FUNCTION formata_moeda(valor NUMERIC)
RETURNS VARCHAR
LANGUAGE SQL
IMMUTABLE
STRICT
PARALLEL SAFE
COST 1
AS $$
    SELECT 'R$ ' || TO_CHAR(valor, 'FM999G990D00');
$$;

-- Função SECURITY DEFINER: permite acesso controlado a tabela sensível
CREATE OR REPLACE FUNCTION consultar_salario(
    p_func_id INT
)
RETURNS NUMERIC
LANGUAGE plpgsql
SECURITY DEFINER
AS $$
DECLARE
    v_salario NUMERIC;
BEGIN
    -- Executa com privilégios do dono, mesmo que o chamador
    -- não tenha permissão direta na tabela
    SELECT salario INTO v_salario
    FROM folha_pagamento
    WHERE funcionario_id = p_func_id;
    RETURN v_salario;
END;
$$;

5. Sobrecarga de Funções (Function Overloading)

O PostgreSQL suporta sobrecarga de funções — você pode criar múltiplas funções com o mesmo nome, desde que tenham assinaturas diferentes (número de parâmetros ou tipos de parâmetros diferentes). O PostgreSQL decide qual função chamar com base nos tipos dos argumentos fornecidos.

5.1 Exemplo: Funções formatar() Sobrecarregadas


-- Função 1: formata um número inteiro
CREATE OR REPLACE FUNCTION formatar(valor INTEGER)
RETURNS VARCHAR
LANGUAGE SQL
IMMUTABLE
AS $$
    SELECT TO_CHAR(valor, 'FM999G990');
$$;

-- Função 2: formata um número decimal (sobrecarregada)
CREATE OR REPLACE FUNCTION formatar(valor NUMERIC)
RETURNS VARCHAR
LANGUAGE SQL
IMMUTABLE
AS $$
    SELECT 'R$ ' || TO_CHAR(valor, 'FM999G990D00');
$$;

-- Função 3: formata uma data (sobrecarregada)
CREATE OR REPLACE FUNCTION formatar(data_val DATE)
RETURNS VARCHAR
LANGUAGE SQL
IMMUTABLE
AS $$
    SELECT TO_CHAR(data_val, 'DD/MM/YYYY');
$$;

-- Função 4: formata data com hora (sobrecarregada)
CREATE OR REPLACE FUNCTION formatar(data_hora TIMESTAMP)
RETURNS VARCHAR
LANGUAGE SQL
IMMUTABLE
AS $$
    SELECT TO_CHAR(data_hora, 'DD/MM/YYYY HH24:MI:SS');
$$;

-- Testando a sobrecarga — o PostgreSQL escolhe a função correta
SELECT formatar(1500);             -- chama formatar(INTEGER)    → "1.500"
SELECT formatar(1500.50);          -- chama formatar(NUMERIC)    → "R$ 1.500,50"
SELECT formatar(CURRENT_DATE);     -- chama formatar(DATE)       → "DD/MM/YYYY"
SELECT formatar(NOW());            -- chama formatar(TIMESTAMP)  → "DD/MM/YYYY HH24:MI:SS"

-- Listar todas as funções com o mesmo nome
SELECT proname, proargtypes::regtype[]
FROM pg_proc
WHERE proname = 'formatar';

5.2 Resolução de Ambiguidade

Quando há múltiplas funções sobrecarregadas, o PostgreSQL escolhe a que melhor corresponde aos tipos dos argumentos. Se houver ambiguidade (ex: INTEGER pode ser convertido implicitamente para NUMERIC), é necessário fazer um cast explícito: 


-- Se houver ambiguidade, use cast explícito:
SELECT formatar(10::INTEGER);    -- força a versão INTEGER
SELECT formatar(10::NUMERIC);    -- força a versão NUMERIC

-- Sobrecarga com número diferente de parâmetros
CREATE OR REPLACE FUNCTION formatar(
    valor NUMERIC,
    simbolo VARCHAR DEFAULT 'R$'
)
RETURNS VARCHAR
LANGUAGE SQL
IMMUTABLE
AS $$
    SELECT simbolo || ' ' || TO_CHAR(valor, 'FM999G990D00');
$$;

SELECT formatar(1500.50);          -- usa DEFAULT → "R$ 1.500,50"
SELECT formatar(1500.50, 'US$');   -- "US$ 1.500,50"
Sobrecarga de procedimentos: Procedures (CREATE PROCEDURE) também suportam sobrecarga no PostgreSQL. Você pode ter múltiplos procedimentos com o mesmo nome, diferenciados pelos tipos e quantidade de parâmetros, exatamente como funções.

6. Passo a Passo: Sistema de Pedidos Completo

Vamos construir um mini sistema de pedidos usando procedures e functions, demonstrando todos os conceitos aprendidos: procedures com controle transacional, functions escalares e de tabela, sobrecarga e atributos do otimizador.

6.1 Criar a Estrutura de Tabelas


-- Tabela de clientes
CREATE TABLE clientes (
    id     SERIAL PRIMARY KEY,
    nome   VARCHAR(100) NOT NULL,
    email  VARCHAR(150) UNIQUE NOT NULL,
    saldo  NUMERIC(12,2) NOT NULL DEFAULT 0
);

-- Tabela de pedidos
CREATE TABLE pedidos (
    id          SERIAL PRIMARY KEY,
    cliente_id  INT NOT NULL REFERENCES clientes(id),
    data_pedido DATE NOT NULL DEFAULT CURRENT_DATE,
    status      VARCHAR(20) NOT NULL DEFAULT 'pendente',
    total       NUMERIC(12,2) NOT NULL DEFAULT 0
);

-- Tabela de itens do pedido
CREATE TABLE itens_pedido (
    id          SERIAL PRIMARY KEY,
    pedido_id   INT NOT NULL REFERENCES pedidos(id) ON DELETE CASCADE,
    produto_id  INT NOT NULL REFERENCES produtos(id),
    quantidade  INT NOT NULL,
    preco_unit  NUMERIC(10,2) NOT NULL
);

-- Inserir clientes de exemplo
INSERT INTO clientes (nome, email, saldo) VALUES
    ('João Silva', 'joao@email.com', 10000.00),
    ('Maria Santos', 'maria@email.com', 5000.00);

6.2 Procedure: Criar Pedido com Controle Transacional


CREATE OR REPLACE PROCEDURE criar_pedido(
    IN  p_cliente_id INT,
    OUT p_pedido_id  INT
)
LANGUAGE plpgsql
AS $$
DECLARE
    v_saldo NUMERIC;
BEGIN
    -- Verifica se cliente existe e tem saldo
    SELECT saldo INTO v_saldo
    FROM clientes
    WHERE id = p_cliente_id
    FOR UPDATE;  -- bloqueia a linha para evitar concorrência

    IF NOT FOUND THEN
        RAISE EXCEPTION 'Cliente ID % não encontrado', p_cliente_id;
    END IF;

    IF v_saldo <= 0 THEN
        RAISE EXCEPTION 'Cliente ID % não possui saldo suficiente', p_cliente_id;
    END IF;

    -- Cria o pedido
    INSERT INTO pedidos (cliente_id)
    VALUES (p_cliente_id)
    RETURNING id INTO p_pedido_id;

    -- Confirma a transação
    COMMIT;
END;
$$;

6.3 Procedure: Adicionar Item ao Pedido e Atualizar Estoque


CREATE OR REPLACE PROCEDURE adicionar_item_pedido(
    p_pedido_id  INT,
    p_produto_id INT,
    p_quantidade INT
)
LANGUAGE plpgsql
AS $$
DECLARE
    v_preco     NUMERIC;
    v_estoque   INT;
    v_status    VARCHAR;
BEGIN
    -- Verifica se o pedido existe e está pendente
    SELECT status INTO v_status
    FROM pedidos
    WHERE id = p_pedido_id
    FOR UPDATE;

    IF NOT FOUND THEN
        RAISE EXCEPTION 'Pedido ID % não encontrado', p_pedido_id;
    END IF;

    IF v_status != 'pendente' THEN
        RAISE EXCEPTION 'Pedido ID % não está pendente (status: %)', p_pedido_id, v_status;
    END IF;

    -- Verifica estoque e obtém preço
    SELECT preco, estoque INTO v_preco, v_estoque
    FROM produtos
    WHERE id = p_produto_id
    FOR UPDATE;

    IF NOT FOUND THEN
        RAISE EXCEPTION 'Produto ID % não encontrado', p_produto_id;
    END IF;

    IF v_estoque < p_quantidade THEN
        RAISE EXCEPTION 'Estoque insuficiente do produto ID % (disponível: %, solicitado: %)',
            p_produto_id, v_estoque, p_quantidade;
    END IF;

    -- Insere o item
    INSERT INTO itens_pedido (pedido_id, produto_id, quantidade, preco_unit)
    VALUES (p_pedido_id, p_produto_id, p_quantidade, v_preco);

    -- Atualiza o estoque
    UPDATE produtos
    SET estoque = estoque - p_quantidade
    WHERE id = p_produto_id;

    -- Atualiza o total do pedido
    UPDATE pedidos
    SET total = (
        SELECT SUM(quantidade * preco_unit)
        FROM itens_pedido
        WHERE pedido_id = p_pedido_id
    )
    WHERE id = p_pedido_id;

    COMMIT;
END;
$$;

6.4 Function: Calcular Total do Pedido com Desconto


-- Função escalar: calcula o total com desconto baseado no valor
CREATE OR REPLACE FUNCTION calcular_total_com_desconto(
    p_pedido_id INT
)
RETURNS NUMERIC
LANGUAGE plpgsql
STABLE
AS $$
DECLARE
    v_total     NUMERIC;
    v_desconto  NUMERIC := 0;
BEGIN
    SELECT total INTO v_total
    FROM pedidos
    WHERE id = p_pedido_id;

    -- Regra de desconto progressivo
    IF v_total > 5000 THEN
        v_desconto := 0.10;  -- 10% para pedidos acima de R$ 5000
    ELSIF v_total > 2000 THEN
        v_desconto := 0.05;  -- 5% para pedidos acima de R$ 2000
    END IF;

    RETURN v_total * (1 - v_desconto);
END;
$$;

-- Função de tabela: listar pedidos de um cliente
CREATE OR REPLACE FUNCTION listar_pedidos_cliente(p_cliente_id INT)
RETURNS TABLE(
    pedido_id   INT,
    data        DATE,
    status      VARCHAR,
    total       NUMERIC,
    total_desc  NUMERIC
)
LANGUAGE plpgsql
STABLE
AS $$
BEGIN
    RETURN QUERY
    SELECT
        p.id,
        p.data_pedido,
        p.status,
        p.total,
        calcular_total_com_desconto(p.id)
    FROM pedidos p
    WHERE p.cliente_id = p_cliente_id
    ORDER BY p.data_pedido DESC;
END;
$$;

6.5 Procedure: Finalizar Pedido


CREATE OR REPLACE PROCEDURE finalizar_pedido(
    p_pedido_id INT
)
LANGUAGE plpgsql
AS $$
DECLARE
    v_total      NUMERIC;
    v_cliente_id INT;
    v_saldo      NUMERIC;
BEGIN
    -- Obtém dados do pedido
    SELECT cliente_id, total INTO v_cliente_id, v_total
    FROM pedidos
    WHERE id = p_pedido_id AND status = 'pendente'
    FOR UPDATE;

    IF NOT FOUND THEN
        RAISE EXCEPTION 'Pedido ID % não encontrado ou já finalizado', p_pedido_id;
    END IF;

    IF v_total <= 0 THEN
        RAISE EXCEPTION 'Pedido ID % não possui itens', p_pedido_id;
    END IF;

    -- Verifica saldo do cliente
    SELECT saldo INTO v_saldo
    FROM clientes
    WHERE id = v_cliente_id
    FOR UPDATE;

    IF v_saldo < v_total THEN
        RAISE EXCEPTION 'Saldo insuficiente (saldo: R$ %, pedido: R$ %)',
            v_saldo, v_total;
    END IF;

    -- Debita do saldo
    UPDATE clientes
    SET saldo = saldo - v_total
    WHERE id = v_cliente_id;

    -- Atualiza status do pedido
    UPDATE pedidos
    SET status = 'finalizado'
    WHERE id = p_pedido_id;

    COMMIT;
END;
$$;

7. Comparação: Procedures e Functions no Oracle vs PostgreSQL

Para quem tem experiência com Oracle PL/SQL, aqui estão as principais diferenças na criação de procedures e functions:

Característica Oracle (PL/SQL) PostgreSQL (PL/pgSQL)
Criar procedure CREATE OR REPLACE PROCEDURE nome AS ... CREATE OR REPLACE PROCEDURE nome() LANGUAGE plpgsql AS $$ ... $$
Chamar procedure EXECUTE nome(); ou dentro de bloco PL/SQL CALL nome(); (não pode usar em SELECT)
Controle de transação COMMIT/ROLLBACK permitido (Oracle trata procedures como transações independentes) COMMIT/ROLLBACK permitido apenas em procedures (não em functions)
Criar function CREATE OR REPLACE FUNCTION nome RETURN tipo AS ... CREATE OR REPLACE FUNCTION nome() RETURNS tipo AS $$ ... $$
Delimitador de corpo Não usa delimitador especial Usa $$ (dollar quoting)
Package (pacotes) Suporta (CREATE PACKAGE) Não possui packages (usa schemas e naming conventions)
Sobrecarga Suporta dentro de packages Suporta nativamente (mesmo schema)
RETURNS TABLE Usa PIPELINED ou retorna SYS_REFCURSOR RETURNS TABLE(col tipo, ...) com RETURN QUERY
Parâmetros DEFAULT Suporta via DEFAULT Suporta via DEFAULT ou :=
RAISE / Exceções RAISE_APPLICATION_ERROR(-20001, 'msg'); RAISE EXCEPTION 'msg';
Atribuição variavel := valor; variavel := valor; ou SELECT ... INTO variavel;

1. Procedure com INOUT: Transferência entre Contas

Crie um procedimento transferir_valor com parâmetro INOUT p_saldo_origem NUMERIC que transfere um valor de uma conta para outra. O procedimento deve verificar se há saldo suficiente, debitar da origem e creditar no destino. Se o saldo for insuficiente, deve lançar uma exceção. Use COMMIT ao final.

Gabarito: 

CREATE TABLE contas (
    id     SERIAL PRIMARY KEY,
    titular VARCHAR(100) NOT NULL,
    saldo  NUMERIC(12,2) NOT NULL DEFAULT 0
);

INSERT INTO contas (titular, saldo) VALUES
    ('João', 5000.00),
    ('Maria', 2000.00);

CREATE OR REPLACE PROCEDURE transferir_valor(
    IN    p_conta_origem  INT,
    IN    p_conta_destino INT,
    IN    p_valor         NUMERIC,
    INOUT p_saldo_origem  NUMERIC
)
LANGUAGE plpgsql
AS $$
DECLARE
    v_saldo_destino NUMERIC;
BEGIN
    -- Bloqueia as duas contas para evitar concorrência
    SELECT saldo INTO p_saldo_origem
    FROM contas WHERE id = p_conta_origem FOR UPDATE;

    IF NOT FOUND THEN
        RAISE EXCEPTION 'Conta origem ID % não encontrada', p_conta_origem;
    END IF;

    IF p_saldo_origem < p_valor THEN
        RAISE EXCEPTION 'Saldo insuficiente (saldo: R$ %, transferência: R$ %)',
            p_saldo_origem, p_valor;
    END IF;

    SELECT saldo INTO v_saldo_destino
    FROM contas WHERE id = p_conta_destino FOR UPDATE;

    IF NOT FOUND THEN
        RAISE EXCEPTION 'Conta destino ID % não encontrada', p_conta_destino;
    END IF;

    -- Realiza a transferência
    UPDATE contas SET saldo = saldo - p_valor WHERE id = p_conta_origem;
    UPDATE contas SET saldo = saldo + p_valor WHERE id = p_conta_destino;

    -- Atualiza o saldo de saída
    SELECT saldo INTO p_saldo_origem FROM contas WHERE id = p_conta_origem;

    COMMIT;
END;
$$;

-- Teste
DO $$
DECLARE
    v_saldo NUMERIC;
BEGIN
    CALL transferir_valor(1, 2, 500.00, v_saldo);
    RAISE NOTICE 'Novo saldo da conta origem: R$ %', v_saldo;
END;
$$;

SELECT * FROM contas;

2. Function: Total de Vendas por Período

Crie uma função total_vendas_por_periodo(data_inicio DATE, data_fim DATE) que retorna o total de vendas (NUMERIC) no período especificado. Se não houver vendas, deve retornar 0. A função deve ser STABLE. Teste com diferentes intervalos de data.

Gabarito: 

CREATE OR REPLACE FUNCTION total_vendas_por_periodo(
    data_inicio DATE,
    data_fim    DATE
)
RETURNS NUMERIC
LANGUAGE plpgsql
STABLE
STRICT
AS $$
DECLARE
    v_total NUMERIC;
BEGIN
    SELECT COALESCE(SUM(valor_total), 0)
    INTO v_total
    FROM vendas
    WHERE data_venda BETWEEN data_inicio AND data_fim;

    RETURN v_total;
END;
$$;

3. Function com RETURNS TABLE: Produtos com Estoque Baixo

Crie uma função produtos_estoque_baixo(limite INT DEFAULT 10) que retorna uma tabela com (id, nome, estoque, categoria) dos produtos cujo estoque está abaixo do limite informado. Ordene por estoque ASC. Use RETURNS TABLE.

Gabarito: 

CREATE OR REPLACE FUNCTION produtos_estoque_baixo(
    limite INT DEFAULT 10
)
RETURNS TABLE(
    id        INT,
    nome      VARCHAR,
    estoque   INT,
    categoria VARCHAR
)
LANGUAGE plpgsql
STABLE
AS $$
BEGIN
    RETURN QUERY
    SELECT p.id, p.nome, p.estoque, p.categoria
    FROM produtos p
    WHERE p.estoque < limite
    ORDER BY p.estoque ASC;
END;
$$;

GROUP BY peb.nome, peb.estoque, peb.categoria;

4. Sobrecarga: Duas Funções formatar_valor()

Crie duas funções sobrecarregadas com o nome formatar_valor. A primeira deve receber um DATE e retornar VARCHAR no formato 'DD de Mon de YYYY'. A segunda deve receber um NUMERIC e um parâmetro opcional simbolo VARCHAR DEFAULT 'R$' e retornar VARCHAR formatado como moeda (ex: 'R$ 1.500,00').

Gabarito: 

-- Função 1: formata data
CREATE OR REPLACE FUNCTION formatar_valor(data_val DATE)
RETURNS VARCHAR
LANGUAGE SQL
IMMUTABLE
STRICT
AS $$
    SELECT TO_CHAR(data_val, 'DD "de" Mon "de" YYYY');
$$;

-- Função 2: formata número como moeda (sobrecarregada)
CREATE OR REPLACE FUNCTION formatar_valor(
    valor   NUMERIC,
    simbolo VARCHAR DEFAULT 'R$'
)
RETURNS VARCHAR
LANGUAGE SQL
IMMUTABLE
STRICT
AS $$
    SELECT simbolo || ' ' || TO_CHAR(valor, 'FM999G990D00');
$$;

-- Testes da sobrecarga

SELECT formatar_valor(999999.99, '€') AS moeda_euro;

-- Usando em consulta

    nome,
    formatar_valor(preco) AS preco_formatado,
    formatar_valor(preco, 'US$') AS preco_usd
FROM produtos;
