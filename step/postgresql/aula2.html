Visão Geral dos Tipos de Dados Oracle

O Oracle oferece uma ampla variedade de tipos de dados, cada um projetado para um propósito específico. Diferente de outros SGBDs, o Oracle possui tipos com comportamentos e limitações próprios. Conhecer esses tipos é essencial para modelar corretamente o banco de dados.

Detalhes Importantes: VARCHAR2 vs VARCHAR vs CHAR

O Oracle recomenda VARCHAR2 ao invés de VARCHAR — embora atualmente sejam sinônimos, a Oracle se reserva o direito de alterar o comportamento de VARCHAR no futuro.

  • CHAR(10) com valor 'ABC' ocupa 10 bytes (7 espaços extras em branco)
  • VARCHAR2(10) com valor 'ABC' ocupa 3 bytes (apenas o necessário)
  • Em comparações, 'ABC' = 'ABC ' retorna TRUE para CHAR (espaços ignorados), mas FALSE para VARCHAR2 

-- Demonstração: diferença entre CHAR e VARCHAR2
CREATE TABLE teste_char_varchar (
  col_char     CHAR(10),
  col_varchar  VARCHAR2(10)
);

INSERT INTO teste_char_varchar VALUES ('ABC', 'ABC');

-- CHAR: LENGTH = 10 (com espaços)
-- VARCHAR2: LENGTH = 3 (sem espaços)
SELECT
  LENGTH(col_char)     AS len_char,
  LENGTH(col_varchar)  AS len_varchar
FROM teste_char_varchar;

-- Comparação: CHAR ignora espaços, VARCHAR2 não
-- Ambas retornam 'IGUAL'
SELECT
  CASE WHEN col_char = 'ABC'    THEN 'IGUAL' ELSE 'DIFERENTE' END AS comp_char,
  CASE WHEN col_varchar = 'ABC' THEN 'IGUAL' ELSE 'DIFERENTE' END AS comp_varchar
FROM teste_char_varchar;

Detalhes: NUMBER — Precisão e Escala

O NUMBER(p,s) é o tipo numérico universal do Oracle. Entenda p (precisão = total de dígitos) e s (escala = dígitos após a vírgula):

  • NUMBER(7,2): máximo 99.999,99 (5 dígitos antes, 2 depois)
  • NUMBER(5): inteiro com até 5 dígitos (escala 0 implícita)
  • NUMBER: ponto flutuante com precisão máxima (38 dígitos)
  • NUMBER(3,-2): arredonda para centenas (ex: 12345 → 12300) 

-- Demonstração: precisão e escala do NUMBER
CREATE TABLE teste_number (
  valor_sem_formato  NUMBER,
  preco              NUMBER(10,2),
  qtd_int            NUMBER(5),
  arredonda_centena  NUMBER(5,-2)
);

INSERT INTO teste_number VALUES (12345.6789, 12345.6789, 123.45, 12345);

SELECT
  valor_sem_formato,        -- 12345.6789
  preco,                    -- 12345.68 (arredondado)
  qtd_int,                  -- 123 (truncado, não arredondado)
  arredonda_centena         -- 12300
FROM teste_number;

DATE e TIMESTAMP — Formatos e Conversões

O Oracle armazena datas em um formato interno próprio. Para exibição, use TO_CHAR e para conversão, TO_DATE. 


-- Trabalhando com DATE no Oracle
SELECT
  SYSDATE                                          AS data_atual,
  SYSTIMESTAMP                                     AS timestamp_atual,
  TO_CHAR(SYSDATE, 'DD/MM/YYYY HH24:MI:SS')        AS formatado_br,
  TO_CHAR(SYSDATE, 'Day, DD "de" Month "de" YYYY') AS data_extenso,
  TO_CHAR(SYSDATE, 'YYYY-MM-DD')                   AS iso_format
FROM DUAL;

-- Converter string para data
SELECT
  TO_DATE('15/08/2025', 'DD/MM/YYYY')              AS data_convertida,
  TO_DATE('2025-08-15 14:30:00', 'YYYY-MM-DD HH24:MI:SS') AS data_hora
FROM DUAL;

-- Operações aritméticas com datas
SELECT
  SYSDATE               AS hoje,
  SYSDATE + 1           AS amanha,
  SYSDATE - 7           AS uma_semana_atras,
  SYSDATE + 1/24        AS uma_hora_depois,
  SYSDATE + 30/1440     AS trinta_minutos_depois
FROM DUAL;

-- Diferença entre datas (resultado em dias)
SELECT
  TO_DATE('01/07/2025', 'DD/MM/YYYY') - TO_DATE('01/01/2025', 'DD/MM/YYYY') AS dias_diferenca
FROM DUAL;

Criando Estrutura de Banco de Treinamento

Vamos criar um banco de dados de exemplo para uma empresa, utilizando os tipos de dados Oracle de forma apropriada e demonstrando constraints (restrições).

Tabela: Departamentos


CREATE TABLE departamentos (
  id_depto     NUMBER(5) PRIMARY KEY,
  nome         VARCHAR2(80) NOT NULL UNIQUE,
  sigla        CHAR(4) NOT NULL,
  orcamento    NUMBER(12,2) DEFAULT 0,
  data_criacao DATE DEFAULT SYSDATE,
  ativo        CHAR(1) DEFAULT 'S' CHECK (ativo IN ('S', 'N'))
);

Tabela: Funcionários


CREATE TABLE funcionarios (
  id_func       NUMBER(10) PRIMARY KEY,
  nome          VARCHAR2(120) NOT NULL,
  email         VARCHAR2(150) UNIQUE,
  cpf           CHAR(11) UNIQUE,
  salario       NUMBER(10,2) CHECK (salario > 0),
  bonus         NUMBER(8,2) DEFAULT 0,
  data_admissao DATE DEFAULT SYSDATE,
  data_nasc     DATE,
  id_depto      NUMBER(5) NOT NULL,
  cargo         VARCHAR2(80),
  foto          BLOB,
  observacoes   CLOB,
  ultima_atual  TIMESTAMP DEFAULT SYSTIMESTAMP,
  CONSTRAINT fk_func_depto FOREIGN KEY (id_depto)
    REFERENCES departamentos(id_depto),
  CONSTRAINT ck_salario_bonus CHECK (bonus <= salario * 0.5)
);

Tabela: Projetos


CREATE TABLE projetos (
  id_projeto   NUMBER(8) PRIMARY KEY,
  nome         VARCHAR2(150) NOT NULL,
  descricao    CLOB,
  data_inicio  DATE NOT NULL,
  data_fim     DATE,
  orcamento    NUMBER(12,2),
  status       VARCHAR2(20) DEFAULT 'PLANEJADO'
               CHECK (status IN ('PLANEJADO','EM ANDAMENTO','CONCLUIDO','CANCELADO')),
  id_gerente   NUMBER(10),
  CONSTRAINT fk_proj_gerente FOREIGN KEY (id_gerente)
    REFERENCES funcionarios(id_func),
  CONSTRAINT ck_data_projeto CHECK (data_fim IS NULL OR data_fim > data_inicio)
);

Tabela: Alocação de Funcionários em Projetos


CREATE TABLE alocacao_projeto (
  id_alocacao    NUMBER(10) PRIMARY KEY,
  id_func        NUMBER(10) NOT NULL,
  id_projeto     NUMBER(8) NOT NULL,
  horas_alocadas NUMBER(4) CHECK (horas_alocadas > 0 AND horas_alocadas <= 200),
  papel          VARCHAR2(50),
  data_entrada   DATE DEFAULT SYSDATE,
  CONSTRAINT fk_aloc_func FOREIGN KEY (id_func)
    REFERENCES funcionarios(id_func),
  CONSTRAINT fk_aloc_proj FOREIGN KEY (id_projeto)
    REFERENCES projetos(id_projeto),
  CONSTRAINT uk_aloc UNIQUE (id_func, id_projeto)
);

Tabela: Vendas (para exercícios de agregação)


CREATE TABLE vendas (
  id_venda    NUMBER(12) PRIMARY KEY,
  regiao      VARCHAR2(50) NOT NULL,
  produto     VARCHAR2(100) NOT NULL,
  categoria   VARCHAR2(50),
  quantidade  NUMBER(8) CHECK (quantidade > 0),
  valor_unit  NUMBER(10,2) CHECK (valor_unit > 0),
  valor_total NUMBER(12,2) GENERATED ALWAYS AS (quantidade * valor_unit) VIRTUAL,
  data_venda  DATE DEFAULT SYSDATE,
  vendedor    VARCHAR2(120),
  canal       VARCHAR2(30) CHECK (canal IN ('ONLINE','LOJA FISICA','TELEVENDAS','REVENDA'))
);

Sequences no Oracle

Diferente do PostgreSQL que oferece SERIAL e do MySQL que oferece AUTO_INCREMENT, o Oracle utiliza SEQUENCES — objetos independentes que geram números sequenciais. Não existe auto-incremento nativo em colunas Oracle (até o 12c, que introduziu GENERATED AS IDENTITY).

Criando uma Sequence


CREATE SEQUENCE seq_funcionarios
  START WITH 1
  INCREMENT BY 1
  MINVALUE 1
  MAXVALUE 9999999999
  NOCACHE
  NOCYCLE;

Usando NEXTVAL e CURRVAL


-- Obter o próximo valor da sequence (avança a sequence)
SELECT seq_funcionarios.NEXTVAL FROM DUAL;

-- Obter o valor atual (sem avançar) — só pode ser usado na mesma sessão após um NEXTVAL
SELECT seq_funcionarios.CURRVAL FROM DUAL;

-- Inserir com sequence
INSERT INTO funcionarios (id_func, nome, email, salario, id_depto) VALUES
  (seq_funcionarios.NEXTVAL, 'João Silva', 'joao@empresa.com.br', 5000, 1);

INSERT INTO funcionarios (id_func, nome, email, salario, id_depto) VALUES
  (seq_funcionarios.NEXTVAL, 'Maria Oliveira', 'maria@empresa.com.br', 6200, 2);

Oracle 12c+: Coluna Identity (substituto do SERIAL)

A partir do Oracle 12c, você pode usar GENERATED AS IDENTITY como alternativa às sequences manuais: 


-- Coluna com identity (similar ao SERIAL do PostgreSQL)
CREATE TABLE categorias (
  id_categoria NUMBER(10) GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  nome          VARCHAR2(80) NOT NULL
);

-- Inserir sem especificar o ID (gerado automaticamente)
INSERT INTO categorias (nome) VALUES ('Eletrônicos');
INSERT INTO categorias (nome) VALUES ('Vestuário');
INSERT INTO categorias (nome) VALUES ('Alimentos');

Consultas com JOINs no Oracle

O Oracle suporta tanto a sintaxe ANSI SQL (INNER JOIN, LEFT JOIN, RIGHT JOIN, FULL JOIN) quanto a sintaxe antiga Oracle com o operador (+) para outer joins. A sintaxe ANSI é recomendada para novos desenvolvimentos.

INNER JOIN


-- Sintaxe ANSI (recomendada)
SELECT
  f.nome       AS funcionario,
  f.cargo,
  d.nome       AS departamento,
  d.sigla
FROM funcionarios f
INNER JOIN departamentos d ON f.id_depto = d.id_depto
ORDER BY d.nome, f.nome;

-- Sintaxe Oracle tradicional (equivalente)
SELECT
  f.nome       AS funcionario,
  f.cargo,
  d.nome       AS departamento,
  d.sigla
FROM funcionarios f, departamentos d
WHERE f.id_depto = d.id_depto
ORDER BY d.nome, f.nome;

LEFT JOIN — Sintaxe ANSI vs Operador (+) Oracle


-- LEFT JOIN ANSI: todos os funcionários, mesmo sem departamento
SELECT
  f.nome       AS funcionario,
  NVL(d.nome, 'SEM DEPARTAMENTO') AS departamento
FROM funcionarios f
LEFT JOIN departamentos d ON f.id_depto = d.id_depto;

-- Sintaxe Oracle com (+) — MESMO resultado
SELECT
  f.nome       AS funcionario,
  NVL(d.nome, 'SEM DEPARTAMENTO') AS departamento
FROM funcionarios f, departamentos d
WHERE f.id_depto = d.id_depto(+);
  -- (+) colocado do lado RIGHT (departamentos)
  -- indica que departamentos é opcional

RIGHT JOIN — Operador (+)


-- RIGHT JOIN ANSI: todos os departamentos, mesmo sem funcionários
SELECT
  d.nome       AS departamento,
  NVL(f.nome, 'Nenhum funcionário') AS funcionario
FROM funcionarios f
RIGHT JOIN departamentos d ON f.id_depto = d.id_depto;

-- Sintaxe Oracle com (+) — note que (+) se move
SELECT
  d.nome       AS departamento,
  NVL(f.nome, 'Nenhum funcionário') AS funcionario
FROM funcionarios f, departamentos d
WHERE f.id_depto(+) = d.id_depto;
  -- (+) colocado do lado LEFT (funcionarios)

FULL OUTER JOIN — Múltiplas tabelas


-- FULL OUTER JOIN com 3 tabelas
SELECT
  d.nome       AS departamento,
  f.nome       AS funcionario,
  p.nome       AS projeto
FROM departamentos d
LEFT JOIN funcionarios f ON d.id_depto = f.id_depto
LEFT JOIN alocacao_projeto ap ON f.id_func = ap.id_func
LEFT JOIN projetos p ON ap.id_projeto = p.id_projeto
ORDER BY d.nome, f.nome, p.nome;

Self-Join — Funcionários e seus gerentes


-- SELF JOIN: listar funcionários com nome do gerente do projeto
SELECT
  f.nome              AS funcionario,
  f.cargo,
  p.nome              AS projeto,
  g.nome              AS gerente_projeto
FROM alocacao_projeto ap
JOIN funcionarios f   ON ap.id_func = f.id_func
JOIN projetos p       ON ap.id_projeto = p.id_projeto
LEFT JOIN funcionarios g ON p.id_gerente = g.id_func
ORDER BY p.nome, f.nome;

Funções Agregadas no Oracle

As funções agregadas do Oracle seguem o padrão SQL e permitem sumarizar dados agrupados. As principais são: COUNT, SUM, AVG, MIN, MAX. O Oracle também oferece funções estatísticas avançadas.

Funções Básicas com GROUP BY


-- Contagem e soma por departamento
SELECT
  d.nome                     AS departamento,
  COUNT(f.id_func)            AS total_funcionarios,
  SUM(f.salario)              AS folha_total,
  ROUND(AVG(f.salario), 2)    AS media_salarial,
  MIN(f.salario)              AS menor_salario,
  MAX(f.salario)              AS maior_salario,
  SUM(f.bonus)                AS total_bonus
FROM funcionarios f
JOIN departamentos d ON f.id_depto = d.id_depto
GROUP BY d.nome
ORDER BY media_salarial DESC;

HAVING — Filtrando Grupos


-- Departamentos com média salarial acima de 6000
SELECT
  d.nome                     AS departamento,
  COUNT(*)                   AS total_funcionarios,
  ROUND(AVG(f.salario), 2)   AS media_salarial
FROM funcionarios f
JOIN departamentos d ON f.id_depto = d.id_depto
GROUP BY d.nome
HAVING AVG(f.salario) > 6000
ORDER BY media_salarial DESC;

Funções Oracle Específicas: MEDIAN, STDDEV, VARIANCE


-- Funções estatísticas avançadas (Oracle-specific)
SELECT
  d.nome                         AS departamento,
  ROUND(MEDIAN(f.salario), 2)    AS mediana_salarial,
  ROUND(STDDEV(f.salario), 2)    AS desvio_padrao,
  ROUND(VARIANCE(f.salario), 2)  AS variancia,
  ROUND(AVG(f.salario), 2)       AS media
FROM funcionarios f
JOIN departamentos d ON f.id_depto = d.id_depto
GROUP BY d.nome
ORDER BY media DESC;

Ordenação — ORDER BY com múltiplas colunas


-- Ordenação múltipla: por departamento e salário decrescente
SELECT
  d.nome   AS departamento,
  f.nome   AS funcionario,
  f.salario,
  f.cargo
FROM funcionarios f
JOIN departamentos d ON f.id_depto = d.id_depto
ORDER BY d.nome ASC, f.salario DESC;

-- Ordenação com NULLS FIRST / NULLS LAST (Oracle-specific)
SELECT nome, bonus
FROM funcionarios
ORDER BY bonus DESC NULLS LAST;

-- Ordenação por expressão
SELECT nome, salario, bonus, (salario + bonus) AS remuneracao_total
FROM funcionarios
ORDER BY 4 DESC;

ROLLUP — Subtotais Hierárquicos

O ROLLUP gera subtotais seguindo uma hierarquia da direita para a esquerda. Por exemplo, ROLLUP(regiao, produto) gera: subtotais por (região, produto), subtotais por (região) e total geral.

Os níveis de agregação são: (região, produto) → (região, ALL) → (ALL, ALL). 


-- ROLLUP: Vendas por região e produto com subtotais hierárquicos
SELECT
  regiao,
  produto,
  SUM(quantidade)   AS total_qtd,
  SUM(valor_total)  AS receita_total
FROM vendas
GROUP BY ROLLUP(regiao, produto)
ORDER BY regiao, produto;

Resultado do ROLLUP (explicado)

O resultado da consulta acima terá linhas como:

ROLLUP com GROUPING — Identificando linhas de subtotal

A função GROUPING(coluna) retorna 1 se a linha é um subtotal para aquela coluna (ou seja, a coluna foi agregada / é NULL devido ao ROLLUP/CUBE), e 0 caso contrário. 


-- ROLLUP com GROUPING para labels descritivos
SELECT
  CASE GROUPING(regiao)
    WHEN 1 THEN '*** TOTAL ***'
    ELSE regiao
  END                         AS regiao,
  CASE GROUPING(produto)
    WHEN 1 THEN '-- SUBTOTAL --'
    ELSE produto
  END                         AS produto,
  SUM(quantidade)             AS total_qtd,
  SUM(valor_total)            AS receita_total,
  GROUPING(regiao)            AS grp_regiao,
  GROUPING(produto)           AS grp_produto
FROM vendas
GROUP BY ROLLUP(regiao, produto)
ORDER BY regiao, produto;

CUBE — Todas as Combinações de Subtotais

O CUBE gera subtotais para todas as combinações possíveis das colunas especificadas. Para CUBE(regiao, produto), os níveis são:

  • (região, produto) — detalhe por região e produto
  • (região, ALL) — subtotal por região
  • (ALL, produto) — subtotal por produto (independente da região)
  • (ALL, ALL) — total geral

-- CUBE: Todas as combinações de subtotais (região e produto)
SELECT
  CASE GROUPING(regiao)
    WHEN 1 THEN 'TODAS REGIÕES'
    ELSE regiao
  END                         AS regiao,
  CASE GROUPING(produto)
    WHEN 1 THEN 'TODOS PRODUTOS'
    ELSE produto
  END                         AS produto,
  SUM(quantidade)             AS total_qtd,
  SUM(valor_total)            AS receita_total
FROM vendas
GROUP BY CUBE(regiao, produto)
ORDER BY
  GROUPING(regiao),
  regiao,
  GROUPING(produto),
  produto;

CUBE com 3 dimensões — Análise Multidimensional


-- CUBE com 3 dimensões: regiao, categoria, canal
-- Gera 2³ = 8 níveis de agregação
SELECT
  CASE GROUPING(regiao)
    WHEN 1 THEN '-- TODAS --'
    ELSE regiao
  END                         AS regiao,
  CASE GROUPING(categoria)
    WHEN 1 THEN '-- TODAS --'
    ELSE categoria
  END                         AS categoria,
  CASE GROUPING(canal)
    WHEN 1 THEN '-- TODOS --'
    ELSE canal
  END                         AS canal,
  SUM(quantidade)             AS total_qtd,
  SUM(valor_total)            AS receita_total,
  GROUPING_ID(regiao, categoria, canal) AS nivel_agregacao
FROM vendas
GROUP BY CUBE(regiao, categoria, canal)
ORDER BY
  GROUPING_ID(regiao, categoria, canal),
  regiao,
  categoria,
  canal;

GROUPING SETS — Controle Exato dos Subtotais

O GROUPING SETS permite especificar exatamente quais combinações de subtotais você deseja. É como um CUBE seletivo. 


-- GROUPING SETS: apenas os subtotais que queremos
SELECT
  regiao,
  produto,
  canal,
  SUM(quantidade)   AS total_qtd,
  SUM(valor_total)  AS receita_total
FROM vendas
GROUP BY GROUPING SETS (
  (regiao, produto, canal),   -- detalhe completo
  (regiao, produto),          -- subtotal por região e produto
  (regiao),                   -- subtotal por região
  ()                          -- total geral
)
ORDER BY regiao, produto, canal;

Aplicação Prática: Relatório de Vendas com ROLLUP


-- Relatório completo de vendas por região e categoria com ROLLUP
SELECT
  DECODE(GROUPING(regiao), 1, 'TOTAL GERAL', regiao)     AS regiao,
  DECODE(GROUPING(categoria), 1, 'SUBTOTAL', categoria)  AS categoria,
  COUNT(*)                                               AS num_vendas,
  SUM(quantidade)                                        AS total_itens,
  TO_CHAR(SUM(valor_total), 'FM999G999G990D00')          AS receita,
  ROUND(AVG(valor_total), 2)                             AS ticket_medio
FROM vendas
GROUP BY ROLLUP(regiao, categoria)
ORDER BY
  GROUPING(regiao),
  regiao,
  GROUPING(categoria),
  categoria;

Diferenças entre Sintaxe Oracle SQL e SQL Padrão

O Oracle SQL tem diversas peculiaridades em relação ao SQL padrão (ANSI/ISO). A tabela a seguir destaca as principais diferenças que você encontrará ao migrar entre SGBDs.

Recurso Oracle SQL Padrão / PostgreSQL MySQL
Data/Hora atual SYSDATE CURRENT_DATE, CURRENT_TIMESTAMP NOW(), CURRENT_DATE
Tabela dummy SELECT SYSDATE FROM DUAL; SELECT CURRENT_DATE; (sem FROM) SELECT NOW(); (sem FROM)
String variável VARCHAR2(n) VARCHAR(n) VARCHAR(n)
Outer join antigo WHERE a.col = b.col(+) Não existe Não existe
Limite de linhas WHERE ROWNUM <= 10 ou FETCH FIRST 10 ROWS ONLY (12c+) LIMIT 10 / FETCH FIRST 10 ROWS ONLY LIMIT 10
Concatenação a || b a || b ou CONCAT(a,b) CONCAT(a,b) ou a || b (modo PIPES_AS_CONCAT)
If/Else em SQL DECODE(expr, v1, r1, v2, r2, default) CASE WHEN expr = v1 THEN r1 ... END IF(expr, v1, v2)
Tratamento de NULL NVL(expr, substituto) COALESCE(expr, substituto) IFNULL(expr, substituto) ou COALESCE()
Truncar data TRUNC(data) (remove hora) DATE_TRUNC('day', data) DATE(data)
Auto incremento SEQUENCE + NEXTVAL ou GENERATED AS IDENTITY (12c+) SERIAL / BIGSERIAL ou GENERATED AS IDENTITY AUTO_INCREMENT
Tipo booleano Não existe em SQL. Usa-se CHAR(1) com CHECK ou NUMBER(1) BOOLEAN BOOLEAN / TINYINT(1)
String com escape q'[texto com 'aspas']' (quote operator) $$texto$$ (dollar quoting) Backslash ou doubling de aspas
Subquery em SELECT Permitido: SELECT (SELECT ...) FROM DUAL; Permitido (escalar) Permitido (escalar)

Exemplos Práticos das Diferenças


-- ====== DIFERENÇA 1: Data atual ======
-- Oracle:
SELECT SYSDATE FROM DUAL;

-- SQL Padrão / PostgreSQL:
-- SELECT CURRENT_DATE;

-- MySQL:
-- SELECT NOW();

-- ====== DIFERENÇA 2: DECODE vs CASE ======
-- Oracle DECODE (específico do Oracle)
SELECT
  nome,
  cargo,
  DECODE(cargo,
    'Desenvolvedor Senior', 'SENIOR',
    'Desenvolvedor Junior', 'JUNIOR',
    'Tech Lead', 'LIDER',
    'OUTROS'  -- default
  ) AS nivel
FROM funcionarios;

-- Equivalente em SQL Padrão (funciona no Oracle também)
SELECT
  nome,
  cargo,
  CASE cargo
    WHEN 'Desenvolvedor Senior' THEN 'SENIOR'
    WHEN 'Desenvolvedor Junior' THEN 'JUNIOR'
    WHEN 'Tech Lead' THEN 'LIDER'
    ELSE 'OUTROS'
  END AS nivel
FROM funcionarios;

-- ====== DIFERENÇA 3: NVL vs COALESCE ======
-- Oracle:
SELECT
  nome,
  NVL(bonus, 0) AS bonus_tratado
FROM funcionarios;

-- SQL Padrão (funciona no Oracle também):
SELECT
  nome,
  COALESCE(bonus, 0) AS bonus_tratado
FROM funcionarios;

-- ====== DIFERENÇA 4: Limite de linhas ======
-- Oracle (12c+):
SELECT nome, salario
FROM funcionarios
ORDER BY salario DESC
FETCH FIRST 5 ROWS ONLY;

-- Oracle (todas versões) com ROWNUM:
SELECT *
FROM (
  SELECT nome, salario
  FROM funcionarios
  ORDER BY salario DESC
)
WHERE ROWNUM <= 5;

-- SQL Padrão / PostgreSQL / MySQL:
-- SELECT nome, salario FROM funcionarios ORDER BY salario DESC LIMIT 5;

-- ====== DIFERENÇA 5: TRUNC date ======
-- Oracle:
SELECT
  TRUNC(SYSDATE)                           AS hoje_sem_hora,
  TRUNC(SYSDATE, 'MM')                     AS primeiro_dia_mes,
  TRUNC(SYSDATE, 'YYYY')                   AS primeiro_dia_ano,
  TRUNC(SYSDATE, 'Q')                      AS primeiro_dia_trimestre
FROM DUAL;

-- SQL Padrão / PostgreSQL:
-- SELECT DATE_TRUNC('day', CURRENT_DATE) ...

ROWNUM e FETCH FIRST — Paginação no Oracle

O Oracle não possui LIMIT. A paginação pode ser feita com ROWNUM ou, a partir do 12c, com OFFSET ... FETCH: 


-- Paginação com OFFSET/FETCH (Oracle 12c+) — sintaxe moderna
SELECT nome, salario, cargo
FROM funcionarios
ORDER BY salario DESC
OFFSET 3 ROWS FETCH NEXT 3 ROWS ONLY;
-- Pula 3 linhas, retorna próximas 3

-- Paginação com ROWNUM (todas versões Oracle)
SELECT *
FROM (
  SELECT f.*, ROWNUM AS rn
  FROM (
    SELECT nome, salario, cargo
    FROM funcionarios
    ORDER BY salario DESC
  ) f
  WHERE ROWNUM <= 6
)
WHERE rn > 3;
-- Mesmo resultado: pula 3, retorna 3

Consulta 1: Funcionários por departamento com média e ranking


SELECT
  d.nome                      AS departamento,
  f.nome                      AS funcionario,
  f.salario,

ORDER BY d.nome, ranking_salarial;

2. Criar consulta com ROLLUP mostrando vendas por região e produto

Usando a tabela vendas_trimestre, escreva uma consulta com ROLLUP que mostre a quantidade total vendida e a receita (quantidade * valor_unitario) agrupada por região e produto, incluindo subtotais por região e total geral.

Gabarito: 

SELECT
  CASE GROUPING(regiao)
    WHEN 1 THEN 'TOTAL GERAL'
    ELSE regiao
  END                               AS regiao,
  CASE GROUPING(produto)
    WHEN 1 THEN 'SUBTOTAL REGIAO'
    ELSE produto
  END                               AS produto,

4. Converter consulta Oracle com (+) para sintaxe padrão LEFT JOIN

Converta a consulta abaixo que usa o operador Oracle (+) para a sintaxe ANSI com LEFT JOIN. A consulta lista todos os funcionários (mesmo os sem departamento) e seus respectivos departamentos: 


SELECT f.nome, d.nome AS departamento
FROM funcionarios f, departamentos d
WHERE f.id_depto = d.id_depto(+);
Gabarito: 

SELECT
  f.nome,
  d.nome AS departamento
FROM funcionarios f
LEFT JOIN departamentos d ON f.id_depto = d.id_depto;

Regra: o operador (+) fica do lado da tabela que pode ter valores nulos (tabela opcional). Para converter: a tabela sem (+) vira a LEFT table, e a tabela com (+) vira a RIGHT table do LEFT JOIN.
