O que é HTTP?

HTTP significa HyperText Transfer Protocol. É o protocolo utilizado para comunicação entre navegadores, aplicações, APIs e servidores na internet.

Sempre que acessamos um site, fazemos login em uma aplicação, enviamos um formulário ou buscamos dados de uma API, estamos utilizando HTTP.

O HTTP funciona através de:

  • Request (requisição)
  • Response (resposta)

Como funciona uma requisição HTTP

Quando o navegador acessa um endereço, ele envia uma requisição para o servidor. O servidor processa essa requisição e retorna uma resposta.

Exemplo:

  1. O navegador acessa: http://localhost:3000
  2. O navegador envia uma requisição HTTP
  3. O servidor Node.js recebe essa requisição
  4. O servidor processa
  5. O servidor retorna uma resposta
  6. O navegador renderiza o conteúdo

Estrutura de uma requisição HTTP

  • Método HTTP
  • URL
  • Headers
  • Body (opcional)

Métodos HTTP

Os métodos HTTP definem qual ação desejamos executar no servidor.

Método Função
GET Buscar dados
POST Criar dados
PUT Atualizar dados
DELETE Remover dados

Exemplo de requisição


GET /produtos HTTP/1.1
Host: localhost:3000

Exemplo de resposta


[
  {
    "id": 1,
    "nome": "Notebook"
  },
  {
    "id": 2,
    "nome": "Mouse"
  }
]

O que é um Servidor?

Um servidor é um software responsável por receber requisições e devolver respostas.

No backend, o servidor normalmente:

  • Recebe requisições HTTP
  • Processa regras de negócio
  • Acessa banco de dados
  • Retorna respostas
  • Controla autenticação
  • Executa lógica da aplicação

Servidor Node.js

O Node.js permite criar servidores utilizando JavaScript.


const http = require('http');

const server = http.createServer((req, res) => {

  res.writeHead(200, {
    'Content-Type': 'text/plain'
  });

  res.end('Servidor funcionando');
});

server.listen(3000, () => {
  console.log('Servidor rodando');
});

Nesse exemplo:

  • createServer cria o servidor
  • req representa a requisição
  • res representa a resposta
  • listen faz o servidor escutar uma porta

O que é REST API?

REST API é um padrão utilizado para construção de APIs.

REST significa:

Representational State Transfer

Uma REST API organiza os endpoints utilizando recursos e métodos HTTP.

Exemplo de endpoints

Endpoint Método Função
/produtos GET Listar produtos
/produtos POST Criar produto
/produtos/1 PUT Atualizar produto
/produtos/1 DELETE Remover produto

Resposta em JSON

APIs normalmente retornam dados em JSON.


{
  "id": 1,
  "nome": "Teclado"
}

O que é Express?

O Express é um framework para Node.js utilizado para facilitar a criação de servidores e APIs.

Sem o Express, precisamos utilizar o módulo HTTP manualmente. Com o Express, o código fica mais simples, organizado e produtivo.

Vantagens do Express

  • Criação simples de rotas
  • Facilidade para trabalhar com APIs
  • Suporte a middlewares
  • Organização de código
  • Grande comunidade
  • Integração com bancos de dados

Instalação do Express


npm install express

Primeiro servidor com Express


const express = require('express');

const app = express();

app.get('/', (req, res) => {
  res.send('Servidor Express funcionando');
});

app.listen(3000, () => {
  console.log('Servidor rodando');
});

Entendendo o código

  • express() cria a aplicação
  • app.get() cria uma rota GET
  • res.send() envia resposta
  • listen() inicia o servidor

Rotas no Express

Rotas definem os caminhos da aplicação.


app.get('/usuarios', (req, res) => {
  res.send('Lista de usuários');
});

app.post('/usuarios', (req, res) => {
  res.send('Usuário criado');
});

O que são Middlewares?

Middlewares são funções executadas entre a requisição e a resposta.

Eles podem:

  • Validar dados
  • Autenticar usuários
  • Modificar requisições
  • Registrar logs
  • Tratar erros

app.use((req, res, next) => {

  console.log('Nova requisição recebida');

  next();
});

O next() permite que a execução continue.

Passo 4 — Criar a API

Abra o arquivo index.js e adicione:


const express = require('express');

const app = express();

app.use(express.json());

const produtos = [
  {
    id: 1,
    nome: 'Notebook'
  },
  {
    id: 2,
    nome: 'Mouse'
  }
];

app.get('/', (req, res) => {
  res.send('API funcionando');
});

app.get('/produtos', (req, res) => {

  res.json(produtos);
});

app.post('/produtos', (req, res) => {

  const novoProduto = req.body;

  produtos.push(novoProduto);

  res.status(201).json({
    mensagem: 'Produto criado',
    produto: novoProduto
  });
});

app.listen(3000, () => {
  console.log('Servidor rodando em http://localhost:3000');
});

🤓 Explicando o código 📝

🔹 express()

O express() cria nossa aplicação backend.


const app = express();

🔹 express.json()

Esse middleware permite que o Express consiga interpretar JSON enviado no body das requisições. 


app.use(express.json());

🔹 Rotas GET

Rotas GET são utilizadas para buscar informações.


app.get('/produtos', (req, res) => {
  res.json(produtos);
});

O método res.json() retorna dados em formato JSON.

🔹 Rotas POST

Rotas POST são utilizadas para criar novos dados.


app.post('/produtos', (req, res) => {

  const novoProduto = req.body;

  produtos.push(novoProduto);

  res.json(novoProduto);
});

🔹 req.body

O req.body contém os dados enviados pelo cliente.

🔹 status HTTP

O método status() define o código da resposta HTTP.

Status Significado
200 Sucesso
201 Recurso criado
404 Não encontrado
500 Erro interno

🔹 listen()

O método listen inicia o servidor em uma porta. 


app.listen(3000);

A porta funciona como um canal de comunicação entre navegador e servidor.

Passo 8 — Criar rota DELETE

Adicione:


app.delete('/produtos/:id', (req, res) => {

  const id = Number(req.params.id);

  const indiceProduto = produtos.findIndex(
    (produto) => produto.id === id
  );

  if (indiceProduto === -1) {

    return res.status(404).json({
      mensagem: 'Produto não encontrado'
    });
  }

  produtos.splice(indiceProduto, 1);

  res.json({
    mensagem: 'Produto removido'
  });
});

Passo 9 — Entendendo req.params

O req.params captura parâmetros da URL.

Exemplo:


/produtos/1

Nesse caso:


req.params.id

retornará:


1

Sua API deve conter:

Utilize JSON nas respostas.

Gabarito: 

const express = require('express');

const app = express();

app.use(express.json());

const usuarios = [
  {
    id: 1,
    nome: 'Maria',
    email: 'maria@email.com'
  }
];

app.get('/usuarios', (req, res) => {

  res.json(usuarios);
});

app.post('/usuarios', (req, res) => {

  const usuario = req.body;

  usuarios.push(usuario);

  res.status(201).json({
    mensagem: 'Usuário criado',
    usuario
  });
});

app.delete('/usuarios/:id', (req, res) => {

  const id = Number(req.params.id);

  const indiceUsuario = usuarios.findIndex(
    (usuario) => usuario.id === id
  );

  if (indiceUsuario === -1) {

    return res.status(404).json({
      mensagem: 'Usuário não encontrado'
    });
  }

  usuarios.splice(indiceUsuario, 1);

  res.json({
    mensagem: 'Usuário removido'
  });
});

app.listen(3000, () => {
  console.log('Servidor rodando');
});
