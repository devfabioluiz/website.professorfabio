Integração Front-End com Back-End

Até agora trabalhamos apenas no lado do servidor com Node.js. Nesta aula vamos conectar o front-end (React com Vite) ao back-end (Node.js com Express), criando uma aplicação completa que troca dados em tempo real.

A comunicação entre front-end e back-end acontece através de requisições HTTP. O front-end envia uma requisição para o servidor, que processa e retorna os dados — geralmente no formato JSON. O front-end então renderiza essas informações na tela.

Fluxo da comunicação

  1. O usuário acessa a página React no navegador
  2. O React faz uma requisição fetch para o servidor Node.js
  3. O Express recebe a requisição e consulta os dados
  4. O servidor retorna os dados em JSON
  5. O React atualiza a interface com os dados recebidos

O que vamos construir

Vamos criar dois projetos lado a lado:

  • Backend: servidor Express com uma rota GET que retorna uma lista de mensagens
  • Frontend: aplicação React criada com Vite que consome essa API e exibe as mensagens

Toda a comunicação será feita via JSON, e vamos aprender a configurar o proxy do Vite para evitar problemas com CORS durante o desenvolvimento.

Passo 1 — Criar o Backend (Node.js + Express)

Dentro da pasta do projeto, crie uma pasta para o backend:


mkdir backend
cd backend
npm init -y
npm install express cors

Agora crie o arquivo server.js dentro da pasta backend: 


type nul > server.js

Abra o arquivo backend/server.js e insira o código abaixo: 


const express = require('express');
const cors = require('cors');

const app = express();

app.use(cors());
app.use(express.json());

const mensagens = [
  { id: 1, titulo: 'Bem-vindo!', texto: 'Esta é a primeira mensagem do servidor.' },
  { id: 2, titulo: 'Integração', texto: 'Front-end e Back-end conectados com sucesso!' },
  { id: 3, titulo: 'React + Node', texto: 'Trabalhando juntos com dados em JSON.' }
];

app.get('/api/mensagens', (req, res) => {
  res.json(mensagens);
});

app.listen(3001, () => {
  console.log('Backend rodando em http://localhost:3001');
});

Teste o backend executando:


node server.js

Acesse http://localhost:3001/api/mensagens no navegador. Você deve ver o JSON com as mensagens.

Pressione Ctrl + C para parar o servidor.

🤓 Explicando o código do backend 📝

Vamos entender cada parte do arquivo server.js:

🔹 Importação dos módulos


const express = require('express');
const cors = require('cors');

Importamos o express para criar o servidor web e o cors para permitir que o front-end acesse o backend de forma segura.

🔹 Middlewares


app.use(cors());
app.use(express.json());

app.use(cors()) libera o acesso de qualquer origem ao nosso backend. app.use(express.json()) permite que o servidor entenda requisições com corpo em JSON (útil para POST, PUT).

🔹 Array de dados


const mensagens = [
  { id: 1, titulo: 'Bem-vindo!', texto: 'Primeira mensagem.' },
  { id: 2, titulo: 'Integração', texto: 'Conectados com sucesso!' },
];

Simulamos um banco de dados com um array de objetos. Em projetos reais, esses dados viriam de um banco como MongoDB ou PostgreSQL.

🔹 Rota GET /api/mensagens


app.get('/api/mensagens', (req, res) => {
  res.json(mensagens);
});

Quando o front-end fizer uma requisição GET para /api/mensagens, o servidor responde com o array de mensagens convertido automaticamente para JSON.

🔹 Inicialização do servidor


app.listen(3001, () => {
  console.log('Backend rodando em http://localhost:3001');
});

O servidor escuta na porta 3001. Escolhemos uma porta diferente da padrão do React (3000) para que os dois possam rodar ao mesmo tempo.

Passo 3 — Configurar o Proxy no Vite

Abra o arquivo frontend/vite.config.js e substitua o conteúdo pelo código abaixo. O proxy vai redirecionar as requisições que começam com /api para o backend na porta 3001. 


import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'

export default defineConfig({
  plugins: [react()],
  server: {
    proxy: {
      '/api': 'http://localhost:3001'
    }
  }
})

🤓 Entendendo o Proxy do Vite 📝

Durante o desenvolvimento, o front-end roda em http://localhost:5173 (porta padrão do Vite) e o backend em http://localhost:3001. Se o React tentar acessar http://localhost:3001/api/mensagens diretamente, o navegador bloqueia por causa da política de CORS (origens diferentes).

O proxy do Vite funciona como um intermediário: quando o React faz uma requisição para /api/mensagens, o Vite redireciona automaticamente para http://localhost:3001/api/mensagens sem que o navegador perceba a mudança de origem.

Dessa forma, durante o desenvolvimento, não precisamos configurar o CORS manualmente no front-end — o proxy resolve o problema para nós.

Em produção, o front-end compilado (HTML + CSS + JS estático) seria servido pelo próprio Express ou por um serviço como Vercel, e aí o CORS precisaria estar configurado corretamente no backend.

Passo 4 — Criar o componente App.jsx

Abra o arquivo frontend/src/App.jsx e substitua todo o conteúdo pelo código abaixo: 


import { useState, useEffect } from 'react'

function App() {
  const [mensagens, setMensagens] = useState([])

  useEffect(() => {
    fetch('/api/mensagens')
      .then((res) => res.json())
      .then((data) => setMensagens(data))
      .catch((err) => console.error('Erro ao buscar mensagens:', err))
  }, [])

  return (
    <div>
      <h1> Mensagens do Servidor</h1>
      {mensagens.length === 0 ? (
        <p>Carregando mensagens...</p>
      ) : (
        <ul>
          {mensagens.map((msg) => (
            <li key={msg.id}>
              <h3>{msg.titulo}</h3>
              <p>{msg.texto}</p>
            </li>
          ))}
        </ul>
      )}
    </div>
  )
}

export default App

🤓 Explicando o código do frontend 📝

🔹 useState


const [mensagens, setMensagens] = useState([])

useState é um hook do React que cria um estado chamado mensagens, inicializado como array vazio. Quando chamamos setMensagens(data), o React re-renderiza o componente automaticamente com os novos dados.

🔹 useEffect


useEffect(() => {
  fetch('/api/mensagens')
    .then((res) => res.json())
    .then((data) => setMensagens(data))
    .catch((err) => console.error('Erro ao buscar mensagens:', err))
}, [])

O useEffect executa o código dentro dele uma única vez quando o componente é montado em tela (o array vazio [] no final garante isso).

Dentro dele, usamos a fetch API do navegador para fazer uma requisição GET para /api/mensagens. O Vite redireciona essa requisição para o backend graças ao proxy configurado anteriormente.

A resposta vem em JSON, e armazenamos os dados no estado mensagens chamando setMensagens(data).
