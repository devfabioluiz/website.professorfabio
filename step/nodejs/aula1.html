Node.js

Introdução, Arquitetura Orientada a Eventos e Configuração de Ambiente

O que é Node.js?

Node.js é um ambiente de execução JavaScript construído sobre o motor V8 do Google Chrome, que permite executar JavaScript fora do navegador, principalmente no lado do servidor (backend).

Antes do Node.js, o JavaScript era utilizado quase exclusivamente para manipulação de interfaces no navegador. Com o surgimento do Node, a linguagem passou a ser usada para criação de servidores, APIs, sistemas distribuídos, automações, aplicações em tempo real e muito mais.

Em outras palavras, o Node.js transformou o JavaScript em uma linguagem full-stack.

Objetivos e Características do Node.js

  • Alta Escalabilidade: Capaz de lidar com milhares de conexões simultâneas.
  • Performance Elevada: Arquitetura baseada em eventos e I/O não bloqueante.
  • Unificação de Linguagem: Permite usar JavaScript no frontend e backend.
  • Grande Ecossistema: Possui o maior repositório de bibliotecas do mundo via NPM.
  • Ideal para aplicações em tempo real: Chats, dashboards, streaming e jogos.

Principais Características Técnicas

  • Single Thread: Executa o código JavaScript em uma única thread principal.
  • Non-blocking I/O: Operações de entrada e saída são assíncronas, não bloqueando a execução.
  • Event Loop: Controla a execução dos eventos e callbacks.
  • Thread Pool: Gerencia operações pesadas em segundo plano.
  • NPM: Gerenciador de pacotes padrão do Node.js.

Arquitetura Orientada a Eventos

O Node.js utiliza uma arquitetura baseada em eventos, onde todas as operações são tratadas como eventos assíncronos. Isso permite que a aplicação continue executando outras tarefas enquanto aguarda respostas externas, como consultas ao banco de dados, leitura de arquivos ou requisições HTTP.

Essa arquitetura é sustentada pelo Event Loop, que é responsável por gerenciar a fila de eventos e callbacks.

Fluxo Simplificado do Event Loop


Request → Event Loop → Callback → Response

Exemplo de Execução Assíncrona


console.log("A");

setTimeout(() => {
  console.log("B");
}, 0);

console.log("C");

Saída esperada:


A
C
B

Isso ocorre porque o setTimeout é tratado como um evento assíncrono e colocado na fila de execução.

Configuração de Ambiente

Para trabalhar com Node.js, é necessário instalar o ambiente em sua máquina. Abaixo estão as instruções para os principais sistemas operacionais.

Windows

  1. Acesse: https://nodejs.org
  2. Baixe a versão LTS (recomendada).
  3. Execute o instalador e siga as instruções padrão.

Verifique a instalação:


node -v
npm -v

Linux (Ubuntu / Debian)


sudo apt update
sudo apt install nodejs npm

Recomenda-se o uso do NVM (Node Version Manager) para melhor controle das versões.

MacOS

Utilizando Homebrew:


brew install node

Criando um Projeto Node.js

Para criar um projeto básico com Node.js, utilize os seguintes comandos: 


mkdir projeto-node
cd projeto-node
npm init -y

Executando um Script Node

Crie um arquivo index.js:


console.log("Node.js funcionando corretamente!");

Execute:


node index.js

Quando Utilizar Node.js?

  • APIs REST
  • Microserviços
  • Chats em tempo real
  • Streaming
  • Dashboards interativos
  • Aplicações altamente escaláveis

Evitar: Aplicações altamente dependentes de processamento intensivo de CPU.
