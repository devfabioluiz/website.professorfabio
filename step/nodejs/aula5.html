JWT, Middleware de Autorização e Proteção de Rotas

Tópicos do nosso plano de curso abordados nessa aula: Autenticação com JWT (JSON Web Token), hash de senha com bcrypt, middleware de autorização, proteção de rotas e controle de acesso.

O que é Autenticação?

Autenticação é o processo de verificar a identidade de um usuário. Normalmente feito por login (email + senha).

Existem duas formas principais:

  • Session (Stateful): o servidor guarda uma sessão na memória
  • Token (Stateless): o servidor gera um token e o cliente o envia em cada requisição

Nesta aula usaremos tokens JWT.

O que é JWT?

JWT (JSON Web Token) é um padrão aberto (RFC 7519) para transmissão de informações entre duas partes como um objeto JSON.

O token é composto por 3 partes separadas por pontos:

  • Header: algoritmo de assinatura (HMAC SHA256, RSA)
  • Payload: dados do usuário (id, email, role)
  • Signature: assinatura que garante a integridade 
eyJhbGciOiJIUzI1NiJ9.eyJpZCI6IjEyMyJ9.4bBcWfXn9J2kM8pVqR3tY7uL1dA0sZxG5hN6mC9vB8w

Fluxo JWT:

  1. Usuário faz login (email + senha)
  2. Servidor valida e gera um token JWT
  3. Cliente armazena o token (localStorage, cookie)
  4. Cliente envia o token no header Authorization
  5. Servidor valida o token e autoriza a requisição

O que é bcrypt?

bcrypt é uma biblioteca de hash de senhas. Ela transforma a senha em um texto irreversível.

Por que usar hash?

  • Nunca salvar senhas em texto puro no banco
  • Se o banco for comprometido, as senhas ficam protegidas
  • bcrypt adiciona um salt automático (proteção contra rainbow table)
const bcrypt = require('bcryptjs');

// Gerar hash da senha
const senhaHash = await bcrypt.hash('123456', 10);

// Comparar senha
const valida = await bcrypt.compare('123456', senhaHash);

O que é Middleware?

Middleware é uma função que intercepta a requisição antes dela chegar ao controller.

No Express, um middleware tem acesso a:

  • req: dados da requisição
  • res: dados da resposta
  • next: função que chama o próximo middleware
const autenticar = (req, res, next) => {
  const token = req.headers.authorization;

  if (!token) {
    return res.status(401).json({ erro: 'Token não informado' });
  }

  // Validar token...

  next(); // Passa para o próximo middleware/controller
};

Passo 3 — Criar Controller de Autenticação

Criar o arquivo:

type nul > src\controllers\authController.js

Dentro de src/controllers/authController.js:

const bcrypt = require("bcryptjs");
const jwt = require("jsonwebtoken");

const Usuario = require("../models/Usuario");

const JWT_SECRET = process.env.JWT_SECRET || "segredo";

const registrar = async (req, res) => {
  const { nome, email, idade, senha } = req.body;

  const existe = await Usuario.findOne({ email });

  if (existe) {
    return res.status(400).json({
      erro: "Email já cadastrado",
    });
  }

  const senhaHash = await bcrypt.hash(senha, 10);

  const usuario = await Usuario.create({
    nome,
    email,
    idade,
    senha: senhaHash,
  });

  const token = jwt.sign(
    { id: usuario._id, email: usuario.email },
    JWT_SECRET,
    { expiresIn: "1d" }
  );

  res.status(201).json({
    mensagem: "Usuário registrado",
    token,
  });
};

const login = async (req, res) => {
  const { email, senha } = req.body;

  const usuario = await Usuario.findOne({ email });

  if (!usuario) {
    return res.status(401).json({
      erro: "Email ou senha inválidos",
    });
  }

  const senhaValida = await bcrypt.compare(
    senha,
    usuario.senha
  );

  if (!senhaValida) {
    return res.status(401).json({
      erro: "Email ou senha inválidos",
    });
  }

  const token = jwt.sign(
    { id: usuario._id, email: usuario.email },
    JWT_SECRET,
    { expiresIn: "1d" }
  );

  res.json({
    mensagem: "Login realizado",
    token,
  });
};

module.exports = {
  registrar,
  login,
};

🤓 Explicando o Código 📝

🔹 bcrypt.hash()

Gera um hash seguro da senha. O número 10 é o salt rounds.

const senhaHash = await bcrypt.hash(senha, 10);

🔹 bcrypt.compare()

Compara a senha digitada com o hash salvo no banco.

const valida = await bcrypt.compare(senha, usuario.senha);

🔹 jwt.sign()

Gera um token JWT com os dados do payload (id, email) e um segredo. O token expira em 1 dia. 

jwt.sign(
  { id: usuario._id, email: usuario.email },
  JWT_SECRET,
  { expiresIn: '1d' }
);

Passo 4 — Criar Middleware de Autenticação

Criar a pasta e o arquivo:

mkdir src\middlewares
type nul > src\middlewares\autenticar.js

Dentro de src/middlewares/autenticar.js:

const jwt = require("jsonwebtoken");

const JWT_SECRET = process.env.JWT_SECRET || "segredo";

const autenticar = (req, res, next) => {
  const authHeader = req.headers.authorization;

  if (!authHeader) {
    return res.status(401).json({
      erro: "Token não informado",
    });
  }

  const token = authHeader.split(" ")[1];

  try {
    const payload = jwt.verify(token, JWT_SECRET);

    req.usuarioId = payload.id;
    req.usuarioEmail = payload.email;

    next();
  } catch (erro) {
    return res.status(401).json({
      erro: "Token inválido ou expirado",
    });
  }
};

module.exports = autenticar;

🤓 Explicando o Código 📝

🔹 Authorization Header

O token é enviado no header no formato:

Authorization: Bearer eyJhbGciOiJIUzI1NiJ9...

🔹 jwt.verify()

Valida a assinatura do token e retorna o payload decodificado.

const payload = jwt.verify(token, JWT_SECRET);

🔹 req.usuarioId e req.usuarioEmail

Após a verificação, o middleware injeta os dados do usuário na requisição para uso nos controllers.

const router = express.Router();

const {
  listarProdutos,

E proteja também as rotas de arquivos. Dentro de src/routes/arquivoRoutes.js: 

const express = require("express");
const router = express.Router();
const upload = require("../config/multer");

const autenticar = require("../middlewares/autenticar");

const {
  uploadArquivo,
  listarArquivos,
  exportarModelo,
  exportarProdutos,
  importarProdutos,
} = require("../controllers/arquivoController");

router.post(
  "/upload",
  autenticar,
  upload.single("arquivo"),
  uploadArquivo
);

router.get("/arquivos", autenticar, listarArquivos);
router.get("/modelo-produtos", exportarModelo);
router.get("/exportar-produtos", exportarProdutos);
router.post(
  "/importar-produtos",
  autenticar,
  upload.single("arquivo"),
  importarProdutos
);

module.exports = router;

🤓 Explicando o Código 📝

🔹 Middleware na rota

O middleware autenticar é colocado ANTES do controller. Se o token for inválido, a requisição é bloqueada e o controller nunca é executado. 

router.post("/produtos", autenticar, criarProduto);

🔹 Rotas públicas vs protegidas

Note que /login, /registrar, /modelo-produtos e /produtos (GET) ficaram sem o middleware — são rotas públicas.

Fluxo Completo

Usuário se registra → Recebe um token → Faz login (ou usa o token já obtido) → Envia o token no header Authorization → Middleware valida o token → Se válido, a requisição segue para o controller → Se inválido, retorna erro 401.
