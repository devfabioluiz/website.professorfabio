Buffers, Streams e Objetos Globais

Tópicos do nosso plano de curso abordados nessa aula: Buffers: criação, leitura, escrita e manipulação; Streams: leitura, escrita, duplex e transform; Objetos globais e funções globais.

O que são Buffers?

Buffer é uma área de memória temporária usada para armazenar dados binários. No Node.js, o Buffer é uma classe nativa que lida diretamente com bytes — essencial para manipular arquivos, imagens, vídeos, streams de rede e dados criptografados.

Pense no Buffer como um contêiner de bytes: ele armazena dados brutos antes de serem processados, transformados ou enviados para algum lugar.

Por que precisamos de Buffers?

  • JavaScript não foi projetado para manipular dados binários
  • Arquivos, imagens e vídeos são sequências de bytes
  • Protocolos de rede (TCP, HTTP) transmitem bytes
  • Criptografia trabalha com dados binários
  • O Buffer preenche essa lacuna no JavaScript

Criando Buffers

A partir de uma string

const buffer = Buffer.from('Olá, Node.js');

console.log(buffer);
console.log(buffer.toString());
console.log(buffer.toString('hex'));
console.log(buffer.toString('base64'));

Resultado:

<Buffer 4f 6c c3 a1 2c 20 4e 6f 64 65 2e 6a 73>
Olá, Node.js
4f6cc3a12c204e6f64652e6a73
T2zDoSwgTm9kZS5qcw==

A partir de um tamanho específico

const bufferVazio = Buffer.alloc(10);
console.log(bufferVazio);

const bufferPreenchido = Buffer.alloc(10, 'A');
console.log(bufferPreenchido.toString());

const bufferNaoInicializado = Buffer.allocUnsafe(10);
console.log(bufferNaoInicializado);
Método Descrição Performance
Buffer.alloc(n) Cria buffer preenchido com zeros Mais lento, mais seguro
Buffer.allocUnsafe(n) Cria buffer sem limpar memória Mais rápido, pode conter dados antigos
Buffer.from(dado) Cria buffer a partir de string/array Médio

Manipulando Buffers

Lendo e escrevendo bytes individuais

const buffer = Buffer.alloc(5);

buffer[0] = 72;
buffer[1] = 101;
buffer[2] = 108;
buffer[3] = 108;
buffer[4] = 111;

console.log(buffer.toString());

Escrevendo números em diferentes formatos

const buffer = Buffer.alloc(8);

buffer.writeUInt32BE(0xDEADBEEF, 0);
buffer.writeUInt32LE(0xCAFEBABE, 4);

console.log(buffer.toString('hex'));

Os sufixos BE (Big Endian) e LE (Little Endian) determinam a ordem dos bytes. Big Endian escreve o byte mais significativo primeiro; Little Endian escreve o menos significativo primeiro.

Fatiando e copiando buffers

const original = Buffer.from('abcdefghij');

const fatiado = original.slice(0, 5);
console.log(fatiado.toString());

const copia = Buffer.alloc(5);
original.copy(copia, 0, 0, 5);
console.log(copia.toString());

const concatenado = Buffer.concat([
  Buffer.from('Hello '),
  Buffer.from('World')
]);
console.log(concatenado.toString());

O que são Streams?

Streams são fluxos de dados processados em pedaços, em vez de carregar tudo na memória de uma vez. Isso é fundamental para eficiência em aplicações Node.js.

Imagine assistir a um vídeo no YouTube: você não baixa o vídeo inteiro antes de assistir — os dados chegam em pequenos pedaços (streams) e são reproduzidos conforme chegam.

Por que usar Streams?

  • Eficiência de memória: não carrega arquivos gigantes na RAM
  • Tempo de resposta: começa processando antes de receber tudo
  • Composição: encadeie operações com .pipe()
  • Contrapressão: controle automático de fluxo entre streams

Tipos de Streams

Tipo Função Método principal
Readable Lê dados .read(), .pipe()
Writable Escreve dados .write(), .end()
Duplex Lê E escreve (ex: socket TCP) Ambos acima
Transform Lê, transforma e escreve ._transform()

Streams de Leitura (Readable)

const fs = require('fs');

const streamLeitura = fs.createReadStream('entrada.txt', {
  encoding: 'utf-8',
  highWaterMark: 16
});

let dados = '';

streamLeitura.on('data', (pedaco) => {
  console.log('Recebendo pedaço:', pedaco.length, 'bytes');
  dados += pedaco;
});

streamLeitura.on('end', () => {
  console.log('Leitura completa!');
  console.log('Total:', dados.length, 'caracteres');
});

streamLeitura.on('error', (erro) => {
  console.log('Erro:', erro.message);
});

🤓 Explicando o código 📝

🔹 highWaterMark

Define o tamanho máximo de cada pedaço (chunk) lido. O valor padrão é 65536 (64 KB). Aqui usamos 16 bytes para vermos vários eventos data.

🔹 Evento 'data'

É disparado toda vez que um pedaço de dados está disponível. Cada pedaço é um Buffer (a menos que encoding seja definido).

🔹 Evento 'end'

É disparado quando não há mais dados para ler.

Streams de Escrita (Writable)

const fs = require('fs');

const streamEscrita = fs.createWriteStream('saida.txt', {
  encoding: 'utf-8'
});

const linhas = [
  'Linha 1\n',
  'Linha 2\n',
  'Linha 3\n'
];

linhas.forEach((linha) => {
  streamEscrita.write(linha);
});

streamEscrita.end('Fim do arquivo\n');

streamEscrita.on('finish', () => {
  console.log('Escrita completa!');
});

Pipe: Conectando Streams

O método .pipe() conecta uma stream de leitura a uma de escrita, transferindo dados automaticamente. É a forma mais elegante de trabalhar com streams. 

const fs = require('fs');

const leitura = fs.createReadStream('entrada.txt');
const escrita = fs.createWriteStream('copia.txt');

leitura.pipe(escrita);

escrita.on('finish', () => {
  console.log('Cópia concluída!');
});

Streams de Transformação (Transform)

Transform streams são as mais interessantes: elas leem, transformam e escrevem dados em tempo real. 

const { Transform } = require('stream');
const fs = require('fs');

const transformarMaiusculo = new Transform({
  transform(pedaco, encoding, callback) {
    const pedacoTransformado = pedaco
      .toString()
      .toUpperCase();

    callback(null, pedacoTransformado);
  }
});

const leitura = fs.createReadStream('entrada.txt');
const escrita = fs.createWriteStream('maiusculo.txt');

leitura
  .pipe(transformarMaiusculo)
  .pipe(escrita);

escrita.on('finish', () => {
  console.log('Arquivo convertido para maiúsculas!');
});

Exemplo Prático: Compactando Arquivos

const fs = require('fs');
const zlib = require('zlib');

const leitura = fs.createReadStream('grande.txt');
const compressao = zlib.createGzip();
const escrita = fs.createWriteStream('grande.txt.gz');

leitura
  .pipe(compressao)
  .pipe(escrita);

escrita.on('finish', () => {
  console.log('Arquivo comprimido!');

  const original = fs.statSync('grande.txt').size;
  const comprimido = fs.statSync('grande.txt.gz').size;

  console.log(`Original: ${original} bytes`);
  console.log(`Comprimido: ${comprimido} bytes`);
  console.log(`Redução: ${(100 - (comprimido / original) * 100).toFixed(1)}%`);
});

Criando uma Stream de Transformação Customizada

Vamos criar uma stream que filtra linhas contendo uma palavra específica: 

const { Transform } = require('stream');

class FiltroStream extends Transform {
  constructor(palavra) {
    super();
    this.palavra = palavra.toLowerCase();
    this.sobra = '';
  }

  _transform(pedaco, encoding, callback) {
    const texto = (this.sobra + pedaco.toString()).split('\n');

    this.sobra = texto.pop();

    const filtrado = texto
      .filter((linha) =>
        linha.toLowerCase().includes(this.palavra)
      )
      .join('\n');

    if (filtrado.length > 0) {
      this.push(filtrado + '\n');
    }

    callback();
  }

  _flush(callback) {
    if (this.sobra &&
        this.sobra.toLowerCase().includes(this.palavra)) {
      this.push(this.sobra + '\n');
    }

    callback();
  }
}

const fs = require('fs');

fs.createReadStream('log.txt')
  .pipe(new FiltroStream('ERRO'))
  .pipe(fs.createWriteStream('apenas-erros.txt'));

Objetos Globais no Node.js

Diferente do navegador, o Node.js NÃO possui window. Em vez disso, o escopo global é global. Mas o Node.js oferece várias variáveis e funções globais que podem ser usadas em qualquer arquivo sem importação. p;

Global Descrição Exemplo
__dirname Caminho absoluto do diretório atual C:\projeto\src
__filename Caminho absoluto do arquivo atual C:\projeto\src\index.js
process Informações do processo em execução process.env, process.argv
console Saída no terminal console.log()
Buffer Classe para dados binários Já vimos nesta aula
setTimeout Executa após um tempo (ms) setTimeout(fn, 1000)
setInterval Executa repetidamente (ms) setInterval(fn, 1000)
clearTimeout Cancela um setTimeout clearTimeout(id)
clearInterval Cancela um setInterval clearInterval(id)

const fs = require("fs");
const path = require("path");
const zlib = require("zlib");
const { Transform } = require("stream");

const logger = require("../utils/logger");

const downloadCSV = (req, res) => {
  const caminho = path.join(__dirname, "../../uploads/dados.csv");

  if (!fs.existsSync(caminho)) {
    logger.erro(`Arquivo não encontrado: ${caminho}`);
    return res.status(404).json({ erro: "Arquivo não encontrado" });
  }

  res.setHeader("Content-Type", "text/csv; charset=utf-8");
  res.setHeader(
    "Content-Disposition",
    "attachment; filename=dados.csv"
  );

  logger.info("Iniciando download com stream");

  const streamLeitura = fs.createReadStream(caminho);

  streamLeitura.pipe(res);

  streamLeitura.on("end", () => {
    logger.info("Download concluído");
  });

  streamLeitura.on("error", (erro) => {
    logger.erro(`Erro no download: ${erro.message}`);
    res.status(500).json({ erro: "Erro ao fazer download" });
  });
};

const downloadCSVComprimido = (req, res) => {
  const caminho = path.join(__dirname, "../../uploads/dados.csv");

  if (!fs.existsSync(caminho)) {
    return res.status(404).json({ erro: "Arquivo não encontrado" });
  }

  const nomeArquivo = "dados.csv.gz";

  res.setHeader("Content-Type", "application/gzip");
  res.setHeader(
    "Content-Disposition",
    `attachment; filename=${nomeArquivo}`
  );

  logger.info("Iniciando download comprimido com stream");

  fs.createReadStream(caminho)
    .pipe(zlib.createGzip())
    .pipe(res);
};

module.exports = {
  downloadCSV,
  downloadCSVComprimido,
};

🤓 Explicando o código 📝

🔹 Por que stream para download?

Se o arquivo tiver 500 MB e você usar fs.readFileSync(), o Node.js tentará carregar 500 MB na memória de uma vez — isso pode travar o servidor. Com streams, os dados fluem em pedaços pequenos, consumindo pouca memória.

🔹 Content-Disposition

O header Content-Disposition: attachment instrui o navegador a baixar o arquivo em vez de tentar exibi-lo.

🔹 zlib.createGzip()

O módulo zlib cria streams de compressão nativas. Ao conectar com .pipe(), os dados são comprimidos em tempo real enquanto são enviados.

🔹 fs.existsSync()

Método síncrono que verifica se um arquivo existe. Útil para validação inicial antes de criar a stream.

Criar transform stream de log para JSON

Crie uma branch chamada dever-de-casa-aula7.

Desenvolva:

  • Uma classe LogToJsonStream que estende Transform
  • Ela deve ler o arquivo logs/app.log (criado na Aula 6)
  • Cada linha do log deve ser convertida para um objeto JSON com os campos: data, nivel (INFO/ERRO), mensagem
  • A saída deve ser salva em logs/app.json como um array de objetos
  • Use streams com .pipe() — NÃO use readFileSync

Dica: Use JSON.parse() para validar o JSON.

Gabarito: 
const { Transform } = require("stream");
const fs = require("fs");

class LogToJsonStream extends Transform {
  constructor() {
    super();
    this.primeiro = true;
  }

  _transform(pedaco, encoding, callback) {
    if (this.primeiro) {
      this.push("[\n");
      this.primeiro = false;
    } else {
      this.push(",\n");
    }

    const linha = pedaco.toString().trim();

    const regex = /\[(.*?)\]\s+(\w+):\s+(.*)/;
    const match = linha.match(regex);

    if (match) {
      const obj = {
        data: match[1],
        nivel: match[2],
        mensagem: match[3],
      };

      this.push("  " + JSON.stringify(obj));
    }

    callback();
  }

  _flush(callback) {
    if (!this.primeiro) {
      this.push("\n]\n");
    } else {
      this.push("[]\n");
    }

    callback();
  }
}

fs.createReadStream("logs/app.log")
  .pipe(new LogToJsonStream())
  .pipe(fs.createWriteStream("logs/app.json"))
  .on("finish", () => {
    console.log("Conversão concluída!");
  });
