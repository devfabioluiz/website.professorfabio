Arquivos Estáticos no Express

Arquivos estáticos são recursos que o servidor entrega exatamente como estão no disco: imagens, arquivos CSS, JavaScript de frontend, fontes, PDFs e qualquer outro arquivo que não precise ser processado pelo servidor.

O Express oferece o middleware express.static() para servir esses arquivos de forma eficiente e com cache automático.

Configuração básica

const express = require('express');
const path = require('path');

const app = express();

app.use(express.static('public'));

Com essa configuração, a pasta public se torna pública. Um arquivo public/css/style.css pode ser acessado pelo navegador como: 

http://localhost:3000/css/style.css

Usando caminho absoluto (recomendado)

const path = require('path');

app.use(
  express.static(path.join(__dirname, '../public'))
);

🤓 Explicando o código 📝

🔹 path.join()

Constrói caminhos de forma segura, independente do sistema operacional (Windows usa \, Linux/Mac usam /). SEMPRE use path.join() em vez de concatenar strings.

🔹 __dirname

Contém o caminho absoluto do diretório onde o arquivo atual está. Isso garante que o caminho funcione independente de onde o comando node foi executado.

🔹 Como funciona o express.static()

  1. O navegador solicita /css/style.css
  2. O Express procura o arquivo na pasta configurada
  3. Se encontrar, envia o arquivo com o Content-Type correto
  4. Se não encontrar, chama next() (próximo middleware)

Múltiplas pastas estáticas

app.use('/css', express.static(path.join(__dirname, '../public/css')));
app.use('/js', express.static(path.join(__dirname, '../public/js')));
app.use('/imagens', express.static(path.join(__dirname, '../public/imagens')));
app.use('/uploads', express.static(path.join(__dirname, '../uploads')));

Cache e performance

app.use(
  express.static(path.join(__dirname, '../public'), {
    maxAge: '7d',
    etag: true,
    immutable: true
  })
);
Opção Descrição
maxAge Tempo que o navegador pode usar o cache (ms ou string como '7d')
etag Gera um hash do arquivo para validação condicional (304 Not Modified)
immutable Informa que o arquivo nunca muda durante seu maxAge

O que é PostgreSQL?

PostgreSQL é um banco de dados relacional open source, considerado um dos mais robustos e completos do mercado. Diferente do MongoDB (NoSQL, documentos), o PostgreSQL organiza dados em tabelas com colunas e linhas, com relacionamentos entre elas.

MongoDB vs PostgreSQL

Característica MongoDB (NoSQL) PostgreSQL (Relacional)
Modelo Documentos (JSON/BSON) Tabelas (linhas e colunas)
Schema Flexível (schemaless) Rígido (schema definido)
Relacionamentos Embute ou referencia Foreign keys e JOINs
Transações Limitado (a partir da 4.0) Completo (ACID nativo)
Quando usar Dados flexíveis, alta escrita Dados estruturados, integridade

  • Usuário: postgres
  • Senha: a que você definir

Criando o banco de dados

CREATE DATABASE meubanco;

Criando tabelas

CREATE TABLE categorias (
  id SERIAL PRIMARY KEY,
  nome VARCHAR(100) NOT NULL UNIQUE,
  descricao TEXT,
  criado_em TIMESTAMP DEFAULT NOW()
);

CREATE TABLE produtos (
  id SERIAL PRIMARY KEY,
  nome VARCHAR(150) NOT NULL,
  preco DECIMAL(10, 2) NOT NULL CHECK (preco > 0),
  estoque INTEGER NOT NULL DEFAULT 0 CHECK (estoque >= 0),
  categoria_id INTEGER REFERENCES categorias(id),
  criado_em TIMESTAMP DEFAULT NOW(),
  atualizado_em TIMESTAMP DEFAULT NOW()
);

🤓 Explicando o código 📝

🔹 SERIAL

Tipo especial do PostgreSQL que cria um campo auto-incrementável. Equivalente ao AUTO_INCREMENT do MySQL.

🔹 PRIMARY KEY

Identificador único de cada linha. Garante que não haja duplicatas e que o campo nunca seja nulo.

🔹 FOREIGN KEY (REFERENCES)

Cria um relacionamento entre tabelas. O categoria_id na tabela produtos referencia o id da tabela categorias. O banco impede que você insira um categoria_id que não existe.

🔹 CHECK

Restrição que valida os dados antes de inserir. CHECK (preco > 0) impede preços negativos ou zero.

Conectando Node.js ao PostgreSQL

Usaremos a biblioteca pg (node-postgres), a mais popular para conectar Node.js ao PostgreSQL. 

npm install pg

Configurando a conexão

const { Pool } = require('pg');

const pool = new Pool({
  host: process.env.PG_HOST || 'localhost',
  port: process.env.PG_PORT || 5432,
  user: process.env.PG_USER || 'postgres',
  password: process.env.PG_PASSWORD,
  database: process.env.PG_DATABASE || 'meubanco'
});

module.exports = pool;

Pool de Conexões

O Pool mantém um conjunto de conexões abertas e reutilizáveis. Isso evita o custo de abrir/fechar conexões a cada requisição. É a forma recomendada para aplicações web.

Executando queries

const pool = require('./database/pg');

async function listarProdutos() {
  const resultado = await pool.query(
    'SELECT * FROM produtos ORDER BY nome'
  );

  return resultado.rows;
}

async function criarProduto(dados) {
  const { nome, preco, estoque, categoria_id } = dados;

  const resultado = await pool.query(
    `INSERT INTO produtos (nome, preco, estoque, categoria_id)
     VALUES ($1, $2, $3, $4)
     RETURNING *`,
    [nome, preco, estoque, categoria_id]
  );

  return resultado.rows[0];
}

async function buscarProdutoPorId(id) {
  const resultado = await pool.query(
    'SELECT * FROM produtos WHERE id = $1',
    [id]
  );

  return resultado.rows[0] || null;
}

async function atualizarProduto(id, dados) {
  const { nome, preco, estoque } = dados;

  const resultado = await pool.query(
    `UPDATE produtos
     SET nome = $1, preco = $2, estoque = $3,
         atualizado_em = NOW()
     WHERE id = $4
     RETURNING *`,
    [nome, preco, estoque, id]
  );

  return resultado.rows[0] || null;
}

async function deletarProduto(id) {
  const resultado = await pool.query(
    'DELETE FROM produtos WHERE id = $1 RETURNING *',
    [id]
  );

  return resultado.rows[0] || null;
}

🤓 Explicando o código 📝

🔹 Parâmetros com $1, $2, $3...

NUNCA concatene valores diretamente na string SQL. Isso abre brecha para SQL Injection. O pg usa parameterized queries: os placeholders $1, $2 são substituídos de forma segura pelo segundo argumento (o array de valores).

🔹 RETURNING *

Clausula específica do PostgreSQL que retorna as linhas afetadas pela operação. Muito útil para INSERT, UPDATE e DELETE — evita uma query extra de SELECT.

🔹 resultado.rows

O pool.query() retorna um objeto com várias propriedades. A mais usada é .rows, que é um array de objetos onde cada objeto é uma linha da tabela.

JOINs: Relacionando Tabelas

JOIN é a operação mais poderosa do SQL. Ela combina dados de duas ou mais tabelas baseadas em um relacionamento. 

SELECT
  p.id,
  p.nome,
  p.preco,
  p.estoque,
  c.nome AS categoria

FROM produtos p

JOIN categorias c
  ON p.categoria_id = c.id

ORDER BY p.nome;

Resultado:

 id |    nome     | preco | estoque |  categoria
----+-------------+-------+---------+-------------
  1 | Mouse       | 120.00|      50 | Periféricos
  2 | Monitor     |1500.00|      10 | Periféricos
  3 | PostgreSQL  |  0.00 |   99999 | Livros

Tipos de JOIN

Tipo Retorna
INNER JOIN Apenas registros com correspondência em ambas as tabelas
LEFT JOIN Todos da tabela esquerda + correspondências da direita (ou NULL)
RIGHT JOIN Todos da tabela direita + correspondências da esquerda (ou NULL)
FULL JOIN Todos os registros de ambas as tabelas 
async function listarProdutosComCategoria() {
  const resultado = await pool.query(`
    SELECT
      p.id,
      p.nome,
      p.preco,
      p.estoque,
      c.nome AS categoria_nome,
      c.id AS categoria_id

    FROM produtos p

    LEFT JOIN categorias c
      ON p.categoria_id = c.id

    ORDER BY c.nome, p.nome
  `);

  return resultado.rows;
}

Passo 7 — Criar Controller para produtos PostgreSQL

Criar src/controllers/produtoPgController.js:

type nul > src\controllers\produtoPgController.js
            
const repository = require("../repositories/produtoPgRepository");
const logger = require("../utils/logger");

const listar = async (req, res) => {
  try {
    const produtos = await repository.listarTodos();
    res.json(produtos);
  } catch (erro) {
    logger.erro(`Erro ao listar produtos PG: ${erro.message}`);
    res.status(500).json({ erro: "Erro ao listar produtos" });
  }
};

const criar = async (req, res) => {
  try {
    const { nome, preco, estoque, categoria_id } = req.body;

    if (!nome || !preco) {
      return res.status(400).json({
        erro: "Nome e preço são obrigatórios",
      });
    }

    const produto = await repository.criar({
      nome,
      preco: Number(preco),
      estoque: Number(estoque) || 0,
      categoria_id: categoria_id || null,
    });

    logger.info(`Produto PG criado: ${nome}`);

    res.status(201).json(produto);
  } catch (erro) {
    logger.erro(`Erro ao criar produto PG: ${erro.message}`);
    res.status(500).json({ erro: "Erro ao criar produto" });
  }
};

const deletar = async (req, res) => {
  try {
    const { id } = req.params;

    const produto = await repository.deletar(id);

    if (!produto) {
      return res.status(404).json({ erro: "Produto não encontrado" });
    }

    res.json({ mensagem: "Produto removido", produto });
  } catch (erro) {
    res.status(500).json({ erro: "Erro ao deletar produto" });
  }
};

module.exports = { listar, criar, deletar };
