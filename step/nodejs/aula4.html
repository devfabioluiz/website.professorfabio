Arquitetura de Aplicações Web

A arquitetura de aplicações web define como os componentes do sistema são organizados, comunicam-se entre si e distribuem responsabilidades. Uma boa arquitetura garante escalabilidade, manutenção, segurança e performance.

No contexto moderno, as aplicações web geralmente seguem o modelo cliente-servidor, com separação clara entre frontend e backend.

Modelo Cliente-Servidor

  • Cliente: Interface com o usuário (browser, app)
  • Servidor: Processamento, regras de negócio e acesso a dados

Arquiteturas Mais Utilizadas

  • Monolítica
  • Arquitetura em Camadas (MVC)
  • Microserviços
  • Arquitetura Orientada a Eventos

Exemplo de Arquitetura MVC


src/
├── controllers/
├── services/
├── models/
├── routes/
└── app.js

Servidor Web em Node.js

O Node.js possui um módulo nativo chamado http, que permite criar servidores web capazes de lidar com requisições HTTP.

Criando um Servidor HTTP Básico


const http = require("http");

const server = http.createServer((req, res) => {
  res.writeHead(200, { "Content-Type": "text/plain" });
  res.end("Servidor Node.js funcionando!");
});

server.listen(3000, () => {
  console.log("Servidor rodando em http://localhost:3000");
});

Tratando Rotas Manualmente


const http = require("http");

const server = http.createServer((req, res) => {
  if (req.url === "/") {
    res.end("Home");
  } else if (req.url === "/sobre") {
    res.end("Sobre");
  } else {
    res.writeHead(404);
    res.end("Página não encontrada");
  }
});

server.listen(3000);

Servidor Web com Express.js

Embora o módulo http seja poderoso, o Express.js facilita enormemente a criação de servidores web, oferecendo abstrações prontas para rotas, middlewares, segurança e validações. 


const express = require("express");
const app = express();

app.get("/", (req, res) => {
  res.send("Servidor com Express funcionando!");
});

app.listen(3000, () => {
  console.log("Servidor rodando em http://localhost:3000");
});

Cliente Web em Node.js

O Node.js também pode atuar como cliente HTTP, realizando requisições para outros servidores. Isso é muito útil para consumo de APIs, microserviços e integrações.

Cliente HTTP com Módulo Nativo


const http = require("http");

http.get("http://api.github.com", (res) => {
  let data = "";

  res.on("data", chunk => data += chunk);
  res.on("end", () => console.log(data));
});

Cliente HTTP com Axios

A biblioteca Axios facilita requisições HTTP com suporte a Promises, interceptadores e tratamento avançado de erros. 


const axios = require("axios");

axios.get("https://jsonplaceholder.typicode.com/posts/1")
  .then(response => {
    console.log(response.data);
  })
  .catch(error => {
    console.error(error);
  });

Cliente HTTP com Fetch (Node 18+)


const response = await fetch("https://jsonplaceholder.typicode.com/posts/1");
const data = await response.json();
console.log(data);
