Tópicos do nosso plano de curso abordados nessa aula: Classe EventEmitter, Manipulação de eventos, Callbacks e tratamento de erros.

O que é a Classe EventEmitter?

A classe EventEmitter é o coração da arquitetura orientada a eventos do Node.js. Ela está no módulo nativo events e permite criar, registrar e disparar eventos personalizados.

Pense no EventEmitter como um sistema de som de uma festa: você "sintoniza" em um evento (com .on()) e quando alguém "toca" esse evento (com .emit()), sua função é executada.

Importando o EventEmitter

const EventEmitter = require('events');

const emissor = new EventEmitter();

Métodos principais

Método Descrição
.on(evento, callback) Registra um listener para o evento (executa sempre)
.once(evento, callback) Registra um listener que executa apenas uma vez
.emit(evento, dados) Dispara o evento, executando todos os listeners
.removeListener(evento, callback) Remove um listener específico
.removeAllListeners(evento) Remove todos os listeners de um evento
.listenerCount(evento) Retorna quantos listeners estão registrados

Exemplo Prático: Sistema de Pedidos

const EventEmitter = require('events');

const pedidos = new EventEmitter();

pedidos.on('pedidoCriado', (pedido) => {
  console.log(`Novo pedido #${pedido.id} recebido`);
  console.log(`Cliente: ${pedido.cliente}`);
  console.log(`Produto: ${pedido.produto}`);
  console.log(`Valor: R$ ${pedido.valor}`);
});

pedidos.on('pedidoCriado', (pedido) => {
  console.log(`[ESTOQUE] Baixando estoque do produto ${pedido.produto}`);
});

pedidos.on('pedidoCancelado', (id) => {
  console.log(`Pedido #${id} foi cancelado`);
});

pedidos.emit('pedidoCriado', {
  id: 101,
  cliente: 'Maria',
  produto: 'Mouse Gamer',
  valor: 250
});

pedidos.emit('pedidoCancelado', 101);

Note que dois listeners diferentes respondem ao mesmo evento pedidoCriado. Cada um executa uma responsabilidade diferente — isso é chamado de separação de preocupações.

Resultado no terminal

Novo pedido #101 recebido
Cliente: Maria
Produto: Mouse Gamer
Valor: R$ 250
[ESTOQUE] Baixando estoque do produto Mouse Gamer
Pedido #101 foi cancelado

Estendendo a Classe EventEmitter

Podemos criar nossas próprias classes que herdam de EventEmitter. Isso é útil para encapsular lógica de eventos em módulos reutilizáveis. 

const EventEmitter = require('events');

class Carrinho extends EventEmitter {
  constructor() {
    super();
    this.itens = [];
  }

  adicionar(produto) {
    this.itens.push(produto);
    this.emit('itemAdicionado', produto);
    this.emit('carrinhoAtualizado', this.itens);
  }

  remover(nome) {
    const indice = this.itens.findIndex((item) => item.nome === nome);

    if (indice === -1) {
      this.emit('erro', { mensagem: 'Item não encontrado' });
      return;
    }

    this.itens.splice(indice, 1);
    this.emit('itemRemovido', nome);
    this.emit('carrinhoAtualizado', this.itens);
  }
}

module.exports = Carrinho;

const Carrinho = require('./Carrinho');

const meuCarrinho = new Carrinho();

meuCarrinho.on('itemAdicionado', (produto) => {
  console.log(`✅ ${produto.nome} adicionado ao carrinho`);
});

meuCarrinho.on('itemRemovido', (nome) => {
  console.log(`🗑️ ${nome} removido do carrinho`);
});

meuCarrinho.on('carrinhoAtualizado', (itens) => {
  console.log(`Total de itens: ${itens.length}`);
});

meuCarrinho.on('erro', (erro) => {
  console.log(`❌ Erro: ${erro.mensagem}`);
});

meuCarrinho.adicionar({ nome: 'Teclado', preco: 200 });
meuCarrinho.adicionar({ nome: 'Monitor', preco: 1500 });
meuCarrinho.remover('Teclado');
meuCarrinho.remover('Cadeira');

O que são Callbacks?

Callback é uma função passada como argumento para outra função, que será executada quando a operação assíncrona terminar.

Esse é o padrão mais antigo do Node.js para lidar com operações assíncronas como leitura de arquivos, requisições HTTP e consultas ao banco.

Exemplo simples de callback

function buscarDados(id, callback) {
  console.log(`Buscando dados do ID ${id}...`);

  setTimeout(() => {
    const dados = {
      id: id,
      nome: 'Produto A',
      preco: 99.90
    };

    callback(null, dados);
  }, 2000);
}

buscarDados(1, (erro, resultado) => {
  if (erro) {
    console.log('Erro:', erro);
    return;
  }

  console.log('Dados recebidos:', resultado);
});

console.log('Essa linha executa antes dos dados chegarem');

Padrão Error-First Callback

No Node.js, o padrão convencional de callbacks é o error-first: o primeiro parâmetro do callback é sempre o erro e o segundo é o resultado. Esse padrão é usado em todos os módulos nativos. 

const fs = require('fs');

fs.readFile('arquivo.txt', 'utf-8', (erro, conteudo) => {
  if (erro) {
    console.log('Erro ao ler arquivo:', erro.message);
    return;
  }

  console.log('Conteúdo:', conteudo);
});

Vantagens do padrão error-first

  • Sempre tratar o erro antes de prosseguir
  • Evita que erros sejam ignorados silenciosamente
  • Padroniza a forma como todos os módulos do Node.js funcionam
  • Facilita a leitura e manutenção do código

Callback Hell e como evitar

Quando encadeamos muitos callbacks, o código fica difícil de ler. Esse problema é conhecido como callback hell ou pirâmide da desgraça.

Callback Hell (não faça isso)

fs.readFile('arquivo1.txt', (erro1, dado1) => {
  if (erro1) return;

  fs.readFile('arquivo2.txt', (erro2, dado2) => {
    if (erro2) return;

    fs.readFile('arquivo3.txt', (erro3, dado3) => {
      if (erro3) return;

      console.log(dado1, dado2, dado3);
    });
  });
});

Refatorando com funções nomeadas

function lerArquivo1() {
  fs.readFile('arquivo1.txt', 'utf-8', (erro, dado1) => {
    if (erro) return console.log(erro);
    lerArquivo2(dado1);
  });
}

function lerArquivo2(dado1) {
  fs.readFile('arquivo2.txt', 'utf-8', (erro, dado2) => {
    if (erro) return console.log(erro);
    lerArquivo3(dado1, dado2);
  });
}

function lerArquivo3(dado1, dado2) {
  fs.readFile('arquivo3.txt', 'utf-8', (erro, dado3) => {
    if (erro) return console.log(erro);
    console.log(dado1, dado2, dado3);
  });
}

lerArquivo1();

Async/Await — A Solução Moderna

Async/Await é a forma moderna de trabalhar com código assíncrono no JavaScript. Ele permite escrever código assíncrono com sintaxe que parece síncrona, eliminando o callback hell.

Transformando callback em Promise

const fs = require('fs');

function lerArquivo(caminho) {
  return new Promise((resolve, reject) => {
    fs.readFile(caminho, 'utf-8', (erro, conteudo) => {
      if (erro) {
        reject(erro);
      } else {
        resolve(conteudo);
      }
    });
  });
}

async function processarArquivos() {
  try {
    const dado1 = await lerArquivo('arquivo1.txt');
    const dado2 = await lerArquivo('arquivo2.txt');
    const dado3 = await lerArquivo('arquivo3.txt');

    console.log(dado1);
    console.log(dado2);
    console.log(dado3);
  } catch (erro) {
    console.log('Erro:', erro.message);
  }
}

processarArquivos();

Comparação: Callback vs Promise vs Async/Await

Abordagem Leitura Tratamento de erro
Callback Difícil (aninhamento) if (erro) return
Promise (.then/.catch) Média (encadeamento) .catch() no final
Async/Await Fácil (parece síncrono) try/catch tradicional

Tratamento de Erros com Async/Await

O bloco try/catch é a forma recomendada de tratar erros ao usar async/await. 

async function buscarUsuario(id) {
  if (!id) {
    throw new Error('ID é obrigatório');
  }

  try {
    const usuario = await Usuario.findById(id);

    if (!usuario) {
      throw new Error('Usuário não encontrado');
    }

    return usuario;
  } catch (erro) {
    console.log('Erro ao buscar usuário:', erro.message);
    throw erro;
  }
}

const EventEmitter = require("events");
const fs = require("fs");

class Logger extends EventEmitter {
  constructor() {
    super();
    this.arquivoLog = "logs/app.log";
  }

  info(mensagem) {
    const log = `[${new Date().toISOString()}] INFO: ${mensagem}`;
    this.emit("log", log);
    this.salvar(log);
  }

  erro(mensagem) {
    const log = `[${new Date().toISOString()}] ERRO: ${mensagem}`;
    this.emit("erro", log);
    this.salvar(log);
  }

  requisicao(metodo, rota) {
    const log = `[${new Date().toISOString()}] ${metodo} ${rota}`;
    this.emit("requisicao", log);
    this.salvar(log);
  }

  salvar(log) {
    const linha = log + "\n";

    fs.appendFile(this.arquivoLog, linha, (erro) => {
      if (erro) {
        console.log("Erro ao salvar log:", erro.message);
      }
    });
  }
}

const logger = new Logger();

logger.on("log", (mensagem) => {
  console.log("📝", mensagem);
});

logger.on("erro", (mensagem) => {
  console.log("❌", mensagem);
});

logger.on("requisicao", (mensagem) => {
  console.log("🌐", mensagem);
});

module.exports = logger;

🤓 Explicando o código 📝

🔹 extends EventEmitter

A classe Logger herda todos os poderes do EventEmitter: .on(), .emit(), .once(), etc. Isso é feito com extends EventEmitter.

🔹 super()

O super() no construtor chama o construtor da classe pai (EventEmitter). Sempre que usamos extends, precisamos chamar super() antes de usar this.

🔹 this.emit()

Dentro dos métodos info(), erro() e requisicao(), usamos this.emit() para disparar eventos. Os listeners registrados com .on() reagem automaticamente.

🔹 fs.appendFile()

O método fs.appendFile() adiciona conteúdo ao final de um arquivo. Se o arquivo não existir, ele é criado automaticamente.

🔹 Padrão Singleton

Note que exportamos logger (minúsculo), uma única instância da classe. Isso significa que todos os arquivos que fizerem require('./utils/logger') receberão a MESMA instância, compartilhando os mesmos eventos.

Passo 7 — Melhorar controllers com async/await e tratamento de erros

Vamos refatorar o src/controllers/authController.js para usar tratamento de erros mais robusto: 

const bcrypt = require("bcryptjs");
const jwt = require("jsonwebtoken");

const Usuario = require("../models/Usuario");
const logger = require("../utils/logger");
const { ErroValidacao, ErroAutenticacao } = require("../utils/erros");

const JWT_SECRET = process.env.JWT_SECRET || "segredo";

const registrar = async (req, res) => {
  try {
    const { nome, email, idade, senha } = req.body;

    if (!nome || !email || !senha) {
      throw new ErroValidacao("Nome, email e senha são obrigatórios");
    }

    const existe = await Usuario.findOne({ email });

    if (existe) {
      throw new ErroValidacao("Email já cadastrado");
    }

    const senhaHash = await bcrypt.hash(senha, 10);

    const usuario = await Usuario.create({
      nome,
      email,
      idade,
      senha: senhaHash,
    });

    const token = jwt.sign(
      { id: usuario._id, email: usuario.email },
      JWT_SECRET,
      { expiresIn: "1d" }
    );

    logger.info(`Usuário registrado: ${email}`);

    res.status(201).json({
      mensagem: "Usuário registrado",
      token,
    });
  } catch (erro) {
    logger.erro(`Falha no registro: ${erro.message}`);

    if (erro instanceof ErroValidacao) {
      return res.status(400).json({ erro: erro.message });
    }

    res.status(500).json({ erro: "Erro interno do servidor" });
  }
};

const login = async (req, res) => {
  try {
    const { email, senha } = req.body;

    if (!email || !senha) {
      throw new ErroValidacao("Email e senha são obrigatórios");
    }

    const usuario = await Usuario.findOne({ email });

    if (!usuario) {
      throw new ErroAutenticacao("Email ou senha inválidos");
    }

    const senhaValida = await bcrypt.compare(senha, usuario.senha);

    if (!senhaValida) {
      throw new ErroAutenticacao("Email ou senha inválidos");
    }

    const token = jwt.sign(
      { id: usuario._id, email: usuario.email },
      JWT_SECRET,
      { expiresIn: "1d" }
    );

    logger.info(`Login realizado: ${email}`);

    res.json({
      mensagem: "Login realizado",
      token,
    });
  } catch (erro) {
    logger.erro(`Falha no login: ${erro.message}`);

    if (erro instanceof ErroValidacao) {
      return res.status(400).json({ erro: erro.message });
    }

    if (erro instanceof ErroAutenticacao) {
      return res.status(401).json({ erro: erro.message });
    }

    res.status(500).json({ erro: "Erro interno do servidor" });
  }
};

module.exports = {
  registrar,
  login,
};

🤓 Explicando o código 📝

🔹 instanceof

O operador instanceof verifica se um erro é de um tipo específico. Isso permite tratar cada tipo de erro de forma diferente. 

if (erro instanceof ErroValidacao) {
  res.status(400).json({ erro: erro.message });
}

🔹 throw

A palavra-chave throw lança um erro, interrompendo o fluxo normal. Dentro de um try/catch, o erro é capturado no bloco catch.

🔹 extends Error

Ao estender a classe Error do JavaScript, criamos erros personalizados que podem carregar informações extras como status HTTP, facilitando o tratamento no controller.

🔹 Singleton com EventEmitter

Como estamos usando o padrão singleton (uma única instância), qualquer módulo que der require('./utils/logger') recebe a MESMA instância. Isso significa que listeners registrados em um lugar reagem a eventos emitidos em outro lugar. Muito útil para sistemas de notificação, logging e eventos globais.

Resumo: Quando usar cada abordagem

Abordagem Quando usar
EventEmitter Sistemas de eventos, logs, notificações, comunicação entre módulos
Callbacks APIs de terceiros que seguem padrão error-first, módulos nativos
Async/Await Todo código novo — é o padrão moderno recomendado

Criar sistema de notificações com EventEmitter

Crie uma branch chamada dever-de-casa-aula6.

Dentro dela, desenvolva:

  • Uma classe Notificador que estende EventEmitter
  • Método enviarEmail(para, assunto, corpo) que emite um evento emailEnviado
  • Método enviarSMS(numero, mensagem) que emite um evento smsEnviado
  • Um listener para cada evento que exibe no console os dados da notificação
  • Um middleware que dispara uma notificação quando um novo usuário é registrado (use o notificador no authController)
Gabarito:

src/utils/notificador.js

const EventEmitter = require("events");

class Notificador extends EventEmitter {
  enviarEmail(para, assunto, corpo) {
    const email = { para, assunto, corpo, data: new Date() };
    this.emit("emailEnviado", email);
  }

  enviarSMS(numero, mensagem) {
    const sms = { numero, mensagem, data: new Date() };
    this.emit("smsEnviado", sms);
  }
}

const notificador = new Notificador();

notificador.on("emailEnviado", (email) => {
  console.log("📧 Email enviado!");
  console.log(`  Para: ${email.para}`);
  console.log(`  Assunto: ${email.assunto}`);
});

notificador.on("smsEnviado", (sms) => {
  console.log("📱 SMS enviado!");
  console.log(`  Número: ${sms.numero}`);
  console.log(`  Mensagem: ${sms.mensagem}`);
});

module.exports = notificador;
