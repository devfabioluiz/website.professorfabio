Fundamentos do HTTP e HTTPS

O protocolo HTTP (HyperText Transfer Protocol) é responsável pela comunicação entre cliente e servidor na web. Ele segue o modelo de requisição e resposta.

O HTTPS é a versão segura do HTTP, utilizando criptografia SSL/TLS para garantir confidencialidade, integridade e autenticação dos dados transmitidos.

Requisições e Respostas HTTP

Uma requisição HTTP é composta por método, URL, headers e body. A resposta contém status code, headers e o corpo da resposta. 


app.get("/produtos", (req, res) => {
  res.json({ produtos: [] });
});

app.post("/produtos", (req, res) => {
  res.status(201).json({ mensagem: "Produto criado" });
});

Códigos de Status HTTP

  • 200 - OK
  • 201 - Created
  • 204 - No Content
  • 400 - Bad Request
  • 401 - Unauthorized
  • 403 - Forbidden
  • 404 - Not Found
  • 500 - Internal Server Error

Arquitetura REST

REST (Representational State Transfer) é um estilo arquitetural utilizado para construção de APIs escaláveis, padronizadas e desacopladas.

Os recursos são identificados por URLs e manipulados por meio de métodos HTTP como GET, POST, PUT e DELETE.

Web Services RESTful

Web services RESTful são serviços que seguem rigorosamente os princípios REST, utilizando HTTP e normalmente JSON como formato de dados.

São amplamente usados na integração entre sistemas, aplicações web, mobile e arquiteturas de microsserviços.

Criação de APIs REST

A seguir temos um exemplo completo de uma API REST simples utilizando Express. 


const express = require("express");
const app = express();

app.use(express.json());

let users = [];

app.post("/users", (req, res) => {
  users.push(req.body);
  res.status(201).json({ mensagem: "Usuário criado" });
});

app.get("/users", (req, res) => {
  res.json(users);
});

app.get("/users/:id", (req, res) => {
  const user = users[req.params.id];
  if (!user) return res.sendStatus(404);
  res.json(user);
});

app.put("/users/:id", (req, res) => {
  users[req.params.id] = req.body;
  res.json({ mensagem: "Usuário atualizado" });
});

app.delete("/users/:id", (req, res) => {
  users.splice(req.params.id, 1);
  res.sendStatus(204);
});

app.listen(3000, () => {
  console.log("API rodando em http://localhost:3000");
});

Boas Práticas em APIs REST

  • Uso correto dos métodos HTTP
  • Padronização de respostas
  • Versionamento da API
  • Tratamento centralizado de erros
  • Validação de dados
  • Separação em camadas (rotas, controllers, services, repositories)
