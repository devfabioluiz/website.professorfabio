MVC e MongoDB

Tópicos do nosso plano de curso abordados nessa aula: Padrão de arquitetura MVC, Integração com banco de dados (MongoDB).

O que é MVC?

MVC é um padrão de arquitetura utilizado para organizar aplicações.

MVC significa:

  • Model
  • View
  • Controller

O principal objetivo do MVC é separar responsabilidades dentro da aplicação.

Isso deixa o código mais organizado, reutilizável, escalável e fácil de manter.

Sem MVC

Quando não utilizamos MVC, normalmente colocamos tudo dentro de um único arquivo:

  • Rotas
  • Regras de negócio
  • Banco de dados
  • Validações
  • Respostas

Isso faz o projeto crescer de forma desorganizada e difícil de manter.

Com MVC

No MVC cada parte possui uma responsabilidade:

Camada Responsabilidade
Model Estrutura e manipulação dos dados
View Interface visual
Controller Receber requisições e controlar regras

Entendendo o fluxo do MVC

  1. O cliente faz uma requisição
  2. Essa requisição chega na rota
  3. A rota chama o Controller
  4. O Controller executa regras de negócio
  5. O Controller conversa com o Model
  6. O Model acessa o banco de dados
  7. O banco retorna os dados
  8. O Controller devolve a resposta

O que é Model?

O Model é uma das partes mais importantes do MVC.

Ele representa os dados da aplicação e também a comunicação com o banco de dados.

O Model é responsável por:

  • Criar estrutura dos dados
  • Definir tipos dos campos
  • Validar informações
  • Salvar dados
  • Buscar dados
  • Atualizar dados
  • Remover dados

Pensando na prática

Imagine um sistema de usuários.

O Model define:

  • Quais informações o usuário possui
  • Como essas informações serão salvas
  • Quais campos são obrigatórios
  • Quais regras os dados precisam seguir

Exemplo de usuário


{
  "nome": "Maria",
  "email": "maria@email.com",
  "idade": 25
}

Nesse caso, o Model poderia definir:

  • nome precisa ser texto
  • email precisa ser único
  • idade precisa ser número

Model no Mongoose

No Node.js normalmente utilizamos o Mongoose para criar Models no MongoDB.

O Mongoose funciona como uma ponte entre Node.js e MongoDB.

Ele facilita:

  • Criação de schemas
  • Validação
  • Consultas
  • Manipulação de dados

Exemplo de Model


const mongoose = require('mongoose');

const usuarioSchema = new mongoose.Schema({
  nome: {
    type: String,
    required: true
  },

  email: {
    type: String,
    required: true,
    unique: true
  },

  idade: {
    type: Number
  }
});

const Usuario = mongoose.model(
  'Usuario',
  usuarioSchema
);

module.exports = Usuario;

Entendendo o código

  • mongoose.Schema() cria a estrutura dos dados
  • type define o tipo do campo
  • required define campo obrigatório
  • unique impede valores repetidos
  • mongoose.model() cria o model

O que é Controller?

O Controller é responsável por controlar o fluxo da aplicação.

Ele:

  • Recebe requisições
  • Executa regras de negócio
  • Chama Models
  • Retorna respostas

Exemplo de Controller


const Usuario = require('../models/Usuario');

const listarUsuarios = async (req, res) => {

  const usuarios = await Usuario.find();

  res.json(usuarios);
};

module.exports = {
  listarUsuarios
};

O que é View?

A View representa a interface visual da aplicação.

Em APIs REST normalmente trabalhamos apenas com JSON.

Porém em aplicações completas, a View pode ser:

  • HTML
  • React
  • Vue
  • Angular

O que é MongoDB?

MongoDB é um banco de dados NoSQL orientado a documentos.

Diferente de bancos relacionais, o MongoDB armazena dados em documentos JSON.

Banco relacional

Exemplo:

id nome idade
1 Maria 25

MongoDB


{
  "_id": "123abc",
  "nome": "Maria",
  "idade": 25
}

Estrutura do MongoDB

MongoDB Banco Relacional
Database Banco de dados
Collection Tabela
Document Registro

Vantagens do MongoDB

  • Flexibilidade
  • Alta performance
  • Facilidade com JSON
  • Ótimo para APIs
  • Integração excelente com JavaScript

O que é Mongoose?

O Mongoose é uma biblioteca utilizada para conectar Node.js ao MongoDB.

Ele permite:

  • Criar models
  • Executar queries
  • Validar dados
  • Organizar o código

Passo 6 — Criar conexão com banco

Dentro de src/database/database.js:


const mongoose = require('mongoose');

const conectarBanco = async () => {

  try {

    await mongoose.connect(
      process.env.MONGO_URL
    );

    console.log('MongoDB conectado');

  } catch (error) {

    console.log(error);
  }
};

module.exports = conectarBanco;

Passo 7 — Criar Model

Dentro de src/models/Usuario.js:


const mongoose = require('mongoose');

const usuarioSchema = new mongoose.Schema({
  nome: {
    type: String,
    required: true
  },

  email: {
    type: String,
    required: true,
    unique: true
  },

  idade: {
    type: Number
  }
});

const Usuario = mongoose.model(
  'Usuario',
  usuarioSchema
);

module.exports = Usuario;

Passo 8 — Criar Controller

Dentro de src/controllers/usuarioController.js: 


const Usuario = require('../models/Usuario');

const listarUsuarios = async (req, res) => {

  const usuarios = await Usuario.find();

  res.json(usuarios);
};

const criarUsuario = async (req, res) => {

  const usuario = await Usuario.create(
    req.body
  );

  res.status(201).json(usuario);
};

module.exports = {
  listarUsuarios,
  criarUsuario
};

Passo 9 — Criar Rotas

Dentro de src/routes/usuarioRoutes.js: 


const express = require('express');

const router = express.Router();

const {
  listarUsuarios,
  criarUsuario
} = require(
  '../controllers/usuarioController'
);

router.get(
  '/usuarios',
  listarUsuarios
);

router.post(
  '/usuarios',
  criarUsuario
);

module.exports = router;

Passo 10 — Criar app.js


const express = require('express');

const usuarioRoutes = require(
  './routes/usuarioRoutes'
);

const app = express();

app.use(express.json());

app.use(usuarioRoutes);

module.exports = app;

Passo 11 — Criar server.js


require('dotenv').config();

const app = require('./src/app');

const conectarBanco = require(
  './src/database/database'
);

conectarBanco();

app.listen(process.env.PORT, () => {

  console.log(
    'Servidor rodando'
  );
});

node server.js

🤓 Explicando o código 📝

🔹 mongoose.connect()

Faz a conexão do Node.js com o MongoDB.


await mongoose.connect(
  process.env.MONGO_URL
);

🔹 Schema

O Schema define a estrutura dos dados.


const usuarioSchema =
  new mongoose.Schema({});

🔹 Model

O Model permite acessar a collection do MongoDB.


const Usuario = mongoose.model(
  'Usuario',
  usuarioSchema
);

🔹 find()

Busca todos os documentos da collection.


Usuario.find();

🔹 create()

Cria um documento no MongoDB.


Usuario.create(req.body);

🔹 express.Router()

Permite separar rotas em arquivos diferentes.


const router = express.Router();

🔹 dotenv

O dotenv permite utilizar variáveis de ambiente.


require('dotenv').config();
