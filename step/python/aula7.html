O que é um CMS?

CMS (Content Management System) é um sistema que permite criar, editar, publicar e gerenciar conteúdo digital sem precisar escrever código toda vez. Exemplos famosos:

CMS Tipo Linguagem
WordPress Blogs e sites institucionais PHP
Shopify E-commerce Ruby
Django Admin Painel administrativo automático Python
Nosso mini CMS Gerenciamento de produtos (e-commerce) Python + Django

Por que construir o próprio CMS?

  • Controle total sobre o design e as funcionalidades
  • Sem dependência de plugins de terceiros
  • Personalização exata para as regras do seu negócio
  • Aprendizado profundo de como sistemas web funcionam

O que vamos construir?

Um mini CMS para e-commerce onde o administrador pode:

  • Listar todos os produtos cadastrados
  • Criar novos produtos com nome, preço, descrição, estoque e imagem
  • Editar informações de produtos existentes
  • Excluir produtos

Tudo isso protegido por login — somente usuários autenticados podem acessar o painel de gerenciamento.

Passo 2 — Criar o Model Produto

Em core/models.py, defina o model com os campos essenciais incluindo ImageField: 


# core/models.py
from django.db import models

class Produto(models.Model):
    nome = models.CharField(max_length=200)
    preco = models.DecimalField(max_digits=10, decimal_places=2)
    descricao = models.TextField(blank=True, null=True)
    estoque = models.PositiveIntegerField(default=0)
    imagem = models.ImageField(
        upload_to='produtos/',
        blank=True,
        null=True
    )
    ativo = models.BooleanField(default=True)
    criado_em = models.DateTimeField(auto_now_add=True)
    atualizado_em = models.DateTimeField(auto_now=True)

    def __str__(self):
        return self.nome

    class Meta:
        ordering = ['-criado_em']
Campo Tipo Descrição
nome CharField Nome do produto
preco DecimalField Preço com 2 casas decimais
descricao TextField Descrição longa (opcional)
estoque PositiveIntegerField Quantidade em estoque
imagem ImageField Foto salva em media/produtos/
ativo BooleanField Produto visível na loja

Passo 3 — Criar o Formulário (ModelForm)

Em core/forms.py:


# core/forms.py
from django import forms
from .models import Produto

class ProdutoForm(forms.ModelForm):
    class Meta:
        model = Produto
        fields = [
            'nome', 'preco', 'descricao',
            'estoque', 'imagem', 'ativo'
        ]
        widgets = {
            'descricao': forms.Textarea(
                attrs={'rows': 4}
            ),
            'preco': forms.NumberInput(
                attrs={'step': '0.01', 'min': '0'}
            ),
        }

Importante: formulários com upload de arquivo precisam de enctype="multipart/form-data" no HTML.

Passo 4 — Criar as Views do CRUD

Em core/views.py, implemente as quatro operações. Todas protegidas com @login_required: 


# core/views.py
from django.shortcuts import (
    render, redirect, get_object_or_404
)
from django.contrib.auth.decorators import login_required
from .models import Produto
from .forms import ProdutoForm

@login_required
def listar_produtos(request):
    produtos = Produto.objects.all()
    return render(
        request,
        'core/lista_produtos.html',
        {'produtos': produtos}
    )

@login_required
def criar_produto(request):
    if request.method == 'POST':
        form = ProdutoForm(request.POST, request.FILES)
        if form.is_valid():
            form.save()
            return redirect('listar_produtos')
    else:
        form = ProdutoForm()
    return render(
        request,
        'core/form_produto.html',
        {'form': form, 'titulo': 'Novo Produto'}
    )

@login_required
def editar_produto(request, pk):
    produto = get_object_or_404(Produto, pk=pk)
    if request.method == 'POST':
        form = ProdutoForm(
            request.POST, request.FILES, instance=produto
        )
        if form.is_valid():
            form.save()
            return redirect('listar_produtos')
    else:
        form = ProdutoForm(instance=produto)
    return render(
        request,
        'core/form_produto.html',
        {
            'form': form,
            'titulo': f'Editar: {produto.nome}',
            'produto': produto
        }
    )

@login_required
def excluir_produto(request, pk):
    produto = get_object_or_404(Produto, pk=pk)
    if request.method == 'POST':
        produto.delete()
        return redirect('listar_produtos')
    return render(
        request,
        'core/confirmar_exclusao.html',
        {'produto': produto}
    )

Passo 6 — Configurar as URLs e autenticação

No core/urls.py, defina as rotas do CRUD:


# core/urls.py
from django.urls import path
from . import views

urlpatterns = [
    path(
        '', views.listar_produtos,
        name='listar_produtos'
    ),
    path(
        'novo/', views.criar_produto,
        name='criar_produto'
    ),
    path(
        'editar/<int:pk>/',
        views.editar_produto,
        name='editar_produto'
    ),
    path(
        'excluir/<int:pk>/',
        views.excluir_produto,
        name='excluir_produto'
    ),
]

No urls.py principal, adicione as rotas de login/logout: 


# urls.py (principal)
from django.contrib.auth import views as auth_views

urlpatterns += [
    path(
        'login/',
        auth_views.LoginView.as_view(
            template_name='core/login.html'
        ),
        name='login'
    ),
    path(
        'logout/',
        auth_views.LogoutView.as_view(
            next_page='/login/'
        ),
        name='logout'
    ),
]

lembrete: o LOGIN_URL e LOGIN_REDIRECT_URL já estão configurados no settings.py desde o Passo 1.

Dever de Casa

Adicione busca de produtos por nome

Modifique a view listar_produtos para filtrar produtos pelo nome usando um parâmetro GET (?q=termo). Adicione o campo de busca no template.

Requisitos:

  • Busca via método GET
  • Filtrar por parte do nome com icontains
  • Campo vazio mostra todos os produtos
  • Termo buscado permanece no campo após a pesquisa
Gabarito — view: 

@login_required
def listar_produtos(request):
    query = request.GET.get('q', '')
    if query:
        produtos = Produto.objects.filter(
            nome__icontains=query
        )
    else:
        produtos = Produto.objects.all()
    return render(
        request,
        'core/lista_produtos.html',
        {'produtos': produtos, 'query': query}
    )
Gabarito — campo de busca no template: 

<form method="get" class="mb-4">
  <div class="input-group">
    <input type="text" name="q"
           class="form-control"
           placeholder="Buscar produto..."
           value="{{ query }}" />
    <button type="submit"
            class="btn btn-outline-primary">
      Buscar
    </button>
  </div>
</form>
