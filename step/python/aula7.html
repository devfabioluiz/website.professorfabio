CMS - Sistema de Gerenciamento de Produtos com Django

Tópicos do nosso plano de curso abordados nessa aula: Models e Banco de Dados, CRUD, Templates, Formulários (com upload de imagem), Views e Roteamento, Autenticação.

O que é um CMS?

CMS (Content Management System) é um sistema que permite criar, editar, publicar e gerenciar conteúdo digital sem precisar escrever código toda vez. Exemplos famosos:

CMS Tipo Linguagem
WordPress Blogs e sites institucionais PHP
Shopify E-commerce Ruby
Django Admin Painel administrativo automático Python
Nosso mini CMS Gerenciamento de produtos (e-commerce) Python + Django

Por que construir o próprio CMS?

  • Controle total sobre o design e as funcionalidades
  • Sem dependência de plugins de terceiros
  • Personalização exata para as regras do seu negócio
  • Aprendizado profundo de como sistemas web funcionam

O que vamos construir?

Um mini CMS para e-commerce onde o administrador pode:

  • Listar todos os produtos cadastrados
  • Criar novos produtos com nome, preço, descrição, estoque e imagem
  • Editar informações de produtos existentes
  • Excluir produtos

Tudo isso protegido por login — somente usuários autenticados podem acessar o painel de gerenciamento.

1. Configurando upload de imagens

Antes de criar o model, precisamos configurar o Django para lidar com arquivos enviados pelo usuário (imagens).

Instalar o Pillow

O Django usa a biblioteca Pillow para processar imagens. Instale no terminal: 


pip install Pillow

Configurar MEDIA_URL e MEDIA_ROOT

No settings.py, adicione estas duas configurações:


# settings.py

import os

# Onde os arquivos enviados serão salvos fisicamente no servidor
MEDIA_ROOT = os.path.join(BASE_DIR, 'media')

# URL pública para acessar os arquivos enviados
MEDIA_URL = '/media/'

2. Criando o Model Produto

Nosso model terá os campos essenciais de um produto de e-commerce, incluindo ImageField para a foto. 


# core/models.py
from django.db import models

class Produto(models.Model):
    nome = models.CharField(max_length=200)
    preco = models.DecimalField(max_digits=10, decimal_places=2)
    descricao = models.TextField(blank=True, null=True)
    estoque = models.PositiveIntegerField(default=0)
    imagem = models.ImageField(upload_to='produtos/', blank=True, null=True)
    ativo = models.BooleanField(default=True)
    criado_em = models.DateTimeField(auto_now_add=True)
    atualizado_em = models.DateTimeField(auto_now=True)

    def __str__(self):
        return self.nome

    class Meta:
        ordering = ['-criado_em']  # mais novos primeiro
        verbose_name = 'Produto'
        verbose_name_plural = 'Produtos'
Campo Tipo Descrição
nome CharField Nome do produto (até 200 caracteres)
preco DecimalField Preço com até 2 casas decimais
descricao TextField Descrição longa (opcional)
estoque PositiveIntegerField Quantidade em estoque
imagem ImageField Foto do produto salva em media/produtos/
ativo BooleanField Produto visível ou não na loja

Depois de criar o model, execute as migrações:


python manage.py makemigrations
python manage.py migrate

Registrar no Admin (opcional)

Para também gerenciar pelo Django Admin:

3. Criando o Formulário

Usaremos ModelForm para gerar automaticamente o formulário a partir do model, incluindo o campo de upload de imagem. 


# core/forms.py
from django import forms
from .models import Produto

class ProdutoForm(forms.ModelForm):
    class Meta:
        model = Produto
        fields = ['nome', 'preco', 'descricao', 'estoque', 'imagem', 'ativo']
        widgets = {
            'descricao': forms.Textarea(attrs={'rows': 4}),
            'preco': forms.NumberInput(attrs={'step': '0.01', 'min': '0'}),
        }
        labels = {
            'nome': 'Nome do Produto',
            'preco': 'Preço (R$)',
            'descricao': 'Descrição',
            'estoque': 'Quantidade em Estoque',
            'imagem': 'Foto do Produto',
            'ativo': 'Produto Ativo',
        }

Importante: para formulários com upload de arquivo, o HTML do form precisa ter o atributo enctype="multipart/form-data". Veremos isso nos templates.

4. Criando as Views do CRUD

Vamos criar as quatro operações fundamentais: Listar, Criar, Editar e Excluir.

4.1 Listar produtos


# core/views.py
from django.shortcuts import render, redirect, get_object_or_404
from django.contrib.auth.decorators import login_required
from .models import Produto
from .forms import ProdutoForm

@login_required
def listar_produtos(request):
    produtos = Produto.objects.all()
    return render(request, 'core/lista_produtos.html', {
        'produtos': produtos
    })

4.2 Criar produto


@login_required
def criar_produto(request):
    if request.method == 'POST':
        form = ProdutoForm(request.POST, request.FILES)
        if form.is_valid():
            form.save()
            return redirect('listar_produtos')
    else:
        form = ProdutoForm()
    return render(request, 'core/form_produto.html', {
        'form': form,
        'titulo': 'Novo Produto'
    })

request.FILES é o que permite receber o arquivo da imagem. Sem ele, o ImageField ficaria vazio.

4.3 Editar produto


@login_required
def editar_produto(request, pk):
    produto = get_object_or_404(Produto, pk=pk)
    if request.method == 'POST':
        form = ProdutoForm(request.POST, request.FILES, instance=produto)
        if form.is_valid():
            form.save()
            return redirect('listar_produtos')
    else:
        form = ProdutoForm(instance=produto)
    return render(request, 'core/form_produto.html', {
        'form': form,
        'titulo': f'Editar: {produto.nome}',
        'produto': produto
    })

Passar instance=produto faz o Django preencher o formulário com os dados atuais do produto. Ao salvar, ele atualiza o registro existente em vez de criar um novo.

4.4 Excluir produto


@login_required
def excluir_produto(request, pk):
    produto = get_object_or_404(Produto, pk=pk)
    if request.method == 'POST':
        produto.delete()
        return redirect('listar_produtos')
    return render(request, 'core/confirmar_exclusao.html', {
        'produto': produto
    })

A exclusão usa POST (nunca GET) para evitar exclusões acidentais por links ou bots. O template de confirmação deve ter um formulário com método POST e um botão de confirmação.

from django.urls import path

    path('', views.listar_produtos, name='listar_produtos'),

URLs de autenticação (login/logout)

No urls.py principal do projeto, adicione as rotas de autenticação: 


# urls.py (principal)
from django.contrib import admin
from django.urls import path, include
from django.contrib.auth import views as auth_views

urlpatterns = [
    path('admin/', admin.site.urls),
    path('', include('core.urls')),
    path('login/', auth_views.LoginView.as_view(
        template_name='core/login.html'
    ), name='login'),
    path('logout/', auth_views.LogoutView.as_view(
        next_page='/login/'
    ), name='logout'),
]

7. Fluxo completo do CMS

Diagrama de navegação

URL View Template Protegido?
/ listar_produtos lista_produtos.html Sim
/novo/ criar_produto form_produto.html Sim
/editar/1/ editar_produto form_produto.html Sim
/excluir/1/ excluir_produto confirmar_exclusao.html Sim
/login/ LoginView login.html Não
/logout/ LogoutView - Sim

Fluxo de uso do CMS

  1. Usuário acessa / → redirecionado para /login/
  2. Faz login → vai para a lista de produtos
  3. Clica em "+ Novo Produto" → preenche o formulário com imagem → Salvar
  4. Volta para a lista e vê o novo produto com a foto
  5. Clica em "Editar" → altera preço ou imagem → Salvar
  6. Clica em "Excluir" → confirma → produto removido
  7. Clica em "Sair" → volta para a tela de login

Dever de Casa

Adicione um sistema de busca de produtos

Modifique a view listar_produtos para aceitar um parâmetro de busca via GET (?q=termo) e filtre os produtos pelo nome. Adicione também um campo de busca no template lista_produtos.html.

Requisitos:

  • O campo de busca deve usar método GET
  • A busca deve filtrar por parte do nome (usando icontains)
  • Se o campo estiver vazio, deve mostrar todos os produtos
  • O termo buscado deve permanecer no campo após a pesquisa
Gabarito — view atualizada: 

# core/views.py - listar_produtos atualizada
@login_required
def listar_produtos(request):
    query = request.GET.get('q', '')
    if query:
        produtos = Produto.objects.filter(nome__icontains=query)
    else:
        produtos = Produto.objects.all()
    return render(request, 'core/lista_produtos.html', {
        'produtos': produtos,
        'query': query
    })
Gabarito — form de busca no template: 

<!-- adicione antes da tabela em lista_produtos.html -->
<form method="get" class="mb-4">
  <div class="input-group">
    <input type="text" name="q" class="form-control"
           placeholder="Buscar produto..."
           value="{{ query }}" />
    <button type="submit" class="btn btn-outline-primary">
      Buscar
    </button>
    {% if query %}
      <a href="{% url 'listar_produtos' %}" class="btn btn-outline-secondary">
        Limpar
      </a>
    {% endif %}
  </div>
</form>
