Padrão MVC (Model-View-Controller)

Arquitetura de Sistemas

Arquitetura define como os componentes de um sistema são organizados e como eles se comunicam.

Padrão MVC

1. O que é o padrão MVC?

MVC (Model-View-Controller) é um padrão que separa responsabilidades em três camadas.

2. Objetivos do MVC

  • Separar responsabilidades
  • Facilitar manutenção
  • Organizar código

3. Model

a. O que é? Representa os dados

b. Responsabilidades: Gerenciar dados e regras de negócio

4. View

a. O que é? Interface com o usuário

b. Responsabilidades: Exibir dados

5. Controller

a. O que é? Controla fluxo da aplicação

b. Responsabilidades: Receber requisições e chamar Model e View

6. Exemplo MVC

  • Model → Banco de dados
  • View → Frontend
  • Controller → API

    Passo 5 - View (resposta)

    Crie o arquivo: app/views/usuario_view.py e insira dentro dele:

    

def resposta_sucesso(msg):

    Passo 6 - Controller (regras)

    Crie o arquivo: app/controllers/usuario_controller.py e insira dentro dele:

    
from app.models import usuario_model
from app.views import usuario_view


def listar_usuarios():
    return usuario_model.listar()


def criar_usuario(nome):
    usuario_model.criar(nome)
    return usuario_view.resposta_sucesso("Usuário criado")

    Passo 7 - Integrar com FastAPI

    Dentro do arquivo main.pyinsira o código abaixo:

    
from fastapi import FastAPI
from app.controllers import usuario_controller

app = FastAPI()

@app.get("/usuarios")
def listar():
    return usuario_controller.listar_usuarios()

@app.post("/usuarios")
def criar(nome: str):
    return usuario_controller.criar_usuario(nome)

    Antes de tudo, você precisa instalar as bibliotecas necessárias para rodar sua API.

    Passo 10 - Crie a estrutura do projeto

    Organize seus arquivos para manter o projeto limpo e escalável:

     project/ │── main.py │── database.py │── models.py │── schemas.py

    Cada arquivo terá uma responsabilidade específica (boa prática profissional).

    Passo 11 - Configure a conexão com o Supabase

    Crie o arquivo database.py para centralizar a conexão com o banco.

    Copie sua URL e chave no painel do Supabase.

     from supabase import create_client

SUPABASE_URL = "https://SEU-PROJETO.supabase.co
"
SUPABASE_KEY = "SUA-CHAVE-ANON"

supabase = create_client(SUPABASE_URL, SUPABASE_KEY)

    Esses dados ficam no painel do Supabase em Settings → API.

    Passo 12 - Crie o model da aplicação

    O model representa como os dados são estruturados dentro da aplicação.

    Crie o arquivo models.py:

     class UserModel: def __init__(self, name: str, email: str): self.name = name self.email = email
def to_dict(self):
    return {
        "name": self.name,
        "email": self.email
    }

    Esse método to_dict() é importante para enviar os dados ao Supabase.

    Passo 13 - Crie o schema para validação

    Os schemas garantem que os dados recebidos pela API estejam corretos.

    Crie o arquivo schemas.py:

     from pydantic import BaseModel

class UserCreate(BaseModel):
name: str
email: str

    Se o usuário enviar dados inválidos, o FastAPI já retorna erro automaticamente.

    Agora criamos o arquivo principal da aplicação: main.py.

     from fastapi import FastAPI

app = FastAPI()

@app.get("/")
def home():
return {"message": "API rodando 🚀"}

    Esse endpoint serve apenas para testar se a API está funcionando.

    Passo 15 - Crie o endpoint para inserir dados

    Vamos criar um endpoint para cadastrar usuários no banco.

     from database import supabase from models import UserModel from schemas import UserCreate

@app.post("/users")
def create_user(user: UserCreate):
user_model = UserModel(user.name, user.email)

response = supabase.table("users").insert(user_model.to_dict()).execute()

return response.data

    Fluxo:

    1. Recebe dados → 2. Valida → 3. Converte → 4. Salva no banco

    Passo 16 - Crie o endpoint para listar dados

    Agora vamos buscar os dados do banco:

     @app.get("/users") def get_users(): response = supabase.table("users").select("*").execute() return response.data

    Esse endpoint retorna todos os usuários cadastrados.

    Passo 17 - Crie o banco de dados da nossa aplicação

    Adicione as seguintes colunas:

    - id → tipo: integer, habilitar auto-increment e primary key

    - name → tipo: text

    - email → tipo: text

    Salve a tabela para que a API consiga inserir dados nela.

    Agora rode sua API com o comando:

    Acesse no navegador:

    Lá você pode testar todos os endpoints de forma interativa.

    Passo 19 - Entenda o fluxo completo

    Agora sua aplicação funciona assim:

    1. Cliente envia requisição (POST /users)

    2. FastAPI recebe os dados

    3. Schema valida

    4. Model organiza

    5. Supabase salva no banco

    6. API retorna resposta

    7. GET /users busca os dados salvos
