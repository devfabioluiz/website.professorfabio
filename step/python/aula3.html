Passo 5 - Model

Acesse o arquivo usuario_model.py e substitua o código por este: 


import requests
import bcrypt
from app.database.supabase_client import SUPABASE_URL, SUPABASE_KEY
import jwt
from datetime import datetime, timedelta
import os
import os
from datetime import datetime, timedelta
from dotenv import load_dotenv

load_dotenv()

SUPABASE_URL = os.getenv("SUPABASE_URL")
SUPABASE_KEY = os.getenv("SUPABASE_KEY")
SECRET_KEY = os.getenv("SECRET_KEY")
EXPIRATION_MINUTES = int(os.getenv("EXPIRATION_MINUTES", 60))

ALGORITHM = "HS256"

def get_headers(extra_headers=None):
    headers = {
        "apikey": SUPABASE_KEY,
        "Authorization": f"Bearer {SUPABASE_KEY}",
        "Content-Type": "application/json"
    }

    if extra_headers:
        headers.update(extra_headers)

    return headers


def listar_usuarios():
    url = f"{SUPABASE_URL}/usuarios?select=*"

    response = requests.get(url, headers=get_headers())

    if response.status_code != 200:
        raise Exception(f"Erro ao listar usuários: {response.text}")

    return response.json()


def buscar_usuario_por_email(email):
    url = f"{SUPABASE_URL}/usuarios?email=eq.{email}&select=*"

    response = requests.get(url, headers=get_headers())

    if response.status_code != 200:
        raise Exception(f"Erro ao buscar usuário: {response.text}")

    usuarios = response.json()

    return usuarios[0] if usuarios else None


def cadastrar_usuario(email, senha):
    if buscar_usuario_por_email(email):
        raise Exception("Já existe um usuário cadastrado com este email.")

    senha_hash = bcrypt.hashpw(
        senha.encode("utf-8"),
        bcrypt.gensalt()
    ).decode("utf-8")

    url = f"{SUPABASE_URL}/usuarios"

    data = {
        "email": email,
        "senha": senha_hash
    }

    response = requests.post(
        url,
        json=data,
        headers=get_headers({"Prefer": "return=representation"})
    )

    if response.status_code not in [200, 201]:
        raise Exception(f"Erro ao cadastrar usuário: {response.text}")

    usuario_criado = response.json()[0]

    return {
        "mensagem": "Usuário cadastrado com sucesso.",
        "usuario": {
            "id": usuario_criado.get("id"),
            "email": usuario_criado.get("email")
        }
    }

def gerar_token(usuario_id, email):
    payload = {
        "sub": usuario_id,
        "email": email,
        "exp": datetime.utcnow() + timedelta(minutes=EXPIRATION_MINUTES)
    }

    token = jwt.encode(payload, SECRET_KEY, algorithm=ALGORITHM)
    return token


def login_usuario(email, senha):
    usuario = buscar_usuario_por_email(email)

    if not usuario:
        raise Exception("Email ou senha inválidos.")

    senha_hash = usuario.get("senha")

    if not bcrypt.checkpw(
        senha.encode("utf-8"),
        senha_hash.encode("utf-8")
    ):
        raise Exception("Email ou senha inválidos.")

    token = gerar_token(usuario.get("id"), usuario.get("email"))

    return {
        "mensagem": "Login realizado com sucesso.",
        "token": token,
        "usuario": {
            "id": usuario.get("id"),
            "email": usuario.get("email")
        }
    }

🤓 Explicando o código 📝

Esse código implementa um pequeno serviço de autenticação de usuários usando Python, Supabase, bcrypt e JWT. A ideia principal é permitir que a aplicação consiga cadastrar usuários, salvar suas senhas de forma segura, buscar usuários no banco de dados, validar login e gerar um token de autenticação.

No início do código, são importadas algumas bibliotecas importantes. A biblioteca requests é usada para fazer requisições HTTP para a API REST do Supabase. A biblioteca bcrypt é usada para gerar e verificar senhas criptografadas. Também são utilizados jwt, datetime e timedelta para geração e controle de expiração de tokens.

O código define constantes importantes como a chave secreta (SECRET_KEY), o algoritmo de criptografia (HS256) e o tempo de expiração do token (60 minutos). Em um sistema real, essa chave secreta deveria ficar em variáveis de ambiente por segurança.

🔹 Função get_headers

Essa função monta os cabeçalhos das requisições HTTP para o Supabase, incluindo autenticação e tipo de conteúdo.

  • apikey: identifica o projeto
  • Authorization: autenticação no formato Bearer
  • Content-Type: define JSON como formato

Ela também permite adicionar cabeçalhos extras dinamicamente, o que é útil em requisições específicas.

🔹 Função listar_usuarios

Essa função busca todos os usuários cadastrados na tabela usuarios do Supabase usando uma requisição GET. Caso haja erro, uma exceção é lançada. Caso contrário, retorna os dados em JSON.

🔹 Função buscar_usuario_por_email

Essa função busca um usuário específico pelo email usando o filtro email=eq.valor. O Supabase retorna uma lista, então o código retorna apenas o primeiro usuário encontrado ou None se não existir.

🔹 Função cadastrar_usuario

Responsável por cadastrar um novo usuário:

  • Verifica se o email já existe
  • Gera um hash seguro da senha com bcrypt
  • Salva os dados no Supabase
  • Retorna os dados do usuário sem expor a senha

O uso do bcrypt garante que a senha nunca seja armazenada em texto puro, aumentando significativamente a segurança.

🔹 Função gerar_token

Essa função cria um token JWT contendo:

  • sub: ID do usuário
  • email: email do usuário
  • exp: tempo de expiração

O token é assinado com uma chave secreta e pode ser usado para autenticar o usuário em requisições futuras.

🔹 Função login_usuario

Essa função realiza o login:

  • Busca o usuário pelo email
  • Compara a senha com bcrypt
  • Gera um token JWT se válido
  • Retorna dados do usuário + token

Um detalhe importante é que a mensagem de erro é sempre a mesma para evitar que alguém descubra quais emails estão cadastrados no sistema.

📌 Resumo geral

  1. Usuário é cadastrado com senha criptografada
  2. Login valida senha usando bcrypt
  3. Token JWT é gerado após autenticação
  4. Token pode ser usado em rotas protegidas

Em resumo, esse código implementa uma camada de autenticação completa, combinando segurança de senha (bcrypt), comunicação com banco via API (Supabase) e autenticação stateless com JWT.

Passo 7 — Middleware Token

Agora acesse o arquivo auth.py e coloque o código abaixo nele: 


from fastapi import Header, HTTPException

API_KEY = "123456"

def verificar_token(authorization: str = Header(None)):
    if not authorization:
        raise HTTPException(status_code=401, detail="Token não enviado")

    token = authorization.replace("Bearer ", "")

    if len(token) < 10:
        raise HTTPException(status_code=401, detail="Token inválido")

    return token

def verificar_api_key(x_api_key: str = Header(None)):
    if x_api_key != API_KEY:
        raise HTTPException(status_code=403, detail="API Key inválida")

Passo 8 — Rotas

Agora acesse o arquivo main.py e substitua pelo código abaixo: 


from fastapi import FastAPI, Depends
from app.controllers import usuario_controller
from app.schemas.usuario_schema import UsuarioCreate, Login
from app.auth.auth import verificar_token

app = FastAPI()

@app.get("/")
def home():
    return {"msg": "API rodando"}

@app.get("/usuarios")
def listar_usuarios():
    return usuario_controller.listar()

@app.post("/cadastro")
def cadastro(user: UsuarioCreate):
    return usuario_controller.cadastrar(user.email, user.senha)

@app.post("/login")
def login(user: Login):
    return usuario_controller.login(user.email, user.senha)

@app.get("/perfil")
def perfil(token: str = Depends(verificar_token)):
    return {"msg": "Usuário autenticado"}
