Django ORM na prática

1. Definindo Models

Um Model é uma classe Python que herda de models.Model. Cada atributo da classe representa uma coluna no banco. 


from django.db import models

class Produto(models.Model):
    nome = models.CharField(max_length=100)
    preco = models.DecimalField(max_digits=8, decimal_places=2)
    estoque = models.IntegerField(default=0)
    ativo = models.BooleanField(default=True)
    criado_em = models.DateTimeField(auto_now_add=True)

    def __str__(self):
        return self.nome

5. Relacionamentos entre Models

O ORM do Django permite criar relacionamentos entre tabelas de forma simples, usando campos especiais.

ForeignKey (1:N — Um para Muitos)

Exemplo: Uma Categoria pode ter vários Produtos, mas cada Produto pertence a apenas uma Categoria. 


class Categoria(models.Model):
    nome = models.CharField(max_length=100)

    def __str__(self):
        return self.nome

class Produto(models.Model):
    nome = models.CharField(max_length=100)
    preco = models.DecimalField(max_digits=8, decimal_places=2)
    categoria = models.ForeignKey(Categoria, on_delete=models.CASCADE)

    def __str__(self):
        return self.nome

O parâmetro on_delete define o que acontece quando o registro pai é deletado:

  • models.CASCADE — deleta os filhos junto com o pai
  • models.PROTECT — impede deletar o pai se houver filhos
  • models.SET_NULL — define o campo como NULL nos filhos
  • models.SET_DEFAULT — define o valor padrão nos filhos

ManyToManyField (N:N — Muitos para Muitos)

Exemplo: Um Aluno pode se matricular em várias Turmas, e cada Turma pode ter vários Alunos. 


class Aluno(models.Model):
    nome = models.CharField(max_length=100)

class Turma(models.Model):
    nome = models.CharField(max_length=100)
    alunos = models.ManyToManyField(Aluno)

Para usar o relacionamento ManyToMany:


turma = Turma.objects.get(id=1)
aluno = Aluno.objects.get(id=1)

turma.alunos.add(aluno)            # adiciona aluno na turma
turma.alunos.all()                 # lista todos os alunos da turma
aluno.turma_set.all()              # lista todas as turmas do aluno
turma.alunos.remove(aluno)         # remove aluno da turma

Django Admin

O Django Admin é um painel administrativo gerado automaticamente que permite gerenciar os dados da aplicação por meio de uma interface web. Ele é um dos recursos mais poderosos do Django, pois com poucas linhas de código você tem um CRUD visual completo para seus Models.

Para que serve o Admin?

  • Cadastrar, editar e excluir registros do banco de dados
  • Gerenciar usuários e permissões
  • Visualizar dados de forma organizada com filtros e buscas
  • Administrar o conteúdo do site sem precisar escrever queries SQL
  • Prototipar rapidamente a administração de dados

Registrando Models no Admin

Para que um Model apareça no painel Admin, é necessário registrá-lo no arquivo admin.py do seu app.

Registro simples:


# admin.py
from django.contrib import admin
from .models import Produto, Categoria

admin.site.register(Produto)
admin.site.register(Categoria)

Registro com personalização (recomendado):


# admin.py
from django.contrib import admin
from .models import Produto

@admin.register(Produto)
class ProdutoAdmin(admin.ModelAdmin):
    list_display = ('nome', 'preco', 'estoque', 'ativo', 'criado_em')
    list_filter = ('ativo', 'criado_em')
    search_fields = ('nome',)
    ordering = ('-criado_em',)
    list_per_page = 20

Autenticação e Login no Django

O Django inclui um sistema de autenticação completo pronto para uso. Ele fornece models de usuário, views de login/logout, formulários e todo o fluxo de autenticação.

Componentes do sistema de autenticação

  • User Model — django.contrib.auth.models.User (usuário, senha, e-mail, permissões)
  • Authenticate — verifica se as credenciais são válidas
  • Login — cria uma sessão para o usuário autenticado
  • Logout — encerra a sessão do usuário
  • LoginRequiredMixin / @login_required — protege views para usuários logados

Fluxo de autenticação

  1. Usuário acessa a página de login e preenche usuário e senha
  2. Django chama authenticate() para validar as credenciais
  3. Se válidas, login() cria a sessão
  4. Usuário é redirecionado para a página principal
  5. Em views protegidas, @login_required verifica a sessão

Login manual (view personalizada)


# views.py
from django.shortcuts import render, redirect
from django.contrib.auth import authenticate, login, logout
from django.contrib.auth.decorators import login_required
from django.contrib import messages

def fazer_login(request):
    if request.method == 'POST':
        username = request.POST.get('username')
        senha = request.POST.get('senha')

        usuario = authenticate(request, username=username, password=senha)

        if usuario is not None:
            login(request, usuario)
            return redirect('pagina_inicial')
        else:
            messages.error(request, 'Usuário ou senha inválidos.')

    return render(request, 'login.html')

@login_required
def pagina_inicial(request):
    return render(request, 'inicial.html')

def fazer_logout(request):
    logout(request)
    return redirect('login')

Passo 6 — Criar as Views (com Login)

Arquivo: loja/views.py


from django.shortcuts import render, redirect
from django.contrib.auth import authenticate, login, logout
from django.contrib.auth.decorators import login_required
from django.contrib import messages
from .models import Produto

def fazer_login(request):
    if request.method == 'POST':
        username = request.POST.get('username')
        senha = request.POST.get('senha')

        usuario = authenticate(request, username=username, password=senha)

        if usuario is not None:
            login(request, usuario)
            return redirect('pagina_inicial')
        else:
            messages.error(request, 'Usuário ou senha inválidos.')

    return render(request, 'loja/login.html')

@login_required
def pagina_inicial(request):
    produtos = Produto.objects.filter(ativo=True).order_by('nome')
    return render(request, 'loja/inicial.html', {'produtos': produtos})

def fazer_logout(request):
    logout(request)
    return redirect('login')

1. Explique o que é ORM com suas palavras

Escreva um pequeno parágrafo explicando o que é Object-Relational Mapping, qual problema ele resolve e cite pelo menos duas vantagens.

Gabarito (resposta sugerida):

ORM (Object-Relational Mapping) é uma técnica que permite manipular bancos de dados usando classes e objetos da linguagem de programação, em vez de escrever SQL diretamente. Ele resolve o problema de ter que misturar SQL com código Python e de ter que reescrever queries ao trocar de banco de dados. Vantagens: evita SQL Injection, aumenta a produtividade, permite trocar de banco sem reescrever código e torna o código mais legível.

4. Relacionamento ForeignKey

Crie dois Models: Autor (nome, email) e Livro (titulo, ano_publicacao, autor). O campo autor em Livro deve ser uma ForeignKey para Autor com on_delete=models.CASCADE. Em seguida, registre ambos no Admin.

Gabarito — models.py: 

from django.db import models

class Autor(models.Model):
    nome = models.CharField(max_length=100)
    email = models.EmailField()

        return self.nome

class Livro(models.Model):
    titulo = models.CharField(max_length=200)
    ano_publicacao = models.IntegerField()
