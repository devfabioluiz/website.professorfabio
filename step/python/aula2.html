REST Api e Configurando uma aplicação com SQLite e FastAPI

O que é REST?

REST (Representational State Transfer) é um padrão arquitetural utilizado para comunicação entre sistemas através do protocolo HTTP.

Principais características do REST

  • Utiliza HTTP
  • Baseado em recursos (resources)
  • Sem estado (stateless)
  • Utiliza JSON para troca de dados

Métodos HTTP no REST

Método Descrição
GET Buscar dados
POST Criar dados
PUT Atualizar dados
DELETE Remover dados

Exemplo REST com FastAPI


from fastapi import FastAPI

app = FastAPI()

usuarios = []

@app.get("/usuarios")
def listar():
    return usuarios

@app.post("/usuarios")
def criar(nome: str):
    usuarios.append(nome)
    return {"msg": "criado"}

@app.put("/usuarios/{id}")
def atualizar(id: int, nome: str):
    usuarios[id] = nome
    return {"msg": "atualizado"}

@app.delete("/usuarios/{id}")
def deletar(id: int):
    usuarios.pop(id)
    return {"msg": "deletado"}

Passo 5 - Criar banco e tabela automaticamente

Coloque o código abaixo dentro do arquivo app/database/db.py

Importante: Esse código precisa ser executado para criar a tabela. Vamos garantir isso no próximo passo.

          
import sqlite3

conn = sqlite3.connect("database.db")
cursor = conn.cursor()

cursor.execute("""
CREATE TABLE IF NOT EXISTS usuarios (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    nome TEXT,
    idade INTEGER
)
""")

conn.commit()
conn.close()

Passo 6 - Criar API com FastAPI

Coloque o código abaixo dentro do arquivo main.py

Importante: Vamos importar o arquivo db.py para garantir que a tabela seja criada automaticamente ao iniciar a aplicação.

          
from fastapi import FastAPI
import sqlite3

# IMPORTANTE: executa a criação do banco/tabela
import app.database.db

app = FastAPI()

def get_connection():
    return sqlite3.connect("database.db")

@app.get("/")
def home():
    return {"msg": "API rodando"}

@app.get("/usuarios")
def listar_usuarios():
    conn = get_connection()
    cursor = conn.cursor()

    cursor.execute("SELECT * FROM usuarios")
    dados = cursor.fetchall()

    conn.close()
    return dados

@app.post("/usuarios")
def criar_usuario(nome: str, idade: int):
    conn = get_connection()
    cursor = conn.cursor()

    cursor.execute(
        "INSERT INTO usuarios (nome, idade) VALUES (?, ?)",
        (nome, idade)
    )

    conn.commit()
    conn.close()

    return {"msg": "Usuário criado"}

Passo 7 - Rodar a aplicação

Use o comando abaixo no Command Prompt (cmd) no Windows para iniciar o servidor FastAPI. O parâmetro --reload é importante para desenvolvimento, pois reinicia o servidor automaticamente sempre que você fizer uma alteração no código.

          
uvicorn main:app --reload

Caso haja algum erro: Se aparecer a mensagem 'uvicorn' não é reconhecido, significa que o ambiente virtual não está ativado ou o comando não está no PATH.

Solução 1 - Ativar ambiente virtual

          
venv\Scripts\activate

Solução 2 - Rodar com Python diretamente

Se ainda não funcionar, use o comando abaixo:

          
python -m uvicorn main:app --reload

Passo 9 - Testando a API com Postman

Agora vamos testar nossa API utilizando o Postman, uma ferramenta muito utilizada no mercado para testar requisições HTTP.

Instalar o Postman

Baixe e instale o Postman no seu computador através do link abaixo:

https://www.postman.com/downloads/

Teste GET - Listar usuários

  • Abra o Postman
  • Selecione o método GET
  • Digite a URL: http://localhost:8000/usuarios
  • Clique em Send

Você verá a lista de usuários cadastrados.

Teste POST - Criar usuário

  • No Postman, selecione o método POST
  • Digite a URL: http://localhost:8000/usuarios?nome=Fabio&idade=30
  • Clique em Send

Você verá a mensagem de sucesso indicando que o usuário foi criado.

Testar novamente o GET

Após criar um usuário, repita o teste GET para ver o novo registro inserido no banco.
