Arquitetura de Sistemas

Arquitetura de software define como os componentes de um sistema são organizados e como eles se comunicam entre si.

Uma boa arquitetura permite que o sistema seja mais fácil de manter, evoluir e escalar.

Padrão MVC

1. O que é MVC?

MVC significa Model - View - Controller.

Esse padrão organiza o sistema em três camadas com responsabilidades diferentes.

2. Objetivos do MVC

  • Separar responsabilidades
  • Organizar melhor o código
  • Facilitar manutenção
  • Facilitar crescimento da aplicação

3. Model

Responsabilidade: lidar com os dados.

  • Acessar banco de dados
  • Criar registros
  • Buscar registros
  • Atualizar ou deletar dados

4. View

Responsabilidade: formatar a resposta.

  • Organizar o retorno da API
  • Padronizar respostas

5. Controller

Responsabilidade: controlar o fluxo da aplicação.

  • Receber requisições
  • Chamar models
  • Retornar resposta via view

Exemplo simplificado

  • Model → banco de dados
  • Controller → lógica da API
  • View → resposta da API

Passo 4 — Conexão com Supabase

Entre no arquivo supabase_client.py e coloque as credenciais: 


# app/database/supabase_client.py
import requests

SUPABASE_URL = "https://xyzcompany.supabase.co"
SUPABASE_KEY = "sb_secret_XXXX"

supabase = {
    "url": SUPABASE_URL,
    "key": SUPABASE_KEY
}

# ou se quiser criar um helper para requests
def supabase_request(path, method="GET", data=None):
    headers = {
        "apikey": SUPABASE_KEY,
        "Authorization": f"Bearer {SUPABASE_KEY}",
        "Content-Type": "application/json",
        "Prefer": "return=representation"
    }
    url = f"{SUPABASE_URL}/{path}"
    if method.upper() == "GET":
        return requests.get(url, headers=headers)
    elif method.upper() == "POST":
        return requests.post(url, json=data, headers=headers)

Passo 6 — Model (acesso aos dados)

Entre no arquivo app/models/usuario_model.py e coloque o código abaixo: 


from app.database.supabase_client import supabase_request
def listar_usuarios():
    response = supabase_request(
        "/usuarios?select=*",
        method="GET"
    )
    if response.status_code != 200:
        raise Exception(f"Erro ao listar usuários: {response.text}")
    return response.json()

def criar_usuario(nome):
    response = supabase_request(
        "/usuarios",
        method="POST",
        data={"nome": nome}
    )
    if response.status_code not in (200, 201):
        raise Exception(f"Erro ao criar usuário: {response.text}")
    return response.json()

Passo 8 — Controller

Entre no arquivo app/controllers/usuario_controller.py e coloque o código abaixo: 



from app.models import usuario_model
from app.views import usuario_view

def listar():
usuarios = usuario_model.listar_usuarios()
return usuario_view.resposta_sucesso("Lista de usuários", usuarios)

def criar(nome):
usuario = usuario_model.criar_usuario(nome)
return usuario_view.resposta_sucesso("Usuário criado", usuario)

Passo 9 — Integrar com FastAPI

Entre no arquivo main.py e coloque o código abaixo: 



from fastapi import FastAPI
from app.controllers import usuario_controller
from app.schemas.usuario_schema import UsuarioCreate

app = FastAPI()

@app.get("/")
def home():
return {"msg": "API funcionando 🚀"}

@app.get("/usuarios")
def listar_usuarios():
return usuario_controller.listar()

@app.post("/usuarios")
def criar_usuario(usuario: UsuarioCreate):
return usuario_controller.criar(usuario.nome)

Passo 13 - Evitar deixar expostos seus dados de acesso

Para evitarmos problemas de acessos indevidos ou o próprio github não deixar subirmos nossa aplicação por estar expondo seus dados de acesso ao supabase, faremos a inserção dos dados pela criação do que chamamos de variável de ambiente (.env) 

pip install python-dotenv
type nul > .env

Agora dentro do arquivo .env coloque o código abaixo:

SUPABASE_URL = "https://xyzcompany.supabase.co"
SUPABASE_KEY = "sb_secret_XXXX"

Agora adicione o .env no seu arquivo .gitignore

Agora adicione em supabase_client.py o código abaixo:


# app/database/supabase_client.py
