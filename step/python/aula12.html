Integração Python + React: FastAPI e Django como Backend

Tópicos do nosso plano de curso abordados nessa aula: Criação de APIs REST, FastAPI, Django REST Framework

Introdução: Python como Backend para React

Nas aulas anteriores, aprendemos a criar APIs REST com FastAPI (Aula 1) e Django REST Framework (Aulas 4 e 9).

Nesta aula vamos conectar esses mundos: usar Python como backend para React. Você verá que o React não se importa com a tecnologia do servidor — ele só precisa de uma API REST que retorne JSON.

Vamos construir a mesma API de duas formas diferentes:

  • Opção A — FastAPI: leve, moderno, mínimo de código
  • Opção B — Django REST Framework: completo, cheio de recursos, ORM incluso

O frontend React será o mesmo para ambas as opções — provando que a escolha do backend não afeta o frontend.

🤓 FastAPI vs Django REST Framework — Quando usar cada um?

Use FastAPI quando:

  • Quer uma API simples e rápida de criar
  • Precisa de alta performance (async nativo)
  • Quer documentação automática (Swagger /docs)
  • O projeto é uma API pura (sem frontend integrado)
  • Valoriza tipo e validação automática com Pydantic

Use Django REST Framework quando:

  • Já tem um projeto Django e quer adicionar uma API
  • Precisa de um admin nativo (django-admin)
  • Quer um ORM maduro e migrations automáticas
  • O sistema envolve modelos de dados complexos com relacionamentos
  • Precisa de autenticação, permissões e filtros já prontos

O que vamos construir

Vamos criar uma API com uma única rota que retorna mensagens:

GET /api/mensagens

[
  { "id": 1, "titulo": "Bem-vindo!", "texto": "Python + React funcionando!" },
  { "id": 2, "titulo": "Integração", "texto": "FastAPI e Django servindo React" },
  { "id": 3, "titulo": "Sucesso", "texto": "Mesmo frontend, dois backends!" }
]

O frontend React vai consumir essa API e exibir as mensagens na tela.

Fluxo da comunicação:

  1. O usuário acessa a página React no navegador
  2. O React faz uma requisição fetch para /api/mensagens
  3. O Vite proxy redireciona para o backend Python (FastAPI ou Django)
  4. O servidor retorna os dados em JSON
  5. O React atualiza a interface com as mensagens recebidas

pip freeze > requirements.txt

Passo 4 — Criar o main.py

type nul > main.py

Abra o main.py e insira o código:

from fastapi import FastAPI
from fastapi.middleware.cors import CORSMiddleware

app = FastAPI()

app.add_middleware(
    CORSMiddleware,
    allow_origins=["*"],
    allow_methods=["*"],
    allow_headers=["*"],
)

MENSAGENS = [
    {"id": 1, "titulo": "Bem-vindo!", "texto": "Python + React com FastAPI!"},
    {"id": 2, "titulo": "Integração", "texto": "Front-end e Back-end conectados com sucesso!"},
    {"id": 3, "titulo": "FastAPI + React", "texto": "Trabalhando juntos com dados em JSON."},
]

@app.get("/api/mensagens")
def listar_mensagens():
    return MENSAGENS

🤓 Explicando o código do FastAPI

from fastapi import FastAPI

Importa o framework FastAPI para criar o servidor web.

CORSMiddleware

Libera o acesso de outras origens (como o React em localhost:5173) à nossa API. Sem isso, o navegador bloquearia as requisições por segurança.

MENSAGENS

Array fixo simulando um banco de dados. Em projetos reais, esses dados viriam de um banco SQLite/PostgreSQL com SQLAlchemy.

@app.get("/api/mensagens")

Define uma rota GET. Quando o frontend acessar /api/mensagens, o FastAPI retorna o array em formato JSON automaticamente.

Passo 5 — Testar o backend

uvicorn main:app --reload --port 8000

Acesse http://localhost:8000/api/mensagens no navegador. Você deve ver o JSON com as mensagens.

Acesse também http://localhost:8000/docs para ver a documentação automática do Swagger gerada pelo FastAPI.

Pressione Ctrl + C para parar o servidor.

🤓 Explicando o código do Django REST

Model

Define a estrutura da tabela no banco de dados. Cada CharField ou TextField vira uma coluna no banco. O Django ORM cuida da criação das tabelas automaticamente via migrations.

Serializer

Converte os objetos Python (models) para JSON e vice-versa. O ModelSerializer já descobre automaticamente os campos a partir do model.

ViewSet

Substitui as funções manuais de CRUD. Com ModelViewSet, ganhamos automaticamente: listar, criar, obter, atualizar e deletar — tudo com apenas 3 linhas de código.

Router

Gera automaticamente todas as URLs do CRUD a partir do ViewSet. Sem router, precisaríamos criar cada rota manualmente no urls.py.

CORS

O django-cors-headers adiciona um middleware que inclui os headers necessários para o navegador permitir requisições de origens diferentes (como o React em localhost:5173).

Frontend React (comum para ambos os backends)

Agora vamos criar o frontend React. Ele será exatamente o mesmo para os dois backends — a única diferença será a porta configurada no proxy.

Passo 1 — Criar o projeto com Vite

Na raiz do repositório:

npm create vite@latest frontend -- --template react

Durante a execução, o Vite pode perguntar algumas coisas. Responda Yes quando perguntar se deseja instalar e iniciar.

Se o servidor de desenvolvimento iniciar automaticamente, pressione Ctrl + C para pará-lo. Vamos configurar o proxy primeiro.

Passo 2 — Configurar o Proxy no Vite

Abra o arquivo frontend/vite.config.js e substitua pelo código abaixo: 

import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'

export default defineConfig({
  plugins: [react()],
  server: {
    proxy: {
      '/api': 'http://localhost:8000'
    }
  }
})

Importante: Se você estiver testando com o FastAPI, a porta é 8000. Se for com Django, use a mesma porta 8000 — já configuramos os dois para rodar na porta 8000.

🤓 Entendendo o Proxy do Vite

Durante o desenvolvimento, o frontend roda em http://localhost:5173 (porta padrão do Vite) e o backend em http://localhost:8000. Se o React tentar acessar http://localhost:8000/api/mensagens diretamente, o navegador bloqueia por causa da política de CORS (origens diferentes).

O proxy do Vite funciona como um intermediário: quando o React faz uma requisição para /api/mensagens, o Vite redireciona automaticamente para http://localhost:8000/api/mensagens sem que o navegador perceba a mudança de origem.

Dessa forma, durante o desenvolvimento, não precisamos nos preocupar com CORS — o proxy resolve o problema para nós. Já configuramos o CORS no backend para quando formos fazer o deploy em produção.

🤓 Explicando o código do frontend

useState([])

Cria um estado chamado mensagens inicializado como array vazio. Quando chamamos setMensagens(data), o React re-renderiza o componente automaticamente com os novos dados.

useEffect(() => {{ ... }}, [])

Executa o código dentro dele uma única vez quando o componente é montado em tela (o array vazio [] garante isso).

fetch('/api/mensagens')

Faz uma requisição GET para a API. Como configuramos o proxy no Vite, essa requisição é redirecionada para http://localhost:8000/api/mensagens.

Renderização condicional

Enquanto os dados não chegam, exibimos "Carregando mensagens...". Assim que as mensagens são carregadas, usamos map para percorrer o array e renderizar cada item.

🔥 Projeto Final / Dever de Casa: Deploy no Vercel

Durante a aula, usamos o proxy do Vite para conectar o frontend ao backend. Isso funciona muito bem para desenvolvimento, mas em produção precisamos de uma abordagem diferente.

Neste projeto, você vai fazer o deploy da API no Vercel e configurar o React para consumir a URL real do backend — sem depender do proxy.

Passo a passo

🌐 Fluxo do deploy

  1. Escolha um dos backends (FastAPI ou Django)
  2. Faça o deploy da API no Vercel
  3. Crie um arquivo .env no frontend com a URL da API
  4. Altere o fetch para usar a URL dinâmica via import.meta.env.VITE_API_URL
  5. Remova o proxy do vite.config.js
  6. Faça o build do frontend e deploy também no Vercel

1. Deploy do backend no Vercel

Acesse vercel.com, faça login com seu GitHub e clique em Add New → Project. Selecione o repositório do seu projeto.

Para FastAPI: O Vercel detecta automaticamente o Python. Configure:

  • Framework: Other
  • Build Command: (deixe vazio)
  • Output Directory: (deixe vazio)
  • Install Command: pip install -r requirements.txt
  • Run Command: uvicorn main:app --host 0.0.0.0 --port $PORT

Para Django: Configure:

  • Framework: Other
  • Build Command: python manage.py migrate
  • Run Command: python manage.py runserver 0.0.0.0:$PORT

Após o deploy, o Vercel fornecerá uma URL como:

https://seu-projeto.vercel.app

Na pasta frontend/, crie um arquivo .env:
