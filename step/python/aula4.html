Padrão MVT (Model-View-Template) com Django

O que é o padrão MVT?

O padrão MVT é usado pelo Django para organizar aplicações. Ele é similar ao MVC, mas com pequenas diferenças:

  • Model: Representa os dados e regras de negócio.
  • View: Contém a lógica da aplicação, recebe requisições e processa dados.
  • Template: Define a interface visual, renderizando HTML.

Fluxo MVT no Django

  1. Usuário faz uma requisição HTTP
  2. Django direciona para a View
  3. View consulta os Models se necessário
  4. View envia os dados para um Template
  5. Template renderiza a resposta HTML para o usuário

  • Acesse o repositório no GitHub

Passo 1 — Criar projeto Django

  • Delete toda a pasta projeto_api do seu projeto
  • Abra o terminal e execute os seguintes comandos:

pip install django djangorestframework
django-admin startproject projeto_django_mvt
cd projeto_django_mvt

O comando startapp cria arquivos como models.py e views.py, mas não cria automaticamente serializers.py nem urls.py. Por isso, precisaremos criá-los manualmente.

Passo 5 — Model

Arquivo: usuarios/models.py


from django.db import models

class Usuario(models.Model):
    nome = models.CharField(max_length=100)

    def __str__(self):
        return self.nome

Passo 6 — Serializer (para API REST)

Arquivo: usuarios/serializers.py

O arquivo serializers.py é usado com o Django REST Framework para transformar os dados do model em JSON e também validar dados recebidos pela API. 


from rest_framework import serializers
from .models import Usuario

class UsuarioSerializer(serializers.ModelSerializer):
    class Meta:
        model = Usuario
        fields = '__all__'

Passo 7 — View

Arquivo: usuarios/views.py


from django.shortcuts import render
from rest_framework.decorators import api_view
from rest_framework.response import Response
from .models import Usuario
from .serializers import UsuarioSerializer

# API: listar usuários
@api_view(['GET'])
def listar_usuarios(request):
    usuarios = Usuario.objects.all()
    serializer = UsuarioSerializer(usuarios, many=True)
    return Response(serializer.data)

# API: criar usuário
@api_view(['POST'])
def criar_usuario(request):
    serializer = UsuarioSerializer(data=request.data)
    if serializer.is_valid():
        serializer.save()
        return Response({"msg": "Usuário criado"})
    return Response(serializer.errors)

# Template HTML
def listar_template(request):
    usuarios = Usuario.objects.all()
    return render(request, 'usuarios/lista.html', {'usuarios': usuarios})

Passo 8 — Criar o template HTML

Arquivo: usuarios/templates/usuarios/lista.html


<!DOCTYPE html>
<html lang="pt-BR">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>Lista de Usuários</title>
  </head>
  <body>
    <h1>Usuários cadastrados</h1>

    <ul>
      {% for usuario in usuarios %}
        <li>{{ usuario.nome }}</li>
      {% empty %}
        <li>Nenhum usuário cadastrado.</li>
      {% endfor %}
    </ul>
  </body>
</html>

Passo 9 — URLs

Arquivo: usuarios/urls.py


from django.urls import path
from . import views

urlpatterns = [
    path('api/usuarios/', views.listar_usuarios),
    path('api/usuarios/criar/', views.criar_usuario),
    path('usuarios/', views.listar_template),
]

Em projeto_django_mvt/urls.py, adicione:


from django.contrib import admin
from django.urls import path, include

urlpatterns = [
    path('admin/', admin.site.urls),
    path('', include('usuarios.urls')),
]

Passo 12 — Testar a API com Postman (ou cURL)

Agora vamos testar nossa API na prática, enviando requisições HTTP. Você pode usar ferramentas como o Postman, Insomnia ou diretamente pelo terminal com o comando curl.

1. Testando listagem de usuários (GET)

Esse endpoint retorna todos os usuários cadastrados:


curl http://localhost:8000/api/usuarios/

Resposta esperada (JSON):


[]

Caso já existam usuários cadastrados, você verá algo como:


[
  {
    "id": 1,
    "nome": "João"
  }
]

2. Testando criação de usuário (POST)

Agora vamos enviar dados para criar um novo usuário:


curl -X POST http://localhost:8000/api/usuarios/criar/ \
-H "Content-Type: application/json" \
-d "{\"nome\": \"Maria\"}"

Resposta esperada:


{
  "msg": "Usuário criado"
}

3. Testando novamente a listagem

Após criar um usuário, execute novamente o GET:


curl http://localhost:8000/api/usuarios/

Agora o usuário criado deverá aparecer na lista.

4. Testando no Postman

  • Abra o Postman
  • Crie uma requisição GET para: http://localhost:8000/api/usuarios/
  • Crie uma requisição POST para: http://localhost:8000/api/usuarios/criar/
  • Na aba Body, selecione raw → JSON
  • Envie o seguinte conteúdo:

{
  "nome": "Carlos"
}

Isso terá o mesmo efeito do comando curl.
