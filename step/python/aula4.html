Padrão MVC (Model-View-Controller)

Arquitetura de Sistemas

Arquitetura define como os componentes de um sistema são organizados e como eles se comunicam.

Padrão MVC

1. O que é o padrão MVC?

MVC (Model-View-Controller) é um padrão que separa responsabilidades em três camadas.

2. Objetivos do MVC

  • Separar responsabilidades
  • Facilitar manutenção
  • Organizar código

3. Model

a. O que é? Representa os dados

b. Responsabilidades: Gerenciar dados e regras de negócio

4. View

a. O que é? Interface com o usuário

b. Responsabilidades: Exibir dados

5. Controller

a. O que é? Controla fluxo da aplicação

b. Responsabilidades: Receber requisições e chamar Model e View

6. Exemplo MVC

  • Model → Banco de dados
  • View → Frontend
  • Controller → API

Passo a passo: Criando uma API em padrão MVC com FastAPI

Passo 1 - Criar uma nova branch no GitHub

  • Acesse o repositório projeto-python-step no site do GitHub
  • No topo da página, localize o seletor de branch (geralmente mostra main)
  • Clique nesse seletor
  • No campo "Find or create branch from main"
  • Digite o nome da nova branch (ex: aula-mvc)
  • Clique em Create branch ou pressione Enter
  • Abra o VS Code e abra o terminal
  • Acesse o projeto cd projeto-python-step
  • Digite o comando git fetch

    • Passo 2 - Acessar a branch pelo VS Code

    • Selecione a pasta do projeto aula-mvc no canto esquerdo inferior do VS Code
    • No canto inferior esquerdo, clique no nome da branch atual (ex: main)
    • Uma lista de branches será exibida
    • Selecione a branch que você criou (ex: aula-mvc)
    • O VS Code irá automaticamente trocar para essa branch
    • No explorador de arquivos do VS Code, localize a pasta app
    • Clique com o botão direito na pasta e selecione Delete (Excluir)
    • Abra o arquivo main.py
    • Selecione todo o conteúdo (Ctrl + A)
    • Limpe tudo que está no arquivo

    Passo 4 - Model (dados)

    Crie o arquivo app/models/usuario_model.py manualmente ou usando o comando type nul > app\models\usuario_model.py e insira dentro dele:

    
usuarios = []


def listar():
    return usuarios


def criar(nome):
    usuarios.append(nome)

    Passo 5 - View (resposta)

    Crie o arquivo arquivo: app/views/usuario_view.py manualmente ou usando o comando type nul > app\models\usuario_model.py e insira dentro dele:

    

def resposta_sucesso(msg):
    return {"msg": msg}

    Passo 6 - Controller (regras)

    Crie o arquivo arquivo: app/controllers/usuario_controller.py manualmente ou usando o comando type nul > app\models\usuario_model.py e insira dentro dele:

    
from app.models import usuario_model
from app.views import usuario_view


def listar_usuarios():
    return usuario_model.listar()


def criar_usuario(nome):
    usuario_model.criar(nome)
    return usuario_view.resposta_sucesso("Usuário criado")

    Passo 7 - Integrar com FastAPI

    Dentro do arquivo main.pyinsira o código abaixo:

    
from fastapi import FastAPI
from app.controllers import usuario_controller

app = FastAPI()

@app.get("/usuarios")
def listar():
    return usuario_controller.listar_usuarios()

@app.post("/usuarios")
def criar(nome: str):
    return usuario_controller.criar_usuario(nome)
