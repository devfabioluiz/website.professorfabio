1. Template Inheritance (Herança de Templates)

Um dos recursos mais poderosos do Django Templates é a herança. Em vez de repetir HTML (navbar, footer, estrutura da página) em cada arquivo, você cria um template base e os outros templates estendem ele, substituindo apenas os blocos que precisam mudar.

Como funciona

Imagine que toda página do seu CMS tem: cabeçalho, menu lateral, área de conteúdo e rodapé. Sem herança, você repetiria esse HTML em cada arquivo. Com herança:

  • Base: define a estrutura comum (HTML, head, nav, footer)
  • Blocos: marca as áreas que podem ser substituídas
  • Filhos: estendem a base e preenchem apenas os blocos necessários 

<!-- base.html -->
<!DOCTYPE html>
<html lang="pt-BR">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>{% block titulo %}CMS Produtos{% endblock %}</title>
</head>
<body>
    <nav>
        <a href="/">Home</a>
        <a href="/produtos/">Produtos</a>
    </nav>

    <main>
        {% block conteudo %}
        <!-- O conteúdo de cada página vai aqui -->
        {% endblock %}
    </main>

    <footer>
        <p>CMS Produtos &copy; 2026</p>
    </footer>
</body>
</html>

Agora, um template filho que lista produtos:


<!-- lista_produtos.html -->
{% extends 'base.html' %}

{% block titulo %}Lista de Produtos{% endblock %}

{% block conteudo %}
    <h1>Produtos Cadastrados</h1>
    <table>
        {% for produto in produtos %}
            <tr>
                <td>{{ produto.nome }}</td>
                <td>R$ {{ produto.preco }}</td>
            </tr>
        {% empty %}
            <tr><td>Nenhum produto cadastrado.</td></tr>
        {% endfor %}
    </table>
{% endblock %}

O Django renderiza o filho dentro da estrutura da base. O resultado final é uma página completa com nav, conteúdo da lista e footer, sem repetir código.

Regras importantes

  • {% extends 'base.html' %} precisa ser a primeira linha do template filho
  • Você pode ter vários blocos na base (titulo, conteudo, scripts, etc.)
  • Blocos podem ter conteúdo padrão (o que aparece se o filho não sobrescrever)
  • Use {{ block.super }} para incluir o conteúdo do bloco pai e adicionar mais

2. Tags de Template Essenciais

2.1 — {% extends %} e {% block %}

Já vimos acima. A combinação que possibilita herança de templates.

2.2 — {% include %} (Partials)

O {% include %} insere o conteúdo de outro template dentro do atual. É ideal para componentes reutilizáveis como cards de produto, mensagens de alerta, ou qualquer pedaço de HTML que aparece em várias páginas. 


<!-- _card_produto.html (partial) -->
<div class="card">
    <h3>{{ produto.nome }}</h3>
    <p>R$ {{ produto.preco }}</p>
    <span>Estoque: {{ produto.estoque }}</span>
</div>

<!-- pagina.html (usando o partial) -->
{% for produto in produtos %}
    {% include '_card_produto.html' %}
{% endfor %}

Por convenção, partials costumam começar com _ (underline) no nome do arquivo.

2.3 — {% url %} (URLs nomeadas)

Em vez de escrever URLs fixas no HTML, use {% url 'nome_da_rota' %}. Se a URL mudar no urls.py, os templates se ajustam automaticamente. 


<!-- ERRADO (quebra se a URL mudar) -->
<a href="/produtos/cadastrar/">Novo Produto</a>

<!-- CERTO -->
<a href="{% url 'produto_cadastrar' %}">Novo Produto</a>

<!-- Com parâmetro -->
<a href="{% url 'produto_editar' produto.id %}">Editar</a>

2.4 — {% csrf_token %}

Obrigatório em todo formulário que usa POST. O Django rejeita requisições POST sem esse token como proteção contra ataques CSRF. 


<form method="POST">
    {% csrf_token %}
    <input type="text" name="nome">
    <button type="submit">Salvar</button>
</form>

2.5 — {% if %} / {% elif %} / {% else %}

Controle de fluxo dentro do template.


{% if user.is_authenticated %}
    <p>Bem-vindo, {{ user.username }}!</p>
{% else %}
    <p>Faça login para continuar.</p>
{% endif %}

{% if produto.estoque == 0 %}
    <span class="sem-estoque">Indisponível</span>
{% elif produto.estoque < 5 %}
    <span class="estoque-baixo">Últimas unidades</span>
{% else %}
    <span class="em-estoque">Disponível</span>
{% endif %}

3. Filtros Nativos do Django

Filtros transformam o valor de uma variável na hora de exibir. Eles são aplicados com o caractere pipe |.

Filtro Descrição Exemplo Resultado
|date:"d/m/Y" Formata datas {{ produto.data|date:"d/m/Y" }} 14/06/2026
|default:"-" Valor padrão se variável for vazia {{ produto.descricao|default:"Sem descrição" }} Sem descrição
|length Número de itens (lista/string) {{ produtos|length }} 12
|truncatechars:30 Corta texto e adiciona "..." {{ texto|truncatechars:30 }} "Descrição longa do prod..."
|linebreaks Converte quebras de linha em
{{ descricao|linebreaks }} Texto com <br>
|lower Converte para minúsculas {{ "PRODUTO"|lower }} produto
|upper Converte para maiúsculas {{ "produto"|upper }} PRODUTO
|title Primeira letra maiúscula {{ "notebook gamer"|title }} Notebook Gamer
|floatformat:2 Formata número decimal {{ produto.preco|floatformat:2 }} 199.90
|pluralize Adiciona "s" no plural {{ qtd }} produto{{ qtd|pluralize }} 5 produtos

Exemplo prático com combinação de filtros


<p>
    <strong>{{ produto.nome|title }}</strong> —
    R$ {{ produto.preco|floatformat:2 }}
    <br>
    Cadastrado em {{ produto.data_criacao|date:"d/m/Y" }}
    <br>
    {{ produto.descricao|default:"Produto sem descrição"|truncatechars:100 }}
</p>

4. Filtros Personalizados

Além dos filtros nativos, você pode criar os seus próprios para lógicas específicas do seu projeto. Vamos criar dois filtros úteis para o CMS:

  • format_moeda: formata um número como moeda brasileira (R$ 1.999,90)
  • label_status: retorna um badge colorido baseado no estoque 

cd produtos
mkdir templatetags
type nul > templatetags/__init__.py
type nul > templatetags/filtros_produto.py
cd ..

O arquivo __init__.py (vazio) transforma a pasta em um pacote Python. 


from django import template

register = template.Library()

@register.filter(name='format_moeda')
def format_moeda(valor):
    """Formata um número como moeda brasileira: 1999.9 → R$ 1.999,90"""
    try:
        valor = float(valor)
        inteiro = int(valor)
        centavos = int(round((valor - inteiro) * 100))

        # Formata parte inteira com separador de milhar
        parte_inteira = f"{inteiro:,}".replace(",", ".")

        if centavos == 0:
            return f"R$ {parte_inteira},00"
        elif centavos < 10:
            return f"R$ {parte_inteira},0{centavos}"
        else:
            return f"R$ {parte_inteira},{centavos}"
    except (ValueError, TypeError):
        return "R$ 0,00"


@register.filter(name='label_status')
def label_status(estoque):
    """Retorna um badge HTML baseado na quantidade em estoque"""
    try:
        estoque = int(estoque)
    except (ValueError, TypeError):
        estoque = 0

    if estoque == 0:
        return '<span class="badge badge-danger">Indisponível</span>'
    elif estoque <= 5:
        return f'<span class="badge badge-warning">Apenas {estoque} un.</span>'
    else:
        return f'<span class="badge badge-success">{estoque} un.</span>'
          

{% extends 'base.html' %}
{% load filtros_produto %}

{% block conteudo %}
    <h1>Produtos</h1>

    {% for produto in produtos %}
        <div class="produto">
            <h3>{{ produto.nome }}</h3>
            <p>{{ produto.preco|format_moeda }}</p>
            <p>{{ produto.estoque|label_status|safe }}</p>
        </div>
    {% endfor %}
{% endblock %}

Atenção: O filtro label_status retorna HTML. Use |safe depois dele para que o Django renderize o HTML em vez de escapar as tags.

5. Arquivos Estáticos (CSS, JS, Imagens)

Até agora usamos Bootstrap via CDN. Para um projeto real, você vai querer seus próprios arquivos CSS e JavaScript. O Django gerencia isso com o sistema de arquivos estáticos. 


{% load static %}
<!DOCTYPE html>
<html lang="pt-BR">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>{% block titulo %}CMS Produtos{% endblock %}</title>

    <!-- Bootstrap CDN -->
    <link rel="stylesheet"
          href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.0/dist/css/bootstrap.min.css">

    <!-- Seu CSS personalizado -->
    <link rel="stylesheet" href="{% static 'css/estilo.css' %}">
</head>
<body>
    <nav class="navbar navbar-expand-lg navbar-dark bg-dark">
        <div class="container">
            <a class="navbar-brand" href="{% url 'home' %}">📦 CMS Produtos</a>
            <div class="navbar-nav">
                <a class="nav-link" href="{% url 'produto_lista' %}">Produtos</a>
                <a class="nav-link" href="{% url 'produto_cadastrar' %}">Novo</a>
            </div>
            <div class="navbar-nav ms-auto">
                {% if user.is_authenticated %}
                    <span class="nav-link">{{ user.username }}</span>
                    <a class="nav-link" href="{% url 'logout' %}">Sair</a>
                {% else %}
                    <a class="nav-link" href="{% url 'login' %}">Entrar</a>
                {% endif %}
            </div>
        </div>
    </nav>

    <main class="container mt-4">
        {% block conteudo %}{% endblock %}
    </main>

    <footer class="bg-dark text-white text-center py-3 mt-5">
        <p>CMS Produtos &copy; 2026 — Todos os direitos reservados</p>
    </footer>

    <script src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.0/dist/js/bootstrap.bundle.min.js"></script>
    <script src="{% static 'js/scripts.js' %}"></script>
{% block scripts %}{% endblock %}
</body>
</html>

1. Crie um partial de paginação

Crie o arquivo _paginacao.html que exibe links de navegação entre páginas. Use o objeto page_obj que o Django fornece quando você usa Paginator na view. O partial deve mostrar: "Página X de Y", links "Anterior" e "Próximo", e o número total de itens.

Gabarito: 

<!-- _paginacao.html -->
{% if page_obj.has_other_pages %}
<nav aria-label="Navegação de páginas">
    <ul class="pagination justify-content-center">

        <!-- Link Anterior -->
        {% if page_obj.has_previous %}
            <li class="page-item">
                <a class="page-link" href="?page={{ page_obj.previous_page_number }}">Anterior</a>
            </li>
        {% else %}
            <li class="page-item disabled">
                <span class="page-link">Anterior</span>
            </li>
        {% endif %}

        <!-- Números de página -->
        {% for num in page_obj.paginator.page_range %}
            {% if page_obj.number == num %}
                <li class="page-item active">
                    <span class="page-link">{{ num }}</span>
                </li>
            {% else %}
                <li class="page-item">
                    <a class="page-link" href="?page={{ num }}">{{ num }}</a>
                </li>
            {% endif %}
        {% endfor %}

        <!-- Link Próximo -->
        {% if page_obj.has_next %}
            <li class="page-item">
                <a class="page-link" href="?page={{ page_obj.next_page_number }}">Próximo</a>
            </li>
        {% else %}
            <li class="page-item disabled">
                <span class="page-link">Próximo</span>
            </li>
        {% endif %}

    </ul>

    <p class="text-center text-muted">
        Página {{ page_obj.number }} de {{ page_obj.paginator.num_pages }}
        — Total: {{ page_obj.paginator.count }} itens
    </p>
</nav>
{% endif %}

2. Crie um filtro customizado para status de pagamento

Crie um filtro chamado status_pagamento que recebe um valor ("pago", "pendente", "cancelado") e retorna um badge HTML com a cor correspondente: verde para pago, amarelo para pendente, vermelho para cancelado.

Gabarito: 

from django import template
from django.utils.safestring import mark_safe

register = template.Library()

@register.filter(name='status_pagamento')
def status_pagamento(valor):
    status_map = {
        'pago':      ('success', '✅ Pago'),
        'pendente':  ('warning', '⏳ Pendente'),
        'cancelado': ('danger',  '❌ Cancelado'),
    }

    cor, label = status_map.get(valor.lower(), ('secondary', valor))

    html = f'<span class="badge badge-{cor}">{label}</span>'
    return mark_safe(html)
