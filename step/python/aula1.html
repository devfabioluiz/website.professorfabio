FastAPI + PostgreSQL

Criando uma API com Python

Neste tutorial iremos criar uma API utilizando o FastAPI, um framework moderno para desenvolvimento de APIs em Python, e o banco de dados PostgreSQL.

Durante o processo iremos configurar o ambiente, estruturar o projeto e implementar operações básicas de CRUD (Criar, Listar, Atualizar e Deletar registros).

1. Criar projeto

O primeiro passo é criar uma pasta que irá armazenar todos os arquivos do projeto.

Em seguida acessamos essa pasta utilizando o terminal. 


mkdir projeto_api
cd projeto_api

O comando mkdir cria uma nova pasta chamada projeto_api. Já o comando cd permite entrar dentro dessa pasta.

2. Criar ambiente virtual

O ambiente virtual é utilizado para isolar as dependências do projeto.

Isso evita conflitos entre bibliotecas instaladas em diferentes projetos Python. 


python -m venv venv
py -m venv venv

Esse comando cria uma pasta chamada venv, que contém um ambiente Python independente.

3. Ativar ambiente virtual

Após criar o ambiente virtual, precisamos ativá-lo para que todas as dependências sejam instaladas nele. 


venv\Scripts\activate.bat

Quando o ambiente virtual estiver ativo, o terminal normalmente exibirá o nome do ambiente entre parênteses, indicando que o Python utilizado agora é o do ambiente virtual.

4. Instalar dependências

Agora iremos instalar as bibliotecas necessárias para o funcionamento da API. 


pip install fastapi uvicorn sqlalchemy psycopg2-binary pydantic

Essas bibliotecas possuem as seguintes funções:

  • FastAPI: Framework para criação de APIs.
  • Uvicorn: Servidor ASGI responsável por executar a aplicação.
  • SQLAlchemy: ORM utilizado para comunicação com o banco de dados.
  • psycopg2: Driver que permite conectar ao PostgreSQL.
  • Pydantic: Biblioteca para validação de dados.

5. Estrutura do projeto

Uma boa organização de pastas facilita a manutenção e evolução do projeto. 


projeto_api
│
├── app
│   ├── main.py
│   ├── database.py
│   ├── models.py
│   ├── schemas.py
│   └── crud.py
│
└── venv

Função de cada arquivo:

  • database.py: configuração da conexão com o banco.
  • models.py: definição das tabelas do banco.
  • schemas.py: definição dos formatos de dados da API.
  • crud.py: funções que executam operações no banco.
  • main.py: arquivo principal onde a API é iniciada.

6. database.py

Esse arquivo é responsável por configurar a conexão com o banco de dados PostgreSQL. 


from sqlalchemy import create_engine
from sqlalchemy.orm import sessionmaker, declarative_base

DATABASE_URL = "postgresql://postgres:senha@localhost/fastapi_db"

engine = create_engine(DATABASE_URL)

SessionLocal = sessionmaker(
    autocommit=False,
    autoflush=False,
    bind=engine
)

Base = declarative_base()

O create_engine cria a conexão com o banco. O SessionLocal é utilizado para criar sessões de acesso ao banco. Já o Base será usado como base para os modelos das tabelas.

7. models.py

Nesse arquivo definimos o modelo da tabela do banco de dados utilizando o SQLAlchemy. 


from sqlalchemy import Column, Integer, String
from .database import Base

class Usuario(Base):

    __tablename__ = "usuarios"

    id = Column(Integer, primary_key=True, index=True)
    nome = Column(String)
    email = Column(String)

Essa classe representa a tabela usuarios no banco de dados.

  • id: chave primária.
  • nome: nome do usuário.
  • email: email do usuário.

8. schemas.py

Os schemas são utilizados para definir o formato dos dados que serão enviados e recebidos pela API. 


from pydantic import BaseModel

class UsuarioCreate(BaseModel):
    nome: str
    email: str

class UsuarioUpdate(BaseModel):
    nome: str
    email: str

class UsuarioResponse(BaseModel):
    id: int
    nome: str
    email: str

    class Config:
        orm_mode = True

O Pydantic garante que os dados enviados para a API estejam no formato correto.

9. crud.py

Esse arquivo contém as funções responsáveis por realizar operações no banco de dados.

Essas operações são chamadas de CRUD:

  • Create (Criar)
  • Read (Ler)
  • Update (Atualizar)
  • Delete (Excluir)

As funções implementam essas operações utilizando o SQLAlchemy. 

from sqlalchemy.orm import Session
from . import models, schemas


def criar_usuario(db: Session, usuario: schemas.UsuarioCreate):

    novo_usuario = models.Usuario(
        nome=usuario.nome,
        email=usuario.email
    )

    db.add(novo_usuario)
    db.commit()
    db.refresh(novo_usuario)

    return novo_usuario


def listar_usuarios(db: Session):
    return db.query(models.Usuario).all()


def buscar_usuario(db: Session, usuario_id: int):
    return db.query(models.Usuario).filter(
        models.Usuario.id == usuario_id
    ).first()


def atualizar_usuario(db: Session, usuario_id: int, dados: schemas.UsuarioUpdate):

    usuario = db.query(models.Usuario).filter(
        models.Usuario.id == usuario_id
    ).first()

    if usuario is None:
        return None

    usuario.nome = dados.nome
    usuario.email = dados.email

    db.commit()
    db.refresh(usuario)

    return usuario


def deletar_usuario(db: Session, usuario_id: int):

    usuario = db.query(models.Usuario).filter(
        models.Usuario.id == usuario_id
    ).first()

    if usuario is None:
        return None

    db.delete(usuario)
    db.commit()

    return usuario

10. main.py

Esse é o arquivo principal da aplicação.

Aqui inicializamos o FastAPI e definimos as rotas da API. 


from fastapi import FastAPI, Depends
from sqlalchemy.orm import Session

from .database import engine, SessionLocal, Base
from . import models, schemas, crud

Base.metadata.create_all(bind=engine)

app = FastAPI()

Também criamos as rotas responsáveis por acessar as funções do CRUD.

  • POST /usuarios → cria usuário
  • GET /usuarios → lista usuários
  • GET /usuarios/{id} → busca usuário
  • PUT /usuarios/{id} → atualiza usuário
  • DELETE /usuarios/{id} → remove usuário

11. Rodar a API

Para iniciar o servidor da API utilizamos o Uvicorn. 


uvicorn app.main:app --reload

A opção --reload faz com que o servidor reinicie automaticamente sempre que o código for alterado.

Após iniciar o servidor, a API estará disponível em:

http://localhost:8000

O FastAPI gera automaticamente uma documentação interativa acessível em:

http://localhost:8000/docs
