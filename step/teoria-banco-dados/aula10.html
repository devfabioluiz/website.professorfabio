Requisitos do Projeto

Parte 1 — Criar Tabelas Normalizadas

Crie as seguintes tabelas com PKs, FKs e constraints apropriadas:

  • autores: id (PK, SERIAL), nome (VARCHAR, NOT NULL), data_nascimento (DATE), nacionalidade (VARCHAR)
  • livros: id (PK, SERIAL), titulo (VARCHAR, NOT NULL), autor_id (FK → autores), ano_publicacao (INTEGER, CHECK > 0), genero (VARCHAR), quantidade (INTEGER, DEFAULT 1, CHECK >= 0), isbn (VARCHAR, UNIQUE)
  • membros: id (PK, SERIAL), nome (VARCHAR, NOT NULL), email (VARCHAR, UNIQUE), telefone (VARCHAR), data_cadastro (DATE, DEFAULT CURRENT_DATE)
  • emprestimos: id (PK, SERIAL), livro_id (FK → livros), membro_id (FK → membros), data_emprestimo (DATE, DEFAULT CURRENT_DATE), data_devolucao_prevista (DATE, NOT NULL), data_devolucao_real (DATE), status (VARCHAR, DEFAULT 'ativo', CHECK status IN ('ativo', 'devolvido', 'atrasado'))
Gabarito — Criação das Tabelas: 

CREATE TABLE autores (
    id              SERIAL PRIMARY KEY,
    nome            VARCHAR(150) NOT NULL,
    data_nascimento DATE,
    nacionalidade   VARCHAR(100)
);

CREATE TABLE livros (
    id              SERIAL PRIMARY KEY,
    titulo          VARCHAR(200) NOT NULL,
    autor_id        INTEGER NOT NULL REFERENCES autores(id) ON DELETE CASCADE,
    ano_publicacao  INTEGER CHECK (ano_publicacao > 0),
    genero          VARCHAR(100),
    quantidade      INTEGER DEFAULT 1 CHECK (quantidade >= 0),
    isbn            VARCHAR(20) UNIQUE
);

CREATE TABLE membros (
    id            SERIAL PRIMARY KEY,
    nome          VARCHAR(150) NOT NULL,
    email         VARCHAR(150) UNIQUE NOT NULL,
    telefone      VARCHAR(20),
    data_cadastro DATE DEFAULT CURRENT_DATE
);

CREATE TABLE emprestimos (
    id                        SERIAL PRIMARY KEY,
    livro_id                  INTEGER NOT NULL REFERENCES livros(id),
    membro_id                 INTEGER NOT NULL REFERENCES membros(id),
    data_emprestimo           DATE DEFAULT CURRENT_DATE,
    data_devolucao_prevista   DATE NOT NULL,
    data_devolucao_real       DATE,
    status                    VARCHAR(20) DEFAULT 'ativo'
                              CHECK (status IN ('ativo', 'devolvido', 'atrasado'))
);

Parte 3 — Consultas com JOINs, GROUP BY e HAVING

Crie as seguintes consultas:

  1. Listar todos os livros com nome do autor (INNER JOIN)
  2. Listar todos os empréstimos ativos com nome do membro e título do livro
  3. Contar quantos livros cada autor publicou (GROUP BY)
  4. Mostrar autores com mais de 2 livros cadastrados (HAVING)
  5. Listar membros com empréstimos atrasados
Gabarito — Consultas: 

-- 1. Livros com nome do autor
SELECT l.titulo, a.nome AS autor, l.genero, l.ano_publicacao
FROM livros l
INNER JOIN autores a ON l.autor_id = a.id
ORDER BY a.nome, l.titulo;

-- 2. Empréstimos ativos com membro e livro
SELECT e.id, m.nome AS membro, l.titulo AS livro,
       e.data_emprestimo, e.data_devolucao_prevista
FROM emprestimos e
INNER JOIN membros m ON e.membro_id = m.id
INNER JOIN livros l ON e.livro_id = l.id
WHERE e.status = 'ativo'
ORDER BY e.data_devolucao_prevista;

-- 3. Quantidade de livros por autor
SELECT a.nome, COUNT(l.id) AS total_livros
FROM autores a
LEFT JOIN livros l ON a.id = l.autor_id
GROUP BY a.id, a.nome
ORDER BY total_livros DESC;

-- 4. Autores com mais de 2 livros
SELECT a.nome, COUNT(l.id) AS total_livros
FROM autores a
INNER JOIN livros l ON a.id = l.autor_id
GROUP BY a.id, a.nome
HAVING COUNT(l.id) > 2
ORDER BY total_livros DESC;

-- 5. Membros com empréstimos atrasados
SELECT m.nome, m.email, l.titulo, e.data_devolucao_prevista,
       CURRENT_DATE - e.data_devolucao_prevista AS dias_atraso
FROM emprestimos e
INNER JOIN membros m ON e.membro_id = m.id
INNER JOIN livros l ON e.livro_id = l.id
WHERE e.status = 'atrasado'
ORDER BY dias_atraso DESC;

Parte 4 — Criar uma View "livros_disponiveis"

A view deve mostrar todos os livros que possuem pelo menos 1 unidade disponível para empréstimo (quantidade > número de empréstimos ativos).

Gabarito — View: 

CREATE VIEW livros_disponiveis AS
SELECT
    l.id,
    l.titulo,
    a.nome AS autor,
    l.genero,
    l.quantidade AS total_exemplares,
    COALESCE(emp.emprestados, 0) AS emprestados,
    l.quantidade - COALESCE(emp.emprestados, 0) AS disponiveis
FROM livros l
INNER JOIN autores a ON l.autor_id = a.id
LEFT JOIN (
    SELECT livro_id, COUNT(*) AS emprestados
    FROM emprestimos
    WHERE status IN ('ativo', 'atrasado')
    GROUP BY livro_id
) emp ON l.id = emp.livro_id
WHERE l.quantidade - COALESCE(emp.emprestados, 0) > 0
ORDER BY disponiveis DESC;

Parte 5 — Criar um Trigger que Atualiza o Status do Livro

Crie um trigger que, ao atualizar a data_devolucao_real de um empréstimo, automaticamente:

  • Define o status como 'devolvido' se a devolução estiver dentro do prazo
  • Define o status como 'atrasado' se a devolução for após a data prevista
  • Incrementa a quantidade disponível do livro quando devolvido
Gabarito — Trigger: 

CREATE OR REPLACE FUNCTION atualizar_status_emprestimo()
RETURNS TRIGGER AS $$
BEGIN
    -- Se a data_devolucao_real foi preenchida (devolução)
    IF NEW.data_devolucao_real IS NOT NULL
       AND (OLD.data_devolucao_real IS NULL OR OLD.data_devolucao_real <> NEW.data_devolucao_real) THEN

        -- Define o status baseado na comparação de datas
        IF NEW.data_devolucao_real <= NEW.data_devolucao_prevista THEN
            NEW.status := 'devolvido';
        ELSE
            NEW.status := 'atrasado';
        END IF;

    END IF;

    RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER trigger_atualizar_status
    BEFORE UPDATE ON emprestimos
    FOR EACH ROW
    EXECUTE FUNCTION atualizar_status_emprestimo();

Parte 6 — Criar uma Procedure "realizar_emprestimo"

A procedure deve receber livro_id e membro_id, verificar se o livro está disponível, reduzir a quantidade em 1 e registrar o empréstimo com data de devolução prevista para 14 dias depois.

Gabarito — Procedure: 

CREATE OR REPLACE PROCEDURE realizar_emprestimo(
    p_livro_id  INTEGER,
    p_membro_id INTEGER
)
LANGUAGE plpgsql
AS $$
DECLARE
    v_disponivel INTEGER;
BEGIN
    -- Verifica se o livro está disponível usando a view
    SELECT disponiveis INTO v_disponivel
    FROM livros_disponiveis
    WHERE id = p_livro_id;

    IF v_disponivel IS NULL OR v_disponivel <= 0 THEN
        RAISE EXCEPTION 'Livro não disponível para empréstimo (ID: %)', p_livro_id;
    END IF;

    -- Registra o empréstimo
    INSERT INTO emprestimos (livro_id, membro_id, data_emprestimo, data_devolucao_prevista)
    VALUES (p_livro_id, p_membro_id, CURRENT_DATE, CURRENT_DATE + INTERVAL '14 days');

    RAISE NOTICE 'Empréstimo realizado com sucesso! Livro: %, Membro: %', p_livro_id, p_membro_id;
END;
$$;

Parte 7 — Criar uma Function que Calcula Multa por Atraso

Crie uma função calcular_multa(emprestimo_id INTEGER) que retorna o valor da multa. Considere R$ 2,00 por dia de atraso, contando a partir do dia seguinte à data de devolução prevista.

Gabarito — Function: 

CREATE OR REPLACE FUNCTION calcular_multa(p_emprestimo_id INTEGER)
RETURNS NUMERIC
LANGUAGE plpgsql
AS $$
DECLARE
    v_dias_atraso  INTEGER;
    v_valor_multa  NUMERIC := 0;
    v_status       VARCHAR;
    v_data_prevista DATE;
    v_data_real     DATE;
BEGIN
    SELECT status, data_devolucao_prevista, data_devolucao_real
    INTO v_status, v_data_prevista, v_data_real
    FROM emprestimos
    WHERE id = p_emprestimo_id;

    IF NOT FOUND THEN
        RAISE EXCEPTION 'Empréstimo com ID % não encontrado', p_emprestimo_id;
    END IF;

    IF v_data_real IS NOT NULL THEN
        -- Devolvido: calcula com base na data real
        v_dias_atraso := v_data_real - v_data_prevista;
    ELSE
        -- Ainda não devolvido: calcula com base na data atual
        v_dias_atraso := CURRENT_DATE - v_data_prevista;
    END IF;

    IF v_dias_atraso > 0 THEN
        v_valor_multa := v_dias_atraso * 2.00;
    END IF;

    RETURN v_valor_multa;
END;
$$;

Parte 8 — Criar Roles e Permissões

Crie duas roles e configure as permissões:

  • bibliotecario: acesso total (SELECT, INSERT, UPDATE, DELETE) em todas as tabelas. Pode executar a procedure realizar_emprestimo.
  • membro: apenas SELECT nas views livros_disponiveis e na própria tabela emprestimos (apenas seus próprios registros, via view).
Gabarito — Roles e Permissões: 

-- Criar roles
CREATE ROLE bibliotecario WITH LOGIN PASSWORD 'biblio123';
CREATE ROLE membro WITH LOGIN PASSWORD 'membro123';

-- Conceder acesso ao banco e schema
GRANT CONNECT ON DATABASE biblioteca TO bibliotecario, membro;
GRANT USAGE ON SCHEMA public TO bibliotecario, membro;

-- bibliotecario: acesso total
GRANT ALL PRIVILEGES ON ALL TABLES IN SCHEMA public TO bibliotecario;
GRANT USAGE ON ALL SEQUENCES IN SCHEMA public TO bibliotecario;
GRANT EXECUTE ON PROCEDURE realizar_emprestimo(INTEGER, INTEGER) TO bibliotecario;

-- membro: apenas consulta à view de livros disponíveis
GRANT SELECT ON livros_disponiveis TO membro;

-- Privilégios para objetos futuros (bibliotecario)
ALTER DEFAULT PRIVILEGES IN SCHEMA public
GRANT ALL PRIVILEGES ON TABLES TO bibliotecario;
ALTER DEFAULT PRIVILEGES IN SCHEMA public
GRANT USAGE ON SEQUENCES TO bibliotecario;

Exame Teórico

Responda às 10 questões a seguir. Cada questão vale 1,0 ponto. O exame cobre todos os módulos do curso.

1. Qual forma normal elimina dependências parciais de uma chave primária composta?

  • a) Primeira Forma Normal (1FN)
  • b) Segunda Forma Normal (2FN)
  • c) Terceira Forma Normal (3FN)
  • d) Forma Normal Boyce-Codd (BCNF)
Resposta: b) Segunda Forma Normal (2FN)

A 2FN elimina dependências parciais, ou seja, atributos não-chave que dependem de apenas parte da chave primária composta. Uma tabela está na 2FN se estiver na 1FN e todos os atributos não-chave dependerem da chave primária inteira.

2. Qual tipo de JOIN retorna todas as linhas da tabela à esquerda, mesmo quando não há correspondência na tabela à direita?

  • a) INNER JOIN
  • b) RIGHT JOIN
  • c) LEFT JOIN
  • d) CROSS JOIN
Resposta: c) LEFT JOIN

O LEFT JOIN (ou LEFT OUTER JOIN) retorna todas as linhas da tabela à esquerda da cláusula JOIN e as linhas correspondentes da tabela à direita. Quando não há correspondência, as colunas da tabela direita são preenchidas com NULL.

3. Qual das alternativas abaixo NÃO é uma função de agregação em SQL?

  • a) COUNT
  • b) AVG
  • c) CONCAT
  • d) SUM
Resposta: c) CONCAT

CONCAT é uma função de string que concatena dois ou mais valores, não sendo uma função de agregação. COUNT, AVG, SUM, MIN e MAX são funções de agregação que operam sobre um conjunto de linhas e retornam um único valor.

4. Qual a principal diferença entre uma TRIGGER e uma PROCEDURE no PostgreSQL?

  • a) Procedures podem retornar valores e triggers não
  • b) Triggers são executadas automaticamente em resposta a eventos (INSERT, UPDATE, DELETE), enquanto procedures são chamadas explicitamente com CALL
  • c) Procedures são mais rápidas que triggers
  • d) Não há diferença — são a mesma coisa
Resposta: b) Triggers são executadas automaticamente em resposta a eventos, enquanto procedures são chamadas explicitamente com CALL

Triggers são gatilhos associados a tabelas que disparam automaticamente quando ocorre um evento (INSERT, UPDATE, DELETE). Procedures são blocos de código armazenados que precisam ser explicitamente invocados com o comando CALL.

5. Qual comando SQL concede permissão de SELECT em uma tabela para uma role?

  • a) ALLOW SELECT ON tabela TO role
  • b) PERMIT SELECT ON tabela TO role
  • c) GRANT SELECT ON tabela TO role
  • d) GIVE SELECT ON tabela TO role
Resposta: c) GRANT SELECT ON tabela TO role

O comando GRANT é o comando SQL padrão para conceder privilégios a roles/usuários. O comando para remover privilégios é REVOKE.

6. Sobre tipos de backup no PostgreSQL, qual ferramenta é usada para backup lógico de um banco específico?

  • a) pg_basebackup
  • b) pg_restore
  • c) pg_dump
  • d) pg_start_backup
Resposta: c) pg_dump

pg_dump é a ferramenta padrão para backup lógico de um banco de dados específico no PostgreSQL. pg_dumpall faz backup de todos os bancos + objetos globais. pg_basebackup é usado para backup físico. pg_restore é usado para restaurar backups em formato customizado.

7. Qual função de janela (window function) retorna o valor da linha anterior dentro de uma partição ordenada?

  • a) LEAD
  • b) LAG
  • c) FIRST_VALUE
  • d) PREV
Resposta: b) LAG

LAG acessa o valor de uma linha anterior dentro da partição, baseado no deslocamento especificado. LEAD acessa a linha seguinte. LAG e LEAD são fundamentais para comparações entre linhas consecutivas sem necessidade de self-join.

8. Qual operador é usado para verificar se uma subconsulta retorna pelo menos uma linha?

  • a) ANY
  • b) IN
  • c) EXISTS
  • d) BETWEEN
Resposta: c) EXISTS

EXISTS retorna TRUE se a subconsulta retornar pelo menos uma linha. É frequentemente usado em subconsultas correlacionadas para verificar a existência de registros relacionados. NOT EXISTS retorna TRUE se a subconsulta não retornar nenhuma linha.

9. Qual propriedade ACID garante que uma transação seja executada completamente ou não seja executada de forma alguma?

  • a) Atomicidade
  • b) Consistência
  • c) Isolamento
  • d) Durabilidade
Resposta: a) Atomicidade

Atomicidade (Atomicity) é a propriedade que garante o "tudo ou nada": todas as operações de uma transação são concluídas com sucesso (COMMIT) ou, se houver falha, todas são desfeitas (ROLLBACK). Consistência mantém a integridade do banco. Isolamento controla a concorrência. Durabilidade garante que dados commitados sobrevivam a falhas.

10. Qual tipo de índice é o padrão no PostgreSQL e é otimizado para consultas de igualdade e intervalo?

  • a) Hash
  • b) GIN
  • c) GiST
  • d) B-tree
Resposta: d) B-tree

B-tree (Balanced Tree) é o tipo de índice padrão no PostgreSQL. É eficiente para consultas de igualdade (=), intervalo (<, >, BETWEEN) e ordenação (ORDER BY). Hash é otimizado apenas para igualdade. GIN é para arrays e full-text search. GiST é para dados geoespaciais.
