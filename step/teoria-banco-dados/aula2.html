Tabelas no PostgreSQL

Uma tabela é a estrutura fundamental de um banco de dados relacional. Ela é composta por colunas (atributos) e linhas (registros/tuplas). Cada tabela representa uma entidade do mundo real — como alunos, cursos, produtos ou pedidos.

Elementos Essenciais de uma Tabela

  • Nome da tabela: identificador único dentro do esquema (schema)
  • Colunas: cada coluna possui um nome e um tipo de dado obrigatório
  • Chave Primária (PRIMARY KEY): identifica unicamente cada linha da tabela
  • Valor Padrão (DEFAULT): valor atribuído automaticamente quando nenhum é fornecido
  • Unicidade (UNIQUE): garante que não haja valores duplicados em uma coluna ou conjunto de colunas

Veja um exemplo completo de definição de tabela: 


CREATE TABLE alunos (
    id              SERIAL PRIMARY KEY,
    nome            VARCHAR(100) NOT NULL,
    email           VARCHAR(100) UNIQUE NOT NULL,
    data_nascimento DATE,
    ativo           BOOLEAN DEFAULT TRUE,
    criado_em       TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

Chave Primária e Constraints

Constraints (restrições) são regras aplicadas às colunas de uma tabela para garantir a integridade dos dados. Elas são verificadas automaticamente pelo SGBD sempre que dados são inseridos ou alterados.

Tipos de Constraints

Constraint Descrição
PRIMARY KEY Identifica unicamente cada linha. Não permite NULL. Apenas uma por tabela.
FOREIGN KEY Garante integridade referencial entre tabelas
NOT NULL Impede valores nulos na coluna
UNIQUE Garante que todos os valores da coluna sejam distintos
CHECK Valida se o valor atende a uma condição booleana
DEFAULT Define um valor padrão quando nenhum é fornecido

Exemplos Práticos de Constraints


-- PRIMARY KEY: identificação única
-- NOT NULL: campo obrigatório
-- UNIQUE: email não pode se repetir
-- CHECK: idade deve ser maior que zero
-- DEFAULT: se não informado, ativo será TRUE

CREATE TABLE funcionarios (
    id       SERIAL PRIMARY KEY,
    nome     VARCHAR(100) NOT NULL,
    email    VARCHAR(100) UNIQUE NOT NULL,
    idade    INTEGER CHECK (idade >= 18),
    salario  NUMERIC(10, 2) CHECK (salario > 0),
    ativo    BOOLEAN DEFAULT TRUE,
    criado   TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- FOREIGN KEY: relaciona funcionario ao departamento
CREATE TABLE departamentos (
    id    SERIAL PRIMARY KEY,
    nome  VARCHAR(80) NOT NULL UNIQUE
);

ALTER TABLE funcionarios
    ADD COLUMN departamento_id INTEGER REFERENCES departamentos(id);

Índices

Um índice é uma estrutura de dados auxiliar que acelera as consultas em uma tabela. Funciona de forma semelhante ao índice de um livro: em vez de percorrer todas as páginas, você consulta o índice e vai direto ao conteúdo desejado.

Por que usar índices?

  • Velocidade em SELECT: consultas com WHERE, JOIN e ORDER BY tornam-se muito mais rápidas
  • Garantia de unicidade: PRIMARY KEY e UNIQUE criam índices automaticamente
  • Melhoria em buscas textuais: índices GIN e GiST para全文检索

Estrutura Interna: B-tree

O PostgreSQL utiliza principalmente a estrutura B-tree (árvore balanceada) como padrão para índices. Nessa estrutura, os dados são organizados de forma hierárquica, permitindo buscas em tempo O(log n).

Criando Índices


-- Índice simples em uma coluna
CREATE INDEX idx_alunos_nome
    ON alunos (nome);

-- Índice único (garante valores distintos)
CREATE UNIQUE INDEX idx_alunos_email
    ON alunos (email);

-- Índice composto (múltiplas colunas)
CREATE INDEX idx_alunos_nome_idade
    ON alunos (nome, idade);

-- Remover um índice
DROP INDEX IF EXISTS idx_alunos_nome;

Atenção: Índices aceleram leituras (SELECT), mas tornam escritas (INSERT, UPDATE, DELETE) mais lentas, pois o índice também precisa ser atualizado. Use com equilíbrio.

Bancos e Tabelas do Sistema

O PostgreSQL mantém metadados sobre toda a estrutura do banco em tabelas internas do sistema. Essas tabelas permitem consultar informações sobre bancos, tabelas, colunas, índices e muito mais — tudo usando SQL.

Principais Tabelas e Views do Sistema

Tabela / View Schema Descrição
pg_database pg_catalog Lista todos os bancos de dados do cluster
pg_tables pg_catalog Lista todas as tabelas (inclui views)
pg_indexes pg_catalog Lista todos os índices
pg_views pg_catalog Lista todas as views
tables information_schema Padrão SQL para listar tabelas
columns information_schema Padrão SQL para listar colunas

Consultando o Catálogo do Sistema


-- Listar todos os bancos de dados
SELECT datname FROM pg_database;

-- Listar todas as tabelas do schema public
SELECT tablename FROM pg_tables
    WHERE schemaname = 'public';

-- Listar colunas de uma tabela específica (via information_schema)
SELECT column_name, data_type, is_nullable
    FROM information_schema.columns
    WHERE table_name = 'alunos';

Introdução ao SQL: DDL, DML, DCL

SQL (Structured Query Language) é a linguagem padrão para interagir com bancos de dados relacionais. A linguagem SQL é dividida em subconjuntos, cada um com um propósito específico:

Subconjunto Significado Propósito Principais Comandos
DDL Data Definition Language Definir e modificar a estrutura do banco CREATE, ALTER, DROP, TRUNCATE, RENAME
DML Data Manipulation Language Manipular os dados dentro das tabelas SELECT, INSERT, UPDATE, DELETE
DCL Data Control Language Controlar acesso e permissões GRANT, REVOKE
TCL Transaction Control Language Controlar transações COMMIT, ROLLBACK, SAVEPOINT

Exemplos de cada subconjunto


-- DDL: Criar estrutura
CREATE TABLE produtos (
    id    SERIAL PRIMARY KEY,
    nome  VARCHAR(100) NOT NULL,
    preco NUMERIC(8, 2)
);

-- DML: Inserir e consultar dados
INSERT INTO produtos (nome, preco)
    VALUES ('Notebook', 3500.00);

SELECT * FROM produtos;

UPDATE produtos SET preco = 3200.00 WHERE id = 1;

DELETE FROM produtos WHERE id = 1;

-- DCL: Conceder permissão
GRANT SELECT, INSERT ON produtos TO usuario_leitura;

-- TCL: Gerenciar transação
BEGIN;
    INSERT INTO produtos (nome, preco) VALUES ('Mouse', 150.00);
COMMIT;

Passo a passo: CREATE TABLE

Vamos criar tabelas de exemplo para um sistema acadêmico. Acompanhe cada passo e execute no seu terminal psql.

Passo 1 — Conectar ao banco de dados


psql -U postgres
\c postgres

Passo 2 — Criar a tabela de cursos


CREATE TABLE cursos (
    id           SERIAL PRIMARY KEY,
    nome         VARCHAR(100) NOT NULL,
    descricao    TEXT,
    carga_horaria INTEGER NOT NULL CHECK (carga_horaria > 0),
    ativo        BOOLEAN DEFAULT TRUE,
    criado_em    TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

Passo 3 — Criar a tabela de alunos com chave estrangeira


CREATE TABLE alunos (
    id              SERIAL PRIMARY KEY,
    nome            VARCHAR(100) NOT NULL,
    email           VARCHAR(100) UNIQUE NOT NULL,
    data_nascimento DATE,

Passo a passo: ALTER TABLE

O comando ALTER TABLE permite modificar a estrutura de uma tabela já existente — adicionar colunas, modificar tipos, adicionar ou remover constraints, renomear tabelas e muito mais.

Adicionar uma nova coluna


-- Adicionar coluna telefone à tabela alunos
ALTER TABLE alunos
    ADD COLUMN telefone VARCHAR(20);

Modificar o tipo de uma coluna


-- Alterar o tipo da coluna telefone para aceitar mais caracteres
ALTER TABLE alunos
    ALTER COLUMN telefone TYPE VARCHAR(30);

Adicionar uma constraint NOT NULL


-- Tornar a coluna telefone obrigatória
ALTER TABLE alunos
    ALTER COLUMN telefone SET NOT NULL;

Adicionar uma constraint UNIQUE


-- Garantir que telefone seja único
ALTER TABLE alunos
    ADD CONSTRAINT unique_telefone UNIQUE (telefone);

Adicionar uma constraint CHECK


-- Adicionar validação de ano na tabela cursos
ALTER TABLE cursos
    ADD CONSTRAINT check_carga_horaria
        CHECK (carga_horaria BETWEEN 10 AND 1000);

Remover uma constraint


-- Remover a constraint de CHECK
ALTER TABLE cursos
    DROP CONSTRAINT check_carga_horaria;

Remover uma coluna


-- Remover coluna telefone (se não for mais necessária)
ALTER TABLE alunos
    DROP COLUMN IF EXISTS telefone;

Renomear uma tabela


-- Renomear a tabela (exemplo)
ALTER TABLE alunos
    RENAME TO estudantes;

-- Renomear de volta
ALTER TABLE estudantes
    RENAME TO alunos;

Passo a passo: DROP TABLE

O comando DROP TABLE remove permanentemente uma tabela e todos os seus dados. Use com cautela — essa operação não pode ser desfeita.

DROP TABLE simples


-- Remove a tabela (gera erro se a tabela não existir)
DROP TABLE alunos;

DROP TABLE com IF EXISTS


-- Remove a tabela apenas se existir (forma segura)
DROP TABLE IF EXISTS alunos;

DROP TABLE com CASCADE

Quando uma tabela é referenciada por outra (via FOREIGN KEY), o PostgreSQL impede a exclusão. Para forçar a exclusão e remover também os objetos dependentes, use CASCADE: 


-- Remove a tabela e todos os objetos que dependem dela
DROP TABLE IF EXISTS cursos CASCADE;

DROP TABLE com RESTRICT (padrão)


-- Comportamento padrão: recusa excluir se houver dependências
DROP TABLE IF EXISTS cursos RESTRICT;
