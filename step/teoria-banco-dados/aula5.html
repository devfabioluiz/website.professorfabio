Módulo 5: Funções de Agregação, Subconsultas e Funções de Janela

Tópicos do nosso plano de curso abordados nessa aula: Função COUNT (com coluna e COUNT(*)), Função AVG (com ALL e DISTINCT), Funções SUM, MIN, MAX, GROUP BY, HAVING (comparação com WHERE), Funções de janela (ROW_NUMBER, RANK, DENSE_RANK, LAG, LEAD, SUM OVER), Subconsultas (subconsultas escalares, de linha, de tabela).

O que são Funções de Agregação?

As funções de agregação realizam cálculos sobre um conjunto de linhas e retornam um único valor como resultado. Elas são fundamentais para gerar relatórios, estatísticas e análises de dados. As principais funções de agregação do SQL são:

  • COUNT: conta o número de linhas
  • AVG: calcula a média aritmética
  • SUM: soma os valores de uma coluna
  • MIN: retorna o menor valor
  • MAX: retorna o maior valor

Função COUNT

A função COUNT conta o número de linhas em um conjunto. Existem duas formas principais de uso:

  • COUNT(coluna): conta apenas os valores não-nulos da coluna especificada
  • COUNT(*): conta todas as linhas, independentemente de valores nulos

-- Conta todos os pedidos da tabela
SELECT COUNT(*) FROM pedidos;

-- Conta apenas os produtos que possuem categoria (ignora NULLs)
SELECT COUNT(categoria) FROM produtos;

-- Conta categorias distintas (sem repetições)
SELECT COUNT(DISTINCT categoria) FROM produtos;

-- Exemplo prático: quantos clientes fizeram pedidos?
SELECT COUNT(DISTINCT cliente_id) AS total_clientes_ativos
FROM pedidos;
Forma Comportamento Exemplo
COUNT(*) Conta todas as linhas, incluindo NULLs SELECT COUNT(*) FROM clientes;
COUNT(coluna) Conta apenas valores não-nulos da coluna SELECT COUNT(email) FROM clientes;
COUNT(DISTINCT coluna) Conta valores distintos não-nulos SELECT COUNT(DISTINCT cidade) FROM clientes;

Função AVG

A função AVG calcula a média aritmética dos valores em uma coluna numérica. Por padrão, utiliza ALL (todos os valores), mas também pode ser usada com DISTINCT. 


-- Média de preço de todos os produtos
SELECT AVG(preco) FROM produtos;

-- Média de preços distintos (ignora preços repetidos)
SELECT AVG(DISTINCT preco) FROM produtos;

-- Média de valor dos pedidos concluídos
SELECT AVG(valor_total) FROM pedidos WHERE status = 'concluido';

Função SUM

A função SUM soma todos os valores de uma coluna numérica. Muito utilizada para totais financeiros, quantidades em estoque, etc. 


-- Soma do valor total de todos os pedidos concluídos
SELECT SUM(valor_total) FROM pedidos WHERE status = 'concluido';

-- Quantidade total de produtos em estoque
SELECT SUM(estoque) AS estoque_total FROM produtos;

-- Faturamento total por mês (supondo coluna mes)
SELECT mes, SUM(valor_total) AS faturamento
FROM vendas
GROUP BY mes;

Funções MIN e MAX

As funções MIN e MAX retornam, respectivamente, o menor e o maior valor de uma coluna. Funcionam com números, datas e até strings (ordem alfabética). 


-- Primeiro e último pedido do histórico
SELECT MIN(data_pedido) AS primeiro_pedido,
       MAX(data_pedido) AS ultimo_pedido
FROM pedidos;

-- Produto mais caro e mais barato
SELECT MIN(preco) AS mais_barato,
       MAX(preco) AS mais_caro
FROM produtos;

-- Cliente com nome mais próximo do início e do fim do alfabeto
SELECT MIN(nome) AS primeiro_alfabetico,
       MAX(nome) AS ultimo_alfabetico
FROM clientes;

Combinando Funções de Agregação

É possível usar várias funções de agregação em uma única consulta, gerando um resumo completo dos dados. 


-- Resumo estatístico da tabela de produtos
SELECT
    COUNT(*)        AS total_produtos,
    COUNT(categoria) AS produtos_com_categoria,
    AVG(preco)      AS preco_medio,
    SUM(estoque)    AS estoque_total,
    MIN(preco)      AS menor_preco,
    MAX(preco)      AS maior_preco
FROM produtos;

Entendendo o GROUP BY

A cláusula GROUP BY agrupa linhas que possuem valores iguais em uma ou mais colunas, permitindo aplicar funções de agregação a cada grupo separadamente. Sem GROUP BY, as funções de agregação operam sobre todas as linhas da tabela; com GROUP BY, elas operam sobre cada grupo.

Agrupamento por uma coluna


-- Quantos produtos existem em cada categoria?
SELECT categoria, COUNT(*) AS quantidade
FROM produtos
GROUP BY categoria
ORDER BY quantidade DESC;

Agrupamento com ordenação


-- Total gasto por cliente, do maior para o menor
SELECT cliente_id, SUM(valor_total) AS total_gasto
FROM pedidos
GROUP BY cliente_id
ORDER BY SUM(valor_total) DESC;

Agrupamento por múltiplas colunas

Quando você agrupa por mais de uma coluna, o SQL cria um grupo para cada combinação única de valores. 


-- Quantos produtos existem em cada combinação de categoria e status?
SELECT categoria, status, COUNT(*) AS quantidade
FROM produtos
GROUP BY categoria, status
ORDER BY categoria, status;

Regra importante do GROUP BY

Toda coluna no SELECT que não está dentro de uma função de agregação DEVE aparecer na cláusula GROUP BY. Caso contrário, o banco de dados retornará um erro, pois não saberá qual valor exibir para aquela coluna dentro do grupo. 


-- CORRETO: cidade está no GROUP BY
SELECT cidade, COUNT(*) FROM clientes GROUP BY cidade;

-- ERRADO: nome não está no GROUP BY nem em função de agregação
-- SELECT nome, cidade, COUNT(*) FROM clientes GROUP BY cidade; -- Isso gera erro!

HAVING vs WHERE — Qual a diferença?

Tanto WHERE quanto HAVING filtram dados, mas em momentos diferentes da execução da consulta:

Cláusula Quando atua Filtra Pode usar funções de agregação?
WHERE Antes do agrupamento Linhas individuais Não
HAVING Depois do agrupamento Grupos inteiros Sim

Pense no WHERE como um filtro de entrada: ele decide quais linhas entram na agregação. O HAVING é um filtro de saída: ele decide quais grupos aparecem no resultado final.

WHERE — filtra antes de agrupar


-- Filtra pedidos com valor acima de 100 (antes de qualquer agrupamento)
SELECT * FROM pedidos WHERE valor_total > 100;

-- Média de preços apenas dos produtos da categoria 'Eletrônicos'
SELECT AVG(preco) FROM produtos WHERE categoria = 'Eletrônicos';

HAVING — filtra depois de agrupar


-- Clientes que gastaram mais de 500 no total
SELECT cliente_id, SUM(valor_total) AS total
FROM pedidos
GROUP BY cliente_id
HAVING SUM(valor_total) > 500;

-- Categorias com mais de 10 produtos
SELECT categoria, COUNT(*) AS qtd
FROM produtos
GROUP BY categoria
HAVING COUNT(*) > 10;

WHERE e HAVING juntos

É comum usar ambos na mesma consulta: WHERE para filtrar linhas antes da agregação e HAVING para filtrar os grupos resultantes. 


-- Clientes que fizeram pelo menos 3 pedidos apenas em 2024
SELECT cliente_id, COUNT(*) AS total_pedidos
FROM pedidos
WHERE data_pedido >= '2024-01-01'
GROUP BY cliente_id
HAVING COUNT(*) >= 3
ORDER BY total_pedidos DESC;

Funções de Janela (Window Functions)

As funções de janela realizam cálculos sobre um conjunto de linhas relacionadas à linha atual, sem reduzir o número de linhas no resultado. Diferente do GROUP BY, que colapsa os grupos em uma única linha, as window functions mantêm cada linha individual e adicionam o resultado do cálculo como uma nova coluna.

A sintaxe básica é: FUNÇÃO() OVER (PARTITION BY coluna ORDER BY coluna)

  • OVER(): define a janela (conjunto de linhas) sobre a qual a função opera
  • PARTITION BY: divide os dados em partições (como um GROUP BY sem colapsar)
  • ORDER BY: ordena as linhas dentro de cada partição

ROW_NUMBER — Numerando linhas

ROW_NUMBER() atribui um número sequencial único para cada linha dentro da sua partição, baseado na ordenação especificada. 


-- Ranking dos funcionários por salário (maior salário = 1)
SELECT nome, salario,
       ROW_NUMBER() OVER (ORDER BY salario DESC) AS ranking
FROM funcionarios;

PARTITION BY — Reiniciando a contagem por grupo


-- Ranking de salário DENTRO de cada departamento
SELECT departamento, nome, salario,
       ROW_NUMBER() OVER (PARTITION BY departamento ORDER BY salario DESC) AS ranking_depto
FROM funcionarios;

RANK e DENSE_RANK — Lidando com empates

RANK() e DENSE_RANK() também ordenam as linhas, mas tratam empates (valores iguais) de forma diferente do ROW_NUMBER:

Função Comportamento com empates Exemplo (valores: 100, 100, 80)
ROW_NUMBER() Números únicos, sem empates 1, 2, 3
RANK() Empates recebem mesma posição; pula números 1, 1, 3
DENSE_RANK() Empates recebem mesma posição; não pula números 1, 1, 2 

-- Comparação entre RANK e DENSE_RANK
SELECT nome, salario,
       RANK() OVER (ORDER BY salario DESC)       AS rank_normal,
       DENSE_RANK() OVER (ORDER BY salario DESC) AS rank_denso
FROM funcionarios;

LAG e LEAD — Acessando linhas vizinhas

LAG() acessa o valor de uma linha anterior dentro da janela. LEAD() acessa o valor da linha posterior. São ideais para comparações sequenciais, como calcular variação entre meses. 


-- Salário atual, anterior e próximo (ordenado por salário)
SELECT nome, salario,
       LAG(salario) OVER (ORDER BY salario)  AS salario_anterior,
       LEAD(salario) OVER (ORDER BY salario) AS salario_proximo
FROM funcionarios;

-- Exemplo prático: variação de vendas mês a mês
SELECT mes, valor_vendas,
       LAG(valor_vendas) OVER (ORDER BY mes) AS vendas_mes_anterior,
       valor_vendas - LAG(valor_vendas) OVER (ORDER BY mes) AS variacao
FROM vendas_mensais;

Funções de agregação como Window Functions

Funções como SUM, AVG e COUNT também podem ser usadas com OVER, produzindo somas acumuladas, médias móveis e contagens progressivas. 


-- Soma acumulada de salários por departamento
SELECT departamento, nome, salario,
       SUM(salario) OVER (PARTITION BY departamento ORDER BY salario) AS soma_acumulada
FROM funcionarios;

-- Média móvel de 3 dias de vendas
SELECT data_venda, valor,
       AVG(valor) OVER (ORDER BY data_venda ROWS BETWEEN 2 PRECEDING AND CURRENT ROW) AS media_movel_3dias
FROM vendas;

Subconsultas (Subqueries)

Uma subconsulta é uma consulta SQL dentro de outra consulta SQL. Elas permitem resolver problemas complexos dividindo-os em etapas. Podem aparecer nas cláusulas SELECT, FROM, WHERE e HAVING.

Subconsulta Escalar — Retorna um único valor

A subconsulta escalar retorna exatamente uma linha e uma coluna. Pode ser usada em qualquer lugar onde um valor único é esperado. 


-- Produto(s) com o maior preço
SELECT nome, preco
FROM produtos
WHERE preco = (SELECT MAX(preco) FROM produtos);

-- Funcionários que ganham acima da média
SELECT nome, salario
FROM funcionarios
WHERE salario > (SELECT AVG(salario) FROM funcionarios);

Subconsulta com IN — Retorna uma lista de valores

Quando a subconsulta retorna múltiplos valores, usamos operadores como IN, NOT IN, ANY ou ALL. 


-- Clientes que já fizeram pelo menos um pedido
SELECT nome FROM clientes
WHERE id IN (SELECT DISTINCT cliente_id FROM pedidos);

-- Produtos que nunca foram vendidos
SELECT nome FROM produtos
WHERE id NOT IN (SELECT DISTINCT produto_id FROM itens_pedido);

Subconsulta com EXISTS — Verifica existência

O operador EXISTS retorna TRUE se a subconsulta retornar pelo menos uma linha. É geralmente mais eficiente que IN para grandes volumes de dados, pois para a execução assim que encontra a primeira correspondência. 


-- Clientes que fizeram pelo menos um pedido (usando EXISTS)
SELECT nome FROM clientes c
WHERE EXISTS (
    SELECT 1 FROM pedidos p
    WHERE p.cliente_id = c.id
);

-- Categorias que possuem produtos com estoque baixo
SELECT nome FROM categorias cat
WHERE EXISTS (
    SELECT 1 FROM produtos p
    WHERE p.categoria_id = cat.id AND p.estoque < 10
);

Subconsulta no FROM — Tabela Derivada

Uma subconsulta na cláusula FROM atua como uma tabela temporária, permitindo consultas sobre o resultado de outra consulta. É obrigatório dar um alias para a subconsulta no FROM. 


-- Categorias cuja média de preço é superior a 100
SELECT *
FROM (
    SELECT categoria, AVG(preco) AS media_preco
    FROM produtos
    GROUP BY categoria
) AS medias
WHERE media_preco > 100;

-- Top 3 clientes que mais gastaram
SELECT nome, total_gasto
FROM (
    SELECT c.nome, SUM(p.valor_total) AS total_gasto
    FROM clientes c
    INNER JOIN pedidos p ON c.id = p.cliente_id
    GROUP BY c.nome
) AS ranking
ORDER BY total_gasto DESC
LIMIT 3;

Subconsulta Correlacionada

Uma subconsulta correlacionada referencia colunas da consulta externa. Ela é reexecutada para cada linha da consulta externa, o que a torna mais lenta, mas extremamente poderosa para certos problemas. 


-- Funcionários que ganham mais que a média do seu próprio departamento
SELECT nome, salario, departamento
FROM funcionarios f1
WHERE salario > (
    SELECT AVG(salario)
    FROM funcionarios f2
    WHERE f2.departamento = f1.departamento
);

Dever de Casa

1. Consulta com COUNT, AVG e SUM agrupada por categoria

Escreva uma consulta que retorne: categoria, quantidade de produtos, preço médio e valor total em estoque (preco * estoque) para cada categoria, ordenado pelo valor total em estoque do maior para o menor.

Gabarito: 

SELECT
    categoria,
    COUNT(*) AS qtd_produtos,
    AVG(preco) AS preco_medio,
    SUM(preco * estoque) AS valor_total_estoque
FROM produtos
GROUP BY categoria
ORDER BY valor_total_estoque DESC;

2. GROUP BY + HAVING filtrando grupos

Escreva uma consulta que retorne as categorias que possuem mais de 5 produtos, mostrando o nome da categoria e a quantidade. Ordene da maior para a menor quantidade.

Gabarito: 

SELECT categoria, COUNT(*) AS quantidade
FROM produtos
GROUP BY categoria
HAVING COUNT(*) > 5
ORDER BY quantidade DESC;

3. ROW_NUMBER particionado por departamento

Escreva uma consulta que numere os funcionários de cada departamento por ordem de salário (do maior para o menor), mostrando departamento, nome, salário e a posição (ranking).

Gabarito: 

SELECT
    departamento,
    nome,
    salario,
    ROW_NUMBER() OVER (PARTITION BY departamento ORDER BY salario DESC) AS posicao
FROM funcionarios
ORDER BY departamento, posicao;

4. Subconsulta para encontrar clientes que compraram mais que a média

Escreva uma consulta que retorne o nome e o total gasto dos clientes cujo total de compras é maior que a média de gastos de todos os clientes.

Gabarito: 

SELECT c.nome, SUM(p.valor_total) AS total_gasto
FROM clientes c
INNER JOIN pedidos p ON c.id = p.cliente_id
GROUP BY c.nome
HAVING SUM(p.valor_total) > (
    SELECT AVG(total_por_cliente)
    FROM (
        SELECT SUM(valor_total) AS total_por_cliente
        FROM pedidos
        GROUP BY cliente_id
    ) AS medias
)
ORDER BY total_gasto DESC;
