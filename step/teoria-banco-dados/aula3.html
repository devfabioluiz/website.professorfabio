Módulo 3: Consultas SELECT, INSERT, UPDATE, DELETE

Tópicos do nosso plano de curso abordados nessa aula: Comando SELECT (cláusulas SELECT, FROM, WHERE, ORDER BY), Palavras-chave IN, BETWEEN, LIKE, Uso de LIMIT, Uso de CASE WHEN, Comando INSERT, Comando UPDATE, Comando DELETE, Transações (BEGIN, COMMIT, ROLLBACK).

O Comando SELECT

O comando SELECT é a principal ferramenta de consulta em SQL. Ele permite recuperar dados de uma ou mais tabelas, filtrar registros, ordenar resultados e muito mais. Sua estrutura básica é: 


SELECT [colunas]
FROM [tabela]
WHERE [condições]
ORDER BY [coluna] [ASC|DESC];

SELECT Simples

Para selecionar todos os registros e todas as colunas de uma tabela, utilize o caractere *:


SELECT * FROM alunos;

Este comando retorna todas as linhas e todas as colunas da tabela alunos.

SELECT com Colunas Específicas

Para consultar apenas colunas específicas, substitua o * pelos nomes das colunas desejadas, separados por vírgula:


SELECT nome, email FROM alunos;

Isso é mais eficiente quando não precisamos de todas as colunas, reduzindo o tráfego de dados.

SELECT com Cláusula WHERE

A cláusula WHERE filtra os registros com base em condições específicas. Apenas as linhas que satisfazem a condição são retornadas: 


SELECT * FROM alunos WHERE idade >= 18;

Retorna apenas os alunos com idade maior ou igual a 18 anos.

SELECT com ORDER BY

A cláusula ORDER BY ordena os resultados. Por padrão, a ordenação é ascendente (ASC). Para ordem descendente, utilize DESC: 


SELECT * FROM alunos ORDER BY nome ASC;
          

SELECT * FROM alunos ORDER BY idade DESC;

Também é possível ordenar por mais de uma coluna:


SELECT * FROM alunos ORDER BY curso ASC, nome ASC;

SELECT com Múltiplas Condições WHERE (AND/OR)

Combine múltiplas condições usando os operadores lógicos AND e OR: 


SELECT * FROM alunos
WHERE idade >= 18 AND curso = 'Python';
          

SELECT * FROM alunos
WHERE curso = 'SQL' OR curso = 'React';
          

SELECT * FROM alunos
WHERE (curso = 'Python' OR curso = 'SQL')
  AND idade >= 18;

Atenção à precedência dos operadores: AND tem prioridade sobre OR. Use parênteses para controlar a ordem de avaliação.

Palavras-chave IN, BETWEEN, LIKE

Estes operadores tornam as consultas mais concisas e expressivas, evitando múltiplas condições OR ou AND desnecessárias.

IN

O operador IN verifica se um valor pertence a uma lista de valores especificados. É equivalente a múltiplas condições OR: 


SELECT * FROM alunos
WHERE curso IN ('Python', 'SQL', 'React');

Também pode ser usado com subconsultas:


SELECT * FROM alunos
WHERE curso_id IN (SELECT id FROM cursos WHERE carga > 40);

Use NOT IN para excluir valores da lista:


SELECT * FROM alunos
WHERE curso NOT IN ('Java', 'C#');

BETWEEN

O operador BETWEEN seleciona valores dentro de um intervalo (inclusivo). É equivalente a valor >= minimo AND valor <= maximo: 


SELECT * FROM alunos
WHERE idade BETWEEN 18 AND 25;

Funciona também com datas e valores textuais:


SELECT * FROM pedidos
WHERE data_pedido BETWEEN '2025-01-01' AND '2025-06-30';
          

SELECT * FROM produtos
WHERE preco NOT BETWEEN 100 AND 500;

LIKE

O operador LIKE busca padrões em colunas de texto. Utiliza dois caracteres curinga:

  • % — representa zero ou mais caracteres
  • _ — representa exatamente um caractere

-- Nomes que começam com "João"
SELECT * FROM alunos WHERE nome LIKE 'João%';
          

-- Emails do Gmail
SELECT * FROM alunos WHERE email LIKE '%@gmail.com';
          

-- Nomes que contêm "Silva" em qualquer posição
SELECT * FROM alunos WHERE nome LIKE '%Silva%';
          

-- Nomes com exatamente 4 letras
SELECT * FROM alunos WHERE nome LIKE '____';

Para buscas case-insensitive no PostgreSQL, utilize ILIKE:


SELECT * FROM alunos WHERE nome ILIKE '%joão%';

LIMIT e CASE WHEN

LIMIT

A cláusula LIMIT restringe o número de registros retornados pela consulta. Muito útil para paginação e para consultar os "Top N" registros: 


-- Os 5 produtos mais caros
SELECT * FROM produtos
ORDER BY preco DESC
LIMIT 5;

Use OFFSET para pular uma quantidade de registros (paginação):


-- Página 2: registros de 6 a 10
SELECT * FROM produtos
ORDER BY preco DESC
LIMIT 5 OFFSET 5;

CASE WHEN

O CASE WHEN permite criar colunas condicionais diretamente na consulta, funcionando como um "if/else" dentro do SQL: 


SELECT
    nome,
    preco,
    CASE
        WHEN preco < 50 THEN 'Barato'
        WHEN preco BETWEEN 50 AND 200 THEN 'Médio'
        ELSE 'Caro'
    END AS categoria_preco
FROM produtos;

Exemplo com múltiplas condições:


SELECT
    nome,
    idade,
    CASE
        WHEN idade < 12 THEN 'Criança'
        WHEN idade BETWEEN 12 AND 17 THEN 'Adolescente'
        WHEN idade BETWEEN 18 AND 60 THEN 'Adulto'
        ELSE 'Idoso'
    END AS faixa_etaria
FROM alunos;

Comando INSERT

O comando INSERT adiciona novos registros a uma tabela. A sintaxe básica especifica a tabela, as colunas e os valores a serem inseridos:

Inserindo um Único Registro


INSERT INTO alunos (nome, email, idade)
VALUES ('Maria', 'maria@email.com', 22);

Se você for fornecer valores para todas as colunas na ordem correta, pode omitir a lista de colunas. Porém, essa prática é desencorajada, pois o código fica frágil a alterações na estrutura da tabela.

Inserindo Múltiplos Registros

Para inserir vários registros de uma só vez, separe os conjuntos de valores com vírgula:


INSERT INTO alunos (nome, email, idade)
VALUES
    ('João', 'joao@email.com', 25),
    ('Ana', 'ana@email.com', 20),
    ('Carlos', 'carlos@email.com', 30);

Inserir múltiplos registros em um único comando é mais eficiente do que vários INSERTs separados.

INSERT com SELECT (copiar dados)

Também é possível inserir dados provenientes de uma consulta:


INSERT INTO alunos_backup (nome, email, idade)
SELECT nome, email, idade FROM alunos
WHERE idade >= 18;

RETURNING

No PostgreSQL, a cláusula RETURNING retorna os valores inseridos:


INSERT INTO alunos (nome, email, idade)
VALUES ('Pedro', 'pedro@email.com', 28)
RETURNING id, nome;

Comando UPDATE

O comando UPDATE modifica registros existentes em uma tabela. A cláusula WHERE é fundamental para especificar quais linhas serão afetadas.

⚠️ Cuidado!

Nunca execute um UPDATE sem WHERE a menos que realmente queira atualizar todos os registros da tabela. Um UPDATE sem filtro altera todas as linhas!

Atualizando um Registro Específico


UPDATE alunos
SET email = 'novo@email.com'
WHERE id = 1;

Atualizando Múltiplas Colunas


UPDATE alunos
SET email = 'contato@email.com',
    idade = 26
WHERE id = 2;

Atualizando com Expressões

É possível usar o valor atual da coluna na expressão de atualização:


-- Aumentar em 10% o preço dos eletrônicos
UPDATE produtos
SET preco = preco * 1.1
WHERE categoria = 'Eletrônicos';

Atualizando com Condições Complexas


UPDATE alunos
SET status = 'Inativo'
WHERE idade < 18
   OR data_ultimo_acesso < '2024-01-01';

RETURNING no UPDATE

Assim como no INSERT, o PostgreSQL permite usar RETURNING para ver os dados após a atualização:


UPDATE produtos
SET preco = preco * 0.9
WHERE categoria = 'Livros'
RETURNING id, nome, preco;

Comando DELETE

O comando DELETE remove registros de uma tabela. Assim como no UPDATE, a cláusula WHERE é essencial para evitar a remoção de todos os dados.

⚠️ Atenção!

DELETE sem WHERE remove TODOS os registros da tabela! Use com extrema cautela. Para remover todos os registros de forma mais eficiente (sem registros de log), prefira TRUNCATE TABLE nome_tabela;.

Removendo um Registro Específico


DELETE FROM alunos WHERE id = 5;

Removendo com Condição


DELETE FROM alunos WHERE idade < 18;

Removendo com Subconsulta


DELETE FROM alunos
WHERE curso_id IN (
    SELECT id FROM cursos WHERE nome = 'Curso Cancelado'
);

RETURNING no DELETE

O PostgreSQL permite usar RETURNING para saber quais registros foram removidos:


DELETE FROM alunos
WHERE idade < 18
RETURNING id, nome, idade;

Transações no PostgreSQL

Uma transação é uma unidade lógica de trabalho que agrupa uma ou mais operações SQL. As transações seguem o princípio ACID:

  • Atomicidade (Atomicity): ou todas as operações são executadas com sucesso, ou nenhuma é aplicada
  • Consistência (Consistency): a transação leva o banco de um estado consistente a outro
  • Isolamento (Isolation): transações concorrentes não interferem entre si
  • Durabilidade (Durability): uma vez confirmada, os efeitos da transação são permanentes

BEGIN, COMMIT e ROLLBACK

Os comandos fundamentais de controle de transação são:


-- Inicia uma transação
BEGIN;

-- Operações da transação
UPDATE contas SET saldo = saldo - 500 WHERE id = 1;
UPDATE contas SET saldo = saldo + 500 WHERE id = 2;

-- Se tudo deu certo, confirma
COMMIT;

-- Se algo deu errado, desfaz
-- ROLLBACK;

Exemplo com ROLLBACK em caso de erro:


BEGIN;

UPDATE produtos SET estoque = estoque - 10 WHERE id = 1;
-- Suponha que a próxima operação falhe
UPDATE produtos SET estoque = estoque + 10 WHERE id = 999;

-- Desfaz todas as alterações desde o BEGIN
ROLLBACK;

SAVEPOINT

SAVEPOINT cria pontos de restauração parciais dentro de uma transação. Permite reverter apenas parte das operações sem desfazer a transação inteira: 


BEGIN;

INSERT INTO alunos (nome, email, idade)
VALUES ('Roberto', 'roberto@email.com', 23);

-- Cria um ponto de salvamento
SAVEPOINT sp1;

UPDATE alunos SET idade = 24 WHERE nome = 'Roberto';

-- Suponha que a atualização foi errada
ROLLBACK TO SAVEPOINT sp1;

-- O INSERT permanece; o UPDATE foi desfeito
COMMIT;

Observação: no PostgreSQL, os comandos BEGIN, COMMIT e ROLLBACK são equivalentes a START TRANSACTION, COMMIT WORK e ROLLBACK WORK respectivamente.

Passo a passo: Criando e populando uma tabela

Vamos colocar em prática tudo o que aprendemos. Criaremos uma tabela funcionarios e executaremos diversas operações.

Passo 1 — Criar a tabela


CREATE TABLE funcionarios (
    id          SERIAL PRIMARY KEY,
    nome        VARCHAR(100) NOT NULL,
    cargo       VARCHAR(50) NOT NULL,
    salario     NUMERIC(10, 2) NOT NULL,
    departamento VARCHAR(50) NOT NULL,
    data_admissao DATE DEFAULT CURRENT_DATE
);

Passo 2 — Inserir dados


INSERT INTO funcionarios (nome, cargo, salario, departamento)
VALUES
    ('Ana Souza',    'Desenvolvedora', 4500.00, 'TI'),
    ('Carlos Lima',  'Analista',       3800.00, 'TI'),
    ('Beatriz Alves','Gerente',        6500.00, 'RH'),
    ('Daniel Moreira','Analista',      3200.00, 'Financeiro'),
    ('Eduarda Costa', 'DevOps',        7200.00, 'TI'),
    ('Felipe Rocha', 'Assistente',     2500.00, 'RH'),
    ('Gabriela Nunes','Contadora',     5000.00, 'Financeiro');

Passo 3 — Consultas de exemplo

Selecionar todos os funcionários:


SELECT * FROM funcionarios;

Funcionários do departamento de TI com salário acima de 4000:


SELECT nome, cargo, salario
FROM funcionarios
WHERE departamento = 'TI'
  AND salario > 4000
ORDER BY salario DESC;

Classificação salarial com CASE WHEN:


SELECT
    nome,
    cargo,
    salario,
    CASE
        WHEN salario < 3000 THEN 'Júnior'
        WHEN salario BETWEEN 3000 AND 6000 THEN 'Pleno'
        ELSE 'Sênior'
    END AS nivel
FROM funcionarios
ORDER BY salario DESC;

Funcionários contratados nos últimos 30 dias:


SELECT * FROM funcionarios
WHERE data_admissao >= CURRENT_DATE - INTERVAL '30 days';

Passo 4 — Atualizações e remoções com transação


BEGIN;

-- Aumentar salário do pessoal de TI em 5%
UPDATE funcionarios
SET salario = salario * 1.05
WHERE departamento = 'TI'
RETURNING nome, salario;

-- Remover assistentes com salário abaixo de 3000
DELETE FROM funcionarios
WHERE cargo = 'Assistente' AND salario < 3000
RETURNING nome;

COMMIT;

Dever de Casa

1. Insira 5 registros na tabela funcionarios e selecione apenas os que têm salário > 3000

Crie 5 novos registros variando cargos, salários e departamentos. Em seguida, escreva uma consulta que retorne apenas os funcionários com salário acima de R$ 3.000,00.

Gabarito: 

INSERT INTO funcionarios (nome, cargo, salario, departamento)
VALUES
    ('Hugo Torres',   'Designer',        2800.00, 'Marketing'),
    ('Iara Mendes',   'Coordenadora',    5500.00, 'RH'),
    ('Jonas Pereira', 'Desenvolvedor',   6200.00, 'TI'),
    ('Karen Duarte',  'Estagiária',      1500.00, 'Financeiro'),
    ('Lucas Braga',   'Analista Senior', 8000.00, 'TI');

SELECT * FROM funcionarios
WHERE salario > 3000
ORDER BY salario DESC;

2. Atualize o salário de um funcionário específico e verifique com SELECT

Escolha um funcionário pelo nome e aumente seu salário em 15%. Confirme a alteração com um SELECT.

Gabarito: 

BEGIN;

UPDATE funcionarios
SET salario = salario * 1.15
WHERE nome = 'Iara Mendes'
RETURNING nome, salario;

-- Verificar se a alteração foi aplicada
SELECT nome, salario FROM funcionarios
WHERE nome = 'Iara Mendes';

COMMIT;

3. Delete funcionários de um departamento específico

Remova todos os funcionários do departamento de Marketing. Antes de executar o DELETE, faça um SELECT para conferir quais registros serão afetados.

Gabarito: 

-- Primeiro, confira quais registros serão deletados
SELECT * FROM funcionarios
WHERE departamento = 'Marketing';

-- Em seguida, execute a remoção
BEGIN;

DELETE FROM funcionarios
WHERE departamento = 'Marketing'
RETURNING id, nome;

-- Se a remoção estiver correta, confirme
COMMIT;
-- Caso contrário, desfaça:
-- ROLLBACK;

4. Crie uma consulta com CASE WHEN classificando funcionários por faixa salarial (Júnior, Pleno, Sênior)

Escreva uma consulta que retorne nome, departamento, salário e uma coluna de classificação seguindo os critérios: salário até R$ 3.000 = Júnior, entre R$ 3.001 e R$ 6.000 = Pleno, acima de R$ 6.000 = Sênior.

Gabarito: 

SELECT
    nome,
    departamento,
    salario,
    CASE
        WHEN salario <= 3000 THEN 'Júnior'
        WHEN salario BETWEEN 3001 AND 6000 THEN 'Pleno'
        ELSE 'Sênior'
    END AS classificacao
FROM funcionarios
ORDER BY salario ASC;
