Docker Compose

1. Fundamentos do Docker Compose

a. História do Docker Compose

O Docker Compose surgiu para facilitar a execução de aplicações que precisam de mais de um container.

Antes dele, era comum executar vários comandos docker run para subir banco de dados, backend, frontend, servidores web e outros serviços.

Com o Docker Compose, conseguimos descrever todos esses serviços em um único arquivo chamado docker-compose.yml.

b. Para que é utilizado o Docker Compose?

O Docker Compose é utilizado para definir, configurar e executar aplicações multi-container.

Ele é muito usado em ambientes de desenvolvimento, testes, estudos e também em alguns cenários controlados de produção.

Cenários comuns de uso:

  • Aplicação Node.js com banco PostgreSQL
  • Frontend com backend e banco de dados
  • API com Redis para cache
  • Ambiente local semelhante ao ambiente de produção
  • Execução de vários serviços com apenas um comando

2. Instalação do Docker Compose

a. Instalação no Windows

No Windows, o Docker Compose normalmente já vem instalado junto com o Docker Desktop.

Para verificar, execute:


docker compose version

Se o comando retornar a versão instalada, o Docker Compose está pronto para uso.

b. Instalação no Linux e MacOS

No MacOS, o Docker Compose também costuma vir junto com o Docker Desktop.

No Linux, dependendo da instalação, pode ser necessário instalar o plugin do Compose. 


sudo apt update
sudo apt install docker-compose-plugin
docker compose version

3. Estrutura do arquivo docker-compose.yml

O arquivo docker-compose.yml é escrito em YAML e descreve os serviços, imagens, portas, volumes, redes e variáveis de ambiente da aplicação.

Exemplo simples:


services:
  web:
    image: nginx
    ports:
      - "8080:80"

Nesse exemplo, criamos um serviço chamado web usando a imagem nginx e mapeando a porta 8080 do computador para a porta 80 do container.

a. Diferença entre versões do docker-compose.yml

i. Versão 1 — formato obsoleto

A versão 1 do formato do Compose era mais simples e antiga. Ela não oferecia o mesmo suporte moderno para redes, volumes e organização de serviços.

Atualmente, seu uso não é recomendado.

ii. Versão 2 — formato moderno

A versão 2 trouxe melhor suporte para redes, volumes e configuração de múltiplos serviços.

Hoje, em muitos projetos atuais, o Compose permite usar o arquivo sem declarar explicitamente a propriedade version.

O importante é organizar bem os serviços dentro da chave services.

4. Principais campos do docker-compose.yml

services

Define os containers que farão parte da aplicação.

image

Define uma imagem pronta que será usada pelo serviço.

build

Indica que a imagem será construída a partir de um Dockerfile.

ports

Faz o mapeamento de portas entre o computador e o container.

volumes

Define persistência de dados ou montagem de diretórios.

environment

Define variáveis de ambiente usadas pelos containers.

depends_on

Indica dependência de inicialização entre serviços.

networks

Define redes personalizadas para comunicação entre containers.

Passo a Passo: Criando uma aplicação multi-container

Passo 0 — Criar branch da aula

No repositório projeto-docker-step, crie uma nova branch: 


git checkout -b aula6-docker-compose

Passo 1 — Criar a estrutura de pastas

Crie a seguinte estrutura no projeto:


mkdir aula6-compose
cd aula6-compose
mkdir app

Passo 2 — Criar package.json

Dentro da pasta app, crie o arquivo package.json. 


{
  "name": "app-compose",
  "version": "1.0.0",
  "main": "index.js",
  "scripts": {
    "start": "node index.js"
  },
  "dependencies": {
    "express": "^4.18.2",
    "pg": "^8.11.3"
  }
}

Passo 3 — Criar aplicação Node.js

Ainda dentro da pasta app, crie o arquivo index.js. 


const express = require('express');
const { Client } = require('pg');

const app = express();
const port = 3000;

const client = new Client({
  host: 'db',
  user: 'postgres',
  password: 'postgres',
  database: 'aula_docker',
  port: 5432,
});

client.connect()
  .then(() => console.log('Conectado ao PostgreSQL'))
  .catch((erro) => console.log('Erro ao conectar no banco:', erro.message));

app.get('/', (req, res) => {
  res.send(`
    <h1>Aplicação multi-container com Docker Compose</h1>
    <p>Backend Node.js rodando junto com PostgreSQL.</p>
  `);
});

app.get('/banco', async (req, res) => {
  try {
    const resultado = await client.query('SELECT NOW()');
    res.json({
      mensagem: 'Banco conectado com sucesso!',
      horario: resultado.rows[0].now
    });
  } catch (erro) {
    res.status(500).json({
      mensagem: 'Erro ao consultar o banco',
      erro: erro.message
    });
  }
});

app.listen(port, () => {
  console.log(`Aplicação rodando na porta ${port}`);
});

Passo 4 — Criar Dockerfile da aplicação

Dentro da pasta app, crie o arquivo Dockerfile. 


FROM node:20-alpine

WORKDIR /app

COPY package*.json ./

RUN npm install

COPY . .

EXPOSE 3000

CMD ["npm", "start"]

Passo 5 — Criar arquivo .dockerignore

Dentro da pasta app, crie o arquivo .dockerignore. 


node_modules
npm-debug.log
.git
.env

Passo 6 — Criar docker-compose.yml

Agora volte para a pasta aula6-compose e crie o arquivo docker-compose.yml. 


services:
  app:
    build: ./app
    container_name: app-node-compose
    ports:
      - "3000:3000"
    depends_on:
      - db
    environment:
      DB_HOST: db
      DB_USER: postgres
      DB_PASSWORD: postgres
      DB_NAME: aula_docker

  db:
    image: postgres:16
    container_name: banco-postgres-compose
    restart: always
    environment:
      POSTGRES_USER: postgres
      POSTGRES_PASSWORD: postgres
      POSTGRES_DB: aula_docker
    ports:
      - "5432:5432"
    volumes:
      - dados-postgres:/var/lib/postgresql/data

volumes:
  dados-postgres:

Passo 7 — Entendendo o docker-compose.yml

  • app: serviço da aplicação Node.js.
  • build: ./app: cria a imagem usando o Dockerfile da pasta app.
  • ports: libera a porta 3000 para acesso no navegador.
  • depends_on: indica que o serviço app depende do serviço db.
  • db: serviço do banco PostgreSQL.
  • volumes: mantém os dados do banco mesmo se o container for removido.

Passo 8 — Subir os containers

Na pasta onde está o arquivo docker-compose.yml, execute: 


docker compose up

Para executar em segundo plano:


docker compose up -d

Passo 9 — Testar no navegador

Acesse:


http://localhost:3000

Depois teste também:


http://localhost:3000/banco

Passo 10 — Listar serviços do Compose


docker compose ps

Passo 11 — Ver logs


docker compose logs
docker compose logs app
docker compose logs db

Passo 12 — Parar os containers


docker compose stop

Passo 13 — Subir novamente


docker compose start

Passo 14 — Remover containers da aplicação


docker compose down

Para remover também os volumes criados:


docker compose down -v

Último Passo — Subindo para o Git


git add .
git commit -m "aula 6 docker compose"
git push -u origin aula6-docker-compose

Dever de Casa

Criar aplicação multi-container com Docker Compose

Crie uma branch chamada dever-docker-compose.

Dentro dela, crie uma aplicação usando Docker Compose com dois serviços:

  • Um serviço Node.js chamado app
  • Um serviço PostgreSQL chamado db

A aplicação Node.js deve ter uma rota principal / exibindo uma mensagem em HTML e uma rota /status retornando um JSON simples.

O banco PostgreSQL deve usar volume nomeado para persistência dos dados.

Obs: Não se esqueça de subir as alterações para o Git na branch do dever.

Gabarito:

1. Criar branch


git checkout -b dever-docker-compose

2. Criar estrutura


mkdir dever-compose
cd dever-compose
mkdir app

3. Criar app/package.json


{
  "name": "dever-compose",
  "version": "1.0.0",
  "main": "index.js",
  "scripts": {
    "start": "node index.js"
  },
  "dependencies": {
    "express": "^4.18.2"
  }
}

4. Criar app/index.js


const express = require('express');

const app = express();
const port = 3000;

app.get('/', (req, res) => {
  res.send(`
    <h1>Dever Docker Compose</h1>
    <p>Aplicação Node.js rodando com Docker Compose.</p>
  `);
});

app.get('/status', (req, res) => {
  res.json({
    status: 'ok',
    mensagem: 'Aplicação funcionando corretamente'
  });
});

app.listen(port, () => {
  console.log(`Servidor rodando na porta ${port}`);
});

5. Criar app/Dockerfile


FROM node:20-alpine

WORKDIR /app

COPY package*.json ./

RUN npm install

COPY . .

EXPOSE 3000

CMD ["npm", "start"]

6. Criar docker-compose.yml


services:
  app:
    build: ./app
    container_name: dever-app-compose
    ports:
      - "3000:3000"
    depends_on:
      - db

  db:
    image: postgres:16
    container_name: dever-db-compose
    environment:
      POSTGRES_USER: postgres
      POSTGRES_PASSWORD: postgres
      POSTGRES_DB: dever_docker
    volumes:
      - dever-postgres:/var/lib/postgresql/data

volumes:
  dever-postgres:

7. Executar aplicação


docker compose up -d

8. Testar no navegador


http://localhost:3000
http://localhost:3000/status

9. Parar e remover


docker compose down

10. Subir para o Git


git add .
git commit -m "dever docker compose"
git push -u origin dever-docker-compose
