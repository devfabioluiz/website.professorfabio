Fundamentos e Implantação

Kubernetes

1. História do surgimento do Kubernetes

O Kubernetes surgiu no Google, inspirado em um sistema interno chamado Borg, que já era utilizado para gerenciar containers em larga escala.

Com o crescimento do uso de containers, especialmente com Docker, surgiu a necessidade de uma ferramenta que automatizasse a orquestração desses containers.

Assim nasceu o Kubernetes, uma plataforma open source para gerenciar, escalar e manter aplicações em containers.

2. Quando utilizar (e quando não utilizar) Kubernetes

Quando utilizar:

  • Aplicações com muitos containers
  • Arquitetura de microsserviços
  • Necessidade de alta disponibilidade
  • Escalabilidade automática
  • Ambientes distribuídos

Quando não utilizar:

  • Projetos pequenos ou simples
  • Aplicações monolíticas simples
  • Ambientes com poucos recursos
  • Quando Docker Compose já resolve

Kubernetes adiciona complexidade. Nem sempre é a melhor escolha para todos os cenários.

3. Principais termos e conceitos

a. Nodes

Máquinas (físicas ou virtuais) que executam containers.

b. Pods

Unidade básica do Kubernetes. Um Pod pode conter um ou mais containers.

c. Replication Controllers

Garantem que um número específico de Pods esteja sempre em execução.

d. Services

Permitem acessar Pods de forma estável, mesmo que eles sejam recriados.

e. Volumes

Permitem persistência de dados entre Pods.

f. Labels

Metadados usados para organizar e selecionar recursos.

g. Kubectl

Ferramenta de linha de comando usada para interagir com o Kubernetes. 


kubectl get pods
kubectl get nodes

4. Arquitetura do Kubernetes

a. Node

Máquina onde os containers são executados.

b. Kubelet

Agente que roda em cada node e garante que os containers estejam funcionando.

c. Kube-Proxy

Gerencia a comunicação de rede entre Pods e serviços.

d. Componentes de controle

i. Etcd

Banco de dados chave-valor que armazena o estado do cluster.

ii. API Server

Porta de entrada do Kubernetes. Todos os comandos passam por ele.

iii. Scheduler

Decide em qual node um Pod será executado.

e. Controller Manager

Controla o estado desejado do cluster, garantindo que tudo funcione como esperado.

5. Planejamento e implantação

a. Hardware físico

Utilizado em grandes empresas com infraestrutura própria.

b. Máquinas virtuais

Muito usado para testes e ambientes corporativos.

c. Nuvem

Forma mais comum atualmente. Serviços como AWS, Azure e Google Cloud oferecem Kubernetes gerenciado.

Exemplos:

  • EKS (AWS)
  • AKS (Azure)
  • GKE (Google Cloud)

Passo a Passo: Executando Kubernetes local

Passo 0 — Criar branch


git checkout -b aula7-kubernetes

Passo 1 — Instalar Minikube

O Minikube permite rodar um cluster Kubernetes localmente.


minikube start

Passo 2 — Verificar cluster


kubectl get nodes

Passo 3 — Criar deployment


kubectl create deployment nginx-deploy --image=nginx

Passo 4 — Ver pods


kubectl get pods

Passo 5 — Expor serviço


kubectl expose deployment nginx-deploy --type=NodePort --port=80

Passo 6 — Acessar aplicação


minikube service nginx-deploy

Passo 7 — Escalar aplicação


kubectl scale deployment nginx-deploy --replicas=3

Passo 8 — Ver pods novamente


kubectl get pods

Passo 9 — Remover recursos


kubectl delete deployment nginx-deploy
kubectl delete service nginx-deploy

Último Passo — Subir para o Git


git add .
git commit -m "aula 7 kubernetes"
git push -u origin aula7-kubernetes
