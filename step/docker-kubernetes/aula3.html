Redes no Docker

1. Modelo de rede no Docker

O Docker possui um modelo de rede próprio que permite que containers se comuniquem entre si e também com o mundo externo.

Quando instalamos o Docker, ele já cria automaticamente uma rede padrão chamada bridge.

Essa rede funciona como um "switch virtual", permitindo que containers conectados a ela se comuniquem usando IP interno.

Por padrão:

  • Todo container recebe um IP interno
  • Containers na mesma rede conseguem se comunicar
  • Para acessar externamente, usamos mapeamento de portas

Configuração do modelo de rede padrão

Podemos visualizar a rede padrão com o comando:


docker network ls

2. Interface de rede virtual

Cada container possui uma interface de rede virtual, semelhante a uma placa de rede em um computador real.

Essa interface permite que o container:

  • Receba um endereço IP
  • Se comunique com outros containers
  • Acesse redes externas

Esse isolamento garante segurança e organização dentro do ambiente Docker.

3. Drivers de rede disponíveis no Docker

a. bridge

É o driver padrão do Docker. Permite comunicação entre containers dentro da mesma máquina.

b. macvlan

Permite que containers tenham um IP próprio na rede física, como se fossem dispositivos reais na rede.

c. host

O container compartilha a rede do host, ou seja, não há isolamento de rede.

d. none

O container não possui rede. Ele fica completamente isolado.

4. Comandos para configuração de rede no Docker

a. docker network ls

Lista todas as redes disponíveis.


docker network ls

b. docker network inspect

Mostra detalhes de uma rede específica.


docker network inspect bridge

c. docker network create

Cria uma nova rede.


docker network create minha-rede

d. docker network connect

Conecta um container a uma rede.


docker network connect minha-rede meu-container

e. docker network disconnect

Remove um container de uma rede.


docker network disconnect minha-rede meu-container

f. docker network rm

Remove uma rede.


docker network rm minha-rede

g. docker network prune

Remove redes não utilizadas.


docker network prune

5. Mapeamento de portas no Docker

O mapeamento de portas permite acessar um container a partir do seu computador.

Exemplo:


docker run -p 8080:80 nginx

Nesse caso:

  • 8080 → porta do seu computador
  • 80 → porta do container

Ao acessar http://localhost:8080, você estará acessando o container.

Passo a Passo: Trabalhando com redes Docker

Passo 0 — Criar branch no GitHub

Crie uma nova branch chamada:


aula3-redes-docker

No terminal:


git fetch
git checkout aula3-redes-docker

Passo 1 — Listar redes existentes


docker network ls

Passo 2 — Criar uma rede personalizada


docker network create rede-app

Passo 3 — Criar dois containers na mesma rede

Container 1:


docker run -d --name app1 --network rede-app nginx

Container 2:


docker run -d --name app2 --network rede-app nginx

Passo 4 — Testar comunicação entre containers

Vamos acessar o container app1 e tentar comunicar com o app2. 


docker exec -it app1 bash
apt update
apt install curl -y
curl http://app2

Se tudo estiver correto, o container app1 conseguirá acessar o app2 usando o nome do container.

Passo 5 — Criar um container com mapeamento de porta


docker run -d -p 8080:80 --name web-app nginx

Acesse:


http://localhost:8080

Passo 6 — Inspecionar rede


docker network inspect rede-app

Passo 7 — Remover containers e rede


docker stop app1 app2 web-app
docker rm app1 app2 web-app
docker network rm rede-app

Último Passo — Subindo para o Git


git add .
git commit -m "aula 3 redes docker"
git push -u origin aula3-redes-docker

Dever de Casa

Criar comunicação entre containers

Crie uma rede chamada rede-dever e dois containers:

  • um container nginx
  • um container ubuntu

O container ubuntu deve acessar o nginx via nome do container.

Obs: Criar branch dever-redes-docker e subir no Git.

Gabarito: 

git checkout -b dever-redes-docker

docker network create rede-dever

docker run -d --name nginx-dever --network rede-dever nginx

docker run -it --name ubuntu-dever --network rede-dever ubuntu bash

apt update
apt install curl -y
curl http://nginx-dever

docker stop nginx-dever ubuntu-dever
docker rm nginx-dever ubuntu-dever
docker network rm rede-dever

git add .
git commit -m "dever redes docker"
git push -u origin dever-redes-docker
