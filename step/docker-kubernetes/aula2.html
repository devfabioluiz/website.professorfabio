Trabalhando com Containers Docker

1. Comandos básicos para trabalhar com containers

Depois de entender o que é Docker, imagem e container, precisamos aprender os principais comandos usados no dia a dia.

Um container é uma aplicação em execução a partir de uma imagem. Por isso, os comandos de container servem para criar, executar, parar, reiniciar, remover e consultar aplicações em execução.

a. docker ps

O comando docker ps mostra os containers que estão em execução no momento. 


docker ps

Para listar também os containers parados, usamos:


docker ps -a

b. docker run

O comando docker run cria e executa um container a partir de uma imagem. 


docker run hello-world

Também podemos executar uma aplicação mapeando portas:


docker run -p 3000:3000 minha-primeira-app-docker

O parâmetro -p conecta uma porta do computador com uma porta do container.

c. docker rm

O comando docker rm remove containers parados.


docker rm ID_DO_CONTAINER

Para remover um container que ainda está rodando, primeiro pare o container ou use o parâmetro -f. 


docker rm -f ID_DO_CONTAINER

d. docker logs

O comando docker logs mostra os logs gerados por um container. 


docker logs ID_DO_CONTAINER

Esse comando é muito usado para verificar erros, mensagens do servidor e informações de inicialização da aplicação.

e. docker start, stop e restart

O comando docker start inicia um container que está parado. 


docker start ID_DO_CONTAINER

O comando docker stop para um container em execução.


docker stop ID_DO_CONTAINER

O comando docker restart reinicia um container.


docker restart ID_DO_CONTAINER

f. Outros comandos úteis para containers

Também existem outros comandos importantes para o uso diário do Docker.

  • docker inspect: mostra detalhes técnicos de um container.
  • docker exec: executa comandos dentro de um container em execução.
  • docker rename: renomeia um container.
  • docker stats: mostra uso de CPU, memória e rede dos containers. 

docker inspect ID_DO_CONTAINER
docker exec -it ID_DO_CONTAINER bash
docker stats

Comandos básicos para trabalhar com imagens

2. O que são imagens Docker?

Uma imagem Docker é um modelo usado para criar containers. Ela contém os arquivos, dependências e configurações necessárias para executar uma aplicação.

Podemos baixar imagens prontas do Docker Hub ou criar nossas próprias imagens usando um arquivo chamado Dockerfile.

a. docker images

O comando docker images lista as imagens disponíveis no computador. 


docker images

b. docker search

O comando docker search pesquisa imagens disponíveis no Docker Hub. 


docker search nginx
docker search node

c. docker pull

O comando docker pull baixa uma imagem para o computador. 


docker pull nginx
docker pull node:20

d. docker build

O comando docker build cria uma imagem a partir de um Dockerfile. 


docker build -t nome-da-imagem .

O parâmetro -t define o nome da imagem. O ponto final indica que o Dockerfile está na pasta atual.

e. docker rmi

O comando docker rmi remove uma imagem Docker.


docker rmi NOME_DA_IMAGEM

Caso exista algum container usando essa imagem, talvez seja necessário remover o container antes.

f. Outros comandos úteis para imagens

  • docker image inspect: mostra detalhes técnicos de uma imagem.
  • docker tag: cria uma nova tag para uma imagem.
  • docker history: mostra as camadas de uma imagem.
  • docker image prune: remove imagens não utilizadas.

docker image inspect node:20
docker history node:20
docker image prune

Passo a Passo: Trabalhando com containers e imagens

Passo 0 — Criar uma nova branch no GitHub

Acesse o repositório projeto-docker-step no GitHub e crie uma nova branch baseada na branch anterior.

O nome da branch será:


aula2-containers-imagens

Depois, no terminal do VSCode, dentro da pasta do projeto, execute:


git fetch
git checkout aula2-containers-imagens

Passo 1 — Verificar a versão do Docker

Antes de começar, confirme se o Docker está funcionando.


docker --version

Passo 2 — Executar um container simples

Execute um container de teste usando a imagem hello-world. 


docker run hello-world

Esse comando baixa a imagem, cria o container, executa a mensagem de teste e finaliza o container.

Passo 3 — Listar containers

Agora liste apenas os containers em execução.


docker ps

Como o container hello-world já terminou, ele provavelmente não aparecerá nessa lista.

Para listar todos os containers, inclusive os parados, use:


docker ps -a

Passo 4 — Baixar uma imagem do Docker Hub

Agora vamos baixar a imagem do Nginx.


docker pull nginx

O Nginx é um servidor web muito usado para servir páginas e aplicações.

Passo 5 — Listar imagens baixadas

Para verificar se a imagem foi baixada, execute:


docker images

Passo 6 — Executar o Nginx em um container

Agora vamos executar o Nginx na porta 8080 do computador.


docker run -d -p 8080:80 --name servidor-nginx nginx

Nesse comando:

  • -d: executa o container em segundo plano.
  • -p 8080:80: conecta a porta 8080 do computador com a porta 80 do container.
  • --name servidor-nginx: define um nome para o container.
  • nginx: é a imagem usada para criar o container.

Depois acesse no navegador:


http://localhost:8080

Passo 7 — Verificar logs do container

Para visualizar os logs do container, execute:


docker logs servidor-nginx

Passo 8 — Parar o container

Para parar o container do Nginx, execute:


docker stop servidor-nginx

Passo 9 — Iniciar novamente o container

Para iniciar novamente o mesmo container, execute:


docker start servidor-nginx

Passo 10 — Reiniciar o container

Para reiniciar o container, execute:


docker restart servidor-nginx

Passo 11 — Remover o container

Primeiro, pare o container:


docker stop servidor-nginx

Depois, remova o container:


docker rm servidor-nginx

Passo 12 — Criar uma aplicação simples com HTML

Crie uma pasta chamada site.


mkdir site

Dentro dela, crie um arquivo chamado index.html.


<!DOCTYPE html>
<html lang="pt-BR">
  <head>
    <meta charset="UTF-8" />
    <title>Site com Docker e Nginx</title>
  </head>
  <body>
    <h1>Meu site rodando com Docker</h1>
    <p>Esta página está sendo servida por um container Nginx.</p>

    <h2>Comandos praticados</h2>
    <ul>
      <li>docker run</li>
      <li>docker ps</li>
      <li>docker logs</li>
      <li>docker stop</li>
      <li>docker start</li>
      <li>docker rm</li>
    </ul>
  </body>
</html>

Passo 13 — Rodar o site com Nginx usando volume

Agora vamos executar o Nginx usando o arquivo HTML criado no projeto. 


docker run -d -p 8080:80 --name site-nginx -v ${PWD}/site:/usr/share/nginx/html nginx

No Windows PowerShell, caso o comando acima não funcione, use:


docker run -d -p 8080:80 --name site-nginx -v ${PWD}\site:/usr/share/nginx/html nginx

Acesse:


http://localhost:8080

Passo 14 — Remover imagem

Primeiro, pare e remova o container:


docker stop site-nginx
docker rm site-nginx

Depois, remova a imagem:


docker rmi nginx

Último Passo — Subindo para o Git

Agora vamos salvar os arquivos criados na branch da aula.


git add .
git commit -m "aula 2 containers e imagens docker"
git push -u origin aula2-containers-imagens

Dever de Casa

Criar um site simples e executá-lo em um container Nginx

Crie uma nova branch chamada dever-aula2-nginx.

Dentro do projeto, crie uma pasta chamada atividade-nginx com um arquivo index.html.

O arquivo HTML deve conter:

  • Um título principal
  • Um parágrafo explicando o que é um container
  • Uma lista com pelo menos 5 comandos Docker estudados nesta aula
  • Uma pequena seção explicando a diferença entre imagem e container

Depois, execute esse HTML usando um container Nginx na porta 8080.

Obs: Não se esqueça de subir as alterações para o Git na branch do dever.

Gabarito:

1. Criar a branch 


git checkout -b dever-aula2-nginx

2. Criar a pasta da atividade 


mkdir atividade-nginx

3. Criar o arquivo atividade-nginx/index.html 


<!DOCTYPE html>
<html lang="pt-BR">
  <head>
    <meta charset="UTF-8" />
    <title>Atividade Docker Nginx</title>
  </head>
  <body>
    <h1>Atividade com Docker e Nginx</h1>

    <p>
      Um container é um ambiente isolado usado para executar uma aplicação
      junto com suas dependências.
    </p>

    <h2>Comandos Docker estudados</h2>

    <ul>
      <li>docker ps</li>
      <li>docker run</li>
      <li>docker rm</li>
      <li>docker logs</li>
      <li>docker start</li>
      <li>docker stop</li>
      <li>docker images</li>
      <li>docker pull</li>
    </ul>

    <h2>Imagem x Container</h2>

    <p>
      A imagem é o modelo usado para criar containers. O container é a imagem em
      execução.
    </p>
  </body>
</html>

4. Executar o arquivo com Nginx 


docker run -d -p 8080:80 --name atividade-nginx -v ${PWD}/atividade-nginx:/usr/share/nginx/html nginx

No Windows PowerShell, caso necessário, use:


docker run -d -p 8080:80 --name atividade-nginx -v ${PWD}\atividade-nginx:/usr/share/nginx/html nginx

5. Acessar no navegador 


http://localhost:8080

6. Verificar containers e logs 


docker ps
docker logs atividade-nginx

7. Parar e remover o container 


docker stop atividade-nginx
docker rm atividade-nginx

8. Subir para o GitHub 


git add .
git commit -m "dever aula 2 docker nginx"
git push -u origin dever-aula2-nginx
