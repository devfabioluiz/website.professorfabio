Volumes no Docker

1. Formas de armazenamento de dados no Docker

Por padrão, os dados dentro de um container são temporários. Se o container for removido, os dados também são perdidos.

Para resolver isso, o Docker oferece diferentes formas de persistência de dados.

a. Volumes de dados (docker volumes)

São a forma mais recomendada de persistência no Docker.

O Docker gerencia esses volumes automaticamente e garante melhor desempenho e segurança.

b. Bind mount (montagem de diretório do host)

Permite mapear uma pasta do computador diretamente para dentro do container.

Muito usado em desenvolvimento, pois permite editar arquivos localmente e refletir no container.

c. Named pipes (Windows)

Usado para comunicação entre processos no Windows. É um caso mais específico e pouco utilizado em aplicações comuns.

d. tmpfs (Linux)

Armazena dados em memória (RAM). É muito rápido, mas os dados são perdidos quando o container é parado.

2. Propriedades dos volumes

  • Persistem mesmo após o container ser removido
  • Podem ser compartilhados entre containers
  • São gerenciados pelo Docker
  • Possuem melhor desempenho que bind mounts em produção

3. Para que servem os volumes

a. Compartilhamento entre containers

Dois ou mais containers podem acessar o mesmo volume e compartilhar dados.

b. Armazenamento remoto

Volumes podem ser usados com drivers externos para armazenar dados fora da máquina local.

c. Backup e migração

Podemos copiar volumes e transferir dados entre ambientes diferentes.

4. Comandos para trabalhar com volumes

a. docker volume create


docker volume create meu-volume

b. docker volume ls


docker volume ls

c. docker volume inspect


docker volume inspect meu-volume

d. docker volume rm


docker volume rm meu-volume

e. docker volume

Grupo de comandos para gerenciamento de volumes.

f. docker system prune

Remove recursos não utilizados, incluindo volumes (com flag específica). 


docker system prune -a --volumes

g. Flags --mount e --volume

Existem duas formas de montar volumes:

Forma antiga (mais simples):


-v meu-volume:/app

Forma moderna (recomendada):


--mount type=volume,source=meu-volume,target=/app

Passo a Passo: Trabalhando com Volumes

Passo 0 — Criar branch


git checkout -b aula4-volumes-docker

Passo 1 — Criar volume


docker volume create dados-app

Passo 2 — Criar container usando volume


docker run -d -p 8080:80 --name nginx-volume \
-v dados-app:/usr/share/nginx/html nginx

Passo 3 — Criar conteúdo dentro do volume


docker exec -it nginx-volume bash

echo "<h1>Site com volume</h1>" > /usr/share/nginx/html/index.html
exit

Acesse:


http://localhost:8080

Passo 4 — Remover container


docker stop nginx-volume
docker rm nginx-volume

Passo 5 — Criar novo container com mesmo volume


docker run -d -p 8080:80 --name nginx-volume2 \
-v dados-app:/usr/share/nginx/html nginx

O conteúdo continuará lá, pois o volume persiste mesmo após remover o container.

Passo 6 — Testar bind mount

Crie uma pasta local:


mkdir site-local

Crie um index.html:


<h1>Site local com bind mount</h1>

Execute:


docker run -d -p 8081:80 --name nginx-bind \
-v ${PWD}/site-local:/usr/share/nginx/html nginx

Acesse:


http://localhost:8081

Passo 7 — Limpar recursos


docker stop nginx-volume2 nginx-bind
docker rm nginx-volume2 nginx-bind
docker volume rm dados-app

Último Passo — Subir para o Git


git add .
git commit -m "aula 4 volumes docker"
git push -u origin aula4-volumes-docker

Dever de Casa

Persistência de dados com Docker

Crie um volume chamado meu-volume-dever e utilize um container nginx para armazenar um arquivo HTML dentro dele.

Depois:

  • Remova o container
  • Crie outro container com o mesmo volume
  • Verifique se o conteúdo continua existindo

Obs: Criar branch dever-volumes-docker e subir no Git.

Gabarito: 

git checkout -b dever-volumes-docker

docker volume create meu-volume-dever

docker run -d -p 8080:80 --name app1 \
-v meu-volume-dever:/usr/share/nginx/html nginx

docker exec -it app1 bash
echo "<h1>Persistência OK</h1>" > /usr/share/nginx/html/index.html
exit

docker stop app1
docker rm app1

docker run -d -p 8080:80 --name app2 \
-v meu-volume-dever:/usr/share/nginx/html nginx

# acessar localhost:8080

docker stop app2
docker rm app2
docker volume rm meu-volume-dever

git add .
git commit -m "dever volumes docker"
git push -u origin dever-volumes-docker
