Dockerfile

1. Fundamentos da estrutura dos containers

a. Organização em camadas

As imagens Docker são construídas em camadas (layers). Cada instrução no Dockerfile cria uma nova camada.

Essas camadas são reaproveitadas (cache), o que torna o processo de build mais rápido quando não há alterações.

b. Imagem base (imagem pai)

Toda imagem começa a partir de uma imagem base. Essa imagem já possui um sistema mínimo e, muitas vezes, uma linguagem instalada.

Exemplos:

  • node:20
  • python:3.11
  • nginx
  • alpine

2. Objetivo do Dockerfile

O Dockerfile é um arquivo de texto com instruções que descrevem como uma imagem Docker deve ser construída.

Ele garante que qualquer pessoa consiga gerar a mesma imagem de forma padronizada.

3. Estrutura do Dockerfile

Um Dockerfile é composto por instruções executadas em sequência.


FROM node:20

WORKDIR /app

COPY . .

RUN npm install

CMD ["node", "index.js"]

4. Instruções do Dockerfile

a. FROM

Define a imagem base.

b. RUN

Executa comandos durante o build.

c. COPY

Copia arquivos do host para o container.

d. ADD

Semelhante ao COPY, mas com funcionalidades extras.

e. LABEL

Adiciona metadados à imagem.

f. ENV

Define variáveis de ambiente.

g. CMD

Define o comando padrão ao iniciar o container.

h. WORKDIR

Define o diretório de trabalho.

i. ARG

Define variáveis usadas no build.

j. ENTRYPOINT

Define o ponto de entrada do container.

k. EXPOSE

Indica a porta usada pela aplicação.

l. VOLUME

Define diretórios persistentes.

m. USER

Define o usuário que executa o container.

n. HEALTHCHECK

Verifica se o container está saudável.

o. SHELL

Define o shell padrão.

5. Otimização do tamanho da imagem

a. Cache de build

Docker reutiliza camadas já construídas. Alterações em camadas iniciais invalidam o cache das próximas.

b. Escolha da imagem base

Usar imagens menores como alpine reduz o tamanho final.

c. Multi-stage build

Permite separar etapas de build e execução, reduzindo o tamanho final da imagem. 


FROM node:20 AS build
WORKDIR /app
COPY . .
RUN npm install

FROM node:20-alpine
WORKDIR /app
COPY --from=build /app .
CMD ["node", "index.js"]

d. .dockerignore

Arquivo usado para ignorar arquivos desnecessários no build.


node_modules
.git
.env

Passo a Passo: Criando Dockerfile otimizado

Passo 0 — Criar branch


git checkout -b aula5-dockerfile

Passo 1 — Criar projeto Node


npm init -y
npm install express

Passo 2 — Criar index.js


const express = require('express');
const app = express();

app.get('/', (req, res) => {
  res.send('Dockerfile funcionando!');
});

app.listen(3000);

Passo 3 — Criar Dockerfile


FROM node:20-alpine

WORKDIR /app

COPY package*.json ./

RUN npm install

COPY . .

EXPOSE 3000

CMD ["node", "index.js"]

Passo 4 — Criar .dockerignore


node_modules
.git

Passo 5 — Build da imagem


docker build -t app-dockerfile .

Passo 6 — Rodar container


docker run -p 3000:3000 app-dockerfile

Acesse:


http://localhost:3000

Passo 7 — Testar multi-stage

Substitua o Dockerfile por versão multi-stage e refaça o build.

Último Passo — Subir para o Git


git add .
git commit -m "aula 5 dockerfile"
git push -u origin aula5-dockerfile
