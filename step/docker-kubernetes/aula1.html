Fundamentos do Docker

1. História do surgimento dos containers

Antes do Docker, muitos sistemas eram publicados diretamente em servidores físicos ou máquinas virtuais. Isso gerava um problema muito comum: a aplicação funcionava no computador do desenvolvedor, mas não funcionava corretamente no servidor.

Os containers surgiram como uma forma de empacotar uma aplicação junto com tudo aquilo que ela precisa para funcionar: bibliotecas, dependências, configurações e arquivos necessários.

A ideia principal é simples: se a aplicação roda dentro de um container, ela tende a rodar da mesma forma em diferentes ambientes.

2. Sistemas de virtualização baseados em hipervisores

A virtualização tradicional usa um software chamado hipervisor. Ele permite criar máquinas virtuais completas dentro de uma máquina física.

Cada máquina virtual possui seu próprio sistema operacional, memória, disco, processos e bibliotecas.

Exemplos de ferramentas de virtualização:

  • VirtualBox
  • VMware
  • Hyper-V

A vantagem é o isolamento. A desvantagem é que máquinas virtuais costumam consumir mais memória, mais processamento e mais espaço em disco.

3. Sistemas de virtualização a nível de kernel

Na virtualização a nível de kernel, não criamos um sistema operacional completo para cada aplicação.

Em vez disso, os containers compartilham o kernel do sistema operacional principal, mas continuam isolados em relação a processos, arquivos, rede e recursos.

Isso torna os containers mais leves e rápidos do que máquinas virtuais tradicionais.

4. Diferença entre virtualização e containerização

A virtualização cria máquinas virtuais completas. Cada máquina virtual precisa de um sistema operacional próprio.

A containerização cria ambientes isolados, mas que compartilham o kernel do sistema operacional da máquina hospedeira.

  • Máquina virtual: mais pesada, possui sistema operacional completo.
  • Container: mais leve, compartilha o kernel do sistema operacional.

5. Conceito de arquitetura de microsserviços

Microsserviços são uma forma de organizar uma aplicação em partes menores e independentes.

Em vez de criar uma única aplicação enorme, dividimos o sistema em serviços menores. Cada serviço pode ser responsável por uma função específica.

Exemplo:

  • Serviço de usuários
  • Serviço de produtos
  • Serviço de pagamentos
  • Serviço de notificações

O Docker é muito usado em arquiteturas de microsserviços porque permite executar cada serviço em um container separado.

6. Componentes do Docker

a. Docker Daemon

O Docker Daemon é o serviço principal do Docker. Ele fica rodando em segundo plano e é responsável por criar, executar, parar e remover containers.

b. Docker CLI

O Docker CLI é a ferramenta de linha de comando usada para enviar comandos ao Docker. 


docker --version
docker ps
docker images

c. Dockerfile

O Dockerfile é um arquivo de instruções usado para criar uma imagem Docker.

Nele informamos qual imagem base será usada, quais arquivos serão copiados, quais comandos serão executados e como a aplicação será iniciada.

d. Image

Uma imagem é um modelo usado para criar containers. Ela contém a aplicação, as dependências e as configurações necessárias.

e. Container

Um container é uma instância em execução de uma imagem.

Podemos pensar assim: a imagem é o molde, e o container é a aplicação rodando a partir desse molde.

f. Docker Registry

O Docker Registry é um local onde imagens Docker podem ser armazenadas e compartilhadas.

O registry mais conhecido é o Docker Hub, onde podemos encontrar imagens prontas de sistemas como Node.js, MySQL, Nginx, Python, Java e muitos outros.

7. Instalação do Docker

a. Instalação do Docker no Windows

No Windows, normalmente usamos o Docker Desktop.

  1. Acesse o site oficial do Docker.
  2. Baixe o Docker Desktop para Windows.
  3. Execute o instalador.
  4. Reinicie o computador, se necessário.
  5. Abra o Docker Desktop.

Em muitos computadores Windows, pode ser necessário ativar o WSL 2 ou a virtualização na BIOS.

b. Instalação do Docker no Linux

No Linux, a instalação pode variar conforme a distribuição. Em distribuições baseadas em Ubuntu, a instalação geralmente é feita pelo terminal. 


sudo apt update
sudo apt install docker.io
docker --version

8. Verificação do funcionamento da instalação

Para verificar se o Docker está instalado corretamente, usamos o comando: 


docker run hello-world

Esse comando baixa uma imagem de teste e executa um container simples. Se tudo estiver funcionando, o Docker exibirá uma mensagem confirmando que a instalação foi realizada com sucesso.

Passo a Passo: Criando o primeiro projeto com Docker

Passo 0 — Criar o repositório no GitHub

Acesse o GitHub e crie um novo repositório com o nome:


projeto-docker-step

Depois, clone o repositório no seu computador usando o terminal do VSCode: 


git clone URL_DO_REPOSITORIO
cd projeto-docker-step

Passo 1 — Criar uma branch para a aula

Dentro do projeto, crie uma branch chamada:


aula1-fundamentos-docker

No terminal, execute:


git checkout -b aula1-fundamentos-docker

Passo 2 — Criar o arquivo da aplicação

Crie um arquivo chamado index.js na raiz do projeto.

Dentro dele, coloque o código abaixo:


const http = require('http');

const servidor = http.createServer((req, res) => {
  res.writeHead(200, { 'Content-Type': 'text/html; charset=utf-8' });

  res.end(`
    <!DOCTYPE html>
    <html lang="pt-BR">
      <head>
        <meta charset="UTF-8" />
        <title>Primeira aplicação com Docker</title>
      </head>
      <body>
        <h1>Minha primeira aplicação Node.js com Docker</h1>
        <p>Essa página está sendo executada dentro de um container Docker.</p>

        <h2>Lista de tecnologias estudadas</h2>
        <ul>
          <li>Node.js</li>
          <li>Docker</li>
          <li>Containers</li>
          <li>GitHub</li>
        </ul>
      </body>
    </html>
  `);
});

servidor.listen(3000, () => {
  console.log('Servidor rodando em http://localhost:3000');
});

Passo 3 — Testar a aplicação sem Docker

Antes de criar o container, podemos testar a aplicação diretamente com Node.js.

No terminal, execute:


node index.js

Depois acesse no navegador:


http://localhost:3000

Passo 4 — Criar o Dockerfile

Agora vamos criar o arquivo que ensinará o Docker a montar a imagem da nossa aplicação.

Crie um arquivo chamado Dockerfile, sem extensão, na raiz do projeto. 


FROM node:20

WORKDIR /app

COPY index.js .

EXPOSE 3000

CMD ["node", "index.js"]

Passo 5 — Entendendo o Dockerfile

  • FROM node:20: usa uma imagem pronta do Node.js.
  • WORKDIR /app: define a pasta de trabalho dentro do container.
  • COPY index.js .: copia o arquivo index.js para dentro do container.
  • EXPOSE 3000: informa que a aplicação usa a porta 3000.
  • CMD ["node", "index.js"]: comando executado quando o container iniciar.

Passo 6 — Criar a imagem Docker

Agora vamos transformar nosso projeto em uma imagem Docker.

No terminal, execute:


docker build -t minha-primeira-app-docker .

O parâmetro -t define o nome da imagem. O ponto final indica que o Dockerfile está na pasta atual.

Passo 7 — Executar o container

Agora vamos executar a aplicação dentro de um container.


docker run -p 3000:3000 minha-primeira-app-docker

O parâmetro -p 3000:3000 conecta a porta 3000 do seu computador com a porta 3000 do container.

Acesse novamente:


http://localhost:3000

Passo 8 — Verificar containers em execução

Para listar os containers que estão rodando, execute:


docker ps

Passo 9 — Parar um container

Caso queira parar o container, primeiro copie o ID dele usando docker ps.

Depois execute:


docker stop ID_DO_CONTAINER

Passo 10 — Verificar imagens criadas

Para listar as imagens Docker disponíveis no seu computador, execute: 


docker images

Último Passo — Subindo para o Git

Depois de criar a aplicação, o Dockerfile e testar o container, salve tudo no Git. 


git add .
git commit -m "aula 1 fundamentos do docker"
git push -u origin aula1-fundamentos-docker

Dever de Casa

Criar uma aplicação Node.js executada com Docker

Crie um repositório no GitHub chamado atividade-docker-node.

Dentro dele, crie uma aplicação simples em Node.js que exiba uma página HTML com:

  • Um título principal
  • Um parágrafo explicando o que é Docker
  • Uma lista com pelo menos 4 comandos Docker
  • Um Dockerfile para executar a aplicação

A aplicação deve rodar na porta 3000.

Obs: Não se esqueça de criar uma branch chamada dever-docker-node e subir as alterações para o GitHub.

Gabarito:

1. Criar o repositório e a branch 


git clone URL_DO_REPOSITORIO
cd atividade-docker-node
git checkout -b dever-docker-node

2. Arquivo: index.js 


const http = require('http');

      <head>
        <meta charset="UTF-8" />
        <title>Atividade Docker</title>
      </head>
      <body>
        <h1>Atividade Docker com Node.js</h1>

        <p>
          Docker é uma ferramenta usada para executar aplicações em containers.
          Ele ajuda a padronizar o ambiente da aplicação e facilita a execução
          em diferentes computadores.
        </p>

        <h2>Comandos Docker</h2>

        <ul>
          <li>docker --version</li>
          <li>docker run hello-world</li>

          <li>docker ps</li>

        </ul>
      </body>
    </html>
  `);
