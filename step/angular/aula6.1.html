Conteúdo Extra - Guardião de Rotas com Token JWT e Permissões da API

Tópicos do nosso plano de curso abordados nessa aula: Navegação e transferência de dados (Guardião de Rotas com Token JWT).

O que vamos aprender?

Nesta aula extra, vamos evoluir o conceito de Route Guards.

Agora, ao invés de apenas verificar se existe um token no localStorage, vamos trabalhar com autenticação real utilizando API.

O que iremos fazer?

  • Fazer login em uma API
  • Receber token JWT
  • Salvar token no localStorage
  • Buscar usuário autenticado
  • Verificar permissões do usuário
  • Proteger rotas com guards

API utilizada

Vamos utilizar a API pública DummyJSON Auth API.

Documentação:


https://dummyjson.com/docs/auth

Fluxo da aplicação


Login → API retorna token →
Token salvo →
Guard verifica token →
Guard busca usuário →
Guard verifica role/permissão →
Acesso liberado ou bloqueado

Estrutura que iremos criar


services/
 ├── auth.service.ts

guards/
 ├── auth.guard.ts
 ├── admin.guard.ts

Passo 3 — Criar AuthService

No arquivo auth.service.ts:


import { Injectable, inject } from '@angular/core';

import { HttpClient } from '@angular/common/http';

@Injectable({
  providedIn: 'root'
})
export class AuthService {

  private http = inject(HttpClient);

  api = 'https://dummyjson.com/auth';

  login(username: string, password: string) {

    return this.http.post(
      `${this.api}/login`,
      {
        username,
        password
      }
    );
  }

  getMe() {

    const token = localStorage.getItem('token');

    return this.http.get(
      `${this.api}/me`,
      {
        headers: {
          Authorization: `Bearer ${token}`
        }
      }
    );
  }
}

Passo 4 — Criar página de login

Exemplo simples de login:


import { Component, inject } from '@angular/core';

import { AuthService } from '../../services/auth.service';

@Component({
  selector: 'app-login',
  standalone: true,

  template: `
    
  `
})
export class Login {

  private authService = inject(AuthService);

  login() {

    this.authService.login(
      'emilys',
      'emilyspass'
    ).subscribe((response: any) => {

      localStorage.setItem(
        'token',
        response.accessToken
      );

    });
  }
}

Passo 5 — Criar Auth Guard

No terminal execute:


ng g guard guards/auth

Escolha a opção CanActivate

No arquivo auth.guard.ts:


import { inject } from '@angular/core';

import {
  CanActivateFn,
  Router
} from '@angular/router';

export const authGuard: CanActivateFn = () => {

  const router = inject(Router);

  const token = localStorage.getItem('token');

  if (!token) {

    router.navigate(['/login']);

    return false;
  }

  return true;
};

Passo 6 — Criar Guard de Permissão

Agora vamos criar um guard responsável por verificar se o usuário é administrador. 


ng g guard guards/admin

No arquivo admin.guard.ts:


import { inject } from '@angular/core';

import {
  CanActivateFn,
  Router
} from '@angular/router';

import { map, catchError, of } from 'rxjs';

import { AuthService } from '../services/auth.service';

export const adminGuard: CanActivateFn = () => {

  const authService = inject(AuthService);

  const router = inject(Router);

  return authService.getMe().pipe(

    map((user: any) => {

      if (user.role === 'admin') {

        return true;
      }

      router.navigate(['/produtos']);

      return false;
    }),

    catchError(() => {

      router.navigate(['/login']);

      return of(false);
    })
  );
};

Como esse guard funciona?

  • Verifica se existe token
  • Busca o usuário autenticado na API
  • Verifica a role do usuário
  • Libera ou bloqueia acesso

Passo 7 — Aplicar os guards nas rotas

No arquivo app.routes.ts:


import { Routes } from '@angular/router';

import { authGuard } from './guards/auth.guard';

import { adminGuard } from './guards/admin.guard';

export const routes: Routes = [

  {
    path: 'login',

    loadComponent: () =>
      import('./pages/login/login').then(
        (m) => m.Login
      ),
  },

  {
    path: 'produtos',

    loadComponent: () =>
      import('./pages/lista-produtos/lista-produtos').then(
        (m) => m.ListaProdutos
      ),
  },

  {
    path: 'painel-adm',

    loadComponent: () =>
      import('./painel-adm/painel-adm').then(
        (m) => m.PainelAdm
      ),

    canActivate: [
      authGuard,
      adminGuard
    ]
  },

  {
    path: '**',
    redirectTo: 'produtos',
  }
];

Dever de Casa

Criar Guard para usuários comuns

Crie um guard chamado userGuard.

Esse guard deve permitir acesso apenas para usuários com: 


role === 'user'

Caso o usuário não possua essa permissão, ele deve ser redirecionado para a página de produtos.

Gabarito:

Crie o arquivo user.guard.ts


import { inject } from '@angular/core';

import {
  CanActivateFn,
  Router
} from '@angular/router';

import { map, catchError, of } from 'rxjs';

import { AuthService } from '../services/auth.service';

export const userGuard: CanActivateFn = () => {

  const authService = inject(AuthService);

  const router = inject(Router);

  return authService.getMe().pipe(

    map((user: any) => {

      if (user.role === 'user') {

        return true;
      }

      router.navigate(['/produtos']);

      return false;
    }),

    catchError(() => {

      router.navigate(['/login']);

      return of(false);
    })
  );
};
